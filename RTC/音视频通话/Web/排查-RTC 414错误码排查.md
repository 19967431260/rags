## 问题：Web RTC 414错误码排查

## 问题详情

**现象**：
Web端RTC音视频通话初始化时返回414错误码，提示token计算错误或过期。问题发生在特定用户身上，其他用户使用相同代码正常。

## 排查过程

1. 确认错误码414含义 → token计算错误或过期
2. 排除网络问题 → 414与网络无关，是精准鉴权问题
3. 确认安全模式未改动 → 只考虑token计算问题
4. 建议业务端埋点排查 → 对比服务端计算的token和前端join房间的token，检查appkey、uid、初始化地址是否一致

## 问题原因

token计算错误，可能是Appkey、token、uid、channelName不匹配，或uid传入了非long参数/负数

## 解决方案

检查服务端获取token的方式，确保Appkey、token、uid、channelName匹配，uid必须是long类型且不能为负数。建议前后端埋点对比token值。
