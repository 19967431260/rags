## 问题：Server 录制分辨率过高导致转码OOM

## 问题详情

**现象**：
客户反馈某个房间的视频录制了7个小时但一个视频都没生成，而另一个项目10个视频都生成了。

## 排查过程

1. 客服确认录制7小时无视频生成 → 确认问题现象
2. 技术值班排查确认是服务器转码进程 OOM → 根因定位
3. 客户调大了部分房间的录制分辨率 → 触发原因
4. 临时方案：降低转码并发线程 → 缓解措施
5. 根本方案：增加服务器资源 → 彻底解决方案

**关键发现**：客户调大了部分房间的录制分辨率，服务器资源不足导致转码进程 OOM（内存溢出）

## 问题原因

客户调大了部分房间的录制分辨率，服务器资源不足导致转码进程 OOM（内存溢出）

## 解决方案

临时方案：降低转码并发线程数，可让整体流程顺利进行但会降低录制文件生成速度。根本解决方案：增加服务器 CPU/内存资源。
