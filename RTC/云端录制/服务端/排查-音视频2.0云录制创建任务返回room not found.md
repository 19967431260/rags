## 问题：服务端 音视频2.0云录制创建任务返回room not found

## 问题详情

**现象**：
调用云录制创建录制请求API时返回room not found错误。客户使用Flutter客户端，在P2P两人都已进入视频聊天后发送录制请求，传递cname参数。环境：AppKey为d58406c033a200ba4f235b5990bf8b71（扁鹊门），测试Key为b025ad31c1ab5e2dcd9fd443bfa3ffa6。

**环境信息**：
- SDK 版本：
- 系统版本 / 设备：
- 平台：服务端

## 排查过程

1. 客户询问cname与channelName关系 → 技术支持确认cname就是channelName
2. 客户反馈调用返回room not found → 技术支持初步判断房间未创建或通话已结束
3. 客户提供channelName: 07557e0c97264eac8792d36a3af21a92, channelId: 6137fff39a514b8fa0e216d3ca047114 → 技术支持查询确认房间在通话中
4. 客户再次调用返回room already has running record task → 发现是重复调用导致，首次调用实际已成功

## 问题原因

重复调用创建录制任务API，首次调用已成功创建录制任务，第二次调用时返回房间已有运行中的录制任务。

## 解决方案

首次调用已成功创建录制任务（返回taskId），无需重复调用。可通过查询房间内事件或录制完成抄送确认录制状态。
