## Q: 视频录像切片数量查询及录制状态

**问题描述**：
1. 房间刚结束时，咨询是否有接口能知道一共有多少个视频录像的切片
2. 咨询录制完成的状态标记

**答案**：
1. 暂时没有接口能查询视频录像切片数量
2. 目前还没有接口支持录制都抄送完成的状态标记
3. 水印功能在极端情况下会出现界面黑屏的情况（浏览器的节能机制可能会限制SDK水印功能的运行）
4. 如果没用混录视频水印，暂时先不建议开启

## Q: 录制分辨率降低问题排查

**问题描述**：
录制视频分辨率降低，咨询排查方法。

**答案**：
1. 客户端可以查看setVideoConfig的设置：https://doc.yunxin.163.com/nertc/guide/jY3OTI2NDM?platform=web
2. 需要协调找个复现场景拿客户端日志
3. 客户端CPU满了不会自动降低分辨率
4. 根据过往经验，现场环境网络质量下降可能会去降分辨率
5. 可以通过SDK方法获取上行分辨率：client.getLocalVideoStats()

## Q: 日志获取及日志上报配置

**问题描述**：
咨询日志获取方法和日志上报配置。

**答案**：
1. 客户端日志获取参考：https://faq.yunxin.163.com/kb/main/#/item/KB0179
2. Web端日志只能本地拿，需要右键保存完整的日志信息
3. 可以开启日志上报：RTC SDK初始化后加入房间前调用enableLogUpload方法
4. 参考文档：https://doc.yunxin.163.com/docs/interface/nertc/web/typedoc/Latest/zh/html/modules/nertc.nertc-1.logger.html#enablelogupload
5. 私有化配置需要加上neRtcServerAddresses.statisticsWebSocketServer地址
6. 配置示例：
```
neRtcServerAddresses: {
  channelServer: "https://rtc.xxx.com:2443/nrtc/getChannelInfos.action",
  statisticsServer: "https://rtc.xxx.com:2443/report/statics/report/common/form",
  roomServer: "https://rtc.xxx.com:2443/v2/sdk/rooms",
  statisticsWebSocketServer: "wss://rtc.xxx.com:2443/lps-websocket/websocket/collect",
}
```

## Q: 屏幕共享视频编码配置

**问题描述**：
咨询屏幕共享的视频编码配置，特别是contentHint设置。

**答案**：
1. 开启设备localStream.open()已经包含publish，调用open之后可以不用再执行publish
2. contentHint设置为motion：表示网络变差或者CPU不足的时候，编码优先保证视频的流畅度，将倾向降低视频的画面分辨率
3. contentHint设置为detail：表示清晰优先，如果分辨率不降，会影响流畅度，会变卡
4. 建议监听SDK network-quality事件，可以实时获取当前用户的上行和下行网络质量
5. 参考文档：https://doc.yunxin.163.com/interactive-streaming/guide/Tg1MzQ1Mzg?platform=web
