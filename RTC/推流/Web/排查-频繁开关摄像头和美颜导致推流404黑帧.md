## 问题：Web 频繁开关摄像头和美颜导致推流404黑帧

## 问题详情

**现象**：
用户非常快速操作开启摄像头->开启美颜->关闭摄像头循环，最后推流失败404或出现黑帧。

## 排查过程

1. 检查操作流程 → 发现用户频繁开关摄像头和美颜
2. 分析SDK行为 → SDK在close()时会主动关闭美颜
3. 发现时序问题 → 用户在开启摄像头过程中执行关闭，SDK缓存动作导致美颜模块数据源异常

**关键发现**：SDK执行缓存的close → 关闭美颜 → 业务层开启美颜(加载旧数据源) → SDK close关闭video → 美颜缺少输入 → 业务层open摄像头 → 美颜使用缓存流 → 编码黑帧

## 问题原因

SDK在用户调用close()关闭摄像头时会主动关闭美颜，频繁操作导致美颜模块缓存了错误的媒体流。

## 解决方案

关闭摄像头前先判断：if (!this.localStream.hasVideo()) { return }，避免在未开启摄像头时执行close。SDK下个版本会内部兼容此执行流程。
