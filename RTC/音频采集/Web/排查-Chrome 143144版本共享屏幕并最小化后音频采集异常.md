## 问题：Web Chrome 143/144版本共享屏幕并最小化后音频采集异常

## 问题详情

**现象**：
直播回放视频中第4分钟开始突然没有声音，持续约5分钟，用户重新进入后恢复正常。会议号0d72664a317349be983840db6aa28b75。用户共享屏幕后需要最小化浏览器查看PPT。

## 排查过程

1. 音频日志显示发送码率相同持续5分钟
2. 上传码率不为0且无丢包，但回放无声音
3. 相当于发送了静音帧
4. 用户共享并最小化标签页3分钟后出现采集不到音频数据
5. 出现问题后用户第二次仍最小化浏览器但正常

## 问题原因

Chrome 143/144版本bug，标签页最小化后触发后台资源限制导致音频采集异常。与Windows电源计划、省电策略、用户手势交互有关。SDK未做特殊处理。

## 解决方案

1. 共享时不要最小化标签页，直接打开PPT页面
2. 等待Chrome 146版本修复该问题
3. 回到前台主动重置麦克风（手势交互）可恢复
参考：https://issues.chromium.org/issues/331092194
