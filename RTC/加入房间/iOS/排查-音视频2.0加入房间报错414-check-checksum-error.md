## 问题：iOS 音视频2.0加入房间报错414 check checksum error

## 问题详情

**现象**：
iOS端调用joinChannel加入互动直播房间时报错：Error Domain=NERtcRemoteErrorDomain Code=414 "check checksum error"

## 排查过程

1. 检查报错信息 → Code=414 check checksum error
2. 确认鉴权模式 → 安全模式，需要传token
3. 检查token来源 → 客户使用IM的refreshToken接口获取
4. 确认问题原因 → IM的refreshToken接口与RTC无关

**关键发现**：客户误用了IM的token接口，应该使用RTC专用的token接口

## 问题原因

客户使用了IM的refreshToken接口获取token，该接口与RTC无关，导致token校验失败

## 解决方案

使用RTC专用的获取token接口：https://doc.yunxin.163.com/nertc/server-apis/TcxNDAxMTI?platform=server#方式二申请-nertc-token。如果是后端生成token，需要确认appsecret、nonce、curtime等参数的有效值，可以使用在线工具对比验证：https://www.iamwawa.cn/jiami.html
