网易云信即时通讯 SDK（NetEase IM SDK，以下简称 NIM SDK）支持本地会话管理功能，包括创建、更新、删除本地会话等基础操作，以及置顶会话等进阶操作。

本地会话列表由 SDK 维护并提供查询、监听变化的接口，当会话变更时，SDK 会自动更新会话列表并通知，您无需手动更新。

如何实现本地会话相关功能请参考 [本地会话管理](https://doc.yunxin.163.com/messaging2/client-apis/zMyODU1OTM?platform=client)。本文介绍 NIM SDK 本地会话相关 API。

## 接口类

`V2NIMLocalConversationService` 类提供创建、删除、更新、获取、置顶本地会话，本地会话消息未读数相关、注册本地会话监听等接口。

## <span id="on">on("EventName")</span>

**接口描述**

注册本地会话相关监听。

注册成功后，当事件发生时，SDK 会触发相关回调通知。

::: note note
- 建议在初始化后调用该方法。
- 全局只需注册一次。
- 该接口为同步接口。
:::

**参数说明**

:::::: div linked-codes
::: code Web/uni-app/小程序
```TypeScript
export interface NIMEBaseServiceInterface<I extends NIMEBaseListener> {
  /**
   * 继承自 eventEmitter3 的监听事件方法
   */
  on<T extends keyof I>(eventName: T, fn: (...args: I[T]) => void): void
  /**
   * 继承自 eventEmitter3 的监听事件方法
   */
  once<T extends keyof I>(eventName: T, fn: (...args: I[T]) => void): void
}
```
| 参数名称 | 类型 | 是否必填 | 说明 |
| ---- | ---- | ---- | ---- |
| `eventName` | T | 是 | 事件名称：<ul><li>`onSyncStarted`：数据同步开始回调。建议在回调完成后操作数据，如果在此期间操作数据，会出现数据不全，只能操作部分数据的情况。</li><li>`onSyncFinished`：数据同步结束回调。此回调后需要用户重新获取本地会话列表。回调后可任意操作相关的本地会话数据。</li><li>`onSyncFailed`：数据同步失败回调。此回调后可以操作本地会话数据，但由于同步失败，本地只能操作已有数据，数据可能不全。在相关错误恢复后，SDK 会逐步按需重建会话数据。</li><li>`onConversationCreated`：本地会话创建回调，返回创建的本地会话对象。当本端或多端同步主动创建本地会话成功时会触发该回调。</li><li>`onConversationDeleted`：本地会话删除回调，返回被删除的本地会话 ID 列表。当本端或多端同步主动删除本地会话成功时会触发该回调。</li><li>`onConversationChanged`：本地会话变更回调，返回变更后的会话列表。当本端或多端同步调用会话置顶、免打扰、更新消息内容成功时会触发该回调。</li><li>`onTotalUnreadCountChanged`：本地会话消息总未读数变更回调，返回变更后的消息未读数。</li><li>`onUnreadCountChangedByFilter`：指定过滤条件的会话消息未读数变更，根据过滤条件返回变更后的消息未读数。</li><li>`onConversationReadTimeUpdated`：同一账号多端登录会话已读时间戳标记时间变更回调，返回同步标记的会话 ID 和标记的时间戳。</li></ul>
| `fn` | function | 是 | 监听事件的回调函数。当事件触发时，会调用该函数进行处理。
:::

## <span id="deleteConversationListByIds">deleteConversationListByIds</span>

**接口描述**

根据会话 ID 批量删除指定的本地会话列表。

本端或多端同步删除每一条本地会话成功后，SDK 均会返回删除成功回调 [`onConversationDeleted`](https://doc.yunxin.163.com/messaging2/client-apis/DAxNjk0Mzc?platform=client#V2NIMLocalConversationListener) 回调，并同步更新缓存和数据库。

如果被删除的本地会话中有消息未读，SDK 还会返回 [`onTotalUnreadCountChanged`](https://doc.yunxin.163.com/messaging2/client-apis/DAxNjk0Mzc?platform=client#V2NIMLocalConversationListener) 和 [`onUnreadCountChangedByFilter`](https://doc.yunxin.163.com/messaging2/client-apis/DAxNjk0Mzc?platform=client#V2NIMLocalConversationListener) 回调。

## <span id="getConversationList">getConversationList</span>

**接口描述**

分页查询所有本地会话列表。

该方法从客户端本地分页获取会话数据，本地历史会话数据无上限，因此建议分页多次获取，直到获取全量会话数据，SDK 会进行数据同步并返回对应回调通知 UI 层。

::: note note
- 查询的本地会话列表结果中，置顶会话排首位。
- 数据同步完成前，可能查询不到完整数据或者查询到的是老数据。
- 查询云端会话时，当正好查询完所有数据时，下次查询时才会提示查询完毕（finished = true）。但是查询本地会话时，当正好查询完所有本地数据时，当次查询时就会提示查询完毕（finished = true）。
