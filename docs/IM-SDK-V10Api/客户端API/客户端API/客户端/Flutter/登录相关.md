## 接口类

`LoginService` 类提供登录、登出、踢出其他设备端、注册登录连接状态监听器等接口。

## <span id="listen">listen</span>

**接口描述**

注册登录相关监听。

注册成功后，当事件发生时，SDK 会触发相关回调通知。

::: note note
- 建议在初始化后、登录前调用该方法。

- 全局只需注册一次。

- 该方法为同步。
:::

**回调事件**

- `onLoginStatus`：登录状态变化回调，返回变更后的登录状态。
- `onLoginFailed`：登录失败回调，返回错误码和错误信息。
- `onKickedOffline`：客户端被踢回调，返回被踢详细信息。 
- `onLoginClientChanged`：多端登录时，其他在线客户端登录状态变化回调，返回变更原因和变更具体信息。
- `onConnectStatus`：登录连接状态变化回调，返回连接状态。 
- `onDisconnected`：登录连接断开回调，返回错误码和错误信息。 
- `onConnectFailed`：登录连接失败回调，返回错误码和错误信息。 
- `onDataSync`：数据同步状态变化回调，返回数据同步类型、同步状态和同步错误信息。

**示例代码**

```dart
final subsriptions = <StreamSubscription>[];
subsriptions.add(NimCore.instance.loginService.onLoginStatus.listen((event) {
}));
subsriptions.add(NimCore.instance.loginService.onLoginFailed.listen((event) {
}));
subsriptions.add(NimCore.instance.loginService.onKickedOffline.listen((event) {
}));
subsriptions.add(NimCore.instance.loginService.onLoginClientChanged.listen((event) {
}));
subsriptions.add(NimCore.instance.loginService.onConnectStatus.listen((event) {
}));
subsriptions.add(NimCore.instance.loginService.onDisconnected.listen((event) {
}));
subsriptions.add(NimCore.instance.loginService.onConnectFailed.listen((event) {
}));
subsriptions.add(NimCore.instance.loginService.onDataSync.listen((event) {
}));
```

## <span id="login">login</span>

**接口描述**

登录 IM。调用该接口后，SDK 会自动连接服务器，传递用户信息，返回登录结果。

网易云信 IM 支持多种登录鉴权方式，具体请参考 [登录及登出 IM](https://doc.yunxin.163.com/messaging2/client-apis/Dk1MTY4MzA?platform=client) 开发指南文档。

::: note note
调用该接口成功后，建议将 `accountId` 和 `token` 保存到本地，方便下次应用启动时实现自动登录。
:::
  
**适用场景**

- 在新设备上首次登录 IM
- 同时在线的其他设备踢下线后再次登录 IM
- 切换 IM 账号后再次登录 IM
- 注销登录后再次登录 IM

**参数说明**

```dart
Future<NIMResult<void>> login(
    String accountId, String token, NIMLoginOption option) async {
  return _platform.login(accountId, token, option);
}
```

参数名称 | 类型 | 是否必填 | 说明 |
---- | ---- | ---- | ---- |
`accountId` | String  | 是 | 云信 IM 为用户分配的唯一登录账号。                  |
`token` | String  | 是 | 云信 IM 登录鉴权 Token（静态），获取方式详见 [获取静态 Token](https://doc.yunxin.163.com/messaging2/guide/Dk1MTY4MzA?platform=client#获取静态-token)。               |
`loginOption` | `NIMLoginOption` | 否 | 登录配置。主要包含如下配置项：<li>是否强制登录</li><li>登录失败后重试次数<li>动态 Token 获取回调<li>鉴权方式 |
 `retryCount` | int | 否 | 登录失败后，重试登录的次数。默认重试 3 次。<br>若登录成功后因网络原因断开连接，则不限次数重连。
 `timeout` | int | 否 | 登录超时时间，单位毫秒，默认为 6000ms。<br>系统会在超时时间内重试登录，重试次数最大为 `retryCount`。 |
  `authType` | [`NIMLoginAuthType`](https://doc.yunxin.163.com/messaging2/client-apis/zExMjk2NzY?platform=client#NIMLoginAuthType) | 否 | 鉴权方式，包括静态 Token 鉴权（默认）、动态 Token 鉴权、通过第三方回调鉴权。
  `forceMode` | bool | 否| 多端登录互踢时，是否强制登录，默认为 false。<li>true：强制登录模式。设置为该模式后，如果在 [网易云信控制台](https://app.yunxin.163.com/global/home) 上配置了 **多端登录互踢策略**，则无论本端是否为登录存续状态（未登出但退出应用），登录成功时将其他在线设备端踢下线。<li>false：非强制登录模式。设置为该模式后，如果在 [网易云信控制台](https://app.yunxin.163.com/global/home) 上配置了 **多端登录互踢策略**：<br><li>本端为登录存续状态，本端登录失败且返回 417 错误码（禁止多端登录）。此时可决定再次登录并设置强制登录模式或放弃登录。<li>本端为非登录存续状态，则登录成功时将其他在线设备端踢下线。
  `syncLevel` | [`NIMDataSyncLevel`](https://doc.yunxin.163.com/messaging2/client-apis/zExMjk2NzY?platform=client#NIMDataSyncLevel) | 是 | 数据同步模式，分为完全同步和基础数据同步。|

**示例代码**

```dart
var options = NIMLoginOption();

final loginResult = await NimCore.instance.loginService.login(
        accountId, token, options);

```
  
**返回值**

NIMResult\<void>
 
**相关回调**

无

## <span id="kickOffline">kickOffline</span>

**接口描述**

主动将同时在线的其他客户端踢下线。

被踢端若调用 `add` 注册登录状态监听器，在被踢下线后，会收到 `onKickedOffline` 回调。

::: note note
如需重新登录，建议先调用 `logout` 登出 IM，再切换到登录界面进行登录。
:::
  
**参数说明**

```dart
Future<NIMResult<void>> kickOffline(NIMLoginClient client) {
  return _platform.kickOffline(client);
}
```

| 参数名称 | 类型 | 是否必填 | 说明 |
| ---- | ---- | ---- | ---- |
| `client` | [`NIMLoginClient`](https://doc.yunxin.163.com/messaging2/client-apis/zExMjk2NzY?platform=client#NIMLoginClient) | 是 | 被踢下线的客户端信息。|

**示例代码**

```dart
await NimCore.instance.loginService.kickOffline();
```

**返回值**

NIMResult\<void>

**相关回调**

`onKickedOffline`：被踢下线回调

## <span id="setReconnectDelayProvider">setReconnectDelayProvider</span>

**接口描述**

设置登录重连延时的回调函数，用于提供重连延迟时间。

调用该接口成功后，当遇到网络连接中断时，SDK 会自动调用回调函数获取重连延迟时间。

**参数说明**

```dart
Future<NIMResult<void>> setReconnectDelayProvider(
    NIMReconnectDelayProvider? provider) {
  return _platform.setReconnectDelayProvider(provider);
}
```

**示例代码**

```dart
NimCore.instance.loginService
                        .setReconnectDelayProvider((time) async {
                      //返回自定义的时间
                      return time + 1;
                    });
```

**返回值**

NIMResult\<void>

**相关回调**

reconnectDelayProvider
