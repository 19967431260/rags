云端会话分组是将多个相关主题或目的的聊天会话组织在一起的一种方式。通过这种方式，用户可以更轻松地管理和查找他们的历史聊天会话。如何实现云端会话分组相关功能请参考开发文档 [云端会话分组管理](https://doc.yunxin.163.com/messaging2/client-apis/DYzMjEwOTg?platform=client)。

本文介绍网易云信即时通讯 SDK（NetEase IM SDK，以下简称 NIM SDK）云端会话管理相关 API。

## API 概览

### 云端会话分组监听

[listen](#listen) | 注册云端会话分组监听器 | v10.6.0
[cancel](#cancel) | 移除云端会话分组监听器 | v10.6.0

### 云端会话分组操作

API | 说明 | 起始版本
---- | ---- | ----

[getConversationGroupListByIds](#getConversationGroupListByIds) | 批量获取指定云端会话分组列表 | v10.6.0

## 接口类

`V2NIMConversationGroupService` 类提供创建、删除、更新、获取云端会话分组，注册云端会话分组监听等接口。

## <span id="listen">listen</span>

**接口描述**

注册云端会话分组监听。

注册成功后，当事件发生时，SDK 会触发相关回调通知。

::: note note
- 建议在初始化后调用该方法。
- 全局只需注册一次。
- 该方法为同步。
:::

**回调事件**

- `onConversationGroupCreated`：云端会话分组创建成功回调，返回创建的云端会话分组对象。
- `onConversationGroupDeleted`：云端会话分组删除回调，返回被删除的云端会话分组 ID 列表。
- `onConversationGroupChanged`：云端会话分组变更回调，返回变更后的云端会话分组对象。
- `onConversationsAddedToGroup`：将云端会话列表添加到会话分组回调，返回云端会话分组 ID 和添加的云端会话对象列表。
- `onConversationsRemovedFromGroup`：将云端会话从会话分组中移除回调，返回云端会话分组 ID 和移除的云端会话 ID 列表。

回调事件详细信息请参考 [会话分组相关监听器](https://doc.yunxin.163.com/messaging2/client-apis/zExMjk2NzY?platform=client#会话分组相关监听器)。


**示例代码**

```Dart
subsriptions.addAll([
  NimCore.instance.conversationGroupService.onConversationGroupChanged.listen((event) {
    // todo onConversationGroupChanged
  }),
  NimCore.instance.conversationGroupService.onConversationGroupCreated.listen((event) {
    // todo onConversationGroupCreated
  }),
  NimCore.instance.conversationGroupService.onConversationGroupDeleted.listen((event) {
    // todo onConversationGroupDeleted
  }),
  NimCore.instance.conversationGroupService.onConversationsAddedToGroup.listen((event){
    // todo onConversationsAddedToGroup
  }),
  NimCore.instance.conversationGroupService.onConversationsRemovedFromGroup.listen((event){
    // todo onConversationsRemovedFromGroup
  })
]);
```

## <span id="cancel">cancel</span>

**接口描述**

取消注册云端会话分组相关监听。

::: note note
该方法为同步。
:::

**示例代码**

```Dart
subsriptions.forEach((subsription) {
      subsription.cancel();
    });
```

## <span id="createConversationGroup">createConversationGroup</span>

**接口描述**

创建一个云端会话分组，并指定分组中的云端会话列表。

创建成功后，SDK 会返回创建成功回调 `onConversationGroupCreated`，并同步缓存和数据库。

::: note note
- 不同的 IM 套餐包中，每个用户账号能创建的会话分组数量上限，以及每个会话分组中的会话数量上限都不同。具体可参考 [各套餐详细对比信息](https://doc.yunxin.163.com/messaging2/concept/DczNjI1ODM?platform=client#套餐包对比) 中的会话分组相关信息。
- 如不需要该分组，建议及时调用 `deleteConversationGroup` 方法删除。
:::

**参数说明**

```Dart
Future<NIMResult<V2NIMConversationGroupResult>> createConversationGroup(
  String name,
  List<String> conversationIds, {
  String? serverExtension,
  })
```
| 参数名称 | 类型 | 是否必填 | 说明 |
| ---- | ---- | ---- | ---- |
| `name` | String | 是 | 云端会话分组名称。不可设置为空，否则返回 191004 参数错误。 |
| `serverExtension` | String | 否 | 云端会话分组服务端扩展字段，多端同步。 |
| `conversationIds` | List\<String> | 否 | 云端会话 ID 列表 <li>可以为空 </li><li>最多传入 1000 个会话 ID。</li> |

**示例代码**

```Dart
NimCore.instance.conversationGroupService.createConversationGroup('name', ['id1', 'id2'],
      serverExtension: 'extension').then((result){
        if (result.isSuccess) {
          // todo conversationGroupService success
        }
  });
```

**返回参数**

NIMResult\<[V2NIMConversationGroupResult](https://doc.yunxin.163.com/messaging2/client-apis/zExMjk2NzY?platform=client#V2NIMConversationGroupResult)>：云端会话分组创建结果，包含云端会话分组信息和创建分组失败的云端会话 ID 列表。

**相关回调**

`onConversationGroupCreated`：创建云端会话分组回调
