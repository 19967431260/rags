该接口可以通过指定高级群 ID、消息 ID 和消息发送者的 IM 账号，查询对应高级群消息的已读未读详情，包括目标消息已读数、目标消息的未读数、已读用户列表和未读用户列表。

## 调用频率

单个应用默认最高调用频率请参考 [频控说明](https://doc.yunxin.163.com/messaging2/server-apis/DUzNjAzMTc?platform=server)。

## 请求信息

GET https://{endpoint}/im/v2/messages/actions/team_read_receipt

### 请求头参数

### 查询参数

参数名称 | 类型 | 是否必选 | 说明 |
---- | ---- | ---- | ----
`team_id` | Long | 是 | 需要查询的高级群 ID，创建群组成功时，网易云信服务器会自动生成并返回。
`team_type` | Integer | 是 | 群组类型，目前仅支持 **team_type=1**，即仅支持高级群。
`sender_id` | String | 是 | 消息发送者账号 ID。
`message_server_id` | Long | 是 | 需要查询的服务端消息 ID（发送群已读业务消息时服务器返回的消息 ID），最大长度 64 位长整型。
`snapshot` | Boolean | 否 | 是否返回已读/未读成员的 IM 账号 ID 列表，默认为 false（不返回）。

## 响应信息

### 响应头参数

响应 Header 的参数说明请参考 [响应 Header](https://doc.yunxin.163.com/messaging2/server-apis/zcwODA3MTU?platform=server#响应-header)。

### 响应体参数

参数名称 | 类型 | 说明 | 是否必返回 |
---- | ---- | ---- | ----
`code` | Integer | 状态码，200 表示请求成功。 | 是
`msg` | String | 提示信息。请求失败时返回错误信息，请求成功时返回 "success"。 | 是
`data` | Object | 返回的 JSON 数据对象，请求失败则返回空对象。 | 是 |
 `read_size` | Integer | 目标消息的已读用户数。 | 是
 `unread_size` | Integer | 目标消息的未读用户数。 | 是
 `read_account_ids` | Array of strings | 目标消息的已读用户账号 ID 列表。 | 否
 `unread_account_ids` | Array of strings | 目标消息的未读用户账号 ID 列表。 | 否
