网易云信服务端支持对指定的消息（包括单聊消息、高级群消息和超大群消息）删除快捷评论。

::: note note
调用添加消息快捷评论接口前，请至网易云信控制台 [开通消息快捷评论](https://doc.yunxin.163.com/messaging2/guide/jU0Mzg0MTU?platform=client#全局功能)。
:::

### 请求体参数

参数名称 | 类型 | 是否必选 | 描述
--- | --- | --- | ---
`operator_id` | String | 是 | 进行删除快捷评论操作的用户账号 ID。
`message` | Object | 是 | 被指定的消息对象。
 `conversation_type` | Integer | 是 | 会话类型。1：单聊。2：高级群。3：超大群。
 `sender_id` | String | 是 | 消息的发送者 ID。
 `receiver_id` | String | 是 | 消息的接收者 ID。
 `message_server_id` | Long | 是 | 消息的服务器 ID。
 `message_client_id` | String | 是 | 消息的客户端 ID。
 `create_time` | Long | 是 | 消息的发送时间。
`extension` | String | 否 | 快捷评论的扩展字段，JSON 格式。例如："{\"k\":\"v\"}"。<br>该字段长度上限以使用的 IM 套餐为准。IM 旗舰版及以上套餐才支持配置字段上限。
`index` | Long | 是 | 快捷评论类型，必须大于 0。可在本地自定义映射关系，例如：<li>表情回复。1：笑脸。2：大笑。UI 层根据 index 展示对应的表情。<li>文本快捷回复等其他回复场景。
`push_config` | Object | 否 | 推送相关配置项。
 `push_enabled` | Boolean | 否 | 该消息是否需要 APNs 推送或 Android 系统通知栏推送，默认为 true（推送）。只有该字段为 true 时，推送相关参数才会生效。
 `push_content` | String | 否 | 推送文案，长度上限 500 位字符。如果不填，则使用默认推送文案。
 `push_payload` | String | 否 | 推送对应的 payload，必须是 JSON 格式，长度上限 2048 位字符。详情请参考 [推送 payload 配置](https://doc.yunxin.163.com/messaging/server-apis/DQyNjc5NjE?platform=server)。
