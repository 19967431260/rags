该接口用于将聊天室设置为整体禁言状态（仅创建者和管理员能发言）。

:::note note
只有聊天室创建者或管理员才能设置整体禁言。
:::

## 请求信息

### 请求 URL

```
PATCH https://{endpoint}/im/v2/chatrooms/{room_id}/actions/chat_banned
```

:::note note
请求 URL 中的 `{endpoint}` 代表服务地址域名，您可以根据用户服务区域选择中国大陆和海外服务地址，并支持搭建高可用主备域名机制。详情请参考 [调用方式](https://doc.yunxin.163.com/messaging2/server-apis/zcwODA3MTU?platform=server#%E6%9C%8D%E5%8A%A1%E5%9C%B0%E5%9D%80) 服务地址章节。
:::

### 请求头参数

请求 Header 的参数说明请参考 [请求 Header](https://doc.yunxin.163.com/messaging2/server-apis/zcwODA3MTU?platform=server#请求-header)。

### 路径参数

参数名称 | 类型 | 是否必选 | 描述
---- | ---- | ---- | ----
`room_id` | Long | 是 | 需要设置整体禁言的聊天室 ID，聊天室创建成功后，网易云信会返回聊天室 ID。

### 请求体参数

参数名称 | 类型 | 是否必选 | 描述
---- | ---- | ---- | ----
`operator_account_id` | String | 是 | 操作者（聊天室创建者或管理员）的 IM 账号 ID。
`chat_banned` | Boolean | 是 | 是否将聊天室设置为整体禁言状态（仅管理员和创建者可以发言）。true：整体禁言；false：取消整体禁言。
`notification_enabled` | Boolean | 否 | 是否发送禁言聊天室通知。true（默认）：发送；false：不发送。
`notification_extension` | String | 否 | 通知事件的扩展字段，长度上限 2048 位字符。

### 请求体示例

```JSON
{
    "notification_enabled": true,
    "chat_banned": true,
    "notification_extension": "culpa tempor labore do",
    "operator_account_id": "yx1"
}
```
