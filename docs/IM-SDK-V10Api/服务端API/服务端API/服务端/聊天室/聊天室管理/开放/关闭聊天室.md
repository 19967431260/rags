若关闭聊天室后需要再次开启，创建者也可以调用此接口重新开放聊天室，该接口还可以设置聊天室定时关闭策略。

## 功能描述

创建聊天室后，聊天室默认处于开放状态，所有用户可自由进入聊天室。如需暂停所有用户可自由进入聊天室的权限，创建者可通过调用本服务端 API 关闭聊天室。

::: note notice
如果关闭聊天室，那么聊天室中所有在线成员将被强制踢下线。
:::

### 请求头参数

### 路径参数

参数名称 | 类型 | 是否必选 | 描述

### 请求体参数

参数名称 | 类型 | 是否必选 | 描述
---- | ---- | ---- | ----
`operator_account_id` | String | 是 | 操作者（聊天室创建者）的 IM 账号 ID。
`valid` | Boolean | 是 | 是否开放聊天室，true（默认）：开放；false：关闭。
`delay_close_policy` | Integer | 否 | 聊天室定时关闭方式。<br>0：不开启定时关闭；1：固定时间关闭（不管聊天室中是否还有用户）；2：空闲关闭（等聊天室中没有用户后固定时间关闭）。<note type=notice><ul><li>如果需要使用聊天室定时关闭功能（聊天室自动销毁机制），需要先在 [网易云信控制台](https://app.yunxin.163.com/global/home) 开启（具体请参考 <a href="https://doc.yunxin.163.com/messaging2/server-apis/jg4MzMyNTY?platform=server" target="_blank">开通和配置聊天室功能</a>）。</li><li>开启该功能后，自动销毁机制的策略可以直接通过该字段进行配置，也可以在 [网易云信控制台](https://app.yunxin.163.com/global/home) 配置（优先以该参数传入的策略为准，若未传入该参数，则以控制台配置的策略为准）。</li><li>仅该功能开通后创建的聊天室支持定时关闭，开通前创建的聊天室不支持定时关闭。</li></ul></note>
`delay_seconds` | Long | 否 | 聊天室定时关闭的时间，单位秒，最大值为 7x24x3600 秒。<note type=note><ul><li>当不开启定时关闭功能（delay_close_policy = 0）时，该参数不生效。</li><li>当选择固定时间关闭或空闲关闭策略时（delay_close_policy = 1 或 2）：<ul><li>若该参数不为空，则以传入的定时时间为准；</li><li>若未传入该参数，则以控制台的定时时间为准。</li></ul></li></ul></note>
