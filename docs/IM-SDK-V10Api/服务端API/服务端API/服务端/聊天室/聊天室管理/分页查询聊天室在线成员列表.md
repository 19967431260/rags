该接口用于分页查询指定聊天室中的在线成员列表。

聊天室中的成员包括固定成员（创建者、管理员以及普通成员）和非固定成员（临时成员，即游客）。

## 调用频率

单个应用默认最高调用频率请参考 [频控说明](https://doc.yunxin.163.com/messaging2/server-apis/DUzNjAzMTc?platform=server)。

## 请求信息

### 请求 URL

```
GET https://{endpoint}/im/v2/chatrooms/{room_id}/actions/list_online_members
```

:::note note
请求 URL 中的 `{endpoint}` 代表服务地址域名，您可以根据用户服务区域选择中国大陆和海外服务地址，并支持搭建高可用主备域名机制。详情请参考 [调用方式](https://doc.yunxin.163.com/messaging2/server-apis/zcwODA3MTU?platform=server#%E6%9C%8D%E5%8A%A1%E5%9C%B0%E5%9D%80) 服务地址章节。
:::

### 查询参数

参数名称 | 类型 | 是否必选 | 说明 | 默认值 |
---- | ---- | ---- | ---- | ----
`member_roles` | String | 否 | 需要查询的聊天室成员类型。0：普通用户；1：创建者；2：管理员；3：普通游客；4：匿名游客 | "0,1,2,3,4"，即查询全部在线成员 |
`offset` | Long | 是 | 查询的偏移量，首次查询传入 0，后续查询传入上一次查询结果中的 offset。 | -
`limit` | Integer | 是 | 每页返回的成员数上限，最大为 100。小于等于 0，或者大于 100，会报 414 错误。 | -
