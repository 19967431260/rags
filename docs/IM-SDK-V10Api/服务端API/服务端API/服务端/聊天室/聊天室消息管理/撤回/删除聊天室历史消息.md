该接口可以在聊天室撤回或删除一条消息。

## 功能描述

该接口通过 `notification_enabled` 是否下发通知来判定是撤回消息还是删除消息。

- `notification_enabled`=true，表示该操作为撤回聊天室消息，撤回后服务端会下发撤回消息的通知，且可以设置 `notification_extension` 通知扩展字段。
- `notification_enabled`=false，表示该操作为删除聊天室消息，删除后服务端不会下发相关通知。

## 调用频率

单个应用默认最高调用频率请参考 [频控说明](https://doc.yunxin.163.com/messaging2/server-apis/DUzNjAzMTc?platform=server)。

## 请求信息

### 请求 URL

```
DELETE https://{endpoint}/im/v2/chatrooms/{room_id}/messages/{message_client_id}
```

:::note note
请求 URL 中的 `{endpoint}` 代表服务地址域名，您可以根据用户服务区域选择中国大陆和海外服务地址，并支持搭建高可用主备域名机制。详情请参考 [调用方式](https://doc.yunxin.163.com/messaging2/server-apis/zcwODA3MTU?platform=server#%E6%9C%8D%E5%8A%A1%E5%9C%B0%E5%9D%80) 服务地址章节。
:::

### 请求头参数

### 路径参数

参数名称 | 类型 | 是否必选 | 描述
---- | ---- | ---- | ----
`room_id` | Long | 是 | 聊天室 ID。
`message_client_id` | String | 是 | 消息客户端 ID。

### 查询参数

参数名称 | 类型 | 是否必选 | 描述
---- | ---- | ---- | ----
`operator_id` | String | 是 | 操作者账号 ID。
`sender_id` | String | 是 | 发送聊天室消息的账号 ID。
`create_time` | Long | 是 | 消息发送时间戳。
`notification_enabled` | Boolean | 是 | 该操作是否需要发送通知。true：下发通知，对应撤回聊天室消息；false：不下发通知，对应删除聊天室消息。
`notification_extension` | String | 否 | 消通知扩展字段，JSON 格式，长度上限 1024 位字符，仅对消息撤回生效，即只有 `notification_enabled` =true 时，该字段才有效。例如："{\"k\":\"v\"}"。
