该接口可以将群组普通成员添加为群管理员，包括高级群和超大群。

::: note notice
- 只有群主才能添加群管理员。
- 可以批量添加群管理员，但单次操作最多不超过 10 人。
:::

## 请求信息

### 请求 URL

```
POST https://{endpoint}/im/v2/teams/{team_id}/actions/add_manager
```

:::note note
请求 URL 中的 `{endpoint}` 代表服务地址域名，您可以根据用户服务区域选择中国大陆和海外服务地址，并支持搭建高可用主备域名机制。详情请参考 [调用方式](https://doc.yunxin.163.com/messaging2/server-apis/zcwODA3MTU?platform=server#%E6%9C%8D%E5%8A%A1%E5%9C%B0%E5%9D%80) 服务地址章节。
:::

### 请求头参数

请求 Header 的参数说明请参考 [请求 Header](https://doc.yunxin.163.com/messaging2/server-apis/zcwODA3MTU?platform=server#请求-header)。

### 请求路径参数

参数名称 | 类型 | 是否必选 | 描述
---- | ---- | ---- | ----
`team_id` | Long | 是 | 群组 ID，创建群组成功时，网易云信服务器会自动生成并返回。

### 请求体参数

参数名称 | 类型 | 是否必选 | 描述
---- | ---- | ---- | ----
`team_type` | Integer | 是 | 群组类型。1：高级群。2：超大群。
`operator_id` | String | 否 | 操作者（群主）账号 ID。<!--超大群只能群主，高级群可以群主或管理员-->
`managers` | Array of strings | 是 | 需要添加为群管理员的群成员列表。单个元素为群成员的账号 ID，一次最多添加 10 个管理员。
`extension` | String | 否 | 自定义扩展字段，即自定义的通知字段，JSON 格式，不会持久化。长度上限 512 个字符。对应客户端 SDK 通知消息附件 `V2NIMMessageNotificationAttachment` 的 `serverExtension` 字段。<note type=notice>该字段仅针对高级群，对超大群无效。

### 请求体示例

```JSON
{
  "team_type": 1,
  "extension": "string",
  "operator_id": "account_id",
  "managers": [
    "manager1",
    "manager2"
  ]
}
```
