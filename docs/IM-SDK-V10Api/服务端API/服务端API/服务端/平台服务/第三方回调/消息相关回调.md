本文介绍会话消息回调、消息撤回回调以及消息更新回调的使用示例以及相关的参数说明。

::: note notice
本文示例代码中的 POST 接入点仅为示例，您在实际使用中需使用您在 [网易云信控制台](https://app.yunxin.163.com/global/home) 配置的第三方回调地址。
:::

## <span id='会话消息回调'>会话消息回调</span>

### 消息体字段说明

::: note notice
- 回调的 JSON 格式消息中并不是每个字段都一定会抄送，请注意对各字段的判空处理。
- 针对聊天室消息，`body` 为空，文本内容、图片内容等均承载在 `attach` 中。
:::

| 名称 | 类型 | 必须 | 说明 |
| ---- | ---- | ---- | ---- |
| `eventType` | Integer | 是 | 参考第三方回调事件类型，包括：点对点消息(1)、群消息(2)、聊天室消息(6)、超大群消息(22)，具体请参考 <a href="https://doc.yunxin.163.com/messaging/server-apis/jI3ODc2ODE?platform=server#第三方回调事件类型">第三方回调事件类型</a>。 |
| `fromAccount` | String | 是 | 消息发送者的用户账号。 |
| `fromNick` | String | 否 | 发送方昵称。 |
| `fromClientType` | String | 是 | 操作者的客户端类型，AOS(1)、IOS(2)、PC(4)、WEB(16)、REST(32)，MAC(64)、HARMONY(65)。 |
| `fromDeviceId` | String | 是 | 发送设备 ID。 |
| `to` | String | 是 | <div><ul><li>若 `eventType` 为 1，则 `to` 为消息接收者的用户账号，字符串类型。</li><li>若 `eventType` 为 `2`，则 `to` 为 `tid`，即群 ID，可转为 Long 型数据。</li><li>若 `eventType` 为 `6`，则 `to` 为 `roomid`，即聊天室 ID，可转为 Long 型数据。</li>若 `eventType` 为 `22`，则 `to` 为 `tid`，即超大群 ID，可转为 Long 型数据。</li></ul></div> |
| `msgTimestamp` | String | 是 | 消息发送时间。 |
| `msgType` | String | 是 | <div><ul> <li>`TEXT`：文本消息</li><li>`PICTURE`：图片消息</li><li>`AUDIO`：语音消息 </li><li>`VIDEO`：视频消息</li><li>`LOCATION`：地理位置</li><li>`FILE`：文件消息</li><li>`TIPS`：提示类型消息</li><li>`CUSTOM`：自定义消息</li> </ul></div> |
| `fromClientIp` | String | 否 | 消息发送方的客户端 IP 地址。 <note type=notice>ipv4 地址或 ipv6 地址。 |
| `fromClientPort` | String | 否 | 消息发送方的客户端端口号。 |
| `msgidClient` | String | 否 | 客户端消息 ID。 |
| `body` | String | 否 | 消息内容，部分消息类型不需要传入该参数，但需传入 `attach`，示例以及具体说明请参考 <a href="#回调的消息格式示例">回调的消息格式示例</a>。 |
| `attach` | String | 否 | 消息附件，部分消息类型不需要传入该参数，但需传入 `body`，示例以及具体说明请参考 <a href="#回调的消息格式示例">回调的消息格式示例</a>。 |
| `ext` | String | 否 | 消息扩展字段。 |

## <span id='消息撤回回调'>消息撤回回调</span>

### 消息体字段说明

::: note notice
回调的 JSON 格式消息中并不是每个字段都会一定抄送，请注意对各字段的判空处理。
:::

| <div style="width:160px">名称</div> | 类型 | 必须 | 说明 |
| ---- | ---- | ---- | ---- |
| `eventType` | Integer | 是 | 值为 `35`，表示是消息撤回回调。 |
| `fromAccount` | String | 是 | 操作者账号。 |
| `fromDeviceId` | String | 是 | 操作者设备 ID。 |
| `fromClientType` | String | 是 | 操作者的客户端类型：AOS(1)、IOS(2)、PC(4)、WEB(16)、REST(32)，MAC(64)、HARMONY(65)。|
| `fromClientIp` | String | 否 | 操作者的客户端 IP 地址。 <note type=notice>ipv4 地址或 ipv6 地址。|
| `fromClientPort` | String | 否 | 操作者的客户端端口号。 |
| `msgFromAccid` | String | 是 | 被撤回消息的消息发送者。 |
| `msgId` | Long | 是 | 被撤回消息的消息 ID（服务器 ID）。 |
| `time` | Long | 是 | 被撤回消息的消息发送时间。 |
| `opeType` | int | 是 | <li>值为 `7`，表示单聊消息撤回。<li>值为 `8`，表示群消息撤回。 |
| `toAccount` | String | 是 | 消息接收者的账号 ID，或者群组 ID。 |
| `msgidClient` | String | 否 | 被撤回消息的消息 ID（客户端 ID）。 |
| `msg` | String | 否 | 消息撤回附言。 |
| `attach` | String | 否 | 消息撤回扩展字段。 |
| `timestamp` | String | 是 | 操作时间，字符串类型 。|

## <span id='消息更新回调'>消息更新回调</span>

### 消息体字段说明

::: note notice
回调的 JSON 格式消息中并不是每个字段都会一定抄送，请注意对各字段的判空处理。
:::

| <div style="width:160px">名称</div> | 类型 | 必须 | 说明 |
| ---- | ---- | ---- | ---- |
| `eventType` | Integer | 是 | <li>值为 `82`，表示是单聊消息更新回调。<li>值为 `83`，表示是高级群消息更新回调。<li>值为 `81`，表示是超大群消息更新回调。 |
| `fromAccount` | String | 是 | 消息发送者账号。 |
| `fromDeviceId` | String | 否 | 消息发送者设备 ID。 |
| `fromClientType` | String | 否 | 消息发送者客户端类型。 |
| `fromClientIp` | String | 否 | 消息发送者客户端 IP。 |
| `fromClientPort` | String | 否 | 消息发送者客户端端口。 |
| `msgId` | Long | 是 | 消息服务器 ID。 |
| `time` | Long | 是 | 消息发送时间。 |
| `opeType` | Integer | 是 | <li>值为 `19`，表示单聊消息更新。<li>值为 `20`，表示高级群消息更新。<li>值为 `21`，表示超大群消息更新。 |
| `toAccount` | String | 是 | 消息接收者账号 ID，或群组 ID。 |
| `msgidClient` | String | 是 | 消息客户端 ID。 |
| `modifyAccountId` | String | 是 | 操作者账号 ID。|
| `subType` | Integer | 否 | 消息子类型。用于对消息进行二级分类，取值范围为大于 0 的正整数。推荐自定义消息类型使用此字段区分不同业务场景，其他消息类型可根据实际需求选择性使用。 |
| `body` | String | 否 | 消息内容，部分消息类型不需要传入该参数，但需传入 `attach`，示例以及具体说明请参考 <a href="#回调的消息格式示例">回调的消息格式示例</a>。 |
| `attach` | String | 否 | 消息附件，部分消息类型不需要传入该参数，但需传入 `body`，示例以及具体说明请参考 <a href="#回调的消息格式示例">回调的消息格式示例</a>。 |
| `ext` | String | 否 | 消息扩展字段。 |
