## <span id='音视频/白板时长消息抄送'>音视频/白板时长消息抄送</span> 

```
5. "eventType"="5" 或 "21"。
```

### 抄送情景说明
抄送时长场景说明：<br>
``` http
1. 主被叫接通，正常结束（通话时长大于0）；
2. 主被叫接通，接通的同时立刻挂断（双方都成功进入频道），抄送时长及主被叫账号。备注：立即挂断时，时长也可能不为0；
3. 主被叫接通后出现异常（如：接通时一方网络出现问题），只有一个人成功进入频道（单人进入，抄送时长为0，status是SINGLE_PARTICIPATE），则只抄送成功进入频道的帐号的时长信息；
4. 特别的，对于时长超长通话（通话时长超过12小时）会按照12小时的间隔将时长信息抄送给客户服务器，少于12小时的音视频会话/互动白板会话均不受影响。
```

## <span id='音视频/白板文件下载信息抄送'>音视频/白板文件下载信息抄送</span> 


```
6. "eventType"="6" 或 "22"。
```

说明：
```
从2018年7月10日（v5.4.0版本）开始，网易云信对音视频录制服务进行录制架构优化，对于超过2小时的音视频通话录制文件，会以2小时（若有需要，可向商务经理发起更改申请）为间隔生成文件并进行抄送，需要开发者在处理抄送时了解上述逻辑并做兼容。
```

### <span id='消息体中的JSON字段说明：'>消息体中的JSON字段说明：</span> 

**注意：音视频/白板文件下载信息抄送，其中fileinfo字段为JSON数组，最多可包含10个数组元素(即10条文件信息/10个channelid)，请开发者注意解析区分。**

<table><thead><tr><th>名称</th><th>类型</th><th>说明</th></tr></thead>
<tr><td> eventType </td><td>String</td><td>值为6，表示是音视频/白板文件下载信息类型的消息
</td></tr>
<tr><td> fileinfo </td><td>String</td><td>
可转为JSONArray，其中的字段释义如下：<br/>
caller：是否是此通通话的发起者，若是则为true，若不是则没有此字段，可转为Boolean值<br/>
channelid：通道号，可转为Long值<br/>
filename：文件名，直接存储，混合录制文件filename带有"-mix"标记<br/>
md5：文件的md5值<br/>
size：文件大小，单位为字符，可转为Long值<br/>
type：文件的类型（扩展名），包括：实时音频录制文件(aac)、白板录制文件(gz)、实时视频录制文件(mp4)、互动直播视频录制文件(flv)<br/>
url：文件的下载地址<br/>
user：用户帐号，若该文件为混合录制文件，则该字段为"0"<br/>
mix：是否为混合录制文件，true：混合录制文件；false：单人录制文件<br/>
vid：点播文件id，注意白板录制文件(gz)无此字段。通过该参数可以调用点播接口查询相关信息。<br/>
pieceindex：录制文件的切片索引，如果单通通话录制时长超过切片时长，则录制文件会被且被切割成多个文件<br/>
timestamp：文件生成的系统时间
</td></tr>
</table>
