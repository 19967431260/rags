**兼容版本**

- 适配 NERtc SDK 5.6.41 版本。

## 2.2.0 (2023-12-22)

**新增特性**

增加 `enableMarkInviteEventDelay` 参数，是否开启邀请消息延迟已读。默认关闭。

不是在收到呼叫时就标记已读，而是在如接听、挂断、超时、取消呼叫等操作后标记已读。在呼叫的有效时间内，如果一通呼叫没有接听，刷新后会再次收到邀请；或者在有效时间内 web 端没有接听的通话在移动端可上线接听。

**兼容版本**

- 适配 NIM SDK V9.8.0 版本。
- 适配 NERTC SDK V5.5.10 版本。

## 2.0.0 (2023-05-29)

**改进优化**

Android、iOS、Web 各端接口和字段对齐，更新后的 API 接口请参见 [呼叫组件 V2.0 API 参考](https://doc.yunxin.163.com/nertccallkit/references/web/typedoc/Latest/zh/index.html)。

::: note note
- 集成了呼叫组件 V2.0 版本的用户能和呼叫组件 V1.x 的用户进行通话。
- 请确保代码中全使用 V2.0 版本的接口或全使用 V1.x 版本的接口， V2.0 版本的接口和 V1.x 版本的接口不要混用。
:::

**兼容版本**

- 适配 NERTC SDK V4.6.29 版本。

## 1.8.0 (2022-11-01)

**新增特性**

- 单击本地预览小窗，切换画布。
- 支持设置主叫在呼叫时是否加入 RTC 房间。

**API 更新**

初始化接口新增 `enableJoinRTCChannelWhenCall` 参数，用于配置主叫在呼叫时是否加入 RTC 房间。默认为 NO，在被叫接听时加入RTC 房间，不提前加入。如设置提前加入房间，能缩短首帧时间，但是会带来通话费用的增加。

**兼容版本**

- 适配 NIM SDK V9.6.1 以上版本。
- 适配 NERTC SDK V4.6.203 版本（组件内置）。

## 1.6.1 (2022-08-22)

**新增特性**

支持音视频通话切换确认。

**新增 API**

| 接口名称           | 接口说明                                                                         |
|:--------------------|:--------------------------------------------------------------------------------------|
| `setEnableSwitchCallTypeConfirm` | 音视频切换确认，默认不开启确认。 |
| `onSwtichCallType` | 音视频切换回调，新增回调中有音视频切换指令类型，如果未开启音视频切换确认配置，可忽略第二个字段。 |

**API 变更**

| 接口名称           | 接口说明                                                                         |
|:--------------------|:--------------------------------------------------------------------------------------|
| `switchCallType` |增加 `state` 参数，用于获取音视频切换确认的状态。 |

**兼容版本**

- 适配 NIM SDK V9.2.5 版本。
- 适配 NERTC SDK V4.6.12 版本。

**升级说明**

V1.5.7 版本可直接升级至 V1.6.1 版本。

## 1.5.7 (2022-07-19)

** 新增特性**

- 支持在初始化时关联呼叫组件 `accId` 对应的 `rtcUid`，以便消息抄送时能获取到对应的用户。
- 支持呼叫时绑定该呼叫对应的 RTC 房间名称（`channelName`）。
- 支持呼叫或接听时直接设置 `rtcToken`。

**API 变更**

| 接口名称           | 接口说明                                                                         |
|:--------------------|:--------------------------------------------------------------------------------------|
|`call`| `call` 接口中增加 `rtcChannelName` 参数，用于绑定 RTC 房间的名称。 |

**升级说明**

V1.5.5 版本可直接升级至 V1.5.7 版本。

**兼容版本**

- 适配 NIM SDK V9.2.5 版本。
- 适配 NERTC SDK V4.6.12 版本。
