自 v4.3.0 起，网易云信音视频呼叫组件支持来电横幅功能。当收到来电时，在屏幕顶部展示轻量横幅，支持一键接听或拒绝，不打断用户当前操作。

## 概述

音视频呼叫组件（CallKit）默认关闭来电横幅功能（与原有全屏来电界面行为一致）。若开启该功能，来电时展示顶部横幅，原全屏来电页不再出现。

- 该功能支持在运行时随时切换，并在 **下一次来电** 时生效（对当前已展示的横幅不生效）。
- 若在横幅展示期间有第二路来电，SDK 自动回复忙线。
- 横幅展示期间正常播放被叫铃声；点击横幅主体进入全屏来电页时，铃声 **不中断**。

## 权限配置

Android 端的横幅使用系统悬浮窗（`SYSTEM_ALERT_WINDOW`）实现，需在 **Android 6.0+** 设备上申请权限。

::: note note
若 Android 端无悬浮窗权限时，SDK 会自动跳转系统设置页（`ACTION_MANAGE_OVERLAY_PERMISSION`）引导用户授权，并通过 callback 返回错误码提示业务层。
:::

```kotlin
// 检查权限
if (!Settings.canDrawOverlays(context)) {
    val intent = Intent(
        Settings.ACTION_MANAGE_OVERLAY_PERMISSION,
        Uri.parse("package:${packageName}")
    )
    startActivity(intent)
}
```

## 启用来电横幅功能

云信音视频组件在收到来电时，会在屏幕顶部展示轻量横幅，支持一键接听或拒绝，不打断用户当前操作。

如果需要启用该功能，可以使用 `enableIncomingBanner` 方法，在 NERtcCallKit 组件初始化后开启该功能：

```kotlin
// 开启
CallKitUI.enableIncomingBanner(true)

// 关闭（恢复全屏来电界面）
CallKitUI.enableIncomingBanner(false)
```

::: note note
`enableIncomingBanner` 是 **运行时方法**，不得通过 `CallKitUIOptions.Builder` 传入，Builder 中的同名属性为内部参数，不作为公开 API。
:::

## 常见问题

**Q：横幅开启后，原有全屏来电界面还会出现吗？**

不会。开启横幅模式后，全屏来电界面被完全替换为横幅展示。若需恢复，调用 `enableIncomingBanner(false)` 即可。

**Q：横幅展示期间收到第二路来电怎么处理？**

SDK 会自动向第二路来电方回复忙线，行为与通话中收到新来电一致。

**Q：Android 没有悬浮窗权限时横幅还会显示吗？**

不会。无悬浮窗权限时，SDK 会降级展示系统通知（Notification）来提示来电，并引导用户前往设置页授权。授权后下次来电即可正常展示横幅。

**Q：点击横幅主体进入全屏来电页，铃声会重新播放吗？**

不会。铃声会保持连续播放，进入全屏来电页时不会中断也不会重新触发。铃声统一在通话结束（接听/拒绝/超时/取消）时停止。
