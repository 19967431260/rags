本文介绍在呼叫组件中如何自定义呼叫铃声。

## 功能介绍

呼叫组件的铃声分为 **呼叫铃声** 和 **被叫铃声**。

- 如果您通过源码依赖 NECallKit 组件，您可以替换铃声资源文件（呼叫铃声、被叫铃声）自定义铃声。
- 如果您通过非源码依赖 NECallKit 组件，目前支持自定义被叫铃声，呼叫铃声暂时不支持自定义。

## 实现方法

### 源码依赖

如果您通过源码依赖 NECallKit 组件，那么您可以直接替换源码中的铃声资源文件（呼叫铃声、被叫铃声）进行自定义铃声。

目前铃声资源文件的路径如下：

[呼叫铃声](https://github.com/netease-kit/NECallKit/blob/main/HarmonyOS/callkit_ui/src/main/resources/rawfile/avchat_connecting.mp3)：**NECallKit/HarmonyOS/callkit_ui/src/main/resources/rawfile/avchat_connecting.mp3**

[被叫铃声](https://github.com/netease-kit/NECallKit/blob/main/HarmonyOS/callkit_ui/src/main/resources/rawfile/avchat_ring.mp3)：**NECallKit/HarmonyOS/callkit_ui/src/main/resources/rawfile/avchat_ring.mp3**

### 非源码依赖

如果您通过非源码依赖 NECallKit 组件，请参考以下步骤自定义被叫铃声。

- 方式一：通过使用文件路径自定义

```typescript
import { NECallUI } from '@necallkit/callkit_ui';

// 设置自定义铃声（使用文件路径）
const ringtonePath = 'your custom ring path';
await NECallUI.getInstance().setCallingBell(ringtonePath);
```

- 方式二：通过使用使用资源文件自定义

```typescript
import { NECallUI } from '@necallkit/callkit_ui';
import { media } from '@kit.MediaKit';
import common from '@ohos.app.ability.common';

// 从资源文件获取文件描述符
const context = getContext(this) as common.UIAbilityContext;
const resourceMgr = context.resourceManager;

// 假设铃声文件位于 rawfile 目录下，文件名为 custom_ringtone.mp3
const fileDescriptor = await resourceMgr.getRawFd('custom_ringtone.mp3');
const avFileDescriptor: media.AVFileDescriptor = {
  fd: fileDescriptor.fd,
  offset: fileDescriptor.offset,
  length: fileDescriptor.length
};

// 设置自定义铃声
await NECallUI.getInstance().setCallingBell(avFileDescriptor);
```
