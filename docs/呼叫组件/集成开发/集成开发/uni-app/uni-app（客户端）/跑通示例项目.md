在视频呼叫示例项目中使用了呼叫组件实现音视频呼叫。本文介绍如何快速跑通音视频呼叫的示例项目，体验音视频呼叫功能。

## 开发环境

示例项目对开发环境的要求如下所示：

- HBuilderX ≥ 3.94
- sass（sass-loader 版本 <= 10.1.1）
- node（16.0.0 <= node 版本 <= 17.0.0, 推荐使用 Node.js 官方 LTS 版本 16.17.0）
- npm（版本请与 node 版本匹配）

## 前提条件

在开始运行示例项目之前，请确保您已完成以下操作：

- 已 [创建应用并获取应用的 App Key](https://doc.yunxin.163.com/console/concept/TIzMDE4NTA?platform=console)。
- 已获取本插件使用中需要的 account 和 token，具体请参考 [注册 IM 账号](https://doc.yunxin.163.com/messaging2/guide/jU0Mzg0MTU?platform=client#第二步注册-im-账号)。
- 使用音视频呼叫（呼叫组件）之前，需要开通以下功能。如何开通请参考 [开通服务](https://doc.yunxin.163.com/console/concept/zc3NDYzNzc?platform=console)。
    - IM 即时通讯。当使用呼叫组件自带的话单功能时，需开通 IM。
    - 信令。用于实现点对点呼叫邀请以及音视频通话。
    - 音视频通话 2.0。用于实现实时音视频通话。

        ::: note note
        建议开通音视频通话的 **调试模式** （[鉴权方式](https://doc.yunxin.163.com/nertc/server-apis/TcxNDAxMTI?platform=server)），调试模式建议只在集成开发阶段使用，请在应用正式上线前改回安全模式。
        :::

    - 如需要抄送，请提前开通消息抄送中的 **话单** 抄送服务，实现在一通通话结束后，发送事件通知消息，标记此次通话是否接通以及通话时间、类型等数据。

::: note notice
如果您同时接入了 IM 即时通讯和呼叫组件，那么需要提前在网易云信控制台开启 **多端登录**，具体请参考 [多端登录与互踢](https://doc.yunxin.163.com/messaging2/guide/DUyNjI2MjQ?platform=client)。
:::

- 示例代码中包含业务登录逻辑，如果您的业务中需要使用登录相关功能，请修改登录相关的逻辑。

### 步骤一：下载源码&配置工程

1. 前往 GitHub 克隆 NECallKit 示例项目源码仓库至您本地。

    ```shell
    git clone git@github.com:netease-kit/NECallKit.git
    ```

2. 安装依赖。

    ```shell
    cd NECallKit/uni-app/NECallKit-app
    npm install
    ```

3. 在 `debug/AppConfig.js` 文件中配置您的 appKey。

    ```javascript
    // 应用Key - - 请替换为你的实际appKey
    appKey: "your_app_key",
    ```

### 步骤二：导入插件

1. 购买 uni-app 原生插件。

    登录 uni 原生插件市场，并访问 [netease-CallKit 插件](https://ext.dcloud.net.cn/plugin?id=14739)，在插件详情页中购买（免费插件也可以在插件市场 0 元购）。购买后才能够云端打包使用插件。购买插件时请选择正确的 appid，以及绑定正确包名。

    <img width='300' src='https://yx-web-nosdn.netease.im/common/97a20c588c3760905b68f9996696503a/unipp原生插件.png' />

2. 制作自定义调试基座 （请使用真机运行自定义基座）。

    购买插件后在项目的 manifest.json 页面的 App 原生插件配置项下单击选择云端插件，选择 netease-CallKit 插件。

    <img width='700' src='https://yx-web-nosdn.netease.im/common/b58e3af8857bfa406396c3a8a9ff5a32/dcloud_uniapp_install_cloud_plugins3.png' />

::: note note
- 自定义基座不是正式版，真正发布时，需要再打正式包。使用自定义基座无法正常升级替换 APK。
- 请尽量不要使用本地插件，插件包超过自定义基座的限制，可能导致调试收费。
:::

### 步骤三：运行项目

1. 制作自定义调试基座，请选择 **传统打包** 方式进行打包。

2. 自定义调试基座成功后，使用自定义基座运行项目。

## 实现音视频通话

1. 分别在两台设备上登录两个用户，一方作为主叫，一方作为被叫。

2. 在主叫方进入通话界面，输入被叫方的 accoudId，发起通话。

3. 被叫方收到通话的请求，点击接听后进行通话。

## 常见问题

### iOS 需要配置麦克风和摄像头权限

需要在项目根目录的 manifest.json 中的 **App 权限配置** 中配置麦克风和摄像头权限。

### 同时集成呼叫组件和推送插件后无法正常呼叫

需要在控制台开通 IM 的多端登录。

### iOS 上架问题

当出现以下报错时，需要在项目根目录的 manifest.json 中设置 `embedSwiftStandardLibraries` 为 true。 

```
Invalid Swift Support. The SwiftSupport folder is empty. Rebuild your app using the current public (GM) version of Xcode and resubmit it.
```
