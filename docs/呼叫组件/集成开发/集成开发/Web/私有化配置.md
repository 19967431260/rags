本文介绍呼叫组件的私有化配置。

## 注意事项

- 呼叫组件集成 IM SDK 和 NERTC SDK 功能，因此呼叫组件的私有化配置需要配置 IM SDK 的私有化配置以及 NERTC SDK 的私有化配置。
- 只有 V1.4.2 及以上的呼叫组件版本才支持私有化配置。
- 私有化场景中，呼叫组件自带的话单功能将不再生效，请 [自行实现话单功能](https://doc.yunxin.163.com/docs/zMwMzkwNTE/DQ5Njk3NTQ?platformId=121990#实现自己的话单)。

## RTC 私有化配置

初始化呼叫组件时，在创建实例接口传入 `rtcConfig` 对象，`neRtcServerAddresses` 对象的参数说明请参考 <a href="https://doc.yunxin.163.com/docs/interface/NERTC_SDK/Latest/Web/api/interfaces/types.joinoptions.html#nertcserveraddresses" target="_blank">neRtcServerAddresses</a>。

```JavaScript
new NECall({
    ...,// 这里标识其他参数
    rtcConfig: {
        ...,// 这里标识其他参数
        neRtcServerAddresses: // 私有化配置
    }

})

```

## IM 私有化配置

初始化 IM 时，在 `login` 接口传入 `privateConf` 对象，`privateConf` 对象的参数说明请参考 <a href="https://doc.yunxin.163.com/docs/TM5MzM5Njk/zE0NDY4Njc?platformId=60179" target="_blank">初始化 IM 中的 `privateConf`</a>。
