本文介绍呼叫组件 Web 端从V1.8.2 版本升级至 V2.0 版本的说明和注意事项。

## V2.0.0
从 V1.8.2 版本升级至 V2.0.0 版本，涉及如下接口变更，您需要结合实际业务场景更新相关的 App 代码：


1. 初始化方式更改为单例模式。

    ```
    // v1.8.2
    import { NECall } from '@xkit-yx/call-kit'
    const neCall = new NECall(初始化参数)

    // v2.0.0
    import { NECall } from '@xkit-yx/call-kit'
    const neCall = NECall.getInstance()
    neCall.setup(初始化参数) // 请参考2.0.0接口文档
    ```

2. 配置通话接听超时时间 [`setTimeout`](https://doc.yunxin.163.com/nertccallkit/api-docs/callkit_Web_zh/CallKit_Web_API_Reference/classes/NECall#settimeout) 的接口参数变更，调用方法调整为：

    ```
    // v2.0.0
    import { NECall } from '@xkit-yx/call-kit'
    const neCall = NECall.getInstance()
    neCall.setTimeout(30) 
    ```
    接口参数由 `callTimeout` 和 `rejectTimeout` 变更为 `time`。
3. 回调名称修改。


    序号 | 变更描述 | 集成修改建议
    ---- | -------------- | ---------
    1 | 呼叫方式已变更的回调由 `onSwtichCallType` 变更为 `onCallTypeChange` | 请修改代码中对应的回调名称。
    2 | 话单发送的通知由 `onMessageSent` 变更为 `onRecordSend` |^^|
