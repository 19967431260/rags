网易云信 IM UIKit 内置了多种基本消息类型，包括文本、图片、语音、视频、文件等，可满足大部分即时通讯场景。当这些基础消息类型无法满足您的业务需求时，您可以使用自定义消息来扩展功能，例如实现电商订单卡片、红包、系统通知等。

## 发送自定义消息

实现自定义消息收发之前，您需先自行实现消息数据的封装和相关解析规则。IM UIKit 可按照您实现的封装和解析规则对消息进行处理，将消息返回给您的应用。

在 uni-app 中，您可以在消息的 `ext` 字段中封装自定义数据。

### 实现自定义消息类

在 uni-app 中，您无需创建单独的类。自定义消息的业务数据可以直接封装为一个 JSON 对象，并赋值给消息体的 `ext` 字段。

```
// 假设这是您的自定义消息数据结构
const customData = {
  type: 1002, // 自定义消息类型，建议从1000开始
  data: {
    title: '这是一条自定义消息',
    content: '用户点击了商品卡片'
  }
};
```

### 增加发送按钮

在 **src/pages/Chat/message/message-input.vue** 的 template 的 **发送更多面板** 中添加一个新的 Icon 组件并绑定 tap 事件。

```
// template
// 参考已有的 Icon 组件，复制修改一下绑定的事件函数 handleSendCustomMsg
<div class="msg-input-button">
    <Icon @tap="sendItemCard" type="icon-shezhi" :size="20" />
</div>
```

### 创建并发送自定义消息

您可以调用 SDK 的 `createCustomMessage` 方法创建自定义消息，并将上一步定义的自定义数据字符串化后作为 `rawAttachment` 参数传入，在 **src/pages/Chat/message/message-input.vue** 的 script 加入对应逻辑。

```
//script
//发送商品卡片
const sendItemCard = () => {
  // 假设会话ID和类型已定义
      const store = uni.$UIKitStore
  
  // 自定义消息数据
  const customData = {
    isCustomMessage: true,
    type: 1002,
    data: {
      title: '这是一条商品卡片',
      url: 'https://example.com/product/123'
    }
  };
  
  // 将自定义数据转为JSON字符串
  const customStr = JSON.stringify(customData);
  
  // 创建自定义消息
  const message = uni.$UIKitNIM.V2NIMMessageCreator.createCustomMessage(
        'this is a custom message' + Date.now(),
        customStr,
      )
  
  // 调用 uikit 封装的 store 方法发送消息
  uni.$UIKitStore.msgStore.sendMessageActive({
    msg: message as unknown as V2NIMMessage,
    conversationId,
    sendBefore: () => {
      scrollBottom()
    }
  }).then(()=>{
      scrollBottom()
  })
};
```

## 自定义消息 UI 展示

若需要在会话界面将自定义消息按照您自定义的 UI 样式进行展示，您需要在 **message-item.vue** 文件中添加相应的渲染逻辑。

### 解析并获取消息中的自定义字段

在 uni-app 中，自定义消息的 UI 渲染通过在 Vue 组件中添加条件渲染来实现。您可以在 IM UIKit 的消息组件中添加自定义模板。

```
<!-- 在 /NEUIKit/pages/Chat/message/message-item.vue 的 script 下 -->

const props = withDefaults(...)
  //在props定义后添加
  // 解析消息体上的 props.attachment 上的自定义字段
 const raw = props?.msg?.attachment?.raw || '{}';
   // 这里根据发送时的标记位进行判断是否为自定义消息
 const isCustomMessage = JSON.parse(raw).isCustomMessage;
```

### 增加渲染逻辑

在 uni-app 中，自定义 UI 渲染的注册过程简化为在消息渲染组件中直接编写逻辑。您只需在 **message-item.vue** 中添加判断条件和对应的 UI 模板即可。

```
<!-- 在 /NEUIKit/pages/Chat/message/message-item.vue 的 template 下 -->
 
<template>
  <div :id="MSG_ID_FLAG + props.msg.messageClientId"
    :key="props.msg.createTime">
    <!-- 其他消息类型的渲染逻辑... -->
    
    <!-- 自定义消息的渲染逻辑 -->
    <!-- 使用 v-if 判断是否为自定义消息 -->
    <div class="msg-common" v-else-if="props.msg.messageType === V2NIMConst.V2NIMMessageType.V2NIM_MESSAGE_TYPE_CUSTOM " :style="{ flexDirection: !props.msg.isSelf ? 'row' : 'row-reverse' }"
            >
        <!-- 自行实现任意的自定义消息 UI 展示 -->
        <div class="custom-card">
        <text class="title">{{itemTitle}}</text>
        <text class="content">{{ itemUrl}}</text>
        <button @click="openLink(raw.custom.data.url)">查看详情</button>
    </div>
    </div>
   </div>
</template>
```
