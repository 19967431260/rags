## 运行鸿蒙应用时配置报错

当运行 **鸿蒙应用** 出现以下报错信息时，需要在 **harmony-configs/build-profile.json5** 文件中，修改配置，将 **default** 改为 **relese**。

```
hvigor ERROR: Invalid product for target 'default'.
Detail: Check the target applyToProducts field for 'default': [ 'default', 'release' ].
```

## 鸿蒙网络权限问题

IM 实例需要网络权限，需要在 **harmony-configs/entry/src/main/module.json5** 中配置网络权限（发送图片和视频也需要权限）。具体请参考 [权限配置指南](https://uniapp.dcloud.net.cn/tutorial/harmony/runbuild.html#permission)。

```
requestPermissions: [
      {
        name: 'ohos.permission.INTERNET',
      },
      { name: 'ohos.permission.GET_NETWORK_INFO' },
      {
        name: 'ohos.permission.MICROPHONE',
        reason: '$string:chat_permission_micro_phone_desc',
        usedScene: {
          abilities: ['FormAbility'],
          when: 'always',
        },
      },
      {
        name: 'ohos.permission.CAMERA',
        reason: '$string:chat_permission_camera_desc',
        usedScene: {
          abilities: ['FormAbility'],
          when: 'always',
        },
      },
    ],
```

## 鸿蒙目录问题

默认情况下，HBuilderX 会在项目内的 **unpackage** 目录下创建鸿蒙工程目录，用于构建鸿蒙的运行包和发行包。

- 调试运行时默认使用的鸿蒙工程目录：**unpackage/dist/dev/app-harmony**
- 发行打包时默认使用的鸿蒙工程目录：**unpackage/dist/build/app-harmony**

鸿蒙工具链对于鸿蒙工程目录的路径有一定的限制条件：不能包含中文字符和某些特殊字符。即允许在路径中使用的字符仅限于大小写英文字母、数字、正斜杠、反斜杠、下划线、连字符、冒号、句点。（如果安装了较新版本的 DevEco Studio，则还允许包含空格。）

更多详细信息请参考 [项目根目录与鸿蒙工程目录](https://uniapp.dcloud.net.cn/tutorial/harmony/runbuild.html#project-path)。

## 无法调用数字人 API

**问题描述**

调用数字人相关 API 后出现以下报错：

![ai数字人.png](https://yx-web-nosdn.netease.im/common/8155f71ffb71c88fef58b9c58ec258e0/ai数字人.png)

**问题原因**

未在云信控制台开通并配置数字人。

**解决方案**

- 若不需要使用数字人功能，在初始化 RootStore 时，将 `aiVisible` 设置为 false，关闭数字人功能。

    ```
    const store = (uni.$UIKitStore = new RootStore(
            nim,
            {
            /**
          是否开启数字人功能，默认 true
        */
        aiVisible: false;
              })
    ```
    
- 若需要使用数字人功能，请提前在云信控制台 [开通并添加 AI 数字人](https://doc.yunxin.163.com/console/concept/TYxMDEzNDg?platform=console) 后再调用数字人相关 API。

## 无法正常打开相册

相册拒绝授权后再次开启授权，由于 uni-app API 兼容问题，部分 Android 机型无法正常打开相册，需要您自行处理。

## iOS 低版本构建后无法运行

如果 iOS 低版本构建后无法运行，需在构建出的资源 `dist/build/app/app-service.js` 文件最前面添加 `var globalThis = Function('return this')();`。

部分 uni-app 兼容问题，已在代码中说明。

## page 结构问题

由于 uni-app 自身的 uni.setNavigationBarTitle 限制，chat 页面的 NavigationBarTitle，需要您自行调整 page 结构设置。

## uni-app 推送问题

uni-app 推送相关问题请参考 [uni-app 离线推送](https://doc.yunxin.163.com/messaging/guide/DQ1NTAxNzI?platform=uniapp)。

## 业务登录问题

您业务侧的登录需要结合网易云信 IM 的登录实现。

在您的登录页完成登录后，将您的业务服务器返回的 account、token 传入 app.vue 里的初始化登录 IM 的方法，待 IM 初始化、连接完成后，再重定向到您需要的界面。

## 无法切换深色模式

目前 Demo 不支持深色模式。如需要深色模式，需要您在业务侧自行适配。

## uni_modules 问题

NEUIKit 中使用到的 uni-app 提供的部分组件（例如 uni-popup），如果使用上产生冲突，您需要将 NEUIKit 的 uni-modules 与您使用的 uni-modules 进行合并，然后修改 NEUIkit 中引用的路径即可。

## 视频/语音模块离线打包问题

参考 uni-app 官网的 [离线打包](https://nativesupport.dcloud.net.cn/AppDocs/usesdk/android.html)。

## H5 断网重连问题

若遇到 H5 断网重连失败的问题，请在 `manifest.json` 中，修改配置，示例如下：

```JSON
"optimization" : {
  "treeShaking" : {
    "enable" : false
  }
}
```
