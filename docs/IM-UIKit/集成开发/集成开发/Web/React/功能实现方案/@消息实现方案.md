## 方案介绍

@消息通过消息的扩展参数，保留@消息的相关内容。

在消息体 `V2NIMMessage` 中可以通过 [`serverExtension`](https://doc.yunxin.163.com/messaging2/client-apis/DAxNjk0Mzc?platform=client#v2nimmessage) 来获取和设置消息体中的远程传输的扩展参数，@消息的内容就以 Object 形式保存在该消息体中，数据格式如下：

```json
//@消息的 key 值
"yxAitMsg": {
    //被@的账号，如果是 @All 则为 ait_all
    "332917623668992": {
        "text": "@昵称01 ",//在消息中@的展示内容
        "segments": [{//在消息中@的展示位置
            "start": 0,//在消息中@的展示起始位置
            "end": 5,//在消息中@的展示终止位置
        }]
    }
}
```

在发送一条@消息时，会将上述@消息的内容，设置到该消息的 `serverExtension` 中。

## 实现流程

1. 创建一条文本消息。

    ```TypeScript
    let textMsg = nim.V2NIMMessageCreator.createTextMessage(value)
    ```

2. 在 UI 上进行操作时，将@消息的相关内容按照上述 Object 形式传入 `serverExtension` 中，并在发送文本消息时，将 `serverExtension` 挂载到 `textMsg` 中，然后调用 [`nim.V2NIMMessageService.sendMessage`](https://doc.yunxin.163.com/messaging2/client-apis/zIwODM2NTM?platform=client#sendmessage) 方法。

    ```TypeScript
    textMsg.serverExtension = JSON.stringify(serverExtension)
    ```

3. 发送消息。

    ::: note note
    其中 `conversationId` 为会话 ID，可通过调用 [`V2NIMConversationIdUtil`](https://doc.yunxin.163.com/messaging2/client-apis/DAxNjk0Mzc?platform=client#V2NIMConversationIdUtil) 的对应函数创建。组成方式：用户账号（accountId）| 会话类型（V2NIMConversationType）| 聊天对象账号（accountId）或群组 ID。
    :::

    ```TypeScript
    this.nim.V2NIMMessageService.sendMessage(
            textMsg,
            conversationId,
            ...
        )
    ```
4. 接收方对接收到的消息进行解析。

    根据消息体的 `serverExtension` 来判断是否包含@消息内容，并结合消息内容与 `yxAitMsg` 中的 `text` 进行正则匹配后高亮展示。

    ```TypeScript
    if (serverExtension) {
            try {
            const extObj: YxServerExt = JSON.parse(serverExtension)
            const yxAitMsg = extObj.yxAitMsg

            if (yxAitMsg) {
                Object.keys(yxAitMsg).forEach((key) => {
                const item = yxAitMsg[key]
                // 正则匹配IMUIkit 使用的是reactStringReplace，您可以根据您的实际相关选择合适的工具函数
                finalText = reactStringReplace(
                    finalText,
                    item.text,
                    (match, i) => {
                    return (
                        <span
                        key={messageClientId + match + i}
                        className={`${_prefix}-mention`}
                        >
                        {match}
                        </span>
                    )
                    }
                )
                })
            }
            } catch {
            //
            }
        }
    ```

5. 在会话列表上展示 **有人@我**。

    在页面启动时，对接收消息进行监听，根据是否有@消息进行处理：

    - 如果有@消息，通知会话列表进行刷新。
    - 当进入到会话，清空该会话中所有@相关记录，并通知刷新会话列表。

    本文以在 `conversation` 上挂载 `aitMsg`（类型为 string[]，内容为@消息的 `idClient`）为例，提供以下示例代码：

    ```TypeScript
    handleConversationWithAit(msgs: V2NIMMessage[]): void {
        msgs.map(async (item) => {
        if (
            item.conversationType ===
            V2NIMConst.V2NIMConversationType.V2NIM_CONVERSATION_TYPE_TEAM 
        ) {
            if (item?.serverExtension) {
            const hasAitMsg = this.hasAitMsg([item.serverExtension])
            // 获取会话的已读时间（getConversationReadTimeActive为IMUIkit封装后的api，sdk接入请调sdk方法）
            const conversationReadTime = await this.getConversationReadTimeActive(
                item.conversationId
            )
            const isCurConversation =
                item.conversationId == this.rootStore.uiStore.selectedConversation

            const isSendFromMe =
                item.senderId === this.rootStore.userStore.myUserInfo?.accountId

            // 当前通过_onReceiveMessages获取的新消息时间戳大于会话的最后已读时间戳，且有@消息，且没有选中当前该会话，则标记为已读
            if (
                item.createTime > conversationReadTime &&
                hasAitMsg &&
                !isCurConversation &&
                !isSendFromMe
            ) {
                const conversation = this.conversations.get(item.conversationId)

                // 如果此时conversation为空，则可能的情况为，该用户删除了该会话后，又收到了@消息（onReceiveMessages会先于onConversationCreated触发），此时需要在onConversationCreated处理@消息相关
                if (conversation) {
                const aitMsgs = conversation.aitMsgs || []

                if (aitMsgs.length === 0) {
                    this.updateConversation([
                    { ...conversation, aitMsgs: [item.messageClientId] },
                    ])
                } else {
                    this.updateConversation([
                    {
                        ...conversation,
                        aitMsgs: [...aitMsgs, item.messageClientId],
                    },
                    ])
                }
                }
            }
            }
        }
        })
    }
    
    private hasAitMsg(serverExtensions: string[]): boolean {
        let flag = false

        if (serverExtensions?.length) {
        serverExtensions?.forEach((item) => {
            try {
            const extObj: YxServerExt = JSON.parse(item || '{}')
            const yxAitMsg = extObj.yxAitMsg
            const account = this.rootStore.userStore.myUserInfo.accountId

            if (yxAitMsg) {
                Object.keys(yxAitMsg)?.forEach((key) => {
                if (key === account || key === AT_ALL_ACCOUNT) {
                    flag = true
                }
                })
            }
            } catch {
            this.logger?.error('parse serverExtension failed: ', item)
            }
        })
        }

        return flag
    }
    ```
