## 前提条件

实现自定义消息收发前，请确保已集成会话消息界面。

## 场景一：发送自定义消息

发送方可以在会话界面的消息发送按钮上绑定发送消息的函数。

以 **发送一条评价消息** 为例：

发送时，可将 **是否为评价消息** 标记在附件信息 `attach` 字段或者其他字段中（可 [提交工单](https://app.yunxin.163.com/global/service/ticket/create) 联系网易云信技术支持工程师获取最贴近您业务的字段），然后在消息接收时进行判断。

- 如果是评价消息，则使用自定义渲染。

- 如果不是评价消息，则返回 null，使用默认的消息渲染逻辑。

```JavaScript
import { compile } from "jsx-web-compiler";
// 此部分代码为右侧发送按钮新增一个发送评价信息按钮，并绑定消息发送函数
 window.$uikit.render(
      ChatContainer,
      {
        actions: [
            //...,
            {
              action: 'evaluation',
              visible: true,
              render: (props) => {
                   return compile(`
                    <button onClick={context.sendEvaluationMessage}>
                      评价
                    </button>
                  `, {
                    sendEvaluationMessage
                  });
                },
            },
        ]
      },
          this.$refs.chat
    );

    // 发送评价消息
    const sendEvaluationMessage = () => {
      // 此处的 nim 对象访问取决于您在初始化 IM SDK 时，将 IM SDK 实例挂在哪个实例上，此处以挂在 window 上为例
      const nim = window.nim;
      // 创建一条自定义消息 具体参考 https://doc.yunxin.163.com/messaging2/client-apis/TY4MDg4MTk?platform=client
      const message = nim.V2NIMMessageCreator.createCustomMessage(
        'this is a custom message' + Date.now(),
        JSON.stringify({
          isEvaluationMessage: true,
        }),
      );
      // 此处的 nim 对象访问取决于您在初始化 IMUIKit 时，将 IMUIKit 实例挂在哪个实例上，此处以挂在 vue 实例上为例
      const store = this.$uikit.context.store;
      const to = store.uiStore.selectedConversation;
      nim.V2NIMMessageService.sendMessage(message, to).then((res) => {
        // 把当前返回的消息体插入消息列表
        store.msgStore._handleSendMsgSuccess(res.message);
        console.log('======finalMessage', res.message);
        const messageClientId = res.message.messageClientId;
        // 让消息滚动到可视区域
        document.getElementById(messageClientId).scrollIntoView();
      });
    }
```

## 场景二：接收自定义消息并渲染

接收方和发送方可从如下回调函数中获取 IM UIKit
渲染一条消息的所有参数，从而自行实现任意的 UI 展示。

如果需要针对特定类型（`type`）的消息进行单独的渲染处理，可以根据 `type` 判断并返回自定义的内容，如果返回 `null` 则，使用 IMUIKit 默认渲染方式。

| <div style="width:180px">回调</div> | <div style="width:100px">类型</div> | 说明 |
| ---- | ---- | ---- |
| `renderP2pCustomMessage` | `((options: RenderP2pCustomMessageOptions) => Element | null)` | 渲染单聊中的自定义消息
| `renderTeamCustomMessage` | `((options: RenderTeamCustomMessageOptions) => Element | null)` | 渲染群聊中的自定义消息
| `renderMessageOuterContent` | (<br>    msg: V2NIMMessageForUI<br> ) => JSX.Element \| null \| undefined | 自定义渲染会话消息，气泡样式也需要自定义
| `renderMessageInnerContent` | (<br>    msg: V2NIMMessageForUI<br> ) => JSX.Element \| null \| undefined | 自定义渲染会话消息，不需要自定义气泡样式

您可以根据您业务需要，决定调用以上自定义消息的任意一种。

**示例代码**：

:::::: div linked-codes
::: code Vue2/Vue3
```TypeScript
this.$uikit.render(
      ChatContainer,
      {
        renderMessageInnerContent:(props) => {
          const raw = props?.attachment?.raw || '{}';
          // 这里根据发送自定义消息时的标记位进行判断
          const isEvaluationMessage = JSON.parse(raw).isEvaluationMessage;
          console.log('============isEvaluationMessage==', isEvaluationMessage);
          // 判断是否是自定义消息
          if (isEvaluationMessage) {
            // 消息 UI 部分自行实现
            return compile(
              ` <div>xxx</div>`,
              {},
            );
          // 其他类型消息 return null,使用 IMUIKit 默认渲染逻辑
          } else {
            return null;
          },
        //...
      },
      this.$refs.chat
    );
```

## 常见问题

### 可通过哪些参数/回调自定义渲染消息的样式？

支持通过如下参数/回调自定义渲染单聊和群聊的自定义消息。

- `renderMessageOuterContent`

    ```JSX
    (
        msg: V2NIMMessageForUI
    ) => JSX.Element | null | undefined
    ```

    自定义渲染会话消息，不需要自定义头像、昵称，气泡样式需要自定义。

- `renderMessageInnerContent`

    ```JSX
    (
        msg: V2NIMMessageForUI
    ) => JSX.Element | null | undefined
    ```

    自定义渲染会话消息，不需要自定义气泡样式、头像、昵称。

- `renderMessageAvatar`

    ```JSX
    (
        msg: V2NIMMessageForUI
    ) => JSX.Element | null | undefined
    ```

    自定义渲染消息头像。

- `renderMessageName`

    ```JSX
    (msg: V2NIMMessageForUI) => JSX.Element | null | undefined
    ```

    自定义渲染消息昵称。

- `renderP2pCustomMessage`

    ```JSX
    ((options: RenderP2pCustomMessageOptions) => Element | null)
    ```

    渲染单聊中的整条自定义消息，头像、昵称、消息体均需要自行实现。

- `renderTeamCustomMessage`

    ```JSX
    ((options: RenderTeamCustomMessageOptions) => Element | null)
    ```

    渲染群聊中的自定义消息，头像、昵称、消息体均需要自行实现。
