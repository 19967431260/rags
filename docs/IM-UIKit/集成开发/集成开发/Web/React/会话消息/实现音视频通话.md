本文介绍如何通过 IM UIKit 引入和初始化呼叫组件到 React 框架的项目中，进而在您的应用中实现一对一音视频通话。会话消息模块（`ChatContainer`）开始支持音视频通话功能。该功能基于网易云信呼叫组件实现。

## 效果预览

实现音视频通话功能后，用户在会话界面的输入区域单击 <img style="width:2.5%" src="https://yx-web-nosdn.netease.im/common/90c6bb337b6fcdcafa3003438ce59ed2/音视频按钮.png" /> 图标，即可快速发起语音通话或视频通话。

<img style="width:100%;border: 1px solid #BFBFBF;" src="https://yx-web-nosdn.netease.im/common/680548b299a9ad77f60c3c09815e95b8/image.png" />

## 前提条件

根据本文操作前，请确保您已经完成了以下设置：

- 因呼叫组件是基于 React 框架开发，请确保项目是基于 React 框架开发的，而不是基于 Vue.js 框架。

- 已开通 IM 即时通讯、音视频通话 2.0 及信令能力。具体操作流程请参考 [开通能力](https://doc.yunxin.163.com/console/concept/zc3NDYzNzc?platform=console)。

- 已 [集成会话消息界面](https://doc.yunxin.163.com/messaging-uikit/guide/DgzMDkwODI?platform=web)。

## 实现流程

::: note note 
- 若您通过跑通 [Demo 源码](https://github.com/netease-kit/nim-uikit-web) 集成 UIKit，那么 NEChatUIKit 中已包含云信呼叫组件，您无需单独引入（忽略第一步）。
- 若您通过组件导入的方式集成 UIKit，那么使用音视频通话功能前，需要先在您的项目中引入呼叫组件。
:::

### 第一步：引入呼叫组件

1. 通过 NPM 方式将 IM UIKit 安装到您的 Web 项目中。

    示例代码如下：

    ```NPM
    npm install @xkit-yx/call-kit-react-ui
    npm install @xkit-yx/call-kit
    ```
    :::note note
    目前仅支持 NPM 方式集成，暂不支持 CDN 方式集成。
    :::

2. 将 `CallViewProvider`、`CallViewProviderRef` 组件以及样式文件导入到您的 React 项目中。

    示例代码如下：

    ```JavaScript
    import { CallViewProvider, CallViewProviderRef } from '@xkit-yx/call-kit-react-ui'
    import '@xkit-yx/call-kit-react-ui/es/style/index'
    ```

### 第二步：初始化呼叫组件

在发起音视频通话前，需先初始化呼叫组件。

1. 在函数组件内，为了引用 `CallViewProvider` 组件，您需要先创建一个 `callViewProviderRef` （类型为 `CallViewProviderRef`）。

    ```JSX
    import { CallViewProvider, CallViewProviderRef } from '@xkit-yx/call-kit-react-ui'
    const callViewProviderRef = useRef<CallViewProviderRef>(null)
    ```

2. 使用 `CallViewProvider` 组件时，将 IM SDK 实例、AppKey 和当前用户账号等参数传递给 `neCallConfig` 配置对象。

3. 使用 `CallViewProvider` 包裹 IMUIKit 根组件。

    示例代码如下：

    ```JSX
    import { useStateContext } from '@xkit-yx/im-kit-ui'
    import { CallViewProvider, CallViewProviderRef } from '@xkit-yx/call-kit-react-ui'
    const callViewProviderRef = useRef<CallViewProviderRef>(null)
    // store 实例和 im sdk 实例
    const { store,nim } = useStateContext()
    <CallViewProvider
        ref={callViewProviderRef}
        neCallConfig={{
            nim: nim.nim, //IM SDK 实例
            appKey: appkey, //您的 App Key
            debug: true,
        }}
        position={{
            x: 400,
            y: 10,
        }}
    >
        <IMApp>
    </CallViewProvider>
    ```

### 第三步：监听音视频通话相关事件

在 `useEffect` 中，使用 `callViewProviderRef.current` 可以获取到 neCall 实例，可对呼叫过程中事件进行监听，例如注册呼叫结束事件 `onMessageSent`，以及设置呼叫超时时间 `setTimeout`。

示例代码如下：

```JSX
 useEffect(() => {
    if (callViewProviderRef.current?.neCall) {
      //注册呼叫结束事件监听
      callViewProviderRef.current?.neCall?.on('onRecordSend', (options) => {
        //@ts-ignore
        store.msgStore.addMsg(options.conversationId, [options])
        document.getElementById(options.messageClientId)?.scrollIntoView({
          block: 'nearest', // 滚动到目标元素的最近可见位置
          inline: 'nearest', // 避免水平方向的滚动
        })
      })
      // 设置呼叫超时时间
      callViewProviderRef.current?.neCall?.setTimeout(30)
      // 接通成功事件
      callViewProviderRef.current?.neCall?.on('onCallConnected', () => {
        // 暂停音视频消息的播放
        pauseAllAudio()
        pauseAllVideo()
      })
    }
  }, [callViewProviderRef.current?.neCall, store.msgStore])
```

### 第四步：发起音视频通话

网易云信已将呼叫组件的语音通话和视频通话能力绑定在 `handleCall` 中。因此您可直接调用 `callViewProviderRef.current.call` 方法，发起一对一语音通话或视频通话。

#### 发起呼叫

```JSX
 const receiverId = nim.V2NIMConversationIdUtil.parseConversationTargetId(
    store.uiStore.selectedConversation
  )
// 发起呼叫
  const handleCall = useCallback(
    async (callType) => {
      try {
        await callViewProviderRef.current?.call?.({
          accId: receiverId,
          callType,
        })
        setCallingVisible(false)
      } catch (error) {
        switch (error.code) {
          // 忙线
          case '105':
            message.error(t('inCallText'))
            break
          // 无网络
          case 'Error_Internet_Disconnected':
            message.error(t('networkDisconnectText'))
            break
          default:
            // 处理其他错误
            message.error(t('callFailed'))
            break
        }
      }
    },
    [receiverId]
  )

<div onClick={() => handleCall('1')}>发起呼叫</div>
```

### 第五步：自定义渲染音视频消息

当 `msg.type` 为 `V2NIMConst.V2NIMMessageType.V2NIM_MESSAGE_TYPE_CALL`，表示音视频消息，使用 `renderP2pCustomMessage` 进行自定义渲染音视频消息内容。如下图所示：

<img alt="image-netease" src="https://yx-web-nosdn.netease.im/common/ad311c309148d283d2e8e0226880f7f6/%E8%87%AA%E5%AE%9A%E4%B9%89%E6%B6%88%E6%81%AF%E5%86%85%E5%AE%B9.png" style="width:40%;border: 1px solid #BFBFBF;">

示例代码如下，您也可以根据您的具体业务场景实现 UI（具体实现可参考 Demo [index.tsx](https://github.com/netease-kit/nim-uikit-web/blob/main/react/src/components/IMApp/index.tsx)）。

```JSX
  const renderP2pCustomMessage = useCallback(
    (options: RenderP2pCustomMessageOptions) => {
      const msg = options.msg

      // msg.type 为 g2 代表的是话单消息 renderP2pCustomMessage 返回 null 就会按照组件默认的逻辑进行展示消息
      if (
        msg.messageType !== V2NIMConst.V2NIMMessageType.V2NIM_MESSAGE_TYPE_CALL
      ) {
        return null
      }

      const attach = msg.attachment as any
      const raw = JSON.parse(attach?.raw || '{}')
      const duration = raw.durations[0]?.duration
      const status = raw.status
      const type = raw.type
      const icon = type == 1 ? 'icon-yuyin8' : 'icon-shipin8'
      const myAccount = store.userStore.myUserInfo.accountId
      //判断当前消息是发出的消息还是接收的消息
      const isSelf = msg.senderId === myAccount
      const account = isSelf ? myAccount : receiverId
      const deleteG2Message = (msg) => {
        store.msgStore.deleteMsgActive([msg])
      }

      return (
        <div className={classNames('wrapper', { 'wrapper-self': isSelf })}>
          <ComplexAvatarContainer account={account} />
          <Dropdown
            key={msg.messageClientId}
            trigger={['contextMenu']}
            getPopupContainer={(triggerNode) =>
              messageActionDropdownContainerRef.current || triggerNode
            }
            overlay={
              <Menu
                onClick={() => deleteG2Message(msg)}
                items={[
                  {
                    label: t('deleteText'),
                    key: 'delete',
                    icon: <DeleteOutlined />,
                  },
                ]}
              />
            }
          >
            <div
              className={classNames('g2-msg-wrapper', {
                'g2-msg-wrapper-self': isSelf,
              })}
              ref={messageActionDropdownContainerRef}
            >
              <div className="appellation">
                {store.uiStore.getAppellation({ account })}
              </div>

              <div
                className={classNames('g2-msg', { 'g2-msg-self': isSelf })}
                onClick={() => handleCall(type.toString())}
              >
                <i className={classNames('iconfont', 'g2-icon', icon)}></i>
                <span>{g2StatusMap[status]}</span>
                {duration ? (
                  <span className="g2-time">
                    {convertSecondsToTime(duration)}
                  </span>
                ) : null}
              </div>
              <div className="time">{Utils.formatDate(msg.createTime)}</div>
            </div>
          </Dropdown>
        </div>
      )
    },
    [
      handleCall,
      receiverId,
      store.uiStore,
      store.msgStore,
      store.userStore.myUserInfo.accountId,
    ]
  )
    //...
    <ChatContainer
        //...
        renderP2pCustomMessage={renderP2pCustomMessage}
    />
```
