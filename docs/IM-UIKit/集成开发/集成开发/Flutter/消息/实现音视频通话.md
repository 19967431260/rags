本文介绍如何通过 IM UIKit 引入和初始化呼叫组件，进而在您的 IM 应用中实现音视频通话。

<div id="scenarios">

## 应用场景

自 IM Flutter UIKit v10.6.1 开始，会话消息模块（nim_chatkit_ui）支持音视频通话消息功能，满足即时通讯应用拨打音视频电话的场景。该功能基于 [网易云信呼叫组件](https://doc.yunxin.163.com/nertccallkit/concept/jY3ODM5OTM?platform=client) 实现。

</div>

<div id="examples">

## 效果预览

实现音视频通话功能后，用户在会话界面的输入区域单击 **+ > 音视频通话**，即可快速发起 **音频通话** 或 **视频通话**。

## 前提条件

根据本文操作前，请确保您已经完成了以下准备工作：

- 开通 **即时通讯**、**音视频通话 2.0** 及 **信令** 产品。具体操作流程请参考 [开通能力](https://doc.yunxin.163.com/console/concept/zc3NDYzNzc?platform=console)。
- 已 [集成 IM UIKit](https://doc.yunxin.163.com/messaging-uikit/guide/Dk0OTEzOTY?platform=flutter) 并 [集成消息界面](https://doc.yunxin.163.com/messaging-uikit/guide/jgyMzk2MDc?platform=flutter)。

::: note note
音视频通话功能集成在会话消息界面，因此必须集成消息界面（nim_chatkit_ui）之后，集成音视频通话（nim_chatkit_callkit）才能正常使用。
:::

## 第一步：引入呼叫组件

在您的工程的 `pubspec.ymal` 中引入呼叫组件。

- 本地依赖：

    ```
    dependencies:
        nim_chatkit_callkit:
            path: ../nim_chatkit_callkit
    ```

- pub 依赖：

    ```
    dependencies:
        nim_chatkit_callkit: ^lastVersion
    ```

## 第二步：初始化组件路由和国际化文案

在应用启动时，需要先初始化路由和国际化文案，在 MaterialApp 中配置以下内容：

```Dart
//初始化路由
navigatorObservers: [
 //...
  NECallKitUI.navigatorObserver
],

//初始化国际化文案
localizationsDelegates: [
  //...
  ChatKitCall.delegate,
  NECallKitUI.delegate,
  //...
],
```

## 第三步：初始化呼叫组件

在登录 IM 后调用 `ChatKitCall.init` 方法初始化呼叫组件。

<div style="width:120px">参数<div> | <div style="width:50px">类型</div> | 必传 | 说明
---- | ---- | ----
`appKey` | String | 是 | 在网易云信控制台创建应用时获取到的 App Key。
`accountId` | String | 是 | 当前用户的 IM 账号（account_id）。</li></ul>
`showIndex` | Integer | 否 | 呼叫消息入口显示位置，默认为 1。
`extraConfig` | [`NEExtraConfig`](https://pub.dev/documentation/netease_callkit_ui/latest/ne_callkit_ui/NEExtraConfig-class.html) | 否 | 额外配置参数，包含 lckConfig 等参数配置。

**示例代码**：

```Dart
ChatKitCall.instance.init(
    appKey: 'appkey',
    accountId: 'accountId');
```
