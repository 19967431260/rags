本文档提供网易云信 IM UIKit Flutter 项目中依赖版本冲突问题的解决方案。

::: note note
网易云信会持续更新 IM UIKit 组件的依赖版本约束，建议您关注官网版本更新动态。
:::

## 问题现象

当集成网易云信 IM UIKit 的 Flutter 项目（例如 `nim_conversationkit_ui`）时，执行 `flutter pub get` 命令后触发版本解析报错，报错信息如下：

```
Because nim_conversationkit_ui >=10.6.1 depends on netease_plugin_core_kit ^10.1.0 which depends on permission_handler ^11.0.1, nim_conversationkit_ui >=10.6.1 requires permission_handler ^11.0.1.
So, because koko_flutter depends on both permission_handler 12.0.1 and nim_conversationkit_ui ^10.6.1, version solving failed.
```

## 问题原因

网易云信 IM UIKit 组件通过 **传递依赖** 关联特定版本的基础工具库（例如 `permission_handler`、`device_info_plus` 等）：

- 版本约束遵循 Dart 规范：`^x.y.z` 表示兼容 `>=x.y.z <(x+1).0.0` 的版本，仅限同一大版本内升级。
- 项目直接声明的基础库版本与 UIKit 要求的版本区间无交集，会导致 pub 包管理工具无法匹配到兼容版本，进而报错。
- 常见冲突库：`permission_handler`、`device_info_plus`、`shared_preferences` 等。

## 解决方案

### 方案一：降级项目基础依赖库版本

当您项目的依赖库版本（比如 permission_handler）与 UIKit 要求的版本区间无交集，发生冲突，则需要您降级项目中的基础依赖库版本。

1. 定位冲突的基础依赖库。

   ```bash
   flutter pub deps --style=tree
   ```

2. 修改项目根目录的 **pubspec.yaml** 文件。将版本号调整至云信 IM UIKit 要求的区间内。

   ```yaml
   dependencies:
     # 将版本降级至兼容区间
     permission_handler: ^11.0.1  # 原: 12.0.1
     nim_conversationkit_ui: ^10.6.1
   ```

3. 重新获取依赖。

   ```bash
   flutter pub get
   ```

### 方案二：依赖库版本强制覆盖

当您项目依赖的第三方库中依赖的基础库与 UIKit 要求的版本区间无交集，发生冲突，则需要您强制覆盖依赖库版本。

强制覆盖可能因基础库 API 变更（如方法移除、参数调整），导致 IM UIKit 相关功能异常（如权限申请失败），使用后请验证部分功能。

1. 添加覆盖配置。在项目 **pubspec.yaml** 文件末尾添加 `dependency_overrides` 节点。

   ```yaml
   dependencies:
     nim_conversationkit_ui: ^10.6.1
     permission_handler: 12.0.1

   # 强制覆盖依赖版本
   dependency_overrides:
     permission_handler: 12.0.1
     # 可添加其他需要覆盖的依赖
   ```

2. 执行依赖更新。
 
   ```bash
   flutter pub get
   ```

## 常见问题

### 如何快速确定 IM UIKit 的依赖版本要求？

您可在项目根目录执行 `flutter pub deps` 命令，查看完整依赖树以定位版本约束，也可直接查阅官网 IM UIKit 组件的依赖说明文档。

### 强制覆盖依赖后出现运行时错误怎么办？

请立即回滚至方案一的降级配置，并联系技术支持，提供错误日志及项目依赖版本信息，我们将协助您快速排查兼容性问题。

## 技术支持

若修复后出现功能异常，或遇到无法解决的问题，请通过 [网易云信技术支持工单系统](https://developer.163.com/support/) 提交问题，我们将为您提供专属技术支持。
