当您需要加速对象存储（ OSS 或 COS）上的静态资源时，可以通过网易云信融合 CDN 加速对象存储的访问域名，实现静态资源的访问加速。本文介绍实现对象存储加速的操作流程和应用场景。

## 背景信息
对象存储源站上存储的静态资源包括静态脚本、图片、附件、音频和视频等。

当终端用户请求访问或下载静态资源时，CDN 会将源站上的静态资源进行加速，将源站上的资源缓存到 CDN 加速节点上，系统自动调用离终端用户最近的 CDN 节点上已缓存的资源，实现资源加速。

网易云信融合 CDN 将网易对象存储 NOS、阿里云对象存储 OSS、腾讯云对象存储 COS 等源站资源，缓存到网易云信遍布全球的加速节点，当终端用户请求访问和获取源站资源时，无需回源，可就近获取 CDN 节点上已经缓存的资源，提高资源访问速度，同时分担源站压力。

## 前提条件
- 已在第三方对象存储控制台开通对象存储服务，并创建 Bucket。
- 待加速的 Bucket 为公开空间，非私有空间。

## 实现流程

### 步骤1 添加加速域名
具体操作步骤请参见[配置分发加速](https://doc.yunxin.163.com/docs/jY3NDM4Nzc/DU1NDk4MTI?platformId=100066)。

### 步骤2 完成源站和回源配置

1. 获取 Bucket 的访问域名。
    - 回源阿里云对象存储 OSS：请登录阿里云 OSS 控制台，获取对应的 Bucket 域名。
    - 回源腾讯云对象存储 COS：请登录腾讯云 COS 控制台，获取对应的访问域名。
2. 在[网易云信控制台](app.yunxin.163.com)的**分发加速配置**中，配置**源站信息**和**回源信息**为对应 Bucket 的访问域名。


    1. 在<a href="https://app.yunxin.163.com/index#/" target="_blank">网易云信控制台</a>的首页单击指定应用名称。

    2. 在左侧导航栏选择**产品功能** > **点播**，单击**分发加速配置** 页签。

        ![域名管理.png](https://yx-web-nosdn.netease.im/common/4451af7e1fc6f8de653a353c9919c903/域名管理.png)


    3. 单击**域名管理**右侧的**子功能配置**。


    4. 在**域名管理 子功能配置**页面，单击**添加域名**。
    5. 在**添加域名**对话框中设置相关参数，参数说明如下表所示。单击**确认添加**。


      
        ![加速对象存储资源.png](https://yx-web-nosdn.netease.im/common/e74de40ae801f1ef556f88ac83fc4de9/加速对象存储资源.png)




    分类 | 配置 | 描述
    ---- | -------------- | ---------
    域名信息 | 加速域名 | 请填写已在 ICP 备案过的域名地址。</br> <note type="note"> 云信控制台不支持添加泛域名。如果您需要配置泛域名，请联系网易云信技术支持，在后台配置泛域名。</note>
    源站信息 | 源站类型 |请选择**域名**。</li></ul>|
    ^^ | 主源站 | 请填写待加速 Bucket 的访问域名。 |
    ^^ | 备源站 | 非必选。回源流量优先回源到主源站。当主源站无法访问时，回源到备源站。 |
    回源信息 | 回源协议 |当您通过客户端请求访问资源时，如果CDN节点上未缓存该资源，则会根据您配置的回源协议到源站获取资源。回源协议包括 HTTP、HTTPS 和协议回源。具体说明如下：<br><ul><li>HTTP：使用 HTTP 协议回源获取资源。</li><li>HTTPS：使用 HTTPS 协议回源获取资源。</li><li>协议回源：当客户端使用 HTTP 访问时，使用 HTTP 回源获取资源。当客户端使用HTTPS访问时，使用 HTTPS 回源获取资源（源站需要支持 HTTPS 访问）。</li></ul> |
    ^^ | 回源 HOST | 请填写待加速 Bucket 的访问域名。 |

    4. 单击**确定**。

### 步骤3 配置 CNAME
配置 CNAME 并验证 CNAME 解析是否生效。具体请参见[配置CNAME](https://doc.yunxin.163.com/docs/jY3NDM4Nzc/TAyNjUxNDg?platformId=100066)。
