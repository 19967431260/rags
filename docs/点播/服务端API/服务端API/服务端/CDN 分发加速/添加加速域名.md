本文介绍如何添加加速域名和回源 HOST，加速客户端访问业务服务器上的业务。

## 背景信息

- 源站：指业务所在的服务器。源站决定了回源请求的具体 IP 地址。
- 回源 HOST：指 CDN 节点在回源过程中，在源站访问的站点域名。当源站有多个业务共用的情况时，可以通过回源 HOST 来区分不同的业务。
- 回源：当用户通过客户端请求访问资源时，如果 CDN 节点上未缓存该资源，则会回源到源站获取资源。

## 前提条件

您已经拥有用于加速的域名，且该域名已在 ICP 备案。

## 操作步骤
1. 在<a href="https://app.yunxin.163.com/index#/" target="_blank">网易云信控制台</a>的首页单击指定应用名称。

2. 在左侧导航栏选择**产品功能** > **点播**，单击**分发加速配置** 页签。

    ![域名管理.png](https://yx-web-nosdn.netease.im/common/4451af7e1fc6f8de653a353c9919c903/域名管理.png)


3. 单击**域名管理**右侧的**子功能配置**。


4. 在**域名管理 子功能配置**页面，单击**添加域名**。
5. 在**添加域名**对话框中设置相关参数，参数说明如下表所示。单击**确认添加**。


    ![添加域名.png](https://yx-web-nosdn.netease.im/common/1f0e49656f4eb6c8839b2f6ada4ea1ff/添加域名.png)




    分类 | 配置 | 描述
    ---- | -------------- | ---------
    域名信息 | 加速域名 | 请填写已在 ICP 备案过的域名地址。</br> <note type="note"> 云信控制台不支持添加泛域名。如果您需要配置泛域名，请联系网易云信技术支持，在后台配置泛域名。</note>
    源站信息 | 源站类型 |<ul><li>IP：客户端使用IP地址访问源站。</li><li>域名：客户端使用域名访问源站。</li></ul>|
    ^^ | 主源站 | <ul><li>如果源站类型为 IP ，此处请填写主源站的 IP 地址。当配置了多个 IP 地址时，CDN 通过轮询选择某个IP地址。</li><li>如果源站类型为域名，此处请填写主源站的域名。如果需要加速对象存储上的资源，请参见[CDN 加速对象存储资源](https://doc.yunxin.163.com/docs/jY3NDM4Nzc/TExNzY3OTU?platformId=100066)。</li></ul> |
    ^^ | 备源站 | 非必选。回源流量优先回源到主源站。当主源站无法访问时，回源到备源站。 |
    回源信息 | 回源协议 |当您通过客户端请求访问资源时，如果CDN节点上未缓存该资源，则会根据您配置的回源协议到源站获取资源。回源协议包括 HTTP、HTTPS 和协议回源。具体说明如下：<br><ul><li>HTTP：使用 HTTP 协议回源获取资源。</li><li>HTTPS：使用 HTTPS 协议回源获取资源。</li><li>协议回源：当客户端使用 HTTP 访问时，使用 HTTP 回源获取资源。当客户端使用HTTPS访问时，使用 HTTPS 回源获取资源（源站需要支持 HTTPS 访问）。</li></ul> |
    ^^ | 回源 HOST | 当源站的同一个 IP 地址上绑定了多个域名或站点时，您需要指定回源时访问的具体域名。<br>如果您未配置回源HOST的值，则默认读取当前配置的加速域名。 |



## 示例

下表以不同类型的源站配置为例，展示源站和回源 HOST之间的关系。

源站类型 | 源站地址 | 回源 HOST|回源请求的地址
---- | -------------- | -------- |---------
域名 | www.example.com | www.example.a.com|www.example.com对应的 IP 地址所在主机上的 www.example.a.com 站点。|
IP | 1.1.1.1 |www.example.a.com|1.1.1.1主机上的 www.example.a.com 站点。|
