# 部门管理

## 本节导读

部门管理帮助你搭建和维护企业组织架构，为成员分配合适的部门归属，便于按组织维度管理权限、配额和使用数据。

**阅读本文后，你将了解**：
- 部门层级结构和 SSO 混合模式的基本概念
- 如何在部门树中导航和查看部门详情
- 如何创建、编辑、删除部门以及划转成员

---

## 概念准备

> 如果你已经了解部门层级和 SSO 概念，可以跳过本节。

### 部门层级

部门采用树状层级结构。系统会为你的租户自动创建一个与租户同名的**根部门**，你可以在根部门下创建子部门，子部门下还可以继续创建更深层级的部门。创建子部门的入口统一在右侧部门详情面板中，点击「新建子部门」按钮即可。

### SSO 混合模式

如果你的企业在「租户设置」中配置了 SSO（Single Sign-On，单点登录），部门管理会自动进入混合模式。此时部门分为两种来源：

| 来源 | 标识 | 管理方式 |
|------|------|---------|
| SSO 同步 | 蓝色「SSO」标签 | 由企业 IdP（身份提供商）管理，结构只读 |
| 本地创建 | 绿色「本地」标签 | 可自由创建、编辑、删除 |

SSO 部门和本地部门在左侧部门树中会分区展示，SSO 部门排列在上方，下方用分隔线标注「本地部门」，再下方排列本地部门。SSO 部门不支持编辑、创建子部门和删除操作，操作区域会显示「SSO 部门，只读」提示。

---

## 功能说明

### 浏览部门树

**功能说明**：左侧部门树展示你企业的完整组织架构，通过它快速导航到任意部门。

**操作步骤**：

1. 在左侧菜单中，点击「部门管理」。
2. 页面分为左右两部分：左侧是部门结构树，右侧是部门详情面板。

![部门管理列表页面](https://skillmarket-prod.nos-jd.163yun.com/uploads/2e7af100-c4eb-4a7c-8baf-17a8789f5f26.png)
3. 在左侧部门树中：
   - 点击部门名称，右侧会显示该部门的详细信息
   - 有子部门的节点左侧显示展开/折叠箭头，点击箭头可展开或收起子部门
   - 每个部门名称右侧会显示成员人数

> **提示**：首次进入页面时，系统会自动选中根部门。

---

### SSO 同步操作

> **提示**：以下内容仅适用于已配置 SSO 的租户。如果你的企业未配置 SSO，可以跳过本节。

如果你的企业配置了 SSO，部门树会自动进入混合模式。SSO 部门由企业 IdP 同步而来，结构只读，你不能对其进行编辑或删除。具体的 SSO 同步操作和同步状态查看，请在「成员管理」模块中了解。

---

### 查看部门详情

**功能说明**：选中一个部门后，右侧面板会显示该部门的基本信息和成员列表。

**操作步骤**：

1. 在左侧部门树中，点击要查看的部门。

2. 右侧面板顶部显示部门名称、创建时间，以及来源标签（SSO 或本地）。
3. 面板中包含两个 Tab，点击切换查看：

#### 基本信息 Tab

展示以下信息卡片：

| 信息项 | 含义 |
|--------|------|
| 部门名称 | 当前部门名称 |
| 层级 | 该部门在组织架构中的层级（根部门 / 第 N 级） |
| 直属成员 | 直接归属于该部门的成员数量（不含子部门成员） |
| 含下级共 | 该部门及所有子部门的成员总数 |
| 子部门数 | 直接下级子部门数量 |
| 部门管理员 | 该部门的管理员姓名（如未设置显示"未设置"，当前仅展示不可编辑） |
| 描述 | 部门职能简介（如未填写则不显示） |

下方如果有子部门，会以卡片网格展示，显示每个子部门的名称、来源标签、成员数和子部门数。点击子部门卡片可以快速跳转到对应部门。
#### 直属成员 Tab

展示直接归属于该部门的成员列表，包含以下操作：

- **搜索**：在搜索框中输入姓名或邮箱进行模糊搜索
- **角色筛选**：通过下拉菜单按角色过滤成员
- **批量操作**：勾选成员后，顶部会显示批量操作栏，支持「批量划转」
> **提示**：SSO 成员的复选框和「划转」按钮不可用，SSO 成员不可被划转。当前登录用户也不能被划转。

---

### 创建子部门

**功能说明**：在已有部门下创建一个子部门。

**操作步骤**：

1. 在左侧部门树中，选中父部门（即要创建子部门的上级部门）。

2. 在右侧面板顶部，点击「新建子部门」按钮。

![新建子部门弹窗](https://skillmarket-prod.nos-jd.163yun.com/uploads/974145d4-9ace-4405-b58c-af6f06b34b87.png)

3. 在弹出的对话框中填写以下信息：
   - **上级部门**：自动显示所选父部门名称，不可修改
   - **部门名称**（必填）：输入部门名称，最多 100 个字符
   - **描述**（选填）：输入部门职能简介，最多 100 个字符
4. 点击「保存」。

**操作结果**：创建成功后，新部门会出现在左侧部门树中父部门下方。

> **警告**：SSO 部门不支持创建子部门，操作区域会显示只读提示。

---

### 编辑部门

**功能说明**：修改本地部门的名称和描述。

**操作步骤**：

1. 在左侧部门树中，选中要编辑的部门。

2. 在右侧面板顶部，点击「编辑」按钮。

![编辑部门弹窗](https://skillmarket-prod.nos-jd.163yun.com/uploads/a3eaf6c5-04a1-4ec4-b234-ba6cbb60ee12.png)

3. 在弹出的对话框中修改部门名称或描述。
4. 点击「保存」。

**操作结果**：修改成功后，部门树和详情面板会同步更新。

> **警告**：编辑部门不会影响成员归属。SSO 部门不支持编辑。

---

### 删除部门

**功能说明**：删除本地部门。系统会根据部门的不同状态，展示不同的处理方式。

**操作步骤**：

1. 在左侧部门树中，选中要删除的部门。

2. 在右侧面板顶部，点击「删除」按钮。

![删除部门确认弹窗](https://skillmarket-prod.nos-jd.163yun.com/uploads/17dabe53-d533-49e9-8f53-02546edd82ef.png)

3. 根据部门状态，会出现以下四种情况：

   **情况一：部门下有子部门**

   弹框标题为「无法删除」，提示你该部门下有多少个子部门，需要先删除所有子部门后才能操作。点击「知道了」关闭弹框。
   **情况二：部门下有直属成员，且有上级部门**

   弹框提示成员数量，并要求选择成员处理方式：

   - **移至上级部门**（默认选项）：成员自动归入当前部门的上级部门
   - **移至指定部门**：展开部门树选择器，手动选择目标部门（不能选当前部门自身）

   选择后点击「确认删除」。
   **情况三：部门下有直属成员，无上级部门**

   成员将直接归入根部门。确认后点击「确认删除」。

   **情况四：部门下无子部门且无成员**

   弹框提示"确定要删除部门 XXX？该操作不可撤销。"，点击「确认删除」。
> **危险**：删除部门不可撤销，请谨慎操作。
>
> **警告**：根部门不可删除。SSO 部门不可删除。

---

### 划转成员

**功能说明**：将成员从一个部门移动到另一个部门。划转后成员角色保持不变，仅归属部门改变。

#### 单个划转

1. 在右侧面板中切换到「直属成员」Tab。

2. 在成员列表中，找到目标成员，点击操作列中的「划转」。
3. 在弹出的对话框中：
   - 顶部显示成员姓名和邮箱
   - 在部门树选择器中选择目标部门（当前部门会被排除，不可选）
4. 点击「确认划转」。

**操作结果**：划转成功后，成员会从当前部门移出，出现在目标部门的直属成员列表中。

> **提示**：SSO 租户中，划转目标部门只能选择本地部门，SSO 部门由企业 IdP 管理不在可选范围内。

#### 批量划转

1. 在「直属成员」Tab 中，勾选要划转的成员。

   > **提示**：SSO 成员和当前登录用户不可被勾选。

2. 顶部出现批量操作栏，显示「已选 N 人」，点击「批量划转」。

3. 在弹出的对话框中选择目标部门，点击「确认划转」。

---

### 编辑成员角色

1. 在「直属成员」Tab 中，找到目标成员，点击操作列中的「编辑角色」。

2. 在弹出的对话框中，通过复选框选择该成员的角色（支持多选）。
3. 点击「保存」。

> **提示**：所有成员（包括 SSO 成员）都可以被编辑角色。当前登录用户不可编辑自身角色。

---

## 注意事项

> **提示**：建议根据企业实际组织架构合理划分部门层级，层级过深不利于管理和成员查找。

> **警告**：同一个父部门下，子部门名称不可重复。

> **危险**：删除部门操作不可撤销。删除前请确认部门下无子部门，并妥善处理直属成员的归属。

# 成员管理

## 本节导读

成员管理模块帮你管理企业内的所有成员，包括邀请新成员加入、调整成员角色、停用或移除成员等。

**阅读本文后，你将了解**：
- 成员有哪些状态，各状态意味着什么
- 如何邀请成员加入企业
- 如何对成员执行停用、启用、移除、角色变更和划转操作
- 如何生成和管理邀请链接

---

## 概念准备

> 如果你已了解成员状态和角色概念，可以跳过本节。

### 成员状态

每位成员在租户内有三种可能的状态：

| 状态 | 标识颜色 | 含义 |
|:----:|:--------:|------|
| 正常 | 绿色圆点 | 成员已加入，可正常使用工作区 |
| 已邀请 | 黄色圆点 | 已发送邀请邮件，对方尚未接受 |
| 已停用 | 灰色圆点 | 成员被停用，无法登录工作区，但数据保留 |

### 成员来源

如果企业配置了 SSO（单点登录），成员列表中会显示来源标识：

| 标识 | 含义 |
|:----:|------|
| SSO（蓝色标签） | 通过企业 IdP 系统同步的成员 |
| 本地（绿色标签） | 直接在平台上邀请加入的成员 |

SSO 成员不支持划转操作，批量操作时也会自动跳过 SSO 成员。

### 角色列表

租户内共有 7 种角色，一名成员可以同时拥有多个角色：

| 角色 | 职责说明 |
|------|---------|
| 租户管理员 | 拥有全部管理权限 |
| 部门管理员 | 管理所属部门的成员和设置 |
| 租户运营 | 负责运营相关工作 |
| 频道主编 | 管理领域频道的内容 |
| 频道小编 | 编辑频道内容 |
| 运维管理员 | 负责系统运维 |
| 普通成员 | 基础使用权限 |

---

## 功能说明

### 成员列表

进入「成员管理」页面后，你会看到一个搜索栏和成员列表。

![成员管理列表页面](https://skillmarket-prod.nos-jd.163yun.com/uploads/5c2895da-4ffc-4250-af74-47121d78896e.png)

**筛选栏**提供以下筛选条件：

- **搜索框**：输入姓名或邮箱进行模糊搜索
- **部门筛选**：下拉选择某个部门，只看该部门下的成员
- **角色筛选**：按角色过滤，支持全部角色及 7 种角色
- **状态筛选**：按正常、已邀请、已停用过滤

**列表各列含义**：

| 列名 | 说明 |
|------|------|
| 勾选框 | 勾选成员进行批量操作（SSO 成员和自己不可勾选） |
| 成员 | 彩色头像 + 姓名，当前登录用户旁显示橙色「当前登录用户」标签 |
| 邮箱 | 成员的注册邮箱 |
| 角色 | 彩色标签展示该成员拥有的所有角色 |
| 部门 | 显示成员所属部门的完整路径 |
| 状态 | 绿色圆点「正常」、黄色圆点「已邀请」、灰色圆点「已停用」 |
| 操作 | 根据成员状态显示不同的操作按钮 |

列表底部显示成员总数和分页控件，每页显示 20 条记录。

> **提示**：当企业启用了 SSO 时，列表底部会额外显示 SSO 用户数和本地用户数。

---

### 邀请成员

点击页面右上角的「邀请成员」按钮，会弹出一个下拉菜单，包含三个选项：

- **邮箱邀请**：输入邮箱地址直接发送邀请
- **生成邀请链接**：创建一个可分享的链接
- **管理邀请链接**：查看和管理已生成的邀请链接
#### 邮箱邀请

1. 点击「邀请成员」按钮，选择「邮箱邀请」。

![邮箱邀请弹窗](https://skillmarket-prod.nos-jd.163yun.com/uploads/49530544-a37e-4501-9933-53cdef7c9e3a.png)

2. 填写邀请信息：
   - **邮箱地址**（必填）：每行输入一个邮箱，单次最多 50 个
   - **角色**（必填）：点击下拉菜单选择角色，支持多选
   - **部门**（可选）：选择成员加入后归属的部门，也可不指定
   - **邀请消息**（可选）：自定义一段欢迎语，会附在邀请邮件中

3. 点击「发送邀请」。

**邀请流程**：

- 如果邮箱已在平台注册，对方会收到一封「加入工作区」的邮件，点击链接即可加入
- 如果邮箱未注册，对方会收到一封「注册并加入」的邮件，完成注册后自动关联到你的租户

> **提示**：如果邀请的邮箱已经是你的租户成员，系统会跳过该邮箱并提示失败数量。

#### 生成邀请链接

适合将链接发到群聊或粘贴到入职文档中，让多人自行加入。

1. 点击「邀请成员」按钮，选择「生成邀请链接」。

2. 配置链接参数：
   - **角色**（必填）：选择通过此链接加入的成员角色，支持多选
   - **部门**（可选）：指定加入的部门
   - **有效期**：可选择 7 天、30 天或永久
   - **使用次数**：可选择不限、10 次、50 次、100 次

3. 点击「生成链接」，页面会显示生成的链接，点击「复制」按钮即可分享。

> **提示**：你可以随时重新生成新链接，旧链接不受影响。

---

### 成员状态操作

#### 停用成员

停用后成员无法登录工作区，但其数据和角色信息会保留。

1. 在成员列表中，找到状态为「正常」的成员，点击操作列的「停用」。

![停用成员确认弹窗](https://skillmarket-prod.nos-jd.163yun.com/uploads/e5e2c623-6dd4-4374-8680-b67768b21621.png)

2. 系统弹出确认对话框，提示「停用后该成员将没有该租户的权限，随时可重新启用」。

3. 点击「确认停用」。

> **警告**：你无法停用自己的账号。

#### 启用成员

将已停用的成员恢复为正常状态。

1. 找到状态为「已停用」的成员，点击操作列的「启用」。

2. 系统立即将成员恢复为「正常」状态，无需额外确认。

#### 撤销邀请

对于已发送但对方尚未接受的邀请，可以将其撤销。

1. 找到状态为「已邀请」的成员，点击操作列的「撤销邀请」。

2. 系统弹出确认对话框，点击「确认」即可撤销。

#### 移除成员

将成员从租户中永久删除。

**前置条件**：只有「已停用」状态的成员才能被移除。如果成员状态为正常，需要先停用，再移除。

1. 找到状态为「已停用」的成员，点击操作列的「移除」。

2. 系统弹出确认对话框，提示「移除后该成员将无法访问工作区」。

3. 点击「确认」。

> **危险**：移除操作不可撤销。移除后成员的租户关联关系会被永久删除。

---

### 编辑成员角色

1. 在成员列表中，点击该成员操作列的「编辑角色」。

![编辑角色弹窗](https://skillmarket-prod.nos-jd.163yun.com/uploads/ca88a0cb-5c28-4568-a81c-0572a3d222be.png)

2. 在弹出的对话框中，勾选要分配的角色（支持多选）。

3. 点击「保存」。

> **提示**：编辑角色会覆盖成员原有的角色设置，而不是追加。

---

### 划转成员

将成员从一个部门移动到另一个部门，角色保持不变。

> **警告**：仅「正常」状态的本地成员支持划转。SSO 成员不可划转。

1. 在成员列表中，点击操作列的「划转」。

![划转成员弹窗](https://skillmarket-prod.nos-jd.163yun.com/uploads/05457437-fbfa-407e-854c-09db8b966745.png)

# 配额管理

## 本节导读

配额管理模块帮助你管理企业的积分资源，包括查看积分余额、向部门分配额度、配置模型档位以及购买资源包。

**阅读本文后，你将了解**：
- 积分（Credits）的核心概念和分配规则
- 如何查看配额总览和部门积分使用情况
- 如何分配、调整和释放部门积分额度
- 如何配置租户级和部门级模型档位
- 如何购买资源包充值积分

---

## 概念准备

> 如果你已了解积分和模型档位的概念，可以跳过本节。

### 积分是什么

积分（Credits）是使用平台 AI 模型和龙虾实例（Lobster Instance）的消耗单位，类似手机话费——先用后扣，用完即止。

**分配结构**：租户获得总积分后，向下分配给各部门，各部门未分配的部分由本级成员共享使用。

- 租户总积分 = 已锁定给各部门的积分 + 未分配池（全租户成员共享）
- 部门额度 = 已锁定给子部门的积分 + 部门内未分配池（部门成员共享）
- 子部门额度之和不超过父部门锁定额度

### 模型档位

平台提供多个模型档位，不同档位的 AI 能力和积分消耗倍率不同。每次对话按所选档位的倍率换算实际积分消耗。

| 档位 | 倍率 | 说明 |
|------|------|------|
| 轻量 | ×0.5 | 基础指令执行，响应快速，积分消耗最低 |
| 经济 | ×0.8 | 均衡性能与成本，适合日常办公助手场景 |
| 性能 | ×1.5 | 增强推理能力，适合复杂分析与代码生成 |
| 极致 | ×2.0 | 最强推理，专为高复杂度任务，积分消耗最高 |

举个例子：同一任务使用「经济」档消耗 800 基础积分，实际扣除 640 积分（800 × 0.8）；使用「性能」档则扣除 1200 积分（800 × 1.5）。

## 配额总览

### 进入页面

1. 在左侧菜单中，点击「资源与计费」分组下的「配额管理」。
2. 页面顶部显示标题「配额管理」和「购买资源包」按钮。
### 四张指标卡片

页面上方「积分余额」区域展示四张指标卡片：

| 指标 | 含义 |
|------|------|
| 已购余额 | 你企业当前持有的积分总数 |
| 已锁定（部门） | 已分配锁定的部门积分，标注占已购余额的百分比 |
| 未分配池 | 尚未分配给部门的剩余积分，供全租户成员共享使用 |
| 累计已消耗 | 已使用的积分总数，标注总消耗进度百分比 |
卡片下方有一条「总消耗进度」条，直观展示已消耗积分占已购余额的比例。

> **警告**：当积分余额不足时，页面会出现红色提示「租户已被关停（余额不足），请尽快充值恢复服务」。此时所有 AI 调用将停止，需要立即购买资源包充值。

## 部门积分分配

「部门积分分配」区域以表格形式展示各部门的积分使用情况，表格包含以下列：部门名称、余额（积分）、已消耗（积分）、使用率、操作。
表格底部有「未分配池（租户级）」一行，显示全租户成员共享的剩余积分。展开有子部门的父部门时，还会显示「未分配池（{部门名}内）」子行。

### 分配额度

对尚未分配额度的部门，可以进行首次分配。

1. 在部门行右侧，点击「分配额度」。

2. 在弹出的「调整积分额度」对话框中：
   - 查看当前锁定额度和未分配池可用数量
   - 在「新余额（积分）」输入框中填写要锁定的积分数
   - 点击「确认」
调高的部分从未分配池划拨；调低的部分自动归还至未分配池。输入框留空则表示不限额度。

### 调整额度

对已分配额度的部门，可以修改其锁定额度。

1. 在部门行右侧，点击「调整」。

2. 在「调整积分额度」对话框中修改积分数，点击「确认」。

![调整积分额度弹窗](https://skillmarket-prod.nos-jd.163yun.com/uploads/c89d1257-8192-4f23-86d1-b2d24dce50ed.png)

> **提示**：调低额度时，如果子部门额度之和超过新额度，需要先调低子部门的分配。

### 释放额度

将部门未消耗的积分归还至未分配池。

1. 在部门行右侧，点击「释放」。

2. 在「释放未用额度」对话框中：
   - 查看当前锁定额度、已消耗积分数和可释放数量
   - 输入要释放的积分数
   - 点击「确认释放」
> **提示**：最多可释放的数量等于当前锁定额度。释放后，该部门可用额度会相应减少。

## 模型配置

### 租户级模型配置

「模型配置」区域以卡片网格形式展示各模型档位的状态。每张卡片包含模型名称、倍率标签、开关按钮、配额上限、已消耗进度条。

![模型配置区域](https://skillmarket-prod.nos-jd.163yun.com/uploads/cd5aea0b-6409-4434-a369-cc4839fb677e.png)
#### 开启或关闭模型

1. 找到要操作的模型卡片，点击右侧的开关按钮。

2. **开启**：直接生效，无需确认。

3. **关闭**：系统会弹出「关闭模型」确认对话框，要求选择处理方式。

##### 关闭模型的三种情况

根据当前使用该模型的龙虾实例数量，系统有不同的处理逻辑：

1. **有龙虾实例正在使用，且有其他可用模型**：弹出迁移确认对话框，列出可迁移的目标模型（单选），选择后点击「确认关闭并迁移」。受影响的实例会自动切换到目标模型，调用不中断。

2. **有龙虾实例正在使用，且没有其他可用模型**：对话框中提示「没有其他可用模型，关闭后实例将停止服务」。如果确认关闭，所有实例的 AI 调用将立即停止。

3. **没有实例正在使用**：直接关闭，无需额外确认。
#### 设置配额上限

1. 在模型卡片中，点击「配额上限」旁的数值（显示为「不限」或具体积分数）。

2. 在「设置配额上限」对话框中输入积分上限，留空表示不限。

3. 点击「确认」。
> **提示**：配额上限设置后，该档位的总消耗不得超出限制。当使用率达到 70% 时，进度条会变为红色并显示警告图标。

### 部门级模型配置

部门级配置可以进一步收窄模型的使用范围，但只能在租户已开放的档位内操作。

1. 在「部门积分分配」表格中，找到目标部门行，点击右侧的「模型」。

2. 在「{部门名} · 模型配置」对话框中：
   - 顶部提示租户已开放的档位数量和名称
   - 租户未开放的档位显示为灰色，标注「租户未开放」，不可配置
   - 租户已开放的档位可以通过开关控制是否对本部门开放
   - 点击「剩余额度」旁的数值可编辑该部门的模型配额上限
3. 完成修改后，点击「保存」。

| 支付成功 | 积分已实时到账，可立即使用 | 「返回配额管理」 |

## 注意事项

> **警告**：调低父部门额度前，需要先调低子部门的分配额度，否则操作会失败。

> **警告**：关闭模型档位会影响正在使用该模型的所有龙虾实例。请务必在关闭前确认是否有实例正在使用，并选择合适的迁移目标。

> **危险**：当所有模型档位都关闭后，龙虾实例的 AI 调用将完全停止。请确保至少保留一个开启的模型档位。

# 龙虾管理

## 本节导读

龙虾管理模块帮助你查看和管理企业内所有龙虾实例（Lobster Instance）的运行状态，并执行启动、停止、重启等日常运维操作。

**阅读本文后，你将了解**：
- 龙虾实例的三种状态及含义
- 蜂巢大屏视图的使用方式

- 运维操作（启动/停止/重启/删除）的具体步骤

---

## 概念准备

> 如果你已了解龙虾实例的概念，可以跳过本节。

**龙虾实例**是部署在成员电脑或平台云端上的 AI 助手运行程序。每个成员使用的 AI 助手背后都对应一个龙虾实例。

龙虾实例按部署位置分为两种类型：

| 类型 | 说明 |
|------|------|
| 云端 | 由平台统一托管，运行在云服务器上 |
| 本地 | 安装在成员自己的电脑上，分为 macOS、Windows、Linux 三种操作系统 |

实例有三种运行状态：

| 状态 | 颜色 | 含义 |
|------|------|------|
| 正常 | 绿色 | 实例正在正常运行，可以正常使用 |
| 异常 | 琥珀色 | 实例运行出错，需要关注和处理（蜂巢大屏中会有呼吸闪烁提示） |
| 停止 | 红色 | 实例已停止，无法使用 |

## 蜂巢大屏视图
![龙虾管理-蜂巢大屏](https://skillmarket-prod.nos-jd.163yun.com/uploads/910efe6d-70b2-4f24-911b-0d5352244ded.png)
蜂巢大屏是龙虾管理的默认视图，用六边形蜂巢网格直观展示所有实例的运行状态。

### 页面概览

1. 在左侧菜单「组织管理」分组下，点击「龙虾管理」。
2. 进入页面后，顶部显示实例统计信息和视图切换按钮，下方默认展示蜂巢大屏视图。
### 统计信息

页面顶部显示四项统计标签：

| 标签 | 含义 |
|------|------|
| 共 X 个 | 当前筛选条件下的实例总数 |
| 正常 X | 状态为「正常」的实例数量 |
| 异常 X | 状态为「异常」的实例数量 |
| 停止 X | 状态为「停止」的实例数量 |

统计数字会随筛选条件实时变化。

### 格子颜色

蜂巢中每个六边形格子代表一个龙虾实例，颜色对应实例状态。页面右下角有颜色图例供参考。

- **绿色格子**：正常运行
- **琥珀色格子**（带闪烁动效）：异常，需尽快处理
- **红色格子**：已停止

### 查看实例详情

将鼠标悬停在某个格子上，会弹出深色背景的信息卡片，显示以下内容：

| 字段 | 说明 |
|------|------|
| 实例名称 | 格子顶部的名称（无名称时显示企业编号） |
| 类型标签 | 云端或本地 |
| 部门 | 实例所属的部门 |
| 用户 | 使用该实例的成员 |
| 系统 | 操作系统（macOS / Windows / Linux） |
| 模型 | 实例使用的 AI 模型 |
| 运行 | 实例持续运行时长（如"3天5小时"） |
| 异常原因 | 仅异常状态显示，描述具体错误信息 |
> **提示**：信息卡片底部有"点击查看操作"的提示，点击格子即可打开运维操作面板。

### 执行运维操作

点击某个格子后，会弹出白色背景的运维操作面板，显示实例名称、类型、当前状态，以及可执行的操作按钮。
操作按钮根据实例当前状态动态显示：

| 操作 | 显示条件 | 说明 |
|------|---------|------|
| 启动 | 实例非正常运行时 | 将已停止或异常的实例启动 |
| 停止 | 实例正常运行时 | 将运行中的实例停止 |
| 重启 | 实例非停止状态时 | 重启实例（停止状态下按钮置灰不可用） |
| 删除 | 仅本地实例 | 删除实例（云端实例不支持删除，按钮置灰不可用） |

点击操作面板外任意区域可关闭面板。

# 用量与计费

## 本节导读

用量与计费模块帮助你全面掌握企业的积分消耗情况，包括模型调用和龙虾实例两类消耗的统计分析与明细下钻，以及积分购买记录查询。

---

## 概念准备

> 如果你已经了解积分和计费维度，可以跳过本节。

积分（Credits）是使用平台 AI 服务时消耗的统一计量单位，类似于手机话费。企业在使用平台过程中，积分会通过以下两类方式消耗：

**模型调用**：成员每次与 AI 模型对话时，系统根据对话内容的 Token 数量和所选模型档位的倍率计算消耗的积分。模型档位越高，单次调用的积分消耗越多。

**龙虾实例**：每个龙虾实例只要处于运行状态，每天都会产生固定积分消耗。不同类型的实例日费率不同（云端实例 500 积分/天，本地实例 300 积分/天），停止状态的实例不计费。

所有消耗数据统一以积分展示，不换算为金额（因不同资源包的购买单价不同，换算没有实际参考意义）。消耗数据采用 T+1 更新机制，即今天的数据会在明天展示。

# 审计日志

## 本节导读

审计日志（Audit Log）自动记录你租户内所有管理操作的痕迹，方便事后追溯和问题排查。

**阅读本文后，你将了解**：
- 审计日志覆盖哪些操作类型
- 如何按条件筛选和查询日志
- 如何查看操作详情和变更对比
- 如何将日志导出为 CSV 文件

---

## 概念准备

> 如果你已了解审计日志的作用，可以跳过本节。

审计日志就像一本"操作日记"。当租户管理员在后台执行管理操作时——比如邀请成员、编辑部门、审核 Skill——系统会自动记录谁、在什么时间、做了什么操作、结果如何。

**审计日志的核心特点**：

- 自动记录：你无需手动开启，所有管理操作都会自动留痕
- 只读查询：日志只能查看和导出，不能编辑或删除
- 全租户可见：所有租户管理员都可以查看本租户的全部日志

### 操作分类与类型

审计日志按业务模块分为 7 个分类，共计 38 种操作类型：

| 模块 | 包含的操作 |
|------|-----------|
| 成员管理 | 邀请成员、停用成员、启用成员、移除成员、变更角色、划转部门 |
| 部门管理 | 创建部门、编辑部门、删除部门 |
| 内容生态 | 新建分类、编辑分类、删除分类、新建文档分类、编辑文档分类、删除文档分类、创建文档、编辑文档、删除文档、发布文档、取消发布文档、隐藏平台教程、显示平台教程 |
| 租户设置 | 修改基本信息、修改审核模式、修改 AI 审核策略 |
| 邀请链接 | 生成邀请链接、撤销邀请链接 |
| Skill 审核 | 审核通过、审核拒绝、下架 Skill、批准下架申请、拒绝下架申请、审核通过版本、拒绝版本 |
| 龙虾管理 | 启动龙虾实例、停止龙虾实例、重启龙虾实例、删除龙虾实例 |

# 频道管理

## 本节导读

频道管理模块帮助你管理企业内部的频道内容生态，包括创建和维护领域频道与普通频道、审核小编提交的频道申请、处理小编资格申请，以及配置频道发现页的推荐位。

**阅读本文后，你将了解**：
- 频道与领域频道的关系和层级结构

---

## 概念准备

> 如果你已了解频道和领域频道的概念，可以跳过本节。

频道（Channel）是平台上内容聚合的基本单元，由小编负责维护。每个频道可以关联多个 Skill 和话题，供成员订阅和浏览。

领域频道（Domain Channel）是频道的上层分类，一个领域频道下包含多个普通频道。领域频道由主编负责管理。

两者关系：一个领域频道可以包含多个频道。主编负责整个领域频道（含下属所有频道），小编仅负责单个频道。

| 角色 | 职责范围 |
|------|---------|
| 主编 | 管理所负责的领域频道及下属所有频道 |
| 小编 | 管理所负责的单个频道 |

---

## 频道管理

在左侧「内容生态」分组下，点击「频道管理」进入频道列表页面。

| 主编 / 小编 | 负责人姓名，未分配时显示「未分配」或「无主编」 |
| 订阅者 | 当前订阅人数 |

### 新建领域频道

领域频道是频道的上层分类容器。

1. 点击页面右上角的「+ 新建领域」按钮。
2. 在弹出的对话框中，填写以下信息：
   - **领域名称**（必填）：领域频道的名称
   - **Slug**（必填）：URL 标识，只能包含小写字母、数字和连字符
   - **简介**（选填）：描述该领域的方向
   - **领域图标**（选填）：建议 64x64px，PNG / JPG / SVG，最大 512KB
   - **领域封面**（选填）：推荐 800x320px，JPG / PNG，最大 2MB
   - **主编**（必填）：搜索并选择具有主编资格的用户
3. 点击「创建领域频道」。
### 新建频道

1. 点击页面右上角的「+ 新建频道」按钮。

![新建频道弹窗](https://skillmarket-prod.nos-jd.163yun.com/uploads/136c29c0-f925-4daa-82cc-23c167c79b9b.png)

2. 在弹出的对话框中，填写以下信息：
   - **频道名称**（必填）
   - **Slug**（必填）：URL 标识，只能包含小写字母、数字和连字符
   - **简介**（选填）
   - **频道图标**（选填）：建议 64x64px，PNG / JPG，最大 512KB
   - **频道封面**（选填）：推荐 800x320px，JPG / PNG，最大 2MB
   - **所属领域**（必填）：选择该频道归属的领域频道
   - **频道小编**（必填）：搜索并选择具有小编资格的用户
3. 点击「保存」。
### 编辑频道或领域

在频道列表中，找到目标频道或领域，点击「编辑」按钮。在弹出的对话框中修改信息后，点击「保存」或「保存更改」完成编辑。

### 更换小编或主编

1. 在频道列表中，找到目标频道或领域。
2. 点击「更换小编」或「更换主编」按钮。
3. 在弹出的对话框中，搜索并选择新的负责人。
4. 点击「确认」。
### 归档与恢复

**归档频道**：在频道列表中，点击「归档」按钮。系统弹出确认对话框，确认后频道将归档。归档后频道对订阅者隐藏，但可以随时恢复。

> **警告**：归档领域频道时，系统仅允许归档无关联频道的领域。如果领域下仍有频道，需要先将频道移走或归档。

**恢复频道**：已归档的频道，点击「恢复」按钮即可重新对订阅者展示。

**删除频道**：删除操作不可恢复。系统仅允许删除已归档的频道，已归档的频道行会显示「删除」按钮。

> **危险**：删除频道是不可逆操作，删除后数据无法恢复。

# Skill 审核

## 本节导读

Skill 审核是保障平台上发布内容质量与安全的核心环节。作为租户管理员或运营人员，你需要审核成员提交的 Skill、管理已发布的 Skill，以及处理 Skill 的公开可见和下架申请。

**阅读本文后，你将了解**：
- Skill 审核的整体流程和审核策略配置
- 如何审核新提交的 Skill（通过或拒绝）
- 如何处理新版本、下架申请和可见性变更
- 如何管理 Skill 的公开可见申请

---

## 概念准备

> 如果你已了解 Skill 和审核模式的概念，可以跳过本节。

### 什么是 Skill 审核

当组织内成员发布新的 Skill 或更新已有 Skill 时，内容需要经过审核后才能对其他成员可见。审核的目的在于确保 Skill 的内容安全、质量达标，符合平台规范。

### 审核策略

系统支持两种审核策略，可在 Skill 审核页面顶部进行切换：

- **人工审核**（默认）：系统会自动对 Skill 进行 AI 内容分析和评分，提供结论参考，但最终通过或拒绝由你人工决定。
- **自动审核**：系统按照预设规则自动执行发布或拒绝操作，无需人工介入。你可以针对 AI 结论为「通过」「候选」「拒绝」的三种情况分别配置处理规则。

### 审核类型

| 审核类型 | 含义 |
|---------|------|
| 新 Skill 审核 | 成员首次提交的 Skill，等待审核通过后发布 |
| 新版本审核 | 已发布 Skill 提交了版本更新，新版本通过后自动替换旧版本 |
| 下架申请 | 作者主动申请下架 Skill，等待审批 |
| 变更可见性审核 | 作者申请修改 Skill 的可见范围（如从「仅个人」改为「组织内」） |

---

## 审核列表页面

### 进入页面

在左侧菜单的「内容生态」分组中，点击「Skill 审核」。
### 页面布局

页面顶部是**审核策略**配置区域，下方依次为 Tab 切换栏、搜索栏和内容列表。

### 审核策略配置

页面顶部卡片展示当前审核策略（人工审核或自动审核）。

**切换审核策略**：

1. 在策略卡片中，选择「人工审核」或「自动审核」。
2. 选择「自动审核」后，可以为三种 AI 结论配置自动处理规则：
   - AI 结论为「通过」时：自动发布上架 / 转人工确认
   - AI 结论为「候选」时：转人工确认 / 自动发布上架 / 自动拒绝
   - AI 结论为「淘汰」时：自动拒绝退回 / 转人工确认
3. 点击「保存策略」按钮使配置生效。

> **提示**：自动发布的 Skill 仍可在「已发布」Tab 中手动下架。策略变更后不影响已处理的条目。
### Tab 切换

页面提供七个 Tab，用于查看不同状态的 Skill：

| Tab | 说明 |
|-----|------|
| 待审核 | 新提交的 Skill，等待审核 |
| 新版本审核 | 已发布 Skill 的版本更新，等待审批 |
| 下架申请 | 作者申请下架，等待审批 |
| 变更可见性审核 | 作者申请修改可见范围，等待审批 |
| 已发布 | 已通过审核的 Skill |
| 已拒绝 | 被拒绝的 Skill（含拒绝原因） |
| 已下架 | 已下架的 Skill |

### 搜索和分页

使用搜索栏可以按 Skill 名称或作者名称进行模糊搜索。列表默认每页显示 20 条记录，超过时可在底部使用「上一页」「下一页」按钮翻页。

## 审核通过和拒绝操作

### 审核新 Skill

1. 点击「待审核」Tab，浏览等待审核的 Skill 列表。

2. 每条记录展示以下信息：
   - Skill 名称和状态徽章
   - 简要描述
   - 作者名称和提交时间
   - AI 审核摘要（综合分和 AI 建议）

3. 点击「查看详情」按钮，打开右侧详情抽屉。

![Skill审核详情抽屉](https://skillmarket-prod.nos-jd.163yun.com/uploads/0566e987-e4a4-4a5e-8eb5-79ef449e677c.png)

详情抽屉包含四个子标签：
   - **基本信息**：分类、许可证、可见范围、标签、简介、安装命令等
   - **文件浏览**：查看 Skill 的文件结构和文件内容
   - **AI 审核结果**：查看 AI 给出的综合评分、安全检测、评分维度、内容特征标签等
   - **版本对比**（仅新版本审核时显示）：对比当前版本与待审版本的差异
4. 审核完成后，在列表卡片上执行操作：
   - 点击「通过」按钮：Skill 立即发布上架。
   - 点击「拒绝」按钮：弹出拒绝对话框。

### 拒绝操作

点击「拒绝」后，系统弹出拒绝对话框。

![拒绝Skill弹窗](https://skillmarket-prod.nos-jd.163yun.com/uploads/094e7272-c963-49bb-9a35-29c022659e6a.png)

1. 在「请填写拒绝原因」文本框中输入拒绝理由（10-500 字符，至少 10 个字符）。
2. 点击「确认拒绝」。
> **提示**：拒绝原因会展示给 Skill 作者，建议说明具体问题和改进方向，帮助作者理解拒绝理由。

### 审核新版本

1. 切换到「新版本审核」Tab。

2. 每条记录展示当前发布版本号和待审版本号。

   > **提示**：建议点击「查看详情 & 版本对比」按钮，在版本对比页面中查阅具体变更内容后再审批。新版本通过后自动替换旧版本，已发布版本在此期间继续对用户展示。

3. 点击「通过新版本」或「拒绝」完成审批（拒绝同样需要填写原因）。

### 处理下架申请

1. 切换到「下架申请」Tab。

   > **警告**：申请下架期间，Skill 仍正常对用户展示。批准后立即下架，用户将无法访问。

2. 查看作者的下架原因后，点击「批准下架」或「拒绝申请」。

### 处理可见性变更

1. 切换到「变更可见性审核」Tab。

2. 每条记录展示 Skill 名称、申请理由、当前可见范围和目标可见范围（如从「仅个人可见」变为「组织内可见」）。

3. 点击「批准」或「拒绝」完成审批。拒绝可见性变更时需要填写拒绝原因（可选）。

## Skill 公开可见管理

当需要将组织内的 Skill 公开给全平台用户访问时，需要通过「Skill 管理」页面提交公开申请，经平台管理员审核后生效。

### 进入页面

在左侧菜单的「内容生态」分组中，点击「Skill 管理」。

### 页面概览

页面标题为「Skill 公开管理」，副标题说明为「申请将租户内 Skill 公开给全平台用户访问，需平台管理员审核后生效」。

页面提供两个子标签：

- **公开申请**：查看和管理公开申请记录
- **撤回审批**：处理成员发起的已公开 Skill 撤回申请

### 提交公开申请

1. 点击页面右上角的「提交公开申请」按钮。

2. 在弹出的对话框中：
   - 系统列出所有可申请公开的 Skill（已发布且可见性为「组织内」的 Skill）
   - 勾选需要申请公开的 Skill
   - 填写申请理由（必填，最多 500 字符）
   - 点击「提交申请」
> **警告**：只有获得公开发布权限的租户才能提交公开申请。如果未开通权限，对话框中会显示提示信息，需联系平台管理员开通。

在「公开申请」Tab 下，可以通过状态筛选查看不同阶段的申请：

| 状态 | 说明 |
|------|------|
| 待审核 | 已提交，等待平台管理员审核 |

| 已拒绝 | 平台拒绝，可点击「重新申请」再次提交 |
| 已撤回 | 你主动撤回了申请 |
| 已下架 | Skill 已被平台下架 |

每条记录提供以下操作：
- **撤回**（待审核状态）：撤回已提交的申请，Skill 保持组织内可见
- **重新申请**（已拒绝/已撤回/已下架状态）：修改理由后重新提交
- **详情**：查看申请的完整信息

### 处理撤回审批

在「撤回审批」Tab 下，查看组织内成员发起的公开 Skill 撤回申请。

每条记录展示 Skill 名称、目标可见性（撤回后的可见范围）和申请理由。

操作方式：
- 点击「批准」：同意撤回，申请提交至平台管理员进行二次审批。平台管理员批准后，全平台用户将无法再访问此 Skill。
- 点击「拒绝」：拒绝撤回，Skill 保持公开状态。需填写拒绝原因（必填，最多 200 字符）。

# 分类管理

## 本节导读

分类用于将企业内部的 Skill 按功能领域进行归类，帮助成员快速找到所需的 Skill。本节介绍如何新增、编辑、删除和排序 Skill 分类。

**阅读本文后，你将了解**：
- 分类的作用和适用场景
- 如何新增、编辑和删除分类
- 如何调整分类的显示顺序
- 删除分类对已关联 Skill 的影响

---

## 概念准备

> 如果你已了解 Skill 分类，可以跳过本节。

**分类（Category）** 是企业内部的 Skill 归类方式。当企业发布了大量 Skill 时，通过分类可以让成员按类别快速查找，提升使用效率。

**租户分类与平台分类是两套独立的体系**。租户分类由企业管理员自行创建和维护，仅在企业内部生效；平台分类由平台统一管理，面向所有用户公开。两者的分类名称、数量和排序互不影响。

---

## 分类列表

进入分类管理页面后，页面以表格形式展示当前租户的所有 Skill 分类。

### 查看分类列表

1. 在左侧菜单中，点击「分类管理」。

![分类管理列表页](https://skillmarket-prod.nos-jd.163yun.com/uploads/8d904064-2648-4504-b831-60be5491b4e4.png)

2. 表格包含以下列：

   | 列名 | 说明 |
   |------|------|
   | 图标 | 分类关联的图标，用于前端展示 |
   | 名称 | 分类的显示名称，如「代码开发」 |
   | Slug | 分类的英文标识符，用于 URL 和系统内部识别，如 `code-development` |
   | 描述 | 对该分类的简要说明 |
   | Skills | 已归入该分类的 Skill 数量 |
   | 排序 | 上下箭头按钮，用于调整分类的显示顺序 |
   | 操作 | 编辑和删除按钮 |

当租户内还没有任何分类时，表格区域会显示"暂无分类"的提示文字，你可以通过页面右上角的「新增分类」按钮创建第一个分类。

---

## 新增分类

1. 在分类管理页面，点击右上角的「新增分类」按钮。

2. 在弹出的对话框中，填写以下信息：

   | 字段 | 是否必填 | 说明 |
   |------|---------|------|
   | 名称 | 必填 | 分类名称，如「数据分析」「代码开发」 |
   | Slug | 必填 | 英文标识符，只能包含小写字母、数字和横线（如 `data-analysis`） |
   | 描述 | 选填 | 对该分类包含的 Skill 类型的简短描述 |
   | 图标 | 选填 | Lucide 图标名称，如 `Code`、`Database`、`Folder`，用于前端展示 |
3. 填写完成后，点击「创建」按钮。

**操作结果**：系统创建分类成功，页面右下角弹出"分类已创建"提示，新分类出现在列表末尾。

> **提示**：Slug 是分类的唯一英文标识符，填写后不可轻易修改。建议在创建前确认好命名，使用小写字母、数字和横线的组合。

---

## 编辑分类

1. 在分类列表中，找到目标分类所在行。

2. 点击操作列中的编辑按钮（铅笔图标）。
3. 在弹出的「编辑分类」对话框中，修改需要更新的字段。

4. 点击「保存」按钮。

**操作结果**：系统保存修改成功，页面右下角弹出"分类已更新"提示。

> **提示**：修改分类名称不会影响已归入该分类的 Skill，Skill 仍保持原有的分类归属。

---

## 删除分类

删除分类是一个不可逆操作，执行前系统会弹出确认对话框。

1. 在分类列表中，找到目标分类所在行。

2. 点击操作列中的删除按钮（垃圾桶图标）。

![删除分类确认弹窗](https://skillmarket-prod.nos-jd.163yun.com/uploads/faec112d-5cd9-4af5-9c4f-081e25045b91.png)

3. 在弹出的确认对话框中，阅读提示信息。对话框会显示：

   - 即将删除的分类名称
   - "该操作不可撤销"的警告说明
   - 关于已关联 Skill 的处理说明
4. 点击「删除」按钮确认删除。

> **危险**：删除分类后无法恢复。如果后续需要使用同名的分类，需要重新创建。

> **警告**：删除分类后，已归入该分类的 Skill 不会被删除，但 Skill 的分类归属会被清空。如果这些 Skill 仍需要分类，请在删除后手动为它们重新分配分类。

---

## 排序分类

分类的显示顺序决定了成员在浏览 Skill 分类列表时看到的排列次序。你可以通过上下箭头按钮调整顺序。

1. 在分类列表中，找到「排序」列。
2. 点击向上箭头，将该分类上移一位。
3. 点击向下箭头，将该分类下移一位。

排序规则如下：

- 列表第一行的分类，向上箭头不可点击（已处于最顶部）
- 列表最后一行的分类，向下箭头不可点击（已处于最底部）
- 每次点击移动一位，如需大幅调整顺序，需要多次点击

---

## 注意事项

> **提示**：建议在 Skill 发布前就规划好分类体系，这样发布 Skill 时可以直接选择已有分类，减少后期整理的工作量。

> **警告**：Slug 字段只能包含小写字母、数字和横线，不能包含中文、空格或大写字母。填写时请仔细检查格式。

> **警告**：如果分类的 Skills 列显示的数量大于 0，说明已有 Skill 归入该分类。删除前请确认是否需要先为这些 Skill 重新分配分类。

# 文档管理

## 本节导读

文档管理模块帮助你为企业内部成员创建和维护教程文档，同时支持管理平台提供的公共教程的可见性。

**阅读本文后，你将了解**：
- 如何创建和编辑教程文档
- 如何管理文档分类
- 如何控制平台教程在工作区内的显示与隐藏

---

## 1. 页面概览

在左侧菜单中点击「文档管理」，进入文档管理页面。页面顶部显示三个选项卡：

- **文档列表**：查看和管理本租户创建的文档
- **分类管理**：管理文档的分类目录
- **平台教程**：控制平台公共教程在本工作区内的可见性

![文档管理-文档列表页面](https://skillmarket-prod.nos-jd.163yun.com/uploads/3b09f3d1-100a-4e1d-a1be-cea09a857508.png)

---

## 2. 文档列表

### 2.1 列表字段说明

切换到「文档列表」选项卡后，页面以表格形式展示所有文档。表格包含以下列：

| 列名 | 说明 |
|------|------|
| 标题 | 文档的标题名称 |
| 分类 | 文档所属的分类名称 |
| 来源 | 「全局」表示平台创建的公共文档，「本租户」表示你创建的私有文档 |
| 状态 | 「已发布」表示成员可查看，「草稿」表示尚未公开 |
| 更新时间 | 文档最近一次修改的日期 |
| 排序 | 调整文档的显示顺序（仅本租户文档可排序） |
| 操作 | 对文档进行编辑、发布/撤回、删除等操作 |

> **提示**：来源为「全局」的文档由平台维护，不支持编辑和删除操作。
### 2.2 发布文档

将草稿文档发布后，工作区成员即可在前台教程页面查看。

1. 在文档列表中找到目标文档，点击操作列的「发布」按钮。

2. 在确认弹框中查看文档标题，点击「确认发布」。

3. 文档状态变为「已发布」，页面右上角出现提示。

### 2.3 撤回文档

将已发布的文档撤回后，工作区成员将无法再查看该文档。

1. 在文档列表中找到目标文档，点击操作列的「撤回」按钮。

2. 在确认弹框中点击「确认撤回」。

3. 文档状态变为「草稿」。

> **警告**：撤回后，已收藏或已打开该文档的成员将无法继续访问。

### 2.4 删除文档

1. 在文档列表中找到目标文档，点击操作列的删除图标。

2. 在确认弹框中点击「删除」。

> **危险**：删除操作不可撤销。删除后文档内容将永久丢失，无法恢复。

### 2.5 调整文档排序

1. 在文档列表中找到目标文档（仅来源为「本租户」的文档支持排序）。
2. 点击「排序」列中的上箭头或下箭头。
3. 文档会与相邻项交换位置，列表顺序即时更新。

> **提示**：排序调整后，前台教程页面的文档列表会同步更新显示顺序。全局文档不支持排序调整。

## 3. 新建与编辑文档

![文档管理-文档编辑页面](https://skillmarket-prod.nos-jd.163yun.com/uploads/4dba0964-2ac0-4c05-b20a-6d11b58521c3.png)

### 3.1 新建文档

1. 在「文档列表」选项卡中，点击页面右上角的「新建文档」按钮。
2. 在弹出的编辑器中填写以下信息：

   | 字段 | 是否必填 | 说明 |
   |------|---------|------|
   | 标题 | 是 | 文档的标题名称 |
   | Slug | 是 | URL 友好的标识符，输入标题后系统会自动生成，你也可以手动修改 |
   | 分类 | 是 | 从下拉列表中选择一个文档分类。标注「(全局)」的为平台分类 |
   | 内容 | 是 | 文档正文，使用 Markdown 格式编写 |
3. 在内容编辑区域，左侧为 Markdown 编辑器，右侧为实时预览。输入内容后，右侧会同步显示排版效果。
4. 如果已有 Markdown 文件，可以点击右上角的「上传 .md 文件」按钮直接导入内容。

5. 填写完成后，点击底部按钮：
   - 「保存草稿」：暂不发布，后续可在列表中找到并继续编辑
   - 「发布」：直接发布，成员立即可以查看
### 3.2 编辑文档

1. 在文档列表中找到目标文档，点击操作列的编辑图标。

2. 系统会加载文档的完整内容，在编辑器中打开。

3. 修改完成后，根据当前文档状态，底部按钮会有所不同：

   | 当前状态 | 可用按钮 |
   |---------|---------|
   | 草稿 | 「保存草稿」和「发布」 |
   | 已发布 | 「转为草稿」和「保存更新」 |

> **提示**：编辑已发布的文档并点击「保存更新」后，修改内容会立即对成员生效，无需重新发布。

# 运营数据

## 本节导读

运营数据页面帮助你了解企业成员使用平台的情况，包括成员活跃度、Skill 的发布与使用热度、以及各部门的安装覆盖和使用频次。

**阅读本文后，你将了解**：
- 运营数据与用量与计费的区别
- 页面上六张 KPI 概览卡片的含义
- Skill 分类分布、热门排行榜和贡献作者排行
- 成员活跃漏斗、使用频次分层和高频使用人员排行
- 各部门的安装覆盖和使用分布情况

---

## 概念准备

> 如果你已了解 ClawHive 和 Skill 的概念，可以跳过本节。

### 运营数据 vs 用量与计费

左侧菜单「资源与计费」分组下有两个功能入口：运营数据和用量与计费。两者定位不同：

| 维度 | 运营数据 | 用量与计费 |
|------|---------|-----------|
| 关注点 | **产品使用质量** | 积分消耗与账单 |
| 核心问题 | 成员用得好不好？Skill 热不热？ | 积分花在哪里了？还剩多少？ |
| 典型数据 | 活跃用户数、Skill 下载量、调用次数 | 积分消耗趋势、模型消耗分布、购买记录 |
| 适用场景 | 推动内部推广、发现高价值 Skill | 成本控制、预算规划 |

简单来说：**运营数据看"人"和"内容"，用量与计费看"钱"**。

---

## 功能说明

### 进入运营数据页面

1. 在左侧菜单中，点击「资源与计费」分组下的「运营数据」。
2. 页面加载后，顶部显示页面标题「运营数据」和一句灰色提示文字"数据 T+1 更新，每天凌晨 2:00 刷新"。

> **提示**：运营数据不是实时数据，而是前一天的数据汇总。例如周三看到的数据截止到周二结束。
![运营数据-概览页](https://skillmarket-prod.nos-jd.163yun.com/uploads/0ff48511-7f1e-47f9-ac72-694b42d12277.png)

# 运维管理

## 本节导读

运维管理模块用于向龙虾实例批量下发运维命令，帮助你高效完成实例维护、技能安装等操作。

**阅读本文后，你将了解**：
- 如何通过四步流程创建并下发运维任务
- 如何查看任务执行历史和每个实例的执行结果
- 如何创建和管理脚本模板
- 新增模板时的参数定义和指令编写方法

---

## 概念准备

> 如果你已了解脚本模板和下发任务，可以跳过本节。

运维管理菜单位于左侧菜单「系统」分组下，由三个子功能组成：

| 子功能 | 说明 |
|--------|------|
| 新建任务 | 选择模板、填写参数、选择实例，批量下发运维命令 |
| 任务历史 | 查看所有已下发任务的执行状态和结果 |
| 脚本模板 | 管理预置的运维操作模板，定义命令和参数 |

**脚本模板**是运维操作的核心。一个模板定义了一类运维命令（如安装 Skill、更新配置），包含命令模板和参数定义。下发任务时，你选择一个模板，填写参数并指定目标实例，系统会自动将参数渲染到命令中，然后批量下发给所有选中实例。
任务下发后，所有实例**并发执行**，单个实例的失败不会影响其他实例。如果部分实例失败或超时，你可以一键重试。

---

## 新建运维任务

新建任务是一个四步流程：选择模板 → 填写参数 → 选择实例 → 确认执行。页面顶部有步骤进度条，显示当前所在的步骤。

### 第一步：选择模板

1. 在左侧菜单中，点击「运维管理」→「新建任务」。
2. 在模板列表中，点击要使用的模板卡片。选中后卡片边框变为红色，显示「已选择」。

3. 如果模板较多，可以使用顶部的搜索框，按模板名称或描述进行搜索。

> **提示**：只有「已启用」状态的模板才会出现在列表中。如果找不到需要的模板，请先到「脚本模板」中创建或启用。

![新建任务-选择模板页面](https://skillmarket-prod.nos-jd.163yun.com/uploads/f32c4320-b70e-4d28-b21f-7532625f9f7c.png)

### 第二步：填写参数

选中模板后，页面左侧显示参数表单，右侧实时预览生成的命令。
1. 根据模板定义的参数，逐一填写。

   - **字符串类型**：直接在输入框中输入内容
   - **技能选择类型**：点击输入框搜索并选择一个 Skill
   - **技能多选类型**：搜索并选择多个 Skill，已选中的 Skill 以标签形式显示

2. 右侧「生成的指令预览」区域会根据你填写的参数实时更新命令内容。你可以在这里确认最终下发的命令是否正确。

> **提示**：标记了红色星号（*）的参数为必填项，必须填写才能进入下一步。

![新建任务-填写参数页面](https://skillmarket-prod.nos-jd.163yun.com/uploads/dd80c52c-526c-4399-bf40-a9ad82708011.png)

### 第三步：选择实例

1. 在实例列表中，勾选需要下发命令的实例。
2. 使用筛选功能快速定位实例：
   - 搜索框：按实例名称、IP 地址或归属用户名搜索
   - 状态筛选：运行中 / 已停止
   - 类型筛选：本地 / 云端
   - 操作系统筛选

3. 列表底部显示已选实例数量。如果选中的实例中包含「已停止」状态的实例，系统会提示这些实例可能因无法连接而超时失败。

> **提示**：已停止的实例行背景为橙色，方便你识别。

### 第四步：确认执行

确认页展示任务摘要，包括使用的模板、执行命令和目标实例列表。
1. 核对以下信息：
   - **操作模板**：确认模板名称正确
   - **执行命令**：在终端风格的黑色区域中查看完整命令
   - **目标实例**：确认实例列表和数量无误
   - **执行策略**：所有实例并发执行，超时阈值为 30 秒

2. 确认无误后，点击右下角的「确认执行」按钮。

3. 任务创建成功后，系统自动跳转到任务详情页面。

> **危险**：点击「确认执行」后，命令将立即下发到所有选中实例，操作不可撤销。请务必在确认页仔细核对命令内容和目标实例。

---

## 查看任务历史

### 任务列表

1. 在左侧菜单中，点击「运维管理」→「任务历史」。
2. 列表展示所有已创建的任务，每行包含以下信息：

   | 列 | 说明 |
   |----|------|
   | 任务 ID | 系统生成的唯一编号 |
   | 模板 | 所使用的脚本模板名称 |
   | 操作人 | 创建该任务的管理员 |
   | 执行时间 | 任务创建时间 |
   | 实例数 | 目标实例数量 |
   | 下发结果 | 命令下发的成功/失败/超时统计 |
   | 执行结果 | 命令实际执行后的成功/失败统计（仅同步模式有意义） |
   | 操作 | 点击「查看详情」进入任务详情页 |

3. 使用顶部筛选器搜索和过滤任务：
   - 搜索框：按模板名称或任务号搜索
   - 状态筛选：执行中 / 全部成功 / 部分失败 / 全部失败

### 任务详情

点击任务列表中的「查看详情」，进入任务详情页。
1. 页面顶部展示任务摘要：
   - 模板名称和任务状态标签
   - 任务编号、创建时间、操作人
   - 如果任务存在失败或超时实例，会显示「重试失败实例」按钮

2. **下发状态**统计：总计、下发成功、下发失败、超时。

3. **执行状态**统计（仅同步模式下有意义）：执行成功（退出码为 0）、执行失败（退出码非 0）、无输出。

4. 页面下方的实例结果列表展示每个实例的下发情况：
   - 实例编号和归属用户
   - 执行耗时
   - 下发状态（等待/执行中/成功/失败/超时）
   - 执行状态（成功或退出码）
   - 点击「查看结果」可在弹框中查看标准输出和标准错误

> **提示**：执行中的任务页面会每 2 秒自动刷新，无需手动刷新。

### 重试失败实例

如果任务完成后存在失败或超时的实例：

1. 在任务详情页右上角，点击「重试失败实例」按钮。

2. 系统会重新向之前失败或超时的实例下发命令。

## 脚本模板管理

### 模板列表

1. 在左侧菜单中，点击「运维管理」→「脚本模板」。
2. 页面展示所有已创建的模板卡片，每个卡片显示：
   - 模板名称和图标
   - 启用/禁用状态标签（内置模板会额外显示「内置」标签）
   - 模板描述
   - 参数数量

3. 点击模板卡片可以展开查看：
   - 参数定义：每个参数的名称、类型、是否必填
   - 指令模板：完整的命令模板内容

4. 模板操作：
   - 点击「编辑」进入编辑页面
   - 非内置模板可以点击「禁用」或「启用」切换状态

### 新增模板

1. 在模板列表页右上角，点击「新增模板」。
2. 填写**基础信息**：
   - **模板名称**（必填）：如「Skill 下发」
   - **图标**：从预设图标（扳手、终端、设置、工具）中选择，或直接输入 emoji 表情
   - **下发模式**：同步（等待执行完成并获取输出）或异步（下发后立即返回）
   - **描述**：简要说明模板用途

3. 在左侧**参数定义**区域，定义模板需要的参数：
   - 点击「添加参数」新增一行
   - 每个参数需设置：参数名（必填）、显示名称、类型、是否必填
   - 参数类型支持三种：字符串、技能选择、技能多选

4. 在右侧**指令配置**区域，编写命令模板：
   - 使用 `{{变量名}}` 引用字符串参数
   - 使用 `{{变量名.字段}}` 引用技能参数的字段（如 `{{skill.installCommand}}`）
   - 使用 `{% for s in 变量名 %}...{% endfor %}` 遍历技能多选数组
   - 系统会自动进行语法检查，并在下方实时显示预览效果

5. 右上角的开关控制模板是否启用。点击「创建模板」完成创建。

### 编辑和删除模板

- **编辑**：在模板列表中点击「编辑」，修改后点击「保存更改」
- **删除**：仅非内置模板可删除。在编辑页面底部点击「删除模板」，确认后删除。删除后无法恢复，但历史任务记录不受影响

> **警告**：内置模板不能删除，也不能通过列表页面的按钮禁用。如需调整内置模板，需进入编辑页面操作。

---

## 注意事项

> **提示**：建议先选择少量实例进行测试，确认命令正确后再全量下发。

> **警告**：已停止的实例无法接收命令，下发时会因超时而失败。如果实例状态显示「已停止」，请在「龙虾管理」中先启动实例。

> **危险**：任务确认执行后立即下发，不可撤销。请务必在第四步确认页仔细核对命令内容和目标实例列表。
