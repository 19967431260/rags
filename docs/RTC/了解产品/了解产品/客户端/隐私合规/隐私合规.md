实时音视频应用在处理用户数据时，必须确保严格遵守隐私保护法规。国家对个人隐私保护的政策要求日益严格，为保证您的音视频应用能顺利上架应用商店，需要特别注意数据采集的合规性。

## 背景信息

根据《个人信息安全规范》（GB/T 35273-2020）的相关要求，实时音视频应用在收集用户麦克风、摄像头等敏感权限信息时，必须向用户明确说明信息处理的目的、方式和范围，并提供完整的《隐私政策》。

您需确保《隐私政策》易于访问，用户在应用的主界面通过 **四次** 以内的操作即可查看。此外，应用必须在用户明确同意《隐私政策》后，才能开始收集摄像头、麦克风等隐私敏感信息。相关详情请参考 [《信息安全技术-个人信息安全规范》](http://openstd.samr.gov.cn/bzgk/gb/newGbInfo?hcno=4568F276E0F8346EB0FBA097AA0CE05E)。

## 第一步：完善隐私政策文件

:::note notice
强烈建议在音视频应用首次启动时，通过弹窗清晰展示隐私政策概要，并提供《隐私政策》的完整链接。特别说明使用摄像头、麦克风、通讯录等敏感权限的目的和用途。用户选择 **同意** 后，应用才能请求相关权限并采集音视频数据。
:::

1. 确保您的音视频应用已提供完整《隐私政策》，并在其中添加如下声明：

    ```
    本应用提供实时音视频通话服务，集成了网易云信 SDK。网易云信将作为数据处理者，按照本公司的指示以及《网易云信隐私政策》（https://yunxin.163.com/clauses?serviceType=3）收集、使用并保护您的麦克风、摄像头等音视频相关个人信息。这些信息仅用于提供实时音视频通话功能，不会用于其他目的。
    ```

2. 如果您的音视频应用有单独的《第三方共享清单》，建议添加以下信息：

    ```
    RTC_Android
    第三方公司名称：杭州网易质云科技有限公司
    共享类型：内嵌SDK
    共享信息：设备信息，包含设备类型、设备名称及型号、设备所运行系统版本信息、AndroidID；日志信息，包含设备运行状态（CPU、内存运行状态）、浏览器类型及版本。
    使用目的：保障云信音视频服务的运行安全和服务体验，了解产品适配度，以便对音视频服务质量进行适配调优，同时为了预防安全风险，准确识别违反法律法规的情况。
    使用场景：音视频通话、多人会议、直播互动等场景，提供音视频通话服务
    共享方式：（外部）接口共享
    第三方隐私链接：https://netease.im/clauses?serviceType=3
    ```

## 第二步：延迟初始化

由于初始化 NERTC SDK 操作，有部分隐私相关 API 需要在用户同意之后才被允许调用。所以，首次启动时，请确保用户阅读您的《隐私政策》并取得用户授权之后，再调用 RTC 的初始化函数。以安卓为例，NERTC SDK 初始化函数 [init](https://doc.yunxin.163.com/nertc/references/android/doxygen/Latest/zh/html/classcom_1_1netease_1_1lava_1_1nertc_1_1sdk_1_1_n_e_rtc.html#a2198fb57cd127a8ae136e1f450236e57) 如下：

```Java
NERtcEx.getInstance().init(getApplicationContext(),Config.APP_KEY,callback,null);
```
