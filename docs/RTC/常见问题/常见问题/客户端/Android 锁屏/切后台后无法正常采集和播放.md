## 问题现象

在使用网易云信音视频功能时，您可能会遇到以下情况：

- **锁屏场景**：Android 设备进入锁屏状态后，短时间内（约 1 分钟）出现远端用户音频无法听到或视频画面无法显示。
- **后台场景**：应用在运行过程中切换到后台时，远端用户的音频播放中断或完全无声。

## 问题原因

该问题主要源于 Android 系统在不同版本中对后台应用权限的收紧策略：

- **Android 9 (API 28) 及更高版本**：系统开始限制后台应用对用户敏感数据（如麦克风、摄像头）的访问权限，以提升用户隐私保护。

    📖 详情参考： [Android 9 行为变更](https://developer.android.com/about/versions/pie/android-9.0-changes-all)

- **Android 14 (API 34) 及更高版本**：系统进一步要求应用 **必须明确声明前台服务的类型**。当应用的 `targetSdkVersion` 设置为 34 或更高时，在 Android 14 设备上运行时会进行严格的兼容性校验。若应用未按照规范配置前台服务类型，在切换到后台时系统会限制相关功能的使用，导致麦克风无法采集、远端音频播放失败等情况。

    📖 详情参考：
    - [Android 14 行为变更](https://developer.android.com/about/versions/14/behavior-changes-14) 
    - [必需的前台服务类型](https://developer.android.com/about/versions/14/changes/fgs-types-required)。

## 解决方案

您需要按照 Android 系统的要求正确配置前台服务，确保应用在锁屏或后台状态下仍能持续进行音视频采集和播放，避免因系统策略导致的功能中断。

### 步骤一：配置 AndroidManifest.xml

首先需要在应用清单文件中添加必要的前台服务权限声明，并为服务组件指定对应的服务类型。对于需要持续进行音视频交互的场景，建议同时声明麦克风、摄像头和媒体播放三种服务类型，以覆盖完整的音视频功能需求。配置示例如下：

```
<manifest ...>
    <!-- 声明前台服务权限 -->
    <uses-permission android:name="android.permission.FOREGROUND_SERVICE" />
    <uses-permission android:name="android.permission.FOREGROUND_SERVICE_MICROPHONE" />
    <uses-permission android:name="android.permission.FOREGROUND_SERVICE_CAMERA" />
    <uses-permission android:name="android.permission.FOREGROUND_SERVICE_MEDIA_PLAYBACK" />
    <application ...>
        ...
        <!-- 指定前台服务类型 -->
        <service
            android:name=".YourForegroundService"
            android:foregroundServiceType="microphone|camera|mediaPlayback" />
        ...
    </application>
</manifest>
```

### 步骤二：在服务初始化时启动前台服务

服务创建后，需要立即将其提升为前台服务。通过调用 `startForeground` 方法并传入通知对象，让系统识别该服务正在执行用户可见的重要任务。

针对不同 Android 版本，需要采用不同的启动方式，具体实现如下：

```java
@Override
public void onCreate() {
    super.onCreate();
    // 获取默认的通知
    Notification notification = getDefaultNotification();
    try {
        // 根据 Android 版本，选择合适的方式处理前台服务
        if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.R) {
            // 对于 Android 11 及以上版本，启动前台服务并指定多种服务类型
            int serviceTypes = ServiceInfo.FOREGROUND_SERVICE_TYPE_MICROPHONE |
                            ServiceInfo.FOREGROUND_SERVICE_TYPE_CAMERA |
                            ServiceInfo.FOREGROUND_SERVICE_TYPE_MEDIA_PLAYBACK;
            this.startForeground(NOTIFICATION_ID, notification, serviceTypes);
        } else {
            // 对于 Android 11 以下版本，无需指定服务类型，简单地启动前台服务即可
            this.startForeground(NOTIFICATION_ID, notification);
        }
    } catch (Exception ex) {
        Log.e(TAG, "Error starting foreground service", ex);
    }
}
```

### 步骤三：监听应用生命周期并启动后台服务

为确保应用进入后台时音视频功能不受影响，需要在应用生命周期回调中主动启动前台服务。当检测到应用即将进入后台状态（如 `onPause` 被调用）时，应启动相应的前台服务以维持功能正常运行。实现方式如下：

```java
@Override
public void onPause() {
    super.onPause();
    startRecordingService();
}

private void startRecordingService() {
    if (joined) {
        // 创建一个 intent
        Intent intent = new Intent(requireContext(), LocalRecordingService.class);
        // 根据 Android 版本，选择合适的方式启动前台服务
        if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.O) {
            // 对于 Android 8 及以上版本，使用 startForegroundService 来启动服务
            requireContext().startForegroundService(intent);
        } else {
            // 对于 Android 8 以下版本，使用 startService 来启动服务
            requireContext().startService(intent);
        }
    }
}
```
