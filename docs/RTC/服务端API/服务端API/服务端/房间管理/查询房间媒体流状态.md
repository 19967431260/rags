查询房间中各用户的媒体流状态，包括是否已静音等信息。例如，通过将获取到的媒体流状态与业务侧用户麦位状态进行对比，可以找出不应该推流的用户。该接口适用于“防漏麦”和“防炸麦”场景。

## <span id="URL">URL</span>

- 请求方法：GET
- URL：

    ```
    GET https://rtc.yunxinapi.com/v2/api/rooms/{cid}/streams
    GET https://rtc.yunxinapi.com/v3/api/rooms/streams?cname={cname}
    ```

## 接口请求频率

默认上限为 5 次/秒，若请求频率超出限制，可能会返回 429 错误码。<br>若您需要上调上限，请参考<a href="https://doc.yunxin.163.com/nertc/server-apis/Tg3MDU1NTE?platform=server#如何处理调用服务端-restful-api-超出频率限制" target="_blank">如何处理调用服务端 RESTful API 超出频率限制</a>。

## <span id="错误码">错误码</span>

| 状态码 | 说明 |
|---|---|
| 200 | API 调用成功。 |
| 400 | 请求参数错误，可能原因是：cid非法。 |
| 401 | 接口鉴权失败。请检查请求 header 中的 CheckSum 字段是否正确。 |
| 404 | cid 对应的房间不存在。 |
| 429 | 请求个数超过频控限制。 |
| 500 | 调用失败，可能原因是房间已关闭。 |
