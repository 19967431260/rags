本文主要介绍如何实现旁路推流。

## <span id="注意事项">注意事项</span>

- 如果需要在互动直播场景中使用 SEI（Supplemental Enhancement Information）功能，在服务端创建推流任务时，需要通过 hostuid 设置主播的 UID。互动直播场景中，仅支持解析主播端的 SEI 自定义数据。
- 同一个音视频房间（即同一个 channelId）可以创建 6 个不同的推流任务。
- 服务端<a href="https://doc.yunxin.163.com/nertc/server-apis/DM0MTg2NTM" target="_blank">创建旁路推流任务</a>接口的 <a href="https://doc.yunxin.163.com/nertc/server-apis/jc2MjQzMzA?platform=server#layout" target="_blank">`layout`</a> 参数为必选参数。纯音频直播场景下，无需设置用户画面布局，可以将 users 中 x、y、width、height 等参数设置为 0。

## <span id="实现方式">实现方式</span>

实现音视频通话的相关流程之后，可以调用服务端接口<a href="https://doc.yunxin.163.com/nertc/server-apis/DM0MTg2NTM?platform=server" target="_blank">创建旁路推流任务</a>发起推流任务。

### 配置推流任务信息

创建推流任务时，需要配置任务 ID、推流地址等推流参数信息，还可以开启直播视频录制。

录制的视频默认存储在点播服务中，您可以通过点播的相关接口查看并下载视频文件。详细信息请参考<a href="/docs/jY3NDM4Nzc/jI0Njk0NzU" target="_blank">点播媒资管理</a>。

推流任务信息相关参数说明：

| 参数名称 | 描述 |

| extraInfo | 自定义的媒体补充增强信息。 |


### 配置推流布局

通过 `layout` 参数可以设置互动直播的画面布局，即自定义房间画面的各路视频布局方式，例如调整画布（Canvas）大小和颜色、各路视频的图像大小、位置等。

当房间中只有一路视频流时，您也可以设置视频透传。

- 详细参数说明请参考 <a href="https://doc.yunxin.163.com/nertc/server-apis/jc2MjQzMzA?platform=server#layout" target="_blank">`layout`</a>。
- 布局参数的配置方式及典型配置示例请参见<a href="https://doc.yunxin.163.com/nertc/server-apis/zI4MTk5NTU" target="_blank">旁路推流画面布局</a>。

### 音视频流配置

<a href="https://doc.yunxin.163.com/nertc/server-apis/jc2MjQzMzA?platform=server#config" target="_blank">`config`</a> 参数用于配置音视频流编码参数等设置。

- 通过 `singleVideoNoTrans` 参数设置视频透传。

    `singleVideoNoTrans` 参数设置为 true 表示开启视频透传，开启后，如果房间中只有一路视频流输入，则不对输入视频流进行转码，不遵循转码布局，直接推流 CDN。视频透传默认为关闭状态。

- 通过 `subAllAudio` 参数设置音频流订阅。

    `subAllAudio` 参数设置为 true 表示互动直播中订阅所有用户的音频流，房间中所有成员的音频流均会被混流后推往 CDN。

- 通过 `audioParam` 参数设置音频编码参数。

    - 通过 `bitRate` 字段设置音频推流码率。单位为 kbps，取值范围为 10~192。语音场景建议设置为 64 及以上码率，音乐场景建议设置为 128 及以上码率。
    - 通过 `sampleRate` 字段设置音频推流采样率。单位为Hz，默认采样率为 48 kHz。
    - 通过 `channels` 字段设置音频推流声道数，默认为2，即双声道。
    - 通过 `codecProfile` 字段设置音频编码规格，默认为0，即 LC-AAC 规格，表示基本音频编码规格。
