<!--keywords: 音色重克隆, 语音合成, TTS, 音色更新, 声音文件上传-->

音色重克隆用于对试听结果不满意的场景，可以指定原有的音色 ID 发起重克隆，更新克隆音色，总的克隆次数不得超过 10。

::: note notice
- **支持的音频格式**：mp3、m4a、wav。
- **文件大小限制**：10M 以内。
- **总克隆次数限制**：不得超过 10 次。
- 需要使用 form-data 格式提交请求。
:::

## 请求信息

### 请求 URI

```
POST https://rtc-ai.yunxinapi.com/v1/api/tts/voice-reclone
```

### 请求头参数

请求 Header 的参数说明请参考 [请求结构](https://doc.yunxin.163.com/nertc/server-apis/TYyNjA3MzQ?platform=server#%E8%AF%B7%E6%B1%82%E5%A4%B4header)。

### 请求体参数

| 参数名称 | 类型 | 是否必选 | 示例 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| file | File | 是 | voice_sample.mp3 | 需要上传的声音文件，目前支持 mp3、m4a、wav 格式，大小在 10M 以内 |
| voiceId | String | 是 | GZ_6FXfkjpzhnZI1 | 需要重克隆的音色 ID |
| desc | String | 否 | 睿智的导游音色 | 音色描述，一般用作音色 ID 的别名，方便记忆 |
| textValidation | String | 否 | 这段时间天气真不错。你今天过得怎么样呢？咱们晚上可以一起去散散步，听听音乐，吹吹风，一起聊聊今天发生的新鲜事。 | 音频复刻校验音色的参数，未填时使用示例里的文本做校验 |

### 请求体示例

```shell
curl -X POST https://rtc-ai.yunxinapi.com/v1/api/tts/voice-reclone \
  -H "Content-Type: multipart/form-data" \
  -F "file=@voice_sample.mp3" \
  -F "voiceId=GZ_6FXfkjpzhnZI1" \
  -F "desc=睿智的导游音色" \
  -F "textValidation=这段时间天气真不错。你今天过得怎么样呢？咱们晚上可以一起去散散步，听听音乐，吹吹风，一起聊聊今天发生的新鲜事。"
```

## 响应信息

### 响应参数

| 参数名称 | 类型 | 说明 | 是否必返回 |
| :--- | :--- | :--- | :--- |
| code | Number | 状态码，200 表示成功。出错时，错误信息在 errMsg 中 | 是 |
| voiceId | String | 音色 ID | 是 |
| demoAudio | String | 试听音色链接 | 是 |
| errMsg | String | 错误信息描述 | 否 |

### 响应体示例

```JSON
{
    "code": 200,
    "voiceId": "GZ_6FXfkjpzhnZI1",
    "demoAudio": "https://yunxin-vc.nos-jd.163yun.com/146298631/GZ_6FXfkjpzhnZI1.wav"
}
```
