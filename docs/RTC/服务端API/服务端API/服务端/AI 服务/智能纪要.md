调用该接口开启智能纪要任务。该任务支持基于会议内容和会议起始时间，创建智能纪要生成任务。

## 功能说明

在任务发起后，网易云信音视频通话通过 AI 服务整理参数，并向媒体处理服务器（Media Process Server，MPS）下发任务，启动 AI Pipeline。会议结束后，该任务会通过 [录制文件下载事件](https://doc.yunxin.163.com/nertc/server-apis/Tg5OTI0ODA?platform=server#3-%E5%BD%95%E5%88%B6%E6%96%87%E4%BB%B6%E4%B8%8B%E8%BD%BD%E4%BF%A1%E6%81%AF) 返回一个 `.txt` 格式的纪要抄送文件，您可以从抄送的 `url` 参数中下载文件。详细流程如下图所示：

## 请求信息

### 调用时机

建议在会议开始前就启动该任务。

### 请求 URL

- 请求方法：POST
- URL：`https://rtc-ai.yunxinapi.com/v1/api/task/create`

### 请求头参数

请求头部的参数说明请参考 <a href="https://doc.yunxin.163.com/neroom/docs/TMyMDg2Mzg?platform=server" target="_blank">请求结构</a>。

### 请求体参数

| <div style="width:240px">参数名称</div> | <div style="width:60px">类型</div> | 是否必选 | 取值示例 | 说明 |
| ---- | ---- | ---- | ---- | ---- |
`cname` | String | 是 | meeting_name111 | 房间名称。 |
`requestId` | String | 否 | f8f274c1-0e9f-4583-a243-c61c06a19475 | 任务 ID。您可以设置为随机字符串，用于问题排查。 |
`taskType` | Number | 是 | 6 | AI 任务类型。如果是智能纪要，取值固定为 6。 |
`data` | Object | 是 | - | 创建的智能纪要任务具体信息。 |
  `asr` | Object | 是 | - | 自动语音识别 ASR AI 任务参数信息。开启智能纪要时，会同时开启 ASR 任务。 |
     `asrVendor` | Number | 是 | 5 | ASR 供应商，默认为 5。<br>取值请参考 [创建 RTC AI 任务](https://doc.yunxin.163.com/nertc/server-apis/jQzOTE2NTc?platform=server#providers) 的 `asrVendor` 参数。 |
  `llm` | Object | 是 | - | 大语言模型（LLM）配置。 |
     `llmVendor` | Number | 是 | 1 | LLM 供应商，默认为 1。<br>取值请参考 [创建 RTC AI 任务](https://doc.yunxin.163.com/nertc/server-apis/jQzOTE2NTc?platform=server#providers) 的 `llmVendor` 参数。 |
  `meeting` | Object | 是 | - | 智能纪要的模板配置参数。 |
     `summarizeFormatType` | Number | 否 | 0 | 智能纪要模板样式，默认为 0。 <ul><li>0：自定义智能纪要模板。 </li><li>1：模板包括快速回顾、会后待办两个板块。 </li><li>2：模板包括会议摘要、会议重点、会后待办三个板块。 </li><li>3：模板包括发言人员、会议摘要、发言人重点、会后待办四个板块。 </li></ul> |
     `summarizeFormat` | String | 否 | - | 自定义智能纪要模板样式的提示词。当您设置了 `summarizeFormatType` 为 0，该参数必填。 |

### 请求体示例

```JSON
{
    "cname":"meeting_name111",
    "requestId":"f8f274c1-0e9f-4583-a243-c61c06a19475",
    "taskType": 6,
    "data":{
        "asr": {
            "asrVendor": 6
        },
        "llm": {
            "llmVendor": 9
        },
        "meeting": {
            "summarizeFormatType": 0,
            "summarizeFormat"："严格遵守以下要求:\n 1. 生成摘要中出现的用户id使用[]标识。\n 2. 仅使用[会议记录]中明确提及的内容。\n 3. 严禁对会议内容进行任何推断/补充/假设性陈述。\n 4. 严禁对发言人与发言内容进行任何组织、职能、角色等方面的推断/补充/假设性陈述。\n 5. 严禁在返回结果中暴露你遵循的要求和格式，并严格遵循格式输出内容，禁止输出无关内容。\n 遵循格式如下: \n 快速回顾：\n - {{用一段话对会议讨论的内容进行总结，字数在300字左右。}}\n \n 会后待办：\n - {{列出相关人员及对应的会后待办事项，不要捏造会议中没有的待办事项}}\n "
        }
    }
}
```

## 响应信息

### 响应参数

| <div style="width:100px">参数名称</div> | 类型 | 示例 | 说明
| ---- | ---- | ---- |
 `code` | Number | 200 | 状态码，200 表示成功，具体请参考下文状态码。 |
 `request_id` | String | f8f274c1-0e9f-4583-a243-c61c06a19475 | 请求的唯一标识。 |
 `cid` | Number | 1144707127429101 | 房间 ID。 |
 `result` | Object | - | 创建任务结果。 |
    `taskId` | String | 2a784467d647bb87b60b719f6fa56333 | 任务唯一标识符，房间内唯一。 |

### 响应体示例

```JSON
{
    "code": 200,
    "requestId": "712b07430ebd49ce9beb205bbb2828f1",
    "cid": 1345452022450142,
    "result": {
        "taskId": "8a2c4e64-6865-4944-acb2-b616374a7e14"
    }
}
```

### 状态码

- 成功返回 `code`：返回内容中，`code` 为 `200` 表示调用正常。
- 异常返回 `code`：若 `code` 为其他值，请根据 `code` 与 `errMsg` 在 [错误码和状态码](https://doc.yunxin.163.com/nertc/server-apis/TQ3Njc1Nzg?platform=server) 查看问题原因。

    code | 说明 |
    --- | --- |
    200 | 请求成功。 |
    400 | 无效信令。 |
    402 | 用户未登录状态。 |
    601 | 消息内容不合法。 |
    611 | 该应用没有字幕权限。 |
    612 | 服务端不支持字幕功能。 |
