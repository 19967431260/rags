
本文介绍如何获取房间内被禁言的成员列表。

## 请求

### URI

```bash
GET https://{endpoint}/neroom/v1/rooms/:room_archive_id/mute_list
```

::: note notice

- {endpoint} 为 NERoom 接入地址的域名，默认为 `roomkit.yunxinapi.com`。如果您的应用主要服务于海外用户，请将域名设置为海外数据中心域名（`roomkit-sg.yunxinapi.com`）。
- Content-Type: application/json; charset=utf-8。
:::

### Header

请求 Header 的设置请参见 <a href="https://doc.yunxin.163.com/neroom/docs/TMyMDg2Mzg?platform=server" target="_blank">请求结构</a>。

### 路径参数

| 参数名称         | 类型    | 是否必须 | 描述                |
| --------------- | ------- | ---- | ----------------- |
| room_archive_id | String  | 是  | 房间ID。          |
| chat            | Boolean | 否  | true 表示查询被禁言的用户。    |
| video           | Boolean | 否  | true 表示查询视频被禁的用户。 |
| audio           | Boolean | 否  | true 表示查询音频被禁的用户 。 |
| page_number     | Number  | 否  | 页码，从1开始，默认1。     |
| page_size       | Number  | 否  | 每页显示的被禁言成员数量，最大1000，默认50。|

### 请求体示例

```curl
   https://roomkit.yunxinapi.com/neroom/v1/rooms/441301/mute_list?chat=true&audio=true&page_number=1&page_size=50
```
