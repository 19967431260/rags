本文介绍如何调用服务端接口修改房间中成员的名称与扩展信息。

## 请求信息

### 请求 URI

```HTTP
PUT https://roomkit.yunxinapi.com/neroom/v4/rooms/{room_archive_id}/members/{user_uuid}
```

如果您的应用主要服务于海外用户，请将 `roomkit.yunxinapi.com` 域名设置修改为海外数据中心域名 `roomkit-sg.yunxinapi.com`。

### 请求头参数

请求 Header 的参数说明请参考 [请求结构](https://doc.yunxin.163.com/meeting/server-apis/TM0MzUxOTU)。

### 请求路径参数

| 名称 | 类型 | 是否必选 | 说明 |
| --- | --- | --- | --- |
| room_archive_id | String | 是 | 房间唯一标识。 |
| user_uuid | String | 是 | 被操作人的 userUuid。 |

### 请求体参数

| 名称 | 类型 | 是否必选 | 说明 |
| --- | --- | --- | --- |
| ext | String | 否 | 您自定义的扩展信息，最大长度为 2048 个字符。 |
| user_name | String | 否 | 成员在房间内的名称。最大长度为 64 个字符。 |

### 请求体示例

```JSON
{
    "user_name":"Name***",
    "ext":"custom"
}
```

## 响应信息

### 响应参数

| 参数名称 | 类型 | 示例 | 说明 |
| --- | --- | --- | --- |
| code | int | 0 | 状态码，0 表示成功，1501 表示操作过期，被忽略。 |
| msg | String | Success | 业务结果描述，Success 表示成功。 |
| ts | Long | 1648021056815 | NERoom 服务器处理该请求的完成时间。该时间为 Unix 时间戳，即从 1970 年 1 月 1 日 0 点 0 分 0 秒开始到现在的秒数。 |
| request_id | String | 7c4b6d9c3e9d42*****cc6e3a4d995 | 请求的唯一标识。 |
| data | Object | Null | 无 |
| cost | String | 48ms | 处理该请求所消耗的时间，单位为毫秒。 |

### 响应示例

```JSON
{
    "code": 0,
    "msg": "Success",
    "ts": 1646104672860,
    "cost": "182ms",
    "requestId": "8e9f2dde7d474d28bfc5639f8cb67f60",
    "data": null
}
```

### 错误码

错误码 | 错误信息 | 说明 | 处理建议
---- | ---- | ---- | ---- |
0 | Success | 请求成功 | 无需处理。
400 | Invalid parameter. | 参数错误 | 检查接口传参。 |
1004 | The room does not exist. | 房间号不存在 | 更换 `room_archive_id`。 |
