查询某个房间的当前在线人员列表。使用该接口，您可以方便地获取某个房间的实时在线人数和人员信息，相关数据可以用于展示在您的业务系统中，方便您根据实时在线人员的情况，进行业务调度等操作。

::: note note
已结束的房间无法通过该接口查询在线人员列表。
:::

## 请求信息

### 请求组成

```HTTP
GET https://{endpoint}/neroom/v3/rooms/:room_archive_id/online_user_list
```
{endpoint} 为房间组件接入地址的域名：

- 默认为中国大陆地区的 `roomkit.yunxinapi.com`。
- 如果您的应用主要服务于其他国家或地区用户，请设置为 `roomkit-sg.yunxinapi.com`。

### 请求头参数

请求 Header 的设置，请参考 <a href="https://doc.yunxin.163.com/neroom/docs/TMyMDg2Mzg?platform=server" target="_blank">请求结构</a>。

### 请求 URL 参数

| 参数名称 | 类型 | 是否必选 | 示例 | 说明 |
| ---- | ---- | ---- | ---- | ---- |
| room_archive_id | String | 是 | 4***251 | 创建房间时，NERoom 服务端自动生成的一个用于标识房间的 ID，全局唯一，最大长度 36 个字符。<br>获取方法请参考 <a href="https://doc.yunxin.163.com/neroom/docs/TM0MTUyNjc?platform=server" target="_blank">创建房间</a> 中的返回参数。 |

**Query 参数**

| 参数名称 | 类型 | 是否必选 | 示例 | 说明 |
| ---- | ---- | ---- | ---- | ---- |
| page_number | Integer | 否 | 1 | 查询页码，从 1 开始。如果参数为空，默认查询第 1 页。 |
| page_size | Integer | 否 | 10 | 每页显示个数，最大支持 50 个。如果参数为空，默认显示 10 个。 |

### 请求示例

```HTTP
https://roomkit.yunxinapi.com/neroom/v3/rooms/123/online_user_list?page_number=1&page_size=10
```
