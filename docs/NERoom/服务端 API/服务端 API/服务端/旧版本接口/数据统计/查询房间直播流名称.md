查询房间的直播流名称（streamName）。

使用场景：直播、点播结束后，如果您需要查询对应的直播、点播用量数据，您可以通过该方法查询直播流名称，再通过<a href="https://doc.yunxin.163.com/live-streaming/docs/jk1NTM3MTg?platform=server" target="_blank">数据统计</a>查询数据信息。

## 注意事项
直播流没开始情况下，直播流名也不存在，创建房间不会初始化直播流，这时无法查询流名，只有当第一个成员进入房间后，会进行初始化操作，这时可以查询流名。

## 接口请求地址

- 请求方法：GET
- URL：https://roomkit.yunxinapi.com/neroom/v3/rooms/:room_archive_id/live_stream_name


## URL 中参数说明
请求 Header 的设置请参见 <a href="https://doc.yunxin.163.com/neroom/docs/TMyMDg2Mzg?platform=server" target="_blank">请求结构</a>。

| 参数名称    | 类型   | 是否必选 | 示例 | 描述                                                 |
|-------------|--------|----------|------|--------------|
|   room_archive_id      | String   | 是  |  4***251 |创建房间时，NERoom 服务端自动生成的一个用于标识房间的 ID，全局唯一，最大长度36个字符。<br>获取方法请参见<a href="https://doc.yunxin.163.com/neroom/docs/TM0MTUyNjc?platform=server" target="_blank">创建房间</a> 中的返回参数。    |
     

## 返回参数
| 参数名称    | 类型    | 示例 | 描述                                                 |
|-------------|--------|-------|--------------|
| code | int |  0 |  状态码，0表示成功，具体请参见<a href="https://doc.yunxin.163.com/neroom/docs/TI3ODc2MTc?platform=server" target="_blank">错误码</a>。            |
|   msg      | String   |  Success   |    业务结果描述，Success表示成功。        |
|    ts     | Long  | 1648021056815  | NERoom 服务器处理该请求的完成时间。该时间为 Unix 时间戳，即从 1970 年 1 月 1 日 0 点 0 分 0 秒开始到现在的秒数。 |
|   request_id      | String   |  7c4b6d9c3e9d42*****cc6e3a4d995   |    请求的唯一标识。        |
|   cost      | String   |  48ms   |    处理该请求所消耗的时间。        |
data      | JsonObject   |  -   |    房间的直播流名称。       |
    stream_name      | String   |  13412341324_1664258351668   |  直播流名称。       |
