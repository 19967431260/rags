本文介绍如何批量获取 NERoom 账号的昵称、头像和 Token信息。



::: note notice
请先 <a href="https://doc.yunxin.163.com/neroom/docs/TY1NzM5MjQ?platform=server" target="_blank">创建 NERoom 账号</a>，再调用本接口。
:::


  

## 请求
### URI

```
GET https://{endpoint}/neroom/v3/users?user_uuids=:user_uuids
```
{endpoint} 为 NERoom 接入地址的域名，默认为 `roomkit.yunxinapi.com`。如果您的应用主要服务于海外用户，请将域名设置为海外数据中心域名（`roomkit-sg.yunxinapi.com`）。

### Header

请求 Header 的设置请参见 <a href="https://doc.yunxin.163.com/neroom/docs/TMyMDg2Mzg?platform=server" target="_blank">请求结构</a>。


### 查询参数


| 参数名称    | 类型   | 是否必选 | 示例 | 描述                                                 |
|-------------|--------|----------|------|--------------|
|   user_uuids      | String   | 是  |  user1%2user2%2user3 |账号列表，以英文逗号分隔后再经过 URL 编码，账号个数上限100。|

### 请求示例
```
GET https://roomkit.yunxinapi.com/neroom/v3/users?user_uuids=user1%2user2%2user3
```

## 响应
### 响应参数
| 参数名称    | 类型   |  示例 | 描述 |
|-------------|--------|-------|--------------|
|code|int|0|状态码，0表示成功，具体请参见错误码。|
|msg|String|Operation succeeded.|业务结果描述|
|ts|Long|1648021056815|NERoom 服务器处理该请求的完成时间。<br>该时间为 Unix 时间戳，即从 1970 年 1 月 1 日 0 点 0 分 0 秒开始到现在的毫秒数。|
|request_id|String|7c4b***e3a4d995|请求的唯一标识。|
|cost|String|48ms|处理该请求所消耗的时间。|
|data| Array of Object	 | | 用户信息数组|
   user_uuid | String | user_uuid_00001 |  用户账号。        |
   user_token | String | 0ZNLD4***2N251D1 | 用户的 Token。 |
   im_token | String | 0ZNLD4***2N251D1 | IM 的 Token。 |
   user_name | String | 用户01 | 用户昵称。如果没有设置参数，则不会返回。 |
   icon | String | https://tupian.jpg | 用户的头像。如果没有设置参数，则不会返回。 |
