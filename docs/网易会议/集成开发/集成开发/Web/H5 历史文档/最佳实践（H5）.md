## 准备工作

### 检查浏览器兼容性

入会前调用 `checkSystemRequirements` 接口判断当前浏览器环境是否支持音视频通话：

```JavaScript
// 如果返回 false 建议更换浏览器
const isSupported = NEMeetingKit.checkSystemRequirements();
if (!isSupported) {
  // 提示用户更换浏览器
  showBrowserNotSupportedTip();
}
```

### 会前设备检测

为确保用户体验，建议在会议开始前进行设备检测，验证摄像头、麦克风是否正常工作，以及扬声器音量是否设置合理：

加入会议的配置项中设置 [NEMeetingOptions.showDeviceTest](https://doc.yunxin.163.com/meetingkit/references/web/typedoc/Latest/zh/h5/types/NEMeetingOptions.html#__type.showDeviceTest) 为 `true` 可开启入会前设备检测：

```JavaScript
const meetingOptions = {
  // 其他配置...
  showDeviceTest: true // 入会之前进行设备检测
};
```

:::note note
已知问题：H5 端无法控制声音播放的设备，由系统控制（即扬声器和听筒随机）。更多详情，请参考 [会前设备检测指南](https://doc.yunxin.163.com/nertc/guide/TI3NzU0MjA?platform=web)。
:::

## 常见挑战

### 浏览器自动播放限制

**问题描述**

大多数现代浏览器为了用户体验，会阻止网页在没有用户交互的情况下自动播放音频。这可能导致加入会议后无法听到其他参会者的声音。

**解决方案**

- 会议组件本身加入会议后，如果远端开启音频，本端会弹窗提示让用户进行交互操作。
- 在页面加载时显示提示，引导用户进行交互。

详情请参考 [处理浏览器自动播放限制](https://doc.yunxin.163.com/nertc/guide/jM3NDE0NTI?platform=web)。

```JavaScript
// 示例：用户交互后加入会议
joinButton.addEventListener('click', () => {
    // 此时进行加入会议，将不会受到自动播放限制
    joinMeeting();
});
```

## 平台相关

### Android **Chrome 内核兼容性问题**

**Chrome 79 以下不支持 H264 解码**，您需要调整为 VP8 编码。

```JavaScript
// 在加入房间前调用
client.adapterRef.mediaCapability.supportedCodecRecv = ['VP8'];
client.adapterRef.mediaCapability.supportedCodecSend = ['VP8'];
```

### Android **系统 Bug 解决方案**

- **Android 12（Chrome 内核 99.0.4844.88）**：存在系统 bug，使用 H264 编码频繁切换摄像头可能导致崩溃。
- **华为手机**：可能存在 H264 编码异常问题。

```JavaScript
// 针对上述情况，统一建议调整为 VP8 编码
function checkAndSetCodec() {
    const userAgent = navigator.userAgent;

    // 检测 Android 12 或华为设备
    if ((userAgent.includes('Android 12') && userAgent.includes('Chrome/99')) | |
        userAgent.includes('HUAWEI')) {
        client.adapterRef.mediaCapability.supportedCodecRecv = ['VP8'];
        client.adapterRef.mediaCapability.supportedCodecSend = ['VP8'];
    }
}
```

### **iOS 16（iPhone 14）特定问题**

- 不能设置 `mediaTypesRequiringUserActionForPlayback` 属性，否则会导致麦克风异常。
- 一次通话结束后建议刷新页面再进行下一次通话。
- 建议通过 `muteAudio/unmuteAudio` 方式控制麦克风，而非开关麦克风。
- 屏幕旋转会导致视频拉伸，建议固定竖屏模式。

### **iOS 15 编码问题**

**iOS 15 系统** H264 编码存在多种问题，需调整为 VP8 编码。

```JavaScript
// iOS 15 系统检测并设置 VP8 编码
function checkIOSAndSetCodec() {
    const userAgent = navigator.userAgent;

    // 检测 iOS 15 系统
    if (userAgent.includes('iPhone OS 15_')) {
        client.adapterRef.mediaCapability.supportedCodecRecv = ['VP8'];
        client.adapterRef.mediaCapability.supportedCodecSend = ['VP8'];
    }
}
```

### **iOS 15-15.3音频播放问题**

存在系统声音播放小的问题（声音可能从听筒输出）：

- 建议使用 WebAudio 进行音频处理，可实现音量增益。
- 使用蓝牙耳机时可能出现声音失真，建议添加提示文案。

    let webConfiguration = WKWebViewConfiguration()

    // 1. 设置支持内联播放（否则默认全屏播放）

    // 注意：在 iPhone 14 (iOS 16)上设置此项可能导致麦克风采集异常
