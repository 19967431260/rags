为确保 H5 端能够正常进行音视频收发，减少音频自动播放受限以及未授予音视频设备权限问题，请按照以下流程实现：
  
## 浏览器兼容性

入会前调用 `checkSystemRequirements` 接口判断当前浏览器环境是否支持音视频通话 RTC：

```JavaScript
// 如果返回 false 建议更换浏览器
const isSupported = NEMeetingKit.checkSystemRequirements();
if (!isSupported) {
  // 提示用户更换浏览器
  showBrowserNotSupportedTip();
}
```

## 会前设备检测

入会前建议进行设备检查，确保音视频设备正常工作：

### 1. 配置会前检测选项

加入会议的配置项中设置 [NEMeetingOptions.showDeviceTest](https://doc.yunxin.163.com/meetingkit/references/web/typedoc/Latest/zh/h5/types/NEMeetingOptions.html#__type.showDeviceTest) 为 `true` 可开启入会前设备检测：

```JavaScript
const meetingOptions = {
  // 其他配置...
  showDeviceTest: true // 入会之前进行设备检测
};
```

### 2. 用户体验优化

为优化用户体验，避免每次都进行设备检测，您可在调用加入会议接口前弹窗提示用户是否需要进行设备检测：

```TypeScript
function joinMeeting(meetingId, password) {
  const neMeetingKit = NEMeetingKit.getInstance()
  const meetingService = neMeetingKit.getMeetingService()
  // 显示设备检测确认弹窗
  showDeviceTestConfirm({
    onConfirm: () => {
      // 用户确认进行设备检测
      meetingService.joinMeeting(param, {
        showDeviceTest: true,
        // 其他配置...
      });
    },
    onCancel: () => {
      // 用户取消设备检测，直接入会
      meetingService.joinMeeting(param, {
        showDeviceTest: false,
        // 其他配置...
      });
    }
  });
}
```

### 4. 异常处理

如果设备检测过程中出现异常，用户选择 **我知道了**，系统会取消加入会议流程，并触发 `onMeetingStatusChanged` 事件：

```TypeScript
const neMeetingKit = NEMeetingKit.getInstance()
const meetingService = neMeetingKit.getMeetingService()

meetingService.addMeetingStatusListener({
  onMeetingStatusChanged: ({status}) = {
    if (status === NEMeetingStatus.MEETING_STATUS_FAILED) {
      // 处理入会失败场景
      handleMeetingJoinFailed(errorCode);
    }
  }
})
```

## 实践建议

- 对首次使用的用户默认启用设备检测。
- 记录用户之前的设备检测结果，避免重复检测。
- 提供明确的设备权限请求说明，提高用户授权的意愿。
- 在设备检测失败时，提供清晰的错误提示和解决方案。
