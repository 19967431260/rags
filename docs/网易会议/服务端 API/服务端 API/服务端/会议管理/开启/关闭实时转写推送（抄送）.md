开启或关闭实时转写推送。

当开启推送后，如果会议打开转写功能，系统将通过回调方式向业务服务器推送相关转写内容。

## 请求信息

### 请求 URL

```
POST https://{endpoint}/scene/meeting/api/v2/{meetingId}/transcription-push
```

- {endpoint} 为 网易会议组件接入地址的域名，默认为 `roomkit.yunxinapi.com`。

- Content-Type：application/json;charset=utf-8。

### 请求头参数

请求 Header 的参数说明请参考 [请求结构](https://doc.yunxin.163.com/meeting/server-apis/TM0MzUxOTU) 描述。

### 请求路径参数

| 参数名称 | 类型 | 是否必选 | 示例 | 说明 |
|----------|------|----------|------|------|
| meetingId | Long | 是 | 1188393 | 会议 ID。 |

### 请求体参数

| 参数名称 | 类型 | 是否必选 | 示例 | 说明 |
|----------|------|----------|------|------|
| enable | Boolean | 是 | true | 推送开关。true-开启推送，false-关闭推送。 |

### 请求体示例

```JSON
{
  "enable": true
}
```

## 响应信息

### 响应参数

| 参数名称 | 类型 | 说明 | 是否必返回 |
|----------|------|------|------------|
| code | Integer | 状态码，0 表示成功。 | 是 |
| msg | String | 响应消息。 | 是 |
| data | Object | 响应数据对象。 | 是 |

### 响应体示例

```JSON
{
  "code": 0,
  "msg": "success",
  "data": {}
}
```
