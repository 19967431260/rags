通过该接口为某个房间的巡检者角色绑定用户。

## 功能描述

如何有效地检查和管理会议中可能会存在的违规内容或其他违规行为一直是一个令会议服务提供商头疼的问题。网易会议组件提供巡检者模式，您可自行搭建巡检页面，通过巡检者的身份加入到会议中进行人工巡查，检查会议中是否存在违规内容或其他违规行为。如果发现了违规内容，您可以通过 [结束会议](https://doc.yunxin.163.com/meeting/server-apis/Dk3MDU5NTc?platform=server) 接口结束会议，从而保障整体会议服务的安全。

:::note note
- 1 个房间可以添加 10 个巡检者用户。如果要添加多个巡检者用户，请重复调用本接口。
- 1 个用户可以绑定多个房间。
:::

## 前提条件

已为巡检者角色的用户 [创建会议账号](https://doc.yunxin.163.com/meeting/server-apis/jU0MDAzNTg?platform=server)。

## 请求信息

### 请求 URL

```
PUT https://roomkit.yunxinapi.com/apps/v2/room/bind/observer
```

### 请求头参数

请求中 Header 的设置请参考 [请求结构](/meeting/server-apis/TM0MzUxOTU) 描述。

### 请求体参数

| 参数名称 | 类型 | 是否必选 | 示例 | 说明 |
| ---- | ---- | ---- | ---- | ---- |
| roomArchiveId | Long | 是 | 1234567 | 请填写待添加巡检者的目标房间的 roomArchiveId，您可以通过 [创建会议](https://doc.yunxin.163.com/meeting/server-apis/TUwMzc3NzY?platform=server) 的返回参数中获取该参数的值。 |
| userUuid | String | 是 | abc | 待绑定的巡检者对应的用户 ID。一次只能填写一个用户 ID。您可以通过 [创建会议账号](https://doc.yunxin.163.com/meeting/server-apis/jU0MDAzNTg?platform=server) 的返回参数中获取该参数的值。 |

### 请求体示例

```JSON
{
    "roomArchiveId": 1234567,
    "userUuid": "abc"
}
```

## 响应信息

### 响应体参数

以下是返回结果中 `data` 属性内包含的参数。其他统一返回参数的相关说明请参考 [返回结果](/meeting/server-apis/TczMTMyMjg)。

| 参数名称 | 类型 | 说明 |
| --- | --- | --- |
| code | Integer | 状态码。0 表示成功。非 0 表示失败。 |
| msg | String | 状态描述。 |
| ts | Long | 当前时间戳，单位为毫秒。 |
| cost | String | 该请求耗费的时间。 |
| requestId | String | 请求 ID。 |

### 响应体示例

```JSON
{
  "code": 0,
  "msg": "Success",
  "ts": 1619068087795,
  "requestId": "6e507107d1f441234731abcdec6d2432",
  "cost": "66ms"
}
```
