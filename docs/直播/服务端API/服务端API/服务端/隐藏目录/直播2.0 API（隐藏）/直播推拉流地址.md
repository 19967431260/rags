## <span id="直播推拉流地址">直播推拉流地址</span>
  
云信直播2.0与直播1.0区别主要是对于流名的管理：在直播1.0中，流名即频道，需要预先创建才可以使用进行推拉流；在直播2.0中不再做这样的限制，可以直接使用符合条件的字符串进行推流（长度不大于128字节，0-9a-z）。当需要提前在流名上进行录制配置时，需要通过录制配置相关接口进行提前配置。另外，直播2.0中不再提供获取流名对应推拉流地址的接口，用户根据文档中的相关规则描述生成推拉流地址。

如果需要使用直播2.0相关功能和接口，需要联系客户经理开通权限。需要注意的是，直播2.0接口不兼容原直播1.0相关接口，即用户开通直播2.0功能后，需要全链路使用直播2.0相关接口，有相关不满足的需求可以与我们沟通反馈。

### <span id="推流地址">推流地址</span>

假设推流域名为push.domain.com，推流地址格式如下：

```
rtmp://push.domain.com/live/streamName?wsSecret=XXXXXX&wsTime=XXXXXX
```
其中 `streamName` 为用户自定义的流名称，长度需要不大于 128 字节，`wsSecret` 和 `wsTime` 为防盗链鉴权字符串，参见下文[防盗链规则](#防盗链规则)。

### <span id="拉流地址">拉流地址</span>

拉流地址通常如下所示：

```
rtmp协议：
rtmp://pull.domain.com/live/streamName?playSecret=XXXXXX&playTime=XXXXXX
hls协议：
http://pullhls.domain.com/live/streamName/playlist.m3u8?playSecret=XXXXXX&playTime=XXXXXX
http-flv协议：
http://pullflv.domain.com/live/streamName.flv?playSecret=XXXXXX&playTime=XXXXXX

```
其中 `pull.domain.com`、`pullhls.domain.com`、`pullflv.domain.com` 为不同拉流协议对应的拉流域名，其中 `streamName` 为用户自定义的流名称，字符串长度不大于 128 字节，`wsSecret` 和 `wsTime` 为防盗链鉴权字符串，参见下文[防盗链规则](#防盗链规则)。

### <span id="防盗链规则">防盗链规则</span>

当用户开通推拉流防盗链鉴权功能时，需要在推拉流地址上附带相关用于鉴权的字符串，推流防盗链或拉流防盗链功能可以分别开通。

规则如下：

```

原推流地址：
path = "rtmp://push.domain.com/live/streamName";
SECRET_KEY = "xxxxxxxxxxxxxxxxxxxxxxxx"; // 防盗链信息加密KEY

wsTime = 1587936717; // 时间戳信息
wsSecret = md5(SECRET_KEY + "/live/" + streamName + wsTime) // 加号代表字符串拼接
  		
假设生成的字符串如下：234d3741132c4a23c8e9f618a829d88b;

则带防盗链鉴权信息的地址为：

"rtmp://rtmp://push.domain.com/live/streamName?wsSecret=234d3741132c4a23c8e9f618a829d88b&wsTime=1587936717";

```

服务端收到推拉流请求后，会使用相同的规则和密钥进行计算，与推拉流地址中的校验字符串进行比较，如果不相等，则推拉流失败。同时，再根据wsTime、收到请求时刻的时间戳及用户配置的防盗链过期时间，进行过期时间的校验。

以上为推流地址防盗链规则，拉流地址可参考此规则。

防盗链Key和有效时长，可以在“云信管理控制台 > 应用详情页 > 直播域名管理”页面进行管理。
