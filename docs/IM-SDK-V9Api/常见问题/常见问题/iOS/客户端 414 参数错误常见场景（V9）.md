本文主要介绍 iOS 客户端 414 参数错误码的常见场景。

## 常见错误场景

### 消息发送

```Objective-C
- (void)sendMessage:(NIMMessage *)message
          toSession:(NIMSession *)session
         completion:(__nullable NIMChatManagerBlock)completion;
```

📝 参数校验：

- ❌ `message` 校验错误：

     - 消息必须为 `NIMMessage` 类型。
     - `message.messageId` 不能为空。
     - `message.from` 为空或者 `message.from` 为字符串类型。
     - `message.messageObject` 不能为空且只能是以下类型：
          
          `NIMImageObject`，`NIMAudioObject`，`NIMLocationObject`，`NIMVideoObject`，`NIMFileObject`，`NIMNotificationObject`，`NIMCustomObject`，`NIMTipObject`，`NIMRobotObject`，`NIMRtcCallRecordObject`

- ❌ `session` 校验错误：

     - `session` 必须为 `NIMSession` 类型。
     - `session.sessionId` 不能为空。

### 消息更新

```Objective-C
- (void)updateMessage:(NIMMessage *)message
           forSession:(NIMSession *)session
           completion:(nullable NIMUpdateMessageBlock)completion;
```

📝 参数校验：

- ❌ `message` 校验错误：

     - 必须为 `NIMMessage` 类型且 `message.session` 有值。
     - `message.messageId` 不能为空。
     - `message.from` 为空或者 `message.from` 为字符串类型。
     - `message.serialID` > 0。

- ❌ `session` 校验错误：

     - `session` 必须为 `NIMSession` 类型。
     - `session.sessionId` 不能为空。


### 消息查询

```Objective-C
- (void)messagesInSession:(NIMSession *)session
                  message:(nullable NIMMessage *)message
                    limit:(NSInteger)limit
               completion:(NIMFetchMessageHistoryBlock)completion;
```

📝 参数校验：

- ❌ `message` 校验错误：

     - 传入的 `message` 必须为 `NIMMessage` 类型，且 `message.session` 必须有值，`message.messageId` 不能为空。
     - `message.from` 不传或者传入字符串类型。

### 消息搜索

```Objective-C
- (void)searchMessages:(NIMSession *)session
                option:(NIMMessageSearchOption *)option
                result:(nullable NIMSearchMessageBlock)result;
```

📝 参数校验：

- ❌ `option` 校验错误：

     - 传入 `option` 必须为 `NIMMessageSearchOption` 类型。
     - 如果需要传入 `option.searchContent`，则必须为字符串。
     - 如果需要传入 `option.fromIds`，则数组中的元素必须为字符串。
     - 如果需要传入 `option.startTime` 和 `option.endTime`，必须保证 `endTime` 大于 `startTime`。

### 消息删除

```Objective-C
- (void)deleteMessage:(NIMMessage *)message;
```

📝 参数校验：

- ❌ `message` 校验错误：

     - 传入的 `message` 必须为 `NIMMessage` 类型，且 `message.session` 必须有值，`message.messageId` 不能为空。
     - `message.from` 不传或者传入字符串类型。

### 创建群组

```Objective-C
- (void)createTeam:(NIMCreateTeamOption *)option
              users:(NSArray<NSString *> *)users
         completion:(nullable NIMTeamCreateHandler)completion;
```

- ❌ `option` 校验错误：

     - `option.type`，`option.joinMode`，`option.inviteMode`，`option.beInviteMode`，`option.updateInfoMode`，`option.updateClientCustomMode` 必须传入枚举对应的数字，不能传枚举值以外的数字。
     - 若使用带 `users` 参数的方法，`users` 需要有值，即必须传入至少一个用户账号。
