<!--keywords: 搜索, 圈组搜索, 圈组搜索功能 -->

网易云信即时通讯 NIM SDK 的[`NIMQChatServerManager`](https://doc.yunxin.163.com/docs/interface/messaging/iOS/doxygen/Latest/zh/df/dac/protocol_n_i_m_q_chat_server_manager-p.html)协议提供搜索圈组的服务器和服务器成员方法，[`NIMQChatChannelManager`](https://doc.yunxin.163.com/docs/interface/messaging/iOS/doxygen/Latest/zh/df/d6b/protocol_n_i_m_q_chat_channel_manager-p.html)协议提供搜索频道和频道成员的方法。

SDK 还支持配置搜索结果的排序规则，如搜索出的服务器列表可按时间和人数等维度排序，也可自定义排序。 






## 应用场景

圈组的搜索能力和配套的搜索结果排序能力，可满足您的应用的特定运营需求，如：

- 通过按人数排序，将热门**社区**（对应圈组服务器）展示在用户搜索结果的前列，方便用户加入热门社区。 
- 通过自定义排序，将广告位性质的**话题**（对应频道）展示在用户搜索结果的前列，增加流量引入。

## 技术原理






### **搜索数据来源**
下图左侧所示事件发生后，将触发相关消息通过消息队列（Message Queue）通知云信搜索服务器，搜索服务器把相关的事件数据保存或更新至 Elasticsearch。

由于整个数据传送过程都是通过消息队列生产消费，且 Elasticsearch 的数据每秒刷新一次，所以整体的搜索数据存在一定的延迟（正常 3 秒内）。


![圈组搜索原理.png](https://yx-web-nosdn.netease.im/common/39419fc15ac424347610acaa689d63c7/圈组搜索原理.png)


### **与服务器的关联逻辑**

服务器创建、更新、查询等方法（包括`createServer`和`updateServer`），入参包含`searchEnable`和`searchType`。


参数 | 说明
---- | -------------- 
`searchEnable` | 服务器是否允许被搜索，`true`-允许，`false`-不允许，默认允许 
`searchType` |  服务器搜索类型，客户自定义，大于等于 0 的整数。 0 代表无类型


  
## 前提条件

在开始圈组搜索功能集成前，请确保您已：

- 已[开通圈组搜索功能](https://doc.yunxin.163.com/messaging/guide/TM1OTU0MTM?platform=iOS#圈组子功能列表说明)。圈组搜索功能需要在开通圈组功能的基础上额外开通后才能使用。
- 已创建圈组服务器。 
- 已创建频道。

## 实现原理

### 搜索服务器

调用[`searchServerByPage`](https://doc.yunxin.163.com/docs/interface/messaging/iOS/doxygen/Latest/zh/df/dac/protocol_n_i_m_q_chat_server_manager-p.html#a5e53e2d14fb78426773590109f55bb97)方法可搜索服务器，返回在`startTime`和`endTime`之间的时间段内创建并与`keyword`匹配的服务器。

该方法的入参如下表所示：

| 参数    | 必传 | 类型                      | 说明                                     |
|:------------|:------|:---------------------------|:------------------------------------------|
| `keyword`    | 是   | NSString                    |  目标检索服务器名称的关键字，最大长度 100 字符，支持模糊搜索。关键字必须为连续字符串，? 和 * 会被忽略。<br>例子：假设服务器的名称为“网易云信专属服务器”，用户使用如下三个关键字进行搜索，keyword1=“网易云信”，keyword2=“专属服”，keyword3=“网专属”,keyword4=“网易 云信”，那么只有 keyword1 和 keyword2 能搜索到该服务器       |
| `asc`        | 是   | BOOL                  | `true`-升序，`false`-降序          |
| `searchType` | 是   | [`NIMQChatSearchServerType`](https://doc.yunxin.163.com/docs/interface/messaging/iOS/doxygen/Latest/zh/d2/ddd/_n_i_m_q_chat_defs_8h.html#a2cbb3a8190efd8b671579db78dfb8014) | 配置搜索类型： <div><ul><li>传 `NIMQChatSearchServerTypeSquare` 为在广场搜索服务器，即[全局搜索服务器](https://doc.yunxin.163.com/messaging/guide/jI1MzQyMDA?platform=iOS#%E5%85%A8%E5%B1%80%E6%90%9C%E7%B4%A2%E6%9C%8D%E5%8A%A1%E5%99%A8)</li><li>传 `NIMQChatSearchServerTypePersonal` 为[搜索已加入的服务器](https://doc.yunxin.163.com/messaging/guide/jI1MzQyMDA?platform=iOS#%E6%90%9C%E7%B4%A2%E5%B7%B2%E5%8A%A0%E5%85%A5%E7%9A%84%E6%9C%8D%E5%8A%A1%E5%99%A8)</li></ul> </div>    |
| `startTime`  | 否   | NSNumber                 | 查询时间范围的开始时间                   |
| `endTime`    | 否   | NSNumber                   | 查询时间范围的结束时间，必须比开始时间晚 |
| `limit`      | 否   | NSNumber                  | 每页数据条数，最大和默认都是 100  |
| `serverTypes`       | 否   |NSArray<NSNumber *>                    | 服务器类型，对应创建服务器时传入的`searchType`                              |
|`sortType`| 否   |  [`NIMQChatSearchServerSortType`](https://doc.yunxin.163.com/docs/interface/messaging/iOS/doxygen/Latest/zh/d2/ddd/_n_i_m_q_chat_defs_8h.html#a387f243d872cf7870b47cfb4b4e337bb)   |   <div><ul><li>服务器的排序条件，默认为按创建时间倒序排序，即最新创建的服务器排在最前</li><li>如需实现搜索结果自定义排序，请参照下文的[搜索结果自定义排序](https://doc.yunxin.163.com/messaging/guide/jI1MzQyMDA?platform=iOS#搜索结果自定义排序)中的说明进行配置</li></ul></div>   |
|`cursor`      | 否  | NSString  | 查询游标，查询的起始位置，上一次查询会返回cursor字段 |

#### **全局搜索服务器**

全局搜索集成圈组的应用内的所有服务器，包括已加入的和未加入的服务器。 

::: note notice
全局搜索仅能搜索到“允许被搜索”的服务器。
:::

- 示例代码

    ```
    NIMQChatSearchServerByPageParam *param = [[NIMQChatSearchServerByPageParam alloc] init];
    param.keyword = @"keyword";
    param.asc = YES;
    param.searchType = NIMQChatSearchServerTypeSquare;
    [NIMSDK.sharedSDK.qchatServerManager searchServerByPage:param completion:^(NSError * _Nullable error, NIMQChatSearchServerByPageResult *_Nullable result) {
        //检索完成
    }];
    ```

### 搜索结果自定义排序

- 创建服务器成功后返回的[`NIMQChatServer`](https://doc.yunxin.163.com/docs/interface/messaging/iOS/doxygen/Latest/zh/dc/d25/interface_n_i_m_q_chat_server.html)包含服务器的自定义排序权重信息 `reorderWeight`（`NIMQChatSearchServerSortTypeCustomWeight`），**默认为 0**。
- 创建频道成功后返回的[`NIMQChatChannel`](https://doc.yunxin.163.com/docs/interface/messaging/iOS/doxygen/Latest/zh/db/df6/interface_n_i_m_q_chat_channel.html)包含频道的自定义排序权重信息 `reorderWeight`（`NIMQChatSearchChannelSortTypeCustomWeight`），**默认为 0**。自定义排序权重值，目前仅支持通过服务器端 API 修改。
- 创建频道分组成功后返回的[`NIMQChatChannelCategory`](https://doc.yunxin.163.com/docs/interface/messaging/iOS/doxygen/Latest/zh/d4/d8d/interface_n_i_m_q_chat_channel_category.html)包含频道的自定义排序权重信息 `reorderWeight`，**默认为 0**。自定义排序权重值，目前仅支持通过服务器端 API 修改。

#### **服务器自定义排序**

实现服务器搜索结果自定义排序的流程如下：


1. 调用服务端 API （[`qchat/batchUpdateServerReorderWeight.action`](https://doc.yunxin.163.com/messaging/guide/TExNDI3ODU?platformId=60353)）修改服务器的自定义排序权重默认值。 

    ::: note notice :::
    调用该 API 单次可修改权重的服务器的数量上限默认为 10。如需调整该上限，请通过云信官网首页的在线聊天、微信和电话等联系方式咨询商务经理。
    :::

2. 调用[`searchServerByPage`](https://doc.yunxin.163.com/docs/interface/messaging/iOS/doxygen/Latest/zh/df/dac/protocol_n_i_m_q_chat_server_manager-p.html#a5e53e2d14fb78426773590109f55bb97)方法搜索服务器。调用时除了必传的入参`keyword`、`searchType`和`asc`外，还需传入`sortType`，且将其设置为`NIMQChatSearchServerSortTypeCustomWeight`。 

    **结果**：

    - 如`asc`为默认倒序，则匹配的服务器列表以权重值倒序排列，相同权重的服务器按`serverId`倒序排列。
    - 如`asc`为正序，则按权重值正序排序，相同权重的按`serverId`倒序排列。

#### **频道自定义排序**

实现频道搜索结果自定义排序的流程如下：

1. 调用服务端 API （[`qchat/batchUpdateChannelReorderWeight.action `](https://doc.yunxin.163.com/messaging/guide/Tg1NDMyNDk?platformId=60353)）修改频道的自定义排序权重默认值。

    ::: note notice :::
    调用该 API 单次可修改权重值的频道的数量上限默认为 10。如需调整该上限，请通过云信官网首页的在线聊天、微信和电话等联系方式咨询商务经理。
    :::
2. 调用[`searchChannelByPage`](https://doc.yunxin.163.com/docs/interface/messaging/iOS/doxygen/Latest/zh/df/d6b/protocol_n_i_m_q_chat_channel_manager-p.html#a1a54b00db00479f95c085d821b8908a7)方法搜索频道。调用时除了必传的入参`keyword` 和`asc`外，还需传入`sortType`，且将其设置为`NIMQChatSearchChannelSortTypeCustomWeight`。 

    **结果**：

    - 如`asc`为默认倒序，则匹配的频道列表以权重值倒序排列，相同权重的频道按`channelId`倒序排列。
    - 如`asc`为正序，则按权重值正序排序，相同权重的按`channelId`倒序排列。

#### **频道分组自定义排序**

实现频道分组搜索结果自定义排序的流程如下：

1. 调用服务端 API（[`qchat/updateChannelCategory.action`](https://doc.yunxin.163.com/messaging/server-apis/Dc2MDQyNjI?platform=server)）修改频道分组的自定义排序权重默认值。


2. 调用 `getCategoriesInServerByPage` 方法查询频道分组。调用时将入参 `sortType` 设置为 **按自定义权重排序**，并传入 `cursor` 实现分页。 

    示例代码如下：

    ```objc
    id<NIMQChatChannelManager> qchatChannelManager = [[NIMSDK sharedSDK] qchatChannelManager];
    NIMQChatGetCategoriesInServerByPageParam * param = [[NIMQChatGetCategoriesInServerByPageParam alloc] init];
    param.serverId = 123456;
    param.limit = @(20);
    // 返回值的排序类型
    // 配置为NIMQChatSearchChannelSortTypeCustomWeight时，使用cursor分页
    // 配置为NIMQChatSearchChannelSortTypeCreateTime时，使用timeTag分页
    param.sortType = NIMQChatSearchChannelSortTypeCustomWeight;
    // 这里配置为NIMQChatSearchChannelSortTypeCustomWeight，用cursor分页
    param.cursor = @"1234567890123";
    // 这里配置为NIMQChatSearchChannelSortTypeCustomWeight，不填时间
    // param.timeTag = 1234567890.123;

    [qchatChannelManager getCategoriesInServerByPage:param
        completion:^(NSError *__nullable error, NIMQChatGetCategoriesInServerByPageResult *__nullable result) {
        // your code
    }];
    ```
