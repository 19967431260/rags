
身份组是拥有完善且灵活的配置机制的权限管理系统，它包含两大元素：权限组和拥有这些权限的用户。身份组适用于在不同维度（例如社区维度和房间维度）对用户进行差异化的权限管控。



::: note note
身份组适用于对各类大型线上社区进行权限管控，具体场景可参见[身份组应用场景](https://doc.yunxin.163.com/messaging/guide/TQ0OTAwMzQ?platform=iOS)。
:::


## 身份组分类

<img style="width:75%" src="https://yx-web-nosdn.netease.im/common/74a7f3a5276ccc2ebf2047792f4fb097/身份组使用流程.png"/>


身份组分为两类，在<a href="https://doc.yunxin.163.com/docs/interface/messaging/iOS/doxygen/Latest/zh/d2/ddd/_n_i_m_q_chat_defs_8h.html#a5874b61a136ff33004c8c7cbb9776b5f" target="_blank">`NIMQChatRoleType`</a>内定义。

|<div style="width:100px">分类</div> | <div style="width:100px">API 关键字</div>|说明 |
|:---- | :-------|:------- |
|@everyone身份组<br>（默认身份组） |`NIMQChatRoleTypeEveryOne` |- 在用户创建圈组服务器时默认自动创建。 |\
|             | |- 所有圈组服务器成员默认自动成为@everyone身份组的成员。       |\
|             | |- @everyone身份组不可删除。|\
|             | |- @everyone身份组的权限配置只有服务器创建者可修改。|\
|             | |- @everyone身份组的权限项都会有默认值（默认开启或关闭）。如默认关闭，服务器创建者可手动开启。具体默认值请参见下文的[身份组权限项](#身份组权限项)。
|   自定义身份组   |`NIMQChatRoleTypeCustom`  |用户可在服务器、频道分组和频道这三个维度创建自定义身份组。<br><ul><li>服务器自定义身份组：即服务器维度的自定义身份组，其权限项对用户在服务器维度的操作进行管控。需要用户手动创建。创建后默认没有成员，需要再手动添加身份组成员。</li><li>频道分组自定义身份组：即频道分组维度的自定义身份组，其权限项可对用户在频道分组维度的操作进行管控。</li><li>频道自定义身份组：即频道维度的自定义身份组，其权限项可对用户在频道维度的操作进行管控。</li></ul><note type=notice>为自定义身份组配置权限时，只有身份组创建者已拥有的权限才是可配置的，且默认开启。创建者没有的权限，在新建身份组中默认关闭，且不可修改</note>|

## 身份组权限项

身份组权限项列表在[`NIMQChatPermissionType`](https://doc.yunxin.163.com/docs/interface/messaging/iOS/doxygen/Latest/zh/d2/ddd/_n_i_m_q_chat_defs_8h.html#aeee4335aecd193652bc2e7e05679ebb0)枚举中定义，分为**服务器专有的权限**和**非服务器专有**（即在服务器维度和频道维度都可配置）。



<div style="width:150px">权限项</div>| 说明    |是否服务器专有  | @everyone 身份组中是否默认开启
:---- | :-------------- |:------ |:------
NIMQChatPermissionTypeManageServer   |  管理服务器的权限，拥有该权限可修改服务器信息（包括基本属性，服务器名称、图标、成员数、邀请模式等） | 是 | 否 
NIMQChatPermissionTypeModifySelfInfo           | 修改自己的服务器成员信息的权限  |是   | 是
NIMQChatPermissionTypeKickOthersInServer     |   将其他成员踢出服务器的权限    |是   | 否
NIMQChatPermissionTypeModifyOthersInfoInServer | 修改他人的服务器成员信息的权限   |是 | 否
NIMQChatPermissionTypeInviteToServer        |   	邀请他人加入服务器的权限   |是 | 是
NIMQChatPermissionTypeHandleServerApply  |  处理加入服务器申请的权限   |是   | 是
NIMQChatPermissionTypeQueryServerInviteApplyHistory  |  查看申请或邀请历史的权限，拥有该权限才可以查询服务器级别的申请或邀请记录|是 | 是
NIMQChatPermissionTypeManageBanServerMember     |   封禁其他成员访问当前服务器的权限   | 是  |否
NIMQChatPermissionTypeManageChannel |   管理频道的权限，拥有该权限可创建、修改或删除频道    | 否   | 否
NIMQChatPermissionTypeManageRole   | 管理角色的权限，拥有该权限可创建、修改或删除身份组|否   |否
NIMQChatPermissionTypeSendMsg       |  在频道内发送消息的权限           |否     | 是   
NIMQChatPermissionTypeRevokeMsg |    撤回他人消息的权限    |否   |否
NIMQChatPermissionTypeDeleteOtherMsg           | 删除他人消息的权限  |否   |否
NIMQChatPermissionTypeRemindOther              |  @某个人的权限   | 否   | 是
NIMQChatPermissionTypeRemindAll                |  @所有人的权限   |否   | 是
NIMQChatPermissionTypeMentionedRole      | @身份组的权限 | 否 | 否
NIMQChatPermissionTypeManageBlackWhiteList     |   管理频道黑白名单的权限   |否  |  否
NIMQChatPermissionTypeRTCChannelConnect   |   在实时互动频道建立用户自己的连接的权限   |否  | 是
NIMQChatPermissionTypeRTCChannelDisconnectOther|   在实时互动频道断开他人连接的权限  |否  | 否
NIMQChatPermissionTypeRTCChannelOpenMicrophone  |  在实时互动频道开启麦克风的权限  |否 |是
NIMQChatPermissionTypeRTCChannelOpenCamera  |   在实时互动频道开启摄像头的权限   |否  | 是
NIMQChatPermissionTypeRTCChannelOpenCloseOtherMicrophone  |  在实时互动频道开启或关闭他人麦克风的权限 |否  | 否

:::



## 成员权限查询与判定


服务器成员可同时属于多个身份组，同一权限在用户所属的多个身份组中的状态（开启、关闭或继承）不同时，用户实际拥有的最终权限遵循特定的判定规则，具体请参见<a href="https://doc.yunxin.163.com/messaging/guide/Tc5MDI5MzI?platform=iOS" target="_blank">成员权限判定。</a>
