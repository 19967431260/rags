
本文提供初始化 NIM SDK 的详细说明。




## 功能介绍


在使用云信 IM 的各项功能之前，必须先对 NIM SDK 进行初始化。一般情况下，在应用的生命周期内，仅需进行一次初始化。

初始化时，可配置 APNs 离线推送服务、会话已读多端同步、群消息已读和融合存储等重要功能。



## 前提条件

开始 NIM SDK 的初始化前，请确保：

- 已[集成 SDK](https://doc.yunxin.163.com/messaging/guide/TI1NTAzNTk?platform=iOS)。
- 已在云信控制台[创建应用](https://doc.yunxin.163.com/console/docs/TIzMDE4NTA?platform=console)，获取 App Key。
- 已[注册云信 IM 账号](https://doc.yunxin.163.com/messaging/guide/TE2Nzg1MDg?platform=iOS#4-注册-im-账号)，获取 accid 和 token。



## 实现初始化


### 步骤1：引入头文件

在项目文件中引入头文件。

```
#import <NIMSDK/NIMSDK.h>
```

### 步骤2（可选）：配置初始化可选项


您可按需对初始化的可选项进行配置。可选项主要分为`NIMSDKConfig`中的参数、SDK 根目录和`NIMServerSetting`中的配置项。这些可选项如果不配置，将使用默认配置。



#### NIMSDKConfig 配置

#### SDK 根目录配置

除上述`NIMSDKConfig`的配置参数外，您还可调用`NIMSDKConfig`的[`setupSDKDir:`](https://doc.yunxin.163.com/docs/interface/messaging/iOS/doxygen/Latest/zh/d6/d13/interface_n_i_m_s_d_k_config.html#a22d776a9e90b585742a798b0505acc73)方法设置 SDK 的根目录。为了更好的应用体验，SDK 需要对应用数据做本地持久化，如聊天记录、用户信息等。若不设置，数据默认放置于 `$Document/NIMSDK` 目录下。

::: note notice
- 该方法必须在 NIM SDK 任意一次 [`sharedSDK`](https://doc.yunxin.163.com/docs/interface/messaging/iOS/doxygen/Latest/zh/de/de3/interface_n_i_m_s_d_k.html#a4e0700424d6e77e47939e3adfbdbe42c) 方法的调用之前调用，否则 SDK 根目录配置无法生效。
- 调用该方法设置 SDK 根目录后，SDK 产生的临时文件**不会**放置在根目录下。
:::

<br>

示例代码如下：

```objc
[[NIMSDKConfig sharedConfig] setupSDKDir:@"your data path"];
```



#### NIMServerSetting 配置

通过<a href="https://doc.yunxin.163.com/docs/interface/messaging/iOS/doxygen/Latest/zh/df/d1d/interface_n_i_m_server_setting.html">`NIMServerSetting`</a>可对云信服务器进行配置，如 CDN 域名下发、是否是专属云、IM 服务器 LBS 域名等。


::: note notice
`NIMServerSetting`需要开发者自己创建并注入。
:::

- **HTTPS 配置**

    `NIMServerSetting`一个主要功能为**是否开启 HTTPS 支持**（[`httpsEnabled`](https://doc.yunxin.163.com/docs/interface/messaging/iOS/doxygen/Latest/zh/df/d1d/interface_n_i_m_server_setting.html#adbd4d1f1abe2da9d636d726486488227)）。该属性的配置将影响使用 SDK 时上传下载文件的模式。

    - `httpsEnabled`默认为 YES，服务器将使用 HTTPS 上传/下载文件，所有返回的 URL，也会尝试将 HTTP 转换为 HTTPS。

    - `httpsEnabled`若设置为 NO，服务器将使用 HTTP 上传，所有返回的 URL 都只使用原 URL，不做任何处理，同时 `NIMSDKConfig`的[`enabledHttpsForInfo`](https://doc.yunxin.163.com/docs/interface/messaging/iOS/doxygen/Latest/zh/d6/d13/interface_n_i_m_s_d_k_config.html#a5968b9d4313ab44aed81af78de39dc7a) 和 [`enabledHttpsForMessage`](https://doc.yunxin.163.com/docs/interface/messaging/iOS/doxygen/Latest/zh/d6/d13/interface_n_i_m_s_d_k_config.html#a3fe4fb10dc7919eecd3a060ee1abba3a)将无效。

    示例代码如下：

    ```
    NIMServerSetting *setting    = [[NIMServerSetting alloc] init];
    setting.httpsEnabled = YES;
    [[NIMSDK sharedSDK] setServerSetting:setting];
    ...
    [[NIMSDK sharedSDK] registerWithOption:option];
    ```
- **海外数据中心配置**

    ::: note note
    如果你的应用主要服务于海外用户，且数据需要写入海外数据中心，请先在云信控制台[选择服务区域](https://doc.yunxin.163.com/messaging/guide/zI3NjMzNjA?platform=iOS#创建应用并选择服务区域)为海外，并配置私有化服务器地址中的 LBS 和 link 域名相关信息，具体参见[配置 LBS 和 Link 域名](https://doc.yunxin.163.com/messaging/guide/zI3NjMzNjA?platform=iOS#ios-端)。
    ::::


#### NIMQChatConfig 配置

通过<a href="https://doc.yunxin.163.com/docs/interface/messaging/iOS/doxygen/Latest/zh/d3/d35/interface_n_i_m_q_chat_config.html">`NIMServerSetting`</a>可配置圈组初始化实例，如圈组消息缓存、自动订阅功能。


- **圈组消息缓存**

    `NIMQChatConfig`一个主要功能为**是否开启圈组消息缓存**（[`enabledMessageCache`](https://doc.yunxin.163.com/docs/interface/messaging/iOS/doxygen/Latest/zh/d3/d35/interface_n_i_m_q_chat_config.html#a3cadc763bdf26763b70e045f6a7ae91b)）。若需要缓存圈组消息，那么需要在使用圈组消息功能前设置该配置项。

    - `enabledMessageCache` 默认为 NO，服务器将不缓存圈组消息。

    - `enabledMessageCache` 若设置为 YES，那么 SDK 可以为 100 个频道缓存消息，每个频道最多缓存 20 条。如果超出，则按消息的创建时间去除最晚的。

    示例代码如下：

    ```
    [NIMQChatConfig sharedConfig].enabledMessageCache = YES;
    ```
- **圈组自动订阅**

    可通过配置 [`autoSubscribe`](https://doc.yunxin.163.com/docs/interface/messaging/iOS/doxygen/Latest/zh/d3/d35/interface_n_i_m_q_chat_config.html) 来选择圈组的订阅模式。

    - `autoSubscribe` 默认为 NO，即不开启自动订阅，使用手动订阅模式。当用户登录到圈组服务器后，需要先订阅相关服务器或频道，才能收到相关频道和服务器的消息、事件和系统通知，退出时手动取消订阅。

    - `autoSubscribe` 若设置为 YES，即使用自动订阅模式。当用户登录到圈组服务器后，无需手动订阅服务器或频道，进入服务器或频道时即可收到消息、事件和系统通知，退出时则自动取消订阅。

    示例代码如下：

    ```
    [NIMQChatConfig sharedConfig].autoSubscribe=YES;
    ```
    


### 步骤3：调用初始化接口



调用 <a href="https://doc.yunxin.163.com/docs/interface/messaging/iOS/doxygen/Latest/zh/de/de3/interface_n_i_m_s_d_k.html#af48773fab3390f4e2f665740bd51560a">`registerWithOption:`</a> 方法初始化 SDK，推荐在应用程序启动时初始化。调用该方法可配置`NIMSDKOption`的参数。

**<a href="https://doc.yunxin.163.com/docs/interface/messaging/iOS/doxygen/Latest/zh/d1/de8/interface_n_i_m_s_d_k_option.html">`NIMSDKOption`</a>参数说明：**

参数   |类型 | 是否必传  |说明   
---   |----|----|
`appKey`   |NSString  | 是 |   云信控制台获取到的 App Key    
`apnsCername`   |NSString | 否  |    APNs 推送证书名，如不需要实现离线推送可不配置  
`pkCername`   |NSString | 否   |   PushKit  推送证书名，如不需要实现离线推送可不配置

:::note note
- `NIMSDKOption`提供类方法[`optionWithAppKey:`](https://doc.yunxin.163.com/docs/interface/messaging/iOS/doxygen/Latest/zh/d1/de8/interface_n_i_m_s_d_k_option.html#ae74febe6df677430d82f3e3a33370455)来返回实例。
- App Key 作为应用的唯一标识，使用相同 App Key 、不同 Bundle Id 的应用，消息仍可以互通。
:::

<br>

**示例代码：**
```
- (BOOL)application:(UIApplication *)application didFinishLaunchingWithOptions:(NSDictionary *)launchOptions {
    ...
    // 推荐在程序启动的时候初始化 NIM SDK    
    NSString *appKey        = @"your app key";
    NIMSDKOption *option    = [NIMSDKOption optionWithAppKey:appKey];
    option.apnsCername      = @"your APNs cer name";
    option.pkCername        = @"your pushkit cer name";
    [[NIMSDK sharedSDK] registerWithOption:option];
    ...
}
```
