
网易云信 NIM SDK 支持群组中的消息收发，群组相关操作的通知消息的接收，以及群消息通知模式的设置。

## 群消息收发

群组中支持收发多种消息类型。

在创建/加入群组后，用户发送和接收消息的接口与单聊消息收发相同，区别在于会话类型（`sessionType`）的参数配置，选择 `team` 即可。

**[`SessionTypeEnum`](https://doc.yunxin.163.com/messaging/references/flutter/dartdoc/Latest/zh/nim_core/NIMSessionType.html)参数说明**
|枚举常量|说明|
:----|:-----
|p2p|单聊
|team|高级群
|superTeam|超大群
|chatRoom|聊天室
|system|系统通知

具体消息收发的流程，请参见[消息收发](https://doc.yunxin.163.com/messaging/docs/TM3ODEwODc?platform=flutter)。



##  <span id="群组通知消息">群组通知消息</span>

NIM SDK 内置支持的会话消息类型（[`NIMMessageType`](https://doc.yunxin.163.com/messaging/references/flutter/dartdoc/Latest/zh/nim_core/NIMMessageType.html)）中的通知消息（`notification`）主要用于群组、聊天室和超大群的事件通知，由服务端下发，客户端无法发送事件通知消息。 

目前支持触发群通知消息的事件如下：

|[`NIMTeamNotificationTypes`](https://doc.yunxin.163.com/messaging/references/flutter/dartdoc/Latest/zh/nim_core/NIMTeamNotificationTypes-class.html)|附件类|事件说明|
|:---|:---|:---|
|acceptInvite |NIMMemberChangeAttachment|接受邀请后入群（需要被邀请人同意的模式）|
|inviteMember |NIMMemberChangeAttachment|邀请成员入群（无需被邀请人同意的模式）|
|addTeamManager | NIMMemberChangeAttachment|添加管理员 |
|kickMember | NIMMemberChangeAttachment|被踢出群|
|transferOwner |NIMMemberChangeAttachment|转让群主|
|passTeamApply |NIMMemberChangeAttachment|申请加入群成功|
|removeTeamManager |NIMMemberChangeAttachment|移除管理员|
|dismissTeam | NIMDismissAttachment|解散群|
|leaveTeam |NIMLeaveTeamAttachment|退出群 |
|muteTeamMember |NIMMuteMemberAttachment|群内禁言/解禁 |
|updateTeam | NIMUpdateTeamAttachment|群信息更新|




## 群通知消息解析步骤

1. 通过 [`NIMMessage.messageType`](https://doc.yunxin.163.com/messaging/references/flutter/dartdoc/Latest/zh/nim_core/NIMMessage/messageType.html) 方法获取消息类型，若为 `notification` ，则为通知消息。
2. 将 [`NIMMessage.messageAttachment`](https://doc.yunxin.163.com/messaging/references/flutter/dartdoc/Latest/zh/nim_core/NIMMessage/messageAttachment.html) 方法获取的附件对象强类型转换为 `NIMTeamNotificationAttachment`。
3. 通过 [`NIMTeamNotificationAttachment.type`](https://doc.yunxin.163.com/messaging/references/flutter/dartdoc/Latest/zh/nim_core/NIMTeamNotificationAttachment/type.html) 方法获取具体的通知类型（`TeamNotificationTypes`）。
4. 根据对应的 [`TeamNotificationTypes`](https://doc.yunxin.163.com/messaging/references/flutter/dartdoc/Latest/zh/nim_core/NIMTeamNotificationTypes-class.html) 将 `NIMMessage.messageAttachment` 得到的附件对象强转为对应的附件类（见上方表格）。


:::note Notice
- 针对 [`NIMMemberChangeAttachment`](https://doc.yunxin.163.com/messaging/references/flutter/dartdoc/Latest/zh/nim_core/NIMMemberChangeAttachment-class.html)，通过 [`targets`](https://doc.yunxin.163.com/messaging/references/flutter/dartdoc/Latest/zh/nim_core/NIMMemberChangeAttachment/targets.html) 方法可以获取该事件的承受者。如 B 被踢出群，则通过 `targets` 可以获取到 B 的用户账号。
- 针对 [`NIMMuteMemberAttachment`](https://doc.yunxin.163.com/messaging/references/flutter/dartdoc/Latest/zh/nim_core/NIMMuteMemberAttachment-class.html)，其为 `NIMMemberChangeAttachment` 的子类，可通过 `targets` 可以获取该事件的承受者，可通过 [`mute`](https://doc.yunxin.163.com/messaging/references/flutter/dartdoc/Latest/zh/nim_core/NIMMuteMemberAttachment/mute.html) 判断是禁言还是被解禁。
- 针对 [`NIMUpdateTeamAttachment`](https://doc.yunxin.163.com/messaging/references/flutter/dartdoc/Latest/zh/nim_core/NIMUpdateTeamAttachment-class.html)，需要解析到具体是哪个群信息被更新。群整体禁言属于此类型。
:::

解析被更新的群信息，示例代码如下：

```
    NimCore.instance.messageService.onMessage.listen((event) {
      print('Test_Observer onMessage ${event.toString()}');
      for (NIMMessage m in event) {
        if(message.messageType = NIMMessageType.notification){
         if (type >= NIMTeamNotificationTypes.inviteMember &&
            type <= NIMTeamNotificationTypes.muteTeamMember) {
               ///TODO  参考支持触发群通知消息的事件
          }
        }
      }
    });
```


##  <span id="群消息免打扰">群消息免打扰</span>

SDK 支持对群消息通知提醒模式的配置。群消息通知提醒模式（[`NIMTeamMessageNotifyTypeEnum`](https://doc.yunxin.163.com/messaging/references/flutter/dartdoc/Latest/zh/nim_core/NIMTeamMessageNotifyTypeEnum.html)）分为：
- all：全部提醒（默认）
- manager：仅群主/管理员消息提醒
- mute：全部不提醒

以上通知提醒指的是云信体系内的[推送与消息提醒](https://doc.yunxin.163.com/messaging/docs/zQ5NzIwODk?platform=flutter)。

:::note notice
群消息的提醒模式不影响群消息的接收和未读数的变化。若将提醒模式设置为全部不提醒，仍然能接收到群消息接收，未读数仍会变化。
:::
###  <span id="设置免打扰">设置免打扰</span>

通过调用 [`muteTeam`](https://doc.yunxin.163.com/messaging/references/flutter/dartdoc/Latest/zh/nim_core/TeamService/muteTeam.html) 来设置指定群消息通知类型。


**参数说明：**

| 参数 | 说明     |
|  ----  | --------- |
|teamId | 群ID |
|notifyType | 通知类型枚举<br/>all：全部提醒（默认）<br/>manager：仅群主/管理员消息提醒<br/>mute：全部不提醒|


**示例代码：**

```
var result = await NimCore.instance.teamService.muteTeam(
    teamId,
    NIMTeamMessageNotifyTypeEnum.mute);
```


###  <span id="查询免打扰状态">查询免打扰状态</span>

可通过 [`NIMTeam.messageNotifyType`](https://doc.yunxin.163.com/messaging/references/flutter/dartdoc/Latest/zh/nim_core/NIMTeam/messageNotifyType.html) 获取对应群的消息通知提醒模式。
