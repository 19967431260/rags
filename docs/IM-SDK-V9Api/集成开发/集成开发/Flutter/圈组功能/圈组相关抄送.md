
圈组支持将消息和系统通知实时同步至您预设的服务器，满足您的应用对于业务数据本地存储的需求。更多抄送相关介绍，请参见<a href="https://doc.yunxin.163.com/messaging/docs/jQ4ODE0NTY?platform=server" target="_blank">消息抄送服务概述</a>。



## 前提条件

已开通消息抄送服务，且已在 IM 即时通讯的**抄送信息类型**中勾选需要抄送的圈组事件，具体请参见[开通消息抄送](https://doc.yunxin.163.com/messaging/docs/jY5MDk1NTQ?platform=server)。 

## 实现方法

### 客户端实现


调用<a href="https://doc.yunxin.163.com/messaging/references/flutter/dartdoc/Latest/zh/nim_core/QChatMessageService/sendMessage.html" target="_blank">`sendMessage`</a>方法</a><a href="https://doc.yunxin.163.com/messaging/docs/jUzMDAyNDU?platform=flutter#实现消息收发" target="_blank">发送圈组消息</a>或调用<a href="https://doc.yunxin.163.com/messaging/references/flutter/dartdoc/Latest/zh/nim_core/QChatMessageService/sendSystemNotification.html" target="_blank">`sendSystemNotification`</a>方法<a href="https://doc.yunxin.163.com/messaging/docs/TQ2MTAwNjk?platform=flutter#收发圈组自定义系统通知" target="_blank">发送自定义系统通知</a>时，可通过以下参数配置抄送。


<div style="width:120px">参数</div> | <div style="width:90px">类型</div> | 说明
---- | -------------- | ---------
`env` | String |   当前消息或自定义系统通知需要抄送到的环境的名称，对应您在云信控制台中配置的自定义抄送的环境名称（如下图），最大 32 个字符 ![自定义抄送环境.png](https://yx-web-nosdn.netease.im/common/67dfaa431a321abfb77e5f238da47b62/自定义抄送环境.png)
`isRouteEnable` |  bool | 是否需要将当前消息或自定义系统通知抄送至指定的环境
  
示例代码如下：


```
 var paramSendSys = QChatSendSystemNotificationParam(serverId: serverId);
    paramSendSys.isRouteEnable = true;
    paramSendSys.env = '环境变量';
    NimCore.instance.qChatMessageService.sendSystemNotification(paramSendSys).then((value){
      if(value.isSuccess){
        //todo system notification send success
      }
    });
```



### 服务端实现


您也可通过服务端实现抄送功能。除了圈组的消息和自定义系统通知，服务端还支持抄送圈组内置系统通知，如圈组登录/登出、创建频道和成员加入服务器等。详情请参见<a href="https://doc.yunxin.163.com/messaging/docs/zQ0NTQ4NTk?platform=server" target="_blank">圈组相关抄送</a>。
