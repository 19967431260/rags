
圈组服务器（server）并非传统意义上的“<a href="https://vibaike.com/4051/" target="_blank">服务器</a>”，它是社群本身，是关系的集合。圈组内所有的内容、兴趣、话题、关系都以服务器为基础进行发展。圈组内任何行为开始前，都应该先创建一个圈组服务器。用户只能在服务器维度邀请好友，换而言之，用户的好友加入的是圈组服务器。

## 服务器数据结构

SDK 内定义服务器的结构为<a href="https://doc.yunxin.163.com/messaging/references/flutter/dartdoc/Latest/zh/nim_core/QChatServer-class.html" target="_blank">`QChatServer`</a>，**部分**重要参数说明如下：



<div style="width:160px">参数</div>| 类型 |说明
---- | -------------- | ---------
`applyMode` |`QChatApplyJoinMode`  | 加入服务器的申请模式：<ul><li>`QChatApplyJoinMode.agreeNeedNot`：申请不需要同意（默认），即用户申请加入服务器后立即自动加入服务器</li><li>`QChatApplyJoinMode.agreeNeed`：申请需要同意，即用户发起加入服务器的申请后，需要服务器管理人员同意后才能加入</li></ul><br><p>申请加入服务器相关文档请参见<a href="https://doc.yunxin.163.com/TM5MzM5Njk/docs/jc4ODY5MDA?platform=flutter#申请加入" target="_blank">申请加入</a>。</p>
`inviteMode` | `QChatInviteMode` | 加入服务器的邀请模式：<ul><li>`QChatInviteMode.agreeNeed`：邀请需要同意（默认），即邀请用户加入服务器后，需要被邀请的用户同意，该用户才能被加入服务器</li><li>`QChatInviteMode.agreeNeedNot`：邀请不需要同意，即邀请用户加入服务器后，被邀请的用户立即自动被加入服务器</li></ul><br><p>邀请用户加入服务器相关文档请参见<a href="https://doc.yunxin.163.com/TM5MzM5Njk/docs/jc4ODY5MDA?platform=flutter#邀请加入" target="_blank">邀请加入</a>。</p>
`channelNum`  |int | 服务器下的频道数量
`createTime` | int | 服务器的创建时间 
`custom` | String  | 服务器的自定义扩展字段 
`icon` | String  | 服务器的图标
`owner` | String | 服务器所有者，默认为创建者
`name` |String | 服务器名称 
`memberNumber`| int |服务器成员数量
