## <span id="客户端反垃圾">客户端反垃圾</span>

Unity SDK 1.4.0 版本开始提供客户端反垃圾功能，该服务需要用户在服务器开通并且通过管理后台配置反垃圾过滤词库，支持关键词和正则表达式两种方式。类`NIM.ToolsAPI`中提供了文本替换了匹配的接口，在需要进行垃圾过滤时提前调用接口对文本进行处理。

### 文本替换

在词库中如果存在与目标文本匹配的项则使用替换字符串进行替换，函数返回替换后的字符串。词库名是在管理后台配置的词库名称。

- API原型

```csharp
/// <summary>
/// 替换在词库中匹配的字符串
/// </summary>
/// <param name="text">目标文本</param>
/// <param name="replace">替换字符串</param>
/// <param name="libName">词库名称</param>
/// <returns>替换后的字符串</returns>
public static string ReplaceTextMatchedKeywords(string text,string replace,string libName)
```
::: note note
- PC 端需要传入`libName`参数，值为`本地拦截库`、`服务端拦截库`或`本地替换库`（三选一）。
- 移动端不需要传入`libName`参数。
:::
### 文本匹配

检查词库中是否存在与目标文本相匹配的模式。

- API原型

```csharp
/// <summary>
/// 字符串是否匹配词库中的模式
/// </summary>
/// <param name="text">目标文本</param>
/// <param name="libName">词库名称</param>
/// <returns></returns>
public static bool IsTextMatchedKeywords(string text,string libName)
```
::: note note
- PC 端需要传入`libName`参数，值为`本地拦截库`、`服务端拦截库`或`本地替换库`（三选一）。
- 移动端不需要传入`libName`参数。
:::


### 发消息设置

消息类增加了`ClientAntiSpamHitting`属性用来指示所发送内容是否命中了客户端发垃圾，该属性设置为*true*所发送内容到达服务器后不会再转发到消息接收方，设置为*false*或者不设置该属性与正常消息流程一致。
