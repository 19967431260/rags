 

网易云信 IM SDK（NetEase Instant Messaging SDK，NIM SDK）为微信小程序提供完善的即时通信功能开发能力和简洁的 API，方便您快速集成即时通信功能。


本文介绍如何快速将 NIM SDK 集成到您的小程序项目中。


::: note note 
由于浏览器环境的全局变量为`window`，而小程序的全局变量为`wx`，其属性不尽相同，为了做到兼容及适配，NIM SDK 会 mock 一些属性，诸如`navigator`、`location`、`io`等，一般不影响用户正常使用。
:::









## 前提条件


已在[微信公众平台](https://mp.weixin.qq.com/?token=&lang=zh_CN)中进入**小程序后台 > 开发 > 开发设置 > 服务器域名**，将以下域名填入指定的 “request 合法域名” / “socket 合法域名” / “uploadFile 合法域名” / “downloadFile 合法域名” 中。

## 集成 SDK


### 集成步骤




1. 通过如下 npm 命令安装最新版 SDK。

    ```
    npm install @yxim/nim-web-sdk@latest
    ```
    
    SDK 解压后可得到以下三个文件。请将文件放置到小程序工程目录中。

    ```
    dist/SDK
    ├── NIM_Web_Chatroom_miniapp.js  提供聊天室功能，小程序适配版（UMD 格式）
    ├── NIM_Web_NIM_miniapp.js  提供 IM 功能，包括单聊、会话、群聊等，小程序适配版（UMD 格式）
    ├── NIM_Web_SDK_miniapp.js  同时提供 IM 功能和聊天室功能的集成包，小程序适配版（UMD 格式）
    ```

2. 在小程序的 js 脚本中，通过 require 方式引入所需的 SDK 文件。


    ::: note note 
    具体如何根据业务需求选择 SDK 文件及相关注意事项，请参见下文的[SDK 文件选择](#SDK文件选择)。
    :::

    以下示例为引入同时包含 IM 对象和聊天室对象的 SDK 文件：

    ```
    // index.js
    // 获取应用实例
    const SDK = require('../../vendors/NIM_Web_SDK_miniapp.js')

    Page({
      // 事件处理函数
      connect() {
        const nim = SDK.NIM.getInstance({
            // ...
        })
      }
    })
    ```



### SDK文件选择



请根据下表的说明引入您所需的 SDK 文件。初始化不同 SDK 文件，需调用不同的 SDK API。



业务需求 | 需引入的模块 | 初始化方法
---- | -------------- | ---------
仅需常规 IM （单聊、群聊等）|  `NIM_Web_NIM_miniapp_vX.X.X.js` | 通过[`NIM.getInstance`](https://doc.yunxin.163.com/docs/interface/messaging/web/typedoc/Latest/zh/NIM/classes/nim.NIM.html#getInstance)方法初始化 IM 实例
仅需聊天室   |  `NIM_Web_Chatroom_miniapp_vX.X.X.js`  |通过[`Chatroom.getInstance`](https://doc.yunxin.163.com/docs/interface/messaging/web/typedoc/Latest/zh/Chatroom/classes/chatroom.Chatroom.html#getInstance)方法初始化聊天室实例
常规 IM 和聊天室都需要   | `NIM_Web_SDK_miniapp_vX.X.X.js`|  <ul><li>通过[`SDK.NIM.getInstance`](https://doc.yunxin.163.com/docs/interface/messaging/web/typedoc/Latest/zh/NIM/classes/nim.NIM.html#getInstance)方法初始化 IM 实例</li><li>通过[`SDK.Chatroom.getInstance`](https://doc.yunxin.163.com/docs/interface/messaging/web/typedoc/Latest/zh/Chatroom/classes/chatroom.Chatroom.html#getInstance)方法初始化聊天室实例</li><ul>

::: note important 
引入 SDK 文件时，**请勿同时引入以下两种文件组合**：

- `NIM_Web_SDK_miniapp_vX.X.X.js` 和 `NIM_Web_Chatroom_miniapp_v.X.X.X.js`
- `NIM_Web_SDK_miniapp_vX.X.X.js` 和 `NIM_Web_NIM_miniapp_vX.X.X.js`
:::

## 相关信息



### SDK 使用须知

#### <span id="实例调用方式">实例调用方式</span>

所有业务均通过 NIM SDK 单例调用，例如：

```javascript
  // 引入SDK类的引用以后，获取SDK对象实例
  var nim = SDK.NIM.getInstance({
    debug: true,
    appKey: appKey,
    account: account,
    token: token,
    // ...
  });
```

以发送点对点消息为例：

```javascript
  var msg = nim.sendText({
    scene: 'p2p',
    to: account,
    text: 'hello',
    done: function sendMsgDone (error, msg) {
      // ...
    }
  });
```

#### <span id="事件通知方式">事件通知方式</span>

SDK 通过两种方式通知上层 API 调用结果：回调(callback)和委托 (delegate)，两种方式都只在主线程触发。

* 一般回调接口直接反映在对应接口的 `done` 参数上，调用时设置即可。
* 委托则需要开发者在合适时机在初始化异步监听函数上进行处理

```javascript
  // 委托通知示例
  var nim = NIM.getInstance({
    // ... 此处省略其他配置
    onmsg: function (msg) {
      // 此处为委托消息事件，消息发送成功后，成功消息也在此处处理
    }
  });


  // 回调通知示例
  var msg = nim.sendText({
    scene: 'p2p',
    to: account,
    text: 'hello',
    done: function sendMsgDone (error, msg) {
      // 此处为回调消息事件，仅仅通知开发者，消息是否发送成功
    }
  });
```
