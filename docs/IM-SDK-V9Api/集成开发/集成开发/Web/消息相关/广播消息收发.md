网易云信 NIM Web SDK 支持接收广播消息。广播消息适用于需要通知所有用户的业务场景，如企业管理层通过内部即时通讯软件向全体员工发送内部通知。


## 技术原理 

广播消息由服务端 API（[`msg.broadcastMsg.action`](https://doc.yunxin.163.com/messaging/guide/Dg1NDYxNTM?platform=server)）发起，SDK 收到广播消息之后触发回调直接往应用上层通知，对应用内的所有用户发送一条广播消息。




<img style="width:75%" src="https://yx-web-nosdn.netease.im/common/ed54b769b586420ca130e76123e05c4a/广播消息收发.png" />


::: note note 
广播消息支持离线存储，如果发送时设置了存离线，则该消息具有有效期(默认7天，可自行配置)。更换设备登录，或者同一设备卸载重装应用，都能重新收到有效期内的广播消息。
:::



## 前提条件 



- 广播消息功能需要单独开通，进行相关开发前，请确保您已在[云信控制台](https://app.yunxin.163.com/)**应用管理 > 产品功能 > IM 即时通讯 > 全局功能**开通**全员广播**功能。
- 请确保您使用的 SDK 为 4.3.0 以上版本，4.3.0 之前版本不支持接收广播消息。
- 已集成 SDK。
- 已[注册云信 IM 账号](https://doc.yunxin.163.com/messaging/guide/DU1MTQxNDU?platform=web#4-注册-im-账号)，获取 accid 和 token。


## 使用限制 

- SDK 不支持发送广播消息，仅支持通过 IM 服务端 API（[`msg.broadcastMsg.action`](https://doc.yunxin.163.com/messaging/guide/Dg1NDYxNTM?platformId=60353)）发送，且一个应用一分钟最多调用10次，一天最多调用 1000000 次，超过会返回 416 错误码。如需上调上限，请在官网首页通过微信、在线消息或电话等方式咨询商务人员。
- 未登录 IM 的用户(例如仅通过独立模式进入聊天室的用户)无法收到广播消息。
- 最多保留最近 10000 条离线广播消息；
- 广播消息不支持推送、漫游和生成会话，SDK 不支持保存广播消息历史记录。
- 广播消息不支持客户端存储。


## 实现流程 


1. 消息接收者在调用<a href="https://doc.yunxin.163.com/docs/interface/messaging/web/typedoc/Latest/zh/NIM/classes/nim.NIM.html#getInstance" target="_blank">`getInstance`</a>方法进行 SDK 初始化时，注册`onBroadCastMsg`和`onBroadCastMsgs`回调。


    回调 |说明
    :---- | :-------------- 
    `onbroadcastmsg` | 接收者**在线**时，收到广播消息的回调
    `onbroadcastmsgs`  | 同步到**离线**广播消息的回调，仅当`NIMGetInstanceOptions.syncBroadcastMsgs=true`时，才会同步离线广播消息

  

    ::: note note
    初始化的完整示例代码，请参见<a href="https://doc.yunxin.163.com/messaging/guide/zE0NDY4Njc?platform=web#示例代码" target="_blank">初始化示例代码</a>。
    :::
    
2. 消息发送者调用[`msg.broadcastMsg.action`](https://doc.yunxin.163.com/messaging/guide/Dg1NDYxNTM?platform=server)发送一条广播消息。
3. SDK 触发`onBroadCastMsg`或`onBroadCastMsgs`回调，接收者通过这两个回到接收广播消息。

    
    <div style="width:100px">广播消息参数</div> |   说明
    ---- |----
    `broadcastId` | 广播消息 ID
    `body` | 广播消息体
    `fromAccid` | 发送该广播消息的账号
    `time` | 广播消息发送时间戳
