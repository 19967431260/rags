

如果您的应用 UI 上需要展示频道未读数，可参考本文介绍的流程获取频道内的消息未读信息并管理其变化。 


::: note notice
游客接收到的消息无已读未读逻辑。不支持对游客展示消息的未读数。
:::

## 未读信息定义

NIM SDK 中的<a href="https://doc.yunxin.163.com/docs/interface/messaging/pc/doxygen/Latest/zh/struct_n_i_m_q_chat_unread_info.html" target="_blank">`NIMQChatUnreadInfo`</a>类定义了圈组频道的未读信息。
</details>

您可通过多个方式获取`NIMQChatUnreadInfo`，具体方法参见本文的[实现频道未读数持续监听](#频道未读数持续监听实现方法 )和[实现频道未读数的直接获取](#实现频道未读数的直接获取)。



## 前提条件

开始集成前，请确保：

- 已注册[`nim::Channel::RegUnreadCb`](https://doc.yunxin.163.com/docs/interface/messaging/pc/doxygen/Latest/zh/classnim_1_1_channel.html#a90cd84b1d1d671d8b8c5a6b49d938d20)，监听频道未读信息变更事件（[`QChatChannelUnreadResp`](https://doc.yunxin.163.com/docs/interface/messaging/pc/doxygen/Latest/zh/structnim_1_1_q_chat_channel_unread_resp.html)），该事件信息中包含未读信息`NIMQChatUnreadInfo`。
- 已在频道内<a href="https://doc.yunxin.163.com/messaging/docs/zYzNzc0OTA?platform=pc#实现消息收发" target="_blank">发送消息</a>。


## 实现流程
  
### 实现频道未读数持续监听 
本文通过以下 UI 简图的场景为例，说明如何完成**频道 1** 未读数的监听、未读数的清零以及清零后的持续监听。 

<img style="width:70%" src="https://yx-web-nosdn.netease.im/common/e90c37bc94e6afa0fafd530d62e92645/WebElite未读管理示例.png" />

#### 步骤1: 监听未读数
您需要订阅 **频道 1** 的未读数才能实现监听。完成如下两个子步骤后，当有新的消息下发到**频道 1** ，SDK 将触发频道未读信息变更事件 `QChatChannelUnreadResp` 并上报。

::: note notice :::
订阅的频道仅在当前登录期间有效（因网络波动导致的断线自动重连除外）。如您调用`Logout`登出后，需要重新订阅。
:::
1. 进入**服务器 1** 后，调用<a href="https://doc.yunxin.163.com/docs/interface/messaging/pc/doxygen/Latest/zh/classnim_1_1_channel.html#a310e32a34028a5c2a2559d5ced3849ad" target="_blank">`GetChannelsByPage`</a>方法刷新频道列表，得到**频道 1**、**频道 2**、**频道 3** 和 **频道 4**。
2. 调用<a href="https://doc.yunxin.163.com/docs/interface/messaging/pc/doxygen/Latest/zh/classnim_1_1_channel.html#af876b23eb1eb00faccbc859e5a352c9a" target="_blank">`nim::Channel::Subscribe`</a>方法订阅**频道 1**。
    调用该方法时入参配置如下：
    参数 | 类型 | 说明
    ---- | -------------- 
    `id_infos`| [`NIMQChatChannelIDInfo`](https://doc.yunxin.163.com/docs/interface/messaging/pc/doxygen/Latest/zh/struct_n_i_m_q_chat_channel_i_d_info.html) * | 频道 ID 列表：传入您需要订阅的频道 ID 与该频道所属的服务器的 ID 组成的未读信息列表
    `ope_type`| [`NIMQChatSubscribeOpeType`](https://doc.yunxin.163.com/docs/interface/messaging/pc/doxygen/Latest/zh/nim__qchat__public__def_8h.html#af0eab9ffe459c7edd90e193aee92ba1f) |操作类型：订阅或取消订阅
    `sub_type` |[`NIMQChatSubscribeType`](https://doc.yunxin.163.com/docs/interface/messaging/pc/doxygen/Latest/zh/nim__qchat__public__def_8h.html#a9ddcfda12a811d11124bdb2798a392d3) | 订阅类型，本场景下需传入`kNIMQChatSubscribeTypeUnreadCount`订阅频道的未读数。 
    
    该两步的示例代码如下：
    ```
    QChatChannelGetChannelsPageParam param;
    param.server_id = 1;
    param.timestamp = 0;
    param.limit = 100;
    param.cb = [](const QChatChannelGetChannelsPageResp& resp) {
        if (resp.res_code != NIMResCode::kNIMResSuccess) {
            // error handling
            return;
        }
        // 记录查到的频道列表
        // ...
    };
    Channel::GetChannelsByPage(param);
    ```
    ```
    QChatChannelSubscribeParam param;
    param.ope_type = kNIMQChatSubscribeOpeTypeSubscribe;
    param.sub_type = kNIMQChatSubscribeTypeMsg;
    NIMQChatChannelIDInfo id_info;
    id_info.server_id = 1;
    id_info.channel_id = 1;
    param.id_infos.push_back(id_info);
    param.cb = [](const QChatChannelSubscribeResp& resp) {
        if (resp.res_code != NIMResCode::kNIMResSuccess) {
            // error handling
            return;
        }
        // process response
        // ...
    };
    Channel::Subscribe(param);
    ```


#### 步骤2：清除未读数
以下说明以清除**频道 1** 的未读数为例，介绍在完成**步骤1: 监听未读数**后，如何清除频道未读数以及清除后对未读数进行持续监听。 
1. 进入**频道 1** 后，记住您应用中“代表该频道的属性”（假设为`currentChannel`）。
2. 调用<a href="https://doc.yunxin.163.com/docs/interface/messaging/pc/doxygen/Latest/zh/classnim_1_1_message.html#a6c7856a2eea486f5f11d7fe0a82b4559" target="_blank">`nim::Message::MarkRead`</a>方法，调用时传`timestamp`参数为已读的最新消息的时间戳（Unix 时间戳），进行未读数的清零操作。



    未读数清零时，SDK 触发`QChatChannelUnreadResp`事件，该事件包含的未读数参数`unread_count` 为`0`。

    ::: note notice
    - 针对同一个频道，调用`nim::Message::MarkRead`方法存在频控限制，200 ms 内只能调用一次。
    - 如果调用`nim::Message::MarkRead`时传`timestamp`参数为`0`，会将频道内所有消息标记为未读。
    :::

    <br>

    示例代码如下：

    ```
    QChatMarkMessageReadParam param;
    param.id_info.server_id = 123456;
    param.id_info.channel_id = 123456;
    param.timestamp = 123456;
    param.cb = [](const QChatMarkMessageReadResp& resp) {
        if (resp.res_code != NIMResCode::kNIMResSuccess) {
            // error handling
            return;
        }
        // process response
        // ...
    };
    Message::MarkRead(param);

    ```




3. 根据您的应用逻辑做出判断和处理：当`currentChannel`与`QChatChannelUnreadResp`事件里的`channel_id`相同时，每当频道 1 有新消息接收时，仍旧会触发该事件。

### 实现频道未读数的直接获取
如果您不想订阅某个频道，而只想获取该频道当前的未读数，完成以下操作即可。
1. 确保您已在当前频道中。
2. 直接调用<a href="https://doc.yunxin.163.com/docs/interface/messaging/pc/doxygen/Latest/zh/classnim_1_1_channel.html#aa36e153f13498309e17be970f0ffe12b" target="_blank">`QueryUnreadInfo`</a>查询当前频道未读信息。

    :::note note
    - 该方法调用存在频控，200ms 内最多可调用一次。
    - 该方法单次最多查询频道数量为 100。
    :::

    <br>

    示例代码：


    ```
    QChatChannelQueryUnreadInfoParam param;
    NIMQChatChannelIDInfo id_info;
    id_info.server_id = 123456;
    id_info.channel_id = 123456;
    param.id_infos.push_back(id_info);
    param.cb = [](const QChatChannelQueryUnreadInfoResp& resp) {
        if (resp.res_code != NIMResCode::kNIMResSuccess) {
            // error handling
            return;
        }
        // process response
        // ...
    };
    Channel::QueryUnreadInfo(param);
    ```


  

## 常见问题

### 1. 什么操作会导致频道未读数增加？
您当前所在的频道中，有用户发送消息或更新消息，对应频道中的当前用户未读信息中的未读数会增加。以下特殊情情况除外：

- 消息发送者是自己
- 不存云端消息历史
- 不需要消息计数
- 未订阅当前频道
 

### 2. 什么操作会导致频道未读数减少？
您当前所在的频道中，有用户删除消息，对应频道中的当前用户未读信息中的未读数会减少。以下特殊情况除外：
- 删除的是自己发的消息，则未读数不变。
- 未订阅当前频道。

### 3. 撤回消息和重发消息是否影响未读数？
这两种操作都不影响未读数。


### 4. 通过`nim::Message::MarkRead`方法标记频道消息已读，对未读数有什么影响？
如果在当前频道通过该方法标记消息已读，会更新对应频道的未读信息。

::: note notice 
调用该方法时传入`timestamp`参数（标记消息已读的时间戳），不仅可能减少未读数，还可能增加未读数。例如将`timestamp`设为`0`时，会标记所有消息未读从而增加未读数。
:::
