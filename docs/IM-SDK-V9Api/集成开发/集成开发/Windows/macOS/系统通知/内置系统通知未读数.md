<!--keywords: 未读数,监听,获取,标记,已读,内置系统通知-->

内置系统通知未读数，指在云信系统中，用户接收到的内置系统通知未读总数。本文介绍如何获取内置系统通知未读数和标记已读并提供相应的示例代码。


## 前提条件
已监听系统通知接收事件的回调，具体请参见<a href="https://doc.yunxin.163.com/messaging/docs/TE2OTMyMzI?platform=pc#监听系统通知" target="_blank">监听系统通知</a>。

## <span id="查询内置系统通知未读数">查询内置系统通知未读数</span>


通过调用 [`QueryUnreadCount`](https://doc.yunxin.163.com/messaging/references/pc/doxygen/Latest/zh/classnim_1_1_system_msg.html#acfc6be592e965c3ce3c67ed18814fbd6) 方法查询内置系统通知未读总数。

**示例代码：**

```
void OnQuerySysmsgUnreadCb(nim::NIMResCode res_code, int unread_count)
{
	if (res_code == 200)
		···
}

void foo()
{
	nim::SystemMsg::QueryUnreadCount(&OnQuerySysmsgUnreadCb);
}
```
::: note note
仅在本地数据库中进行查询。
:::


## <span id="标记所有内置系统通知已读">标记所有内置系统通知已读</span>

通过调用 [`ReadAllAsync`](https://doc.yunxin.163.com/messaging/references/pc/doxygen/Latest/zh/classnim_1_1_system_msg.html#a3567dbdf27597839f4719bf2e046facb) 方法将所有的内置系统通知标记为已读，即调用后内置系统通知未读数为零。

**示例代码：**

```
void SysMsgReadAllCb(nim::NIMResCode code, int unread)
{

}
void foo()
{
	nim::SystemMsg::ReadAllAsync(&SysMsgReadAllCb);
}
```
