
圈组支持将消息和系统通知实时同步至您预设的服务器，满足您的应用对于业务数据本地存储的需求。更多抄送相关介绍，请参见<a href="https://doc.yunxin.163.com/messaging/guide/jQ4ODE0NTY?platform=server" target="_blank">消息抄送服务概述</a>。



## 前提条件

已开通消息抄送服务，且已在 IM 即时通讯的**抄送信息类型**中勾选需要抄送的圈组事件，具体请参见[开通消息抄送](https://doc.yunxin.163.com/messaging/guide/jY5MDk1NTQ?platform=server)。

## 实现方法

### 客户端实现


调用<a href="https://doc.yunxin.163.com/docs/interface/messaging/android/doxygen/Latest/zh/interfacecom_1_1netease_1_1nimlib_1_1sdk_1_1qchat_1_1_q_chat_message_service.html#a50851a1367f29b3162f2ae3afcf48624" target="_blank">`sendMessage`</a>方法</a><a href="https://doc.yunxin.163.com/messaging/guide/TE1MjI2MDI?platform=android#%E5%AE%9E%E7%8E%B0%E6%B6%88%E6%81%AF%E6%94%B6%E5%8F%91" target="_blank">发送圈组消息</a>或调用<a href="https://doc.yunxin.163.com/docs/interface/messaging/android/doxygen/Latest/zh/interfacecom_1_1netease_1_1nimlib_1_1sdk_1_1qchat_1_1_q_chat_message_service.html#a4ed011f932cfa8b849adacc0caefe47c" target="_blank">`sendSystemNotification`</a>方法<a href="https://doc.yunxin.163.com/messaging/guide/TA4NjcwNTc?platform=android#%E5%8F%91%E9%80%81%E8%87%AA%E5%AE%9A%E4%B9%89%E7%B3%BB%E7%BB%9F%E9%80%9A%E7%9F%A5" target="_blank">发送自定义系统通知</a>时，可通过以下参数配置抄送。

<div style="width:100px">参数</div> | <div style="width:100px">类型</div> | 说明
---- | -------------- | ---------
`setEnv` | String |  设置当前消息或自定义系统通知需要抄送到的环境的名称，对应您在云信控制台中配置的自定义抄送的环境名称（如下图），最大 32 个字符 ![自定义抄送环境.png](https://yx-web-nosdn.netease.im/common/e10e439e8a4c2d192c3af42d53a13e5e/环境名称.png)
`isRouteEnable` | Boolean | 是否需要将当前消息或自定义系统通知抄送至指定的环境，默认 true
  




### 服务端实现


您也可通过服务端实现抄送功能。除了圈组的消息和自定义系统通知，服务端还支持抄送圈组内置系统通知，如圈组登录/登出、创建频道和成员加入服务器等。详情请参见<a href="https://doc.yunxin.163.com/messaging/guide/zQ0NTQ4NTk?platform=server" target="_blank">圈组相关抄送</a>。
