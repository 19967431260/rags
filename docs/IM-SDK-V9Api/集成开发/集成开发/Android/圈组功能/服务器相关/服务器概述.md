圈组服务器（server）并非传统意义上的 IT 基础设施服务器，而是指社群本身，是关系的集合。圈组内所有的内容、兴趣、话题、关系都以服务器为基础进行发展。圈组内任何行为开始前，都应该先创建一个圈组服务器。用户只能在服务器维度邀请好友，换而言之，用户的好友加入的是圈组服务器。

## 数据结构

SDK 内定义服务器的结构为 <a href="https://doc.yunxin.163.com/docs/interface/messaging/android/doxygen/Latest/zh/interfacecom_1_1netease_1_1nimlib_1_1sdk_1_1qchat_1_1model_1_1_q_chat_server.html" target="_blank">`QChatServer`</a>，**部分** 重要参数说明如下：

<div style="width:180px">参数</div> | <div style="width:160px">类型</div> | 说明
---- | ---- | ----
`getapplyMode` | `QChatApplyJoinMode` | 加入服务器的申请模式：<ul><li>`QChatApplyJoinMode.agreeNeedNot`：申请不需要同意（默认），即用户申请加入服务器后立即自动加入服务器</li><li>`QChatApplyJoinMode.agreeNeed`：申请需要同意，即用户发起加入服务器的申请后，需要服务器管理人员同意后才能加入</li></ul>申请加入服务器相关文档请参考 <a href="https://doc.yunxin.163.com/messaging/guide/DIzODU1MDQ?platform=android#申请加入" target="_blank">申请加入</a>。
`getInviteMode` | `QChatInviteMode` | 加入服务器的邀请模式：<ul><li>`QChatInviteMode.agreeNeed`：邀请需要同意（默认），即邀请用户加入服务器后，需要被邀请的用户同意，该用户才能被加入服务器</li><li>`QChatInviteMode.agreeNeedNot`：邀请不需要同意，即邀请用户加入服务器后，被邀请的用户立即自动被加入服务器</li></ul>邀请用户加入服务器相关文档请参考 <a href="https://doc.yunxin.163.com/messaging/guide/DIzODU1MDQ?platform=android#邀请用户加入" target="_blank">邀请加入</a>。
`getChannelNum` | int | 服务器下的频道数量
`getCreateTime` | long | 服务器的创建时间
`getCustom` | String | 服务器的自定义扩展字段
`getIcon` | String | 服务器的图标
`getOwner` | String | 服务器所有者，默认为创建者
`getName` | String | 服务器名称
`getMemberNumber` | int | 服务器成员数量
