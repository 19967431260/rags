<!--keywords: 未读数,监听,获取,标记,已读,内置系统通知-->

内置系统通知未读数，指在云信系统中，用户接收到的内置系统通知未读总数。本文介绍如何获取内置系统通知未读数和标记已读并提供相应的示例代码。


## 技术原理

网易云信 NIM SDK 提供内置系统通知未读数变更事件的监听接口。用户在处理系统通知后，SDK 会触发该事件，收到未读数变更信息。您可以调用[`SystemMessageObserver`](https://doc.yunxin.163.com/docs/interface/messaging/android/doxygen/Latest/zh/interfacecom_1_1netease_1_1nimlib_1_1sdk_1_1msg_1_1_system_message_observer.html)
接口的 `observeUnreadCountChange` 方法监听该事件。

内置系统通知对象 [`SystemMessage`](https://doc.yunxin.163.com/docs/interface/messaging/android/doxygen/Latest/zh/classcom_1_1netease_1_1nimlib_1_1sdk_1_1msg_1_1model_1_1_system_message.html) 中通过属性 `unread` 标志该条系统通知是否未读。



## 监听内置系统通知未读数

只有在注册监听内置系统通知未读数变化事件后，用户才会收到对应的系统通知。

通过调用 [`observeUnreadCountChange`](https://doc.yunxin.163.com/docs/interface/messaging/android/doxygen/Latest/zh/interfacecom_1_1netease_1_1nimlib_1_1sdk_1_1msg_1_1_system_message_observer.html#abed94cee1c4a12f822d035da9ba00298) 方法监听内置系统通知未读数变化。

**示例代码：**
```
NIMClient.getService(SystemMessageObserver.class)
    .observeUnreadCountChange(sysMsgUnreadCountChangedObserver, register);

private Observer<Integer> sysMsgUnreadCountChangedObserver = new Observer<Integer>() {
        @Override
        public void onEvent(Integer unreadCount) {
            // 更新未读数变化
        }
    };
```


## <span id="查询内置系统通知未读数">查询内置系统通知未读数</span>

### <span id="查询内置系统通知未读总数">查询内置系统通知未读总数</span>

通过调用 [`querySystemMessageUnreadCountBlock`](https://doc.yunxin.163.com/docs/interface/messaging/android/doxygen/Latest/zh/interfacecom_1_1netease_1_1nimlib_1_1sdk_1_1msg_1_1_system_message_service.html#a5af3dac15385e0ef7c1e0bf1ddcd01f3) 方法查询内置系统通知未读总数。

::: note note
该接口为同步接口，仅在本地数据库中进行查询。
:::

**示例代码：**

```
int unread = NIMClient.getService(SystemMessageService.class).querySystemMessageUnreadCountBlock();
```

### <span id="查询指定类型的内置系统通知未读数">查询指定类型的内置系统通知未读数</span>


通过调用 [`querySystemMessageUnreadCountByType`](https://doc.yunxin.163.com/docs/interface/messaging/android/doxygen/Latest/zh/interfacecom_1_1netease_1_1nimlib_1_1sdk_1_1msg_1_1_system_message_service.html#a4cb91fde20b69a33360be7a837e59a93) 方法查询指定类型的内置系统通知未读数。

其中，入参 `types` 为待指定的系统通知类型集合，内置系统通知类型具体请参见[`SystemMessageType`](https://doc.yunxin.163.com/docs/interface/messaging/android/doxygen/Latest/zh/enumcom_1_1netease_1_1nimlib_1_1sdk_1_1msg_1_1constant_1_1_system_message_type.html)。

::: note note
该接口为同步接口，仅在本地数据库中进行查询。
:::

**示例代码：**
```
List<SystemMessageType> types = new ArrayList<>();
types.add(SystemMessageType.AddFriend);

// 查询“添加好友”类型的系统通知未读数总和
int unread = NIMClient.getService(SystemMessageService.class)
	.querySystemMessageUnreadCountByType(types);
```

## <span id="标记内置系统通知已读">标记内置系统通知已读</span>

### <span id="标记所有内置系统通知为已读">标记所有内置系统通知为已读</span>

通过调用 [`resetSystemMessageUnreadCount`](https://doc.yunxin.163.com/docs/interface/messaging/android/doxygen/Latest/zh/interfacecom_1_1netease_1_1nimlib_1_1sdk_1_1msg_1_1_system_message_service.html#a9fcd57ee48612f8ac936fdb28100a7d2) 方法将所有的内置系统通知标记为已读，即调用后内置系统通知未读数为零。

**示例代码：**

```
// 进入过系统通知列表后，可调用此函数将未读清零
NIMClient.getService(SystemMessageService.class).resetSystemMessageUnreadCount();
```

### <span id="标记指定类型的内置系统通知为已读">标记指定类型的内置系统通知为已读</span>

通过调用 [`resetSystemMessageUnreadCountByType`](https://doc.yunxin.163.com/docs/interface/messaging/android/doxygen/Latest/zh/interfacecom_1_1netease_1_1nimlib_1_1sdk_1_1msg_1_1_system_message_service.html#add9577ece28dc098c2bff37277bf31d2) 方法将指定类型的内置系统通知标记为已读。

其中，入参 `types` 为待指定的系统通知类型集合，内置系统通知类型具体请参见[`SystemMessageType`](https://doc.yunxin.163.com/docs/interface/messaging/android/doxygen/Latest/zh/enumcom_1_1netease_1_1nimlib_1_1sdk_1_1msg_1_1constant_1_1_system_message_type.html)。

**示例代码：**

```
List<SystemMessageType> types = new ArrayList<>();
types.add(SystemMessageType.AddFriend);

// 将“添加好友”类型的系统通知标记为已读
NIMClient.getService(SystemMessageService.class).resetSystemMessageUnreadCountByType(types);
```


### <span id="标记单条内置系统通知为已读">标记单条内置系统通知为已读</span>

通过调用 [`setSystemMessageRead`](https://doc.yunxin.163.com/docs/interface/messaging/android/doxygen/Latest/zh/interfacecom_1_1netease_1_1nimlib_1_1sdk_1_1msg_1_1_system_message_service.html#aa0c98e0ea18ed0ed6166cb093bcc207b) 方法将指定的某条内置系统通知标记为已读。


**示例代码：**

```
NIMClient.getService(SystemMessageService.class).setSystemMessageRead(messageId);
```

您也可以调用 [`SystemMessage.setUnread`](https://doc.yunxin.163.com/docs/interface/messaging/android/doxygen/Latest/zh/classcom_1_1netease_1_1nimlib_1_1sdk_1_1msg_1_1model_1_1_system_message.html#a12e08d8507d57f3ae79fcd134e7d3c00) 方法设置该条内置系统通知是否为已读。
