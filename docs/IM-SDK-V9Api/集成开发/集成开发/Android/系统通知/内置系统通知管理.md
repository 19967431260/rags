<!--keywords: 内置系统通知,通知,监听,获取,删除,更新-->

网易云信 NIM SDK 支持接收和存储内置系统通知。同时提供处理、查询、删除内置系统通知、修改通知状态等内置系统通知管理功能。


## 技术原理


内置系统通知是云信系统内建的通知，由云信服务器推送给用户或群组，用于云信系统类的事件通知。

网易云信 NIM SDK 的 [`SystemMessageService`](https://doc.yunxin.163.com/docs/interface/messaging/android/doxygen/Latest/zh/interfacecom_1_1netease_1_1nimlib_1_1sdk_1_1msg_1_1_system_message_service.html) 提供系统通知操作相关接口， [`SystemMessageObserver`](https://doc.yunxin.163.com/docs/interface/messaging/android/doxygen/Latest/zh/interfacecom_1_1netease_1_1nimlib_1_1sdk_1_1msg_1_1_system_message_observer.html) 提供系统通知相关观察者通知接口，帮助您快速实现和使用云信系统的系统通知功能。


## <span id="监听内置系统通知">监听内置系统通知</span>

只有在注册监听内置系统通知相关事件后，用户才会收到对应的系统通知。
  

```mermaid

  flowchart LR
    classDef default fill:#337EFF,stroke:#337EFF,stroke-width:0px,color:#FFFFFF;
    
    A("监听内置系统通知接收事件") --> B("产生触发内置系统通知的事件") --> C("接收具体类型的内置系统通知")
```

通过调用 [`observeReceiveSystemMs`](https://doc.yunxin.163.com/docs/interface/messaging/android/doxygen/Latest/zh/interfacecom_1_1netease_1_1nimlib_1_1sdk_1_1msg_1_1_system_message_observer.html#a5646f06b01f775f9329a7d1fa8965aab) 方法监听内置系统通知接收事件。

示例代码如下：
```
//注册内置系统通知接收事件观察者
NIMClient.getService(SystemMessageObserver.class)
	.observeReceiveSystemMsg(new Observer<SystemMessage>() {
            @Override
            public void onEvent(SystemMessage message) {
	            // 收到内置系统通知，可以做相应操作
            }
        }, register);

```

目前云信内置的能触发系统通知的事件包括：

通知类型|说明
:----|:-----
`TeamInvite`|邀请用户加入高级群
`ApplyJoinTeam`|用户申请加入高级群
`DeclineTeamInvite`|用户拒绝加入高级群邀请
`RejectTeamApply`|拒绝用户的加入高级群申请
`SuperTeamInvite`|邀请用户加入超大群
`SuperTeamApply`|用户申请加入超大群
`SuperTeamInviteReject`|用户拒绝加入超大群邀请
`SuperTeamApplyReject`|拒绝用户的加入超大群申请
`AddFriend`|对方（请求/已经）加你为好友
`undefined`|自定义事件



## <span id="查询内置系统通知">查询内置系统通知</span>

### <span id="查询内置系统通知列表">查询内置系统通知列表</span>

SDK 提供同步和异步两种类型的查询接口，均查询本地的内置系统通知：
- 异步接口 [`querySystemMessages`](https://doc.yunxin.163.com/docs/interface/messaging/android/doxygen/Latest/zh/interfacecom_1_1netease_1_1nimlib_1_1sdk_1_1msg_1_1_system_message_service.html#aee4113df3453854a3d7f65d7877b3d4e)
- 同步接口 [`querySystemMessagesBlock`](https://doc.yunxin.163.com/docs/interface/messaging/android/doxygen/Latest/zh/interfacecom_1_1netease_1_1nimlib_1_1sdk_1_1msg_1_1_system_message_service.html#aeb25626c264b22061219c932914e851c)

**参数说明：**

|参数|说明|
|:---|:---|
|offset |数据库查询偏移量，如100，则代码从第100条开始查询|
|limit  |数据库查询系统通知的数量|

**示例代码：**

```
// 异步调用
// 从10条开始，查询10条系统通知
NIMClient.getService(SystemMessageService.class).querySystemMessages(10, 10)
        .setCallback(new RequestCallback<List<SystemMessage>>() {
    @Override
    public void onSuccess(List<SystemMessage> param) {
        // 查询成功
    }

    @Override
    public void onFailed(int code) {
		// 查询失败
    }

    @Override
    public void onException(Throwable exception) {
		// error
    }
});

//同步调用
// 参数offset为当前已经查了offset条，limit为要继续查询limit条。
List<SystemMessage> temps = NIMClient.getService(SystemMessageService.class).querySystemMessagesBlock(offset, limit); 

```


### <span id="查询指定类型内置系统通知列表">查询指定类型内置系统通知列表</span>


SDK 提供同步和异步两种类型的查询接口，均查询本地的指定类型的内置系统通知：
- 异步接口 [`querySystemMessageByType`](https://doc.yunxin.163.com/docs/interface/messaging/android/doxygen/Latest/zh/interfacecom_1_1netease_1_1nimlib_1_1sdk_1_1msg_1_1_system_message_service.html#a54c00388cb4d1e82c13c6ab271963f65) 
- 同步接口 [`querySystemMessageByTypeBlock`](https://doc.yunxin.163.com/docs/interface/messaging/android/doxygen/Latest/zh/interfacecom_1_1netease_1_1nimlib_1_1sdk_1_1msg_1_1_system_message_service.html#a7574235e9bdf6d78baacab32d313e9de)


**参数说明：**

|参数|说明|
|:---|:---|
|types  |待查询的系统通知类型列表，具体系统类型请参见[`SystemMessageType`](https://doc.yunxin.163.com/docs/interface/messaging/android/doxygen/Latest/zh/enumcom_1_1netease_1_1nimlib_1_1sdk_1_1msg_1_1constant_1_1_system_message_type.html)|
|offset |数据库查询偏移量，如100，则代码从第100条开始查询|
|limit  |数据库查询系统通知的数量|

**示例代码：**

```
//异步接口

List<SystemMessageType> types = new ArrayList<>();
types.add(SystemMessageType.AddFriend);

// 只查询“添加好友”类型的系统通知, 从头开始查询，查询3条
NIMClient.getService(SystemMessageService.class).querySystemMessageByType(types, 0, 3)
        .setCallback(new RequestCallback<List<SystemMessage>>() {
    @Override
    public void onSuccess(List<SystemMessage> param) {
	     // 查询成功
    }

    @Override
    public void onFailed(int code) {
		// 查询失败
    }

    @Override
    public void onException(Throwable exception) {
		// error
    }
});


//同步接口

List<SystemMessageType> types = new ArrayList<>();
types.add(SystemMessageType.AddFriend);

// 只查询“添加好友”类型的系统通知, 从头开始查询，查询3条
List<SystemMessage> temps = NIMClient.getService(SystemMessageService.class)
    .querySystemMessageByTypeBlock(types, 0, 3);
```

### 查询所有未读内置系统通知

通过调用 [`querySystemMessageUnread`](https://doc.yunxin.163.com/docs/interface/messaging/android/doxygen/Latest/zh/interfacecom_1_1netease_1_1nimlib_1_1sdk_1_1msg_1_1_system_message_service.html#aef3b606106f889a3526852e0299ef196) 方法查询所有未读的内置系统通知。

内置系统通知对象[`SystemMessage`](https://doc.yunxin.163.com/docs/interface/messaging/android/doxygen/Latest/zh/classcom_1_1netease_1_1nimlib_1_1sdk_1_1msg_1_1model_1_1_system_message.html) 中通过属性 `unread` 标志该条内置系统通知是否未读。

您也可以调用 [`isUnread`](https://doc.yunxin.163.com/docs/interface/messaging/android/doxygen/Latest/zh/classcom_1_1netease_1_1nimlib_1_1sdk_1_1msg_1_1model_1_1_system_message.html#a07f3848b5163cd9b8df3da6f4114b89f) 方法查询某条内置系统通知是否已读，或者调用 [`setUnread`](https://doc.yunxin.163.com/docs/interface/messaging/android/doxygen/Latest/zh/classcom_1_1netease_1_1nimlib_1_1sdk_1_1msg_1_1model_1_1_system_message.html#a12e08d8507d57f3ae79fcd134e7d3c00) 方法设置该条通知为已读或未读。

示例代码如下：

```
NIMClient.getService(SystemMessageService.class).querySystemMessageUnread();
```

## <span id="删除内置系统通知">删除内置系统通知</span>

### <span id="删除所有内置系统通知">删除所有内置系统通知</span>

通过调用 [`clearSystemMessages`](https://doc.yunxin.163.com/docs/interface/messaging/android/doxygen/Latest/zh/interfacecom_1_1netease_1_1nimlib_1_1sdk_1_1msg_1_1_system_message_service.html#a636bfca903b265e3429e2d8d4fcfa05d) 方法删除所有的内置系统通知。示例代码如下：

```
NIMClient.getService(SystemMessageService.class).clearSystemMessages();
```

### <span id="删除指定类型的内置系统通知">删除指定类型的内置系统通知</span>

通过调用 [`clearSystemMessagesByType`](https://doc.yunxin.163.com/docs/interface/messaging/android/doxygen/Latest/zh/interfacecom_1_1netease_1_1nimlib_1_1sdk_1_1msg_1_1_system_message_service.html#a0c55c2994cded31ffcd56e548b297756) 方法删除指定类型的内置系统通知。

其中入参需要传入待查询的系统通知类型列表，具体系统类型请参见[`SystemMessageType`](https://doc.yunxin.163.com/docs/interface/messaging/android/doxygen/Latest/zh/interfacecom_1_1netease_1_1nimlib_1_1sdk_1_1msg_1_1_system_message_service.html#a0c55c2994cded31ffcd56e548b297756)

示例代码如下：

```
List<SystemMessageType> types = new ArrayList<>();
types.add(SystemMessageType.AddFriend);

// 只删除“添加好友”类型的系统通知
NIMClient.getService(SystemMessageService.class).clearSystemMessagesByType(types);
```

### <span id="删除指定的单条内置系统通知">删除指定的单条内置系统通知</span>

通过调用 [`deleteSystemMessage`](https://doc.yunxin.163.com/docs/interface/messaging/android/doxygen/Latest/zh/interfacecom_1_1netease_1_1nimlib_1_1sdk_1_1msg_1_1_system_message_service.html#a0618813cf425b66bc284ccee90e3dd9f) 方法删除指定的单条内置系统通知。

其中入参需要传入需要删除的系统通知 ID。示例代码如下：

```
NIMClient.getService(SystemMessageService.class).deleteSystemMessage(message.getMessageId());
```

## <span id="设置内置系统通知状态">设置内置系统通知状态</span>

SDK 的内置系统通知状态通过 [`SystemMessageStatus`](https://doc.yunxin.163.com/docs/interface/messaging/android/doxygen/Latest/zh/enumcom_1_1netease_1_1nimlib_1_1sdk_1_1msg_1_1constant_1_1_system_message_status.html#a0260f4565e85c28848fada4a6645110e) 来定义，目前主要内置了以下五种状态。除此之外，还提供五个自定义扩展类型，供第三方开发者使用。
- `init`：未处理状态
- `passed`：已通过验证
- `ignored`：已忽略
- `declined`：已拒绝
- `expired`：已过期

当用户处理过系统通知后，相应的系统通知状态会自动变更；也可以主动调用 [`setSystemMessageStatus`](https://doc.yunxin.163.com/docs/interface/messaging/android/doxygen/Latest/zh/interfacecom_1_1netease_1_1nimlib_1_1sdk_1_1msg_1_1_system_message_service.html#a9f7fca7a02a82dbbc1490e11009cd5ef) 方法手动变更某个系统通知的状态。变更后，您可以调用 [`getStatus`](https://doc.yunxin.163.com/docs/interface/messaging/android/doxygen/Latest/zh/classcom_1_1netease_1_1nimlib_1_1sdk_1_1msg_1_1attachment_1_1_net_call_attachment.html#a2c2969522dae9c20e63f7c74a24f2f1f) 方法查询该系统通知的状态。

**参数说明：**

|参数|说明|
|:---|:---|
|messageId |系统通知的 ID|
|status    |待设置的状态|


**示例代码：**

```
// 以设置系统通知状态为已过期为例
SystemMessageStatus status = SystemMessageStatus.expired;
NIMClient.getService(SystemMessageService.class).setSystemMessageStatus(message.getMessageId(), status);
```
