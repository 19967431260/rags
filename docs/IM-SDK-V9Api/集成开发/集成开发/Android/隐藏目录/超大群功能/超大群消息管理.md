
网易云信 NIM SDK 支持超大群中的消息收发，消息撤回，消息回复，群相关操作的通知消息的接收，以及群消息通知模式的设置。





## 超大群消息收发

### **技术原理**

超大群中的消息收发流程与单聊流程基本相同，但是超大群消息收发有单独的接口，使用自己的服务。

发送消息通过`SuperTeamService`服务，接收消息通过`SuperTeamServiceObserver`服务。

### **前提条件**

在实现超大群消息收发之前，请确保：
- 已通过服务端 API [创建超大群](https://doc.yunxin.163.com/messaging/guide/jk1ODEwNDY?platform=server)。
- 用户双方在同一个超大群中。
- 已了解[消息类型](https://doc.yunxin.163.com/messaging/guide/zU5ODg2NDU?platform=android#%E6%B6%88%E6%81%AF%E7%B1%BB%E5%9E%8B)的使用限制。

### **API 使用限制**

::: note important
发送消息（sendMessage）的方法调用存在频控，一分钟内默认最多可调用 300 次。
:::

## 超大群消息回复

### **技术原理**

没有回复对象的消息被称为 thread 消息；有回复对象的消息被称为回复消息。

thread 消息可以被多条消息回复，这些消息都是回复消息；回复消息的回复对象可以是 thread 消息，也可以是回复消息。如果一条回复消息的回复对象是 thread 消息，则该回复消息对应的 thread 消息就是其回复对象；否则该回复消息对应的 thread 消息和被回复消息对应的 thread 消息相同。

所有对应相同 thread 消息的回复消息组成一个 thread 会话。

:::note note
若未开通回复功能，回复时系统会自动将所发消息转换为一条普通消息。若需要 thread 回复能力，请联系商务经理开通。
:::

超大群中的消息回复流程与单聊流程基本相同，但是超大群消息回复有单独的接口，使用自己的服务。

回复消息通过`SuperTeamService`服务，接收回复消息通过`SuperTeamServiceObserver`服务。

## 超大群消息撤回

### **技术原理**

NIM Android SDK 支持超大群中的消息撤回功能。超大群成员可以在一定时间内（默认 2 分钟，可在云信控制台配置）撤回超大群消息。

超大群中的消息撤回流程与单聊流程基本相同，但是超大群消息撤回有单独的接口，使用自己的服务（`SuperTeamServiceObserver`服务）。

### **实现方法**

通过调用 [`SuperTeamService.revokeMessage`](https://doc.yunxin.163.com/docs/interface/%E5%8D%B3%E6%97%B6%E9%80%9A%E8%AE%AFAndroid%E7%AB%AF/IM_Android/com/netease/nimlib/sdk/superteam/SuperTeamService.html#revokeMessage-com.netease.nimlib.sdk.msg.model.IMMessage-java.lang.String-java.util.Map-boolean-java.lang.String-) 方法撤回已发送的消息。

##  <span id="超大群通知消息">超大群通知消息</span>

NIM SDK 内置支持的会话消息类型（[`MsgTypeEnum`](https://doc.yunxin.163.com/docs/interface/%E5%8D%B3%E6%97%B6%E9%80%9A%E8%AE%AFAndroid%E7%AB%AF/IM_Android/com/netease/nimlib/sdk/msg/constant/MsgTypeEnum.html)）中的通知消息（`notification`）主要用于群组、聊天室和超大群的事件通知，由服务端下发，客户端无法发送事件通知消息。 

目前支持触发超大群通知消息的事件如下：

|[`NotificationType`](https://doc.yunxin.163.com/docs/interface/%E5%8D%B3%E6%97%B6%E9%80%9A%E8%AE%AFAndroid%E7%AB%AF/IM_Android/com/netease/nimlib/sdk/msg/constant/NotificationType.html)|事件说明|
|:---|:---|:---|
|SUPER_TEAM_INVITE_ACCEPT|接受邀请后入群（需要被邀请人同意的模式）|
|SUPER_TEAM_INVITE|邀请成员入群（无需被邀请人同意的模式）|
|SUPER_TEAM_ADD_MANAGER|添加管理员 |
|SUPER_TEAM_KICK |被踢出群|
|SUPER_TEAM_CHANGE_OWNER |转让群主|
|SUPER_TEAM_APPLY_PASS|申请加入群成功|
|SUPER_TEAM_REMOVE_MANAGER|移除管理员|
|SUPER_TEAM_DISMISS|解散群|
|SUPER_TEAM_LEAVE|退出群 |
|SUPER_TEAM_MUTE_TLIST|群内禁言/解禁 |
|SUPER_TEAM_UPDATE_T_INFO|群信息更新|


超大群通知消息解析步骤请参考[群通知消息解析步骤](https://doc.yunxin.163.com/messaging/guide/DQ5NzIwMjg?platform=android#%E7%BE%A4%E9%80%9A%E7%9F%A5%E6%B6%88%E6%81%AF%E8%A7%A3%E6%9E%90%E6%AD%A5%E9%AA%A4)


##  <span id="超大群消息免打扰">超大群消息免打扰</span>

SDK 支持对超大群消息通知提醒模式的配置。超大群消息通知提醒模式分为：
- All：全部提醒（默认）
- Manager：仅群主/管理员消息提醒
- Mute：全部不提醒

以上通知提醒指的是云信体系内的[推送与消息提醒](https://doc.yunxin.163.com/messaging/guide/zc1OTI2MTM?platform=android)。
