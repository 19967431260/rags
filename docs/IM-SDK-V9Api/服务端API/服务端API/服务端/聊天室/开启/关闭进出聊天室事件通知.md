在创建聊天室时，可设置开启/关闭人员进出聊天室事件通知。在聊天室创建成功后，若还需要修改进出聊天室事件的通知状态，可以调用此接口。

目前人员进出聊天室事件通知主要包括以下事件：
- 主动退出聊天室
- 聊天室掉线
- 进入聊天室



## URL 

```http
POST https://api.yunxinapi.com/nimserver/chatroom/updateInOutNotification.action HTTP/1.1
Content-Type:application/x-www-form-urlencoded;charset=utf-8
```




## 请求参数

- POST 请求中 Headers 的设置请参考[API调用方式](/docs/TM5MzM5Njk/jk3MzY2MTI)。

- POST 请求中 Body 的设置如下：

<table>
            <thead>
        <tr> <th style="width:180px">参数</th><th style="width:100px">类型</th><th>必填</th><th>说明</th></tr></thead>
<tr> <td> roomid </td><td> Long</td><td>是</td><td>
           聊天室 ID</td></tr>
  <tr> <td> close </td><td> Boolean</td><td>是</td><td>开启或关闭进出聊天室事件通知<br>
           true：关闭进出聊天室通知，false：开启进出聊天室通知</td></tr>
</table>

## 示例

### **请求示例（curl）**

```curl
curl -X POST  -H 'appkey: fe416640***7ad2547'  -H 'cache-control: no-cache'  -H 'checksum: 18f5435a7bf**7f75265495034'  -H 'content-type: application/x-www-form-urlencoded'   -H 'curtime: 1508481877'   -H 'nonce: 12345' -d 'close=false&roomid=3001' "https://api.yunxinapi.com/nimserver/chatroom/updateInOutNotification.action"
```

### **请求成功返回示例**


```json
"Content-Type": "application/json; charset=utf-8"
{
  "code": 200
}

```

### **请求失败返回示例**

```json
"Content-Type": "application/json; charset=utf-8"
{
  "code": 414
}

```
