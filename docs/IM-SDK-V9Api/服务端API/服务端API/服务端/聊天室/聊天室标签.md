通过聊天室标签功能，可将聊天室消息向聊天室 **部分成员** 下发，也支持仅接收特定的消息。本文介绍 IM 服务端相关标签功能。

::: note note
- 聊天室标签在 SDK 实例登录时设置。聊天室标签功能的具体介绍，请参考 [聊天室标签功能](https://doc.yunxin.163.com/messaging/server-apis/TMxOTI0MDA?platform=server)。
- 除本文介绍的相关 API 外，IM 服务端还支持在发送聊天室消息时通过 `notifyTargetTags` 参数设置将消息限制发送给哪些用户，如果缺失则会发给聊天室所有用户。请参考 [发送聊天室消息](https://doc.yunxin.163.com/messaging/server-apis/zMzOTQxNjE?platform=server#发送聊天室消息)。
:::

## <span id='标签禁言'>通过标签批量禁言用户</span>

将特定标签的登录用户设置为在特定时间段内禁言。

### 请求 URL

```HTTP
POST https://api.yunxinapi.com/nimserver/chatroom/tagTemporaryMute.action HTTP/1.1
Content-Type:application/x-www-form-urlencoded;charset=utf-8
```

### 请求头参数

请求头（Headers）参数的设置，请参考 [API 调用方式](/messaging/server-apis/jk3MzY2MTI)。

### 请求体参数

<table>
<thead>
<tr> <th>参数</th><th>类型</th><th>是否必需</th><th>说明</th></tr>
</thead>
<tr> <td> roomid </td><td> Long</td><td>是</td><td>聊天室 ID</td></tr>
<tr> <td> operator </td><td> string</td><td>是</td><td>操作者的账号 ID（accid），必须是创建者或者管理员</td></tr>
<tr> <td> targetTag </td><td> string</td><td>是</td><td>目标标签</td></tr>
<tr> <td> needNotify </td><td> boolean</td><td>否</td><td>true/false，是否发送禁言通知，默认 true</td></tr>
<tr> <td> notifyExt </td><td> string</td><td>否</td><td>禁言通知通知扩展字段</td></tr>
<tr> <td> muteDuration </td><td> int</td><td>是</td><td>禁言时长，单位秒，最长 30 天，若等于 0 表示取消禁言</td></tr>
<tr> <td> notifyTargetTags </td><td> string</td><td>否</td><td>禁言通知的目标标签表达式，若缺失则发送给设置了 targetTag 的用户</td></tr>
</table>

### **请求示例**

```cURL
curl -X POST -H 'appkey: fe416640**9e1847ad2547' -H 'cache-control: no-cache' -H 'checksum: 18f5435a7bf2**7f75265495034' -H 'content-type: application/x-www-form-urlencoded'   -H 'curtime: 1508481877'   -H 'nonce: 12345' -d 'targetTag=abc&roomid=3001&operator=xxx&muteDuration=123' "https://api.yunxinapi.com/nimserver/chatroom/tagTemporaryMute.action"
```

### **响应示例**

```JSON
"Content-Type": "application/json; charset=utf-8"
{
    "code": 200,
    "desc": {
        "muteDuration": 100   //禁言时长，若取消禁言，则返回上次禁言的剩余禁言时长
    }
}
```

### **状态码**

该 API 在 HTTPS Body 中返回请求的状态码，状态码详情请参考 [状态码](https://doc.yunxin.163.com/messaging/server-apis/TM5NTk2Mzc?platform=server)。

## <span id='查询某个标签下的在线用户数'>查询标签下的在线用户数</span>

查询某个标签下的在线人数。

::: note notice
单个账号多端登录情况下，在线用户数为 1。
:::

### 请求 URL

```HTTP
POST https://api.yunxinapi.com/nimserver/chatroom/tagMembersCount.action HTTP/1.1
Content-Type:application/x-www-form-urlencoded;charset=utf-8
```

### 请求头参数

请求头（Headers）参数的设置，请参考 [API 调用方式](/messaging/server-apis/jk3MzY2MTI)。

### 请求体参数

<table>
<thead>
<tr> <th>参数</th><th>类型</th><th>是否必需</th><th>说明</th></tr>
</thead>
<tr> <td> roomid </td><td> Long</td><td>是</td><td>聊天室 ID</td></tr>
<tr> <td> tag </td><td> string</td><td>是</td><td>标签</td></tr>
</table>

### **请求示例**

```cURL
curl -X POST -H 'appkey: fe416640c8**1847ad2547' -H 'cache-control: no-cache' -H 'checksum: 18f5435a7bf**97f75265495034' -H 'content-type: application/x-www-form-urlencoded'   -H 'curtime: 1508481877'   -H 'nonce: 12345' -d 'tag=abc&roomid=3001' "https://api.yunxinapi.com/nimserver/chatroom/tagMembersCount.action"
```

### **响应示例**

```JSON
"Content-Type": "application/json; charset=utf-8"
{
  "data": {
    "tag": "abc",
    "onlineUserCount":222
  },
  "code": 200
}
```

### **状态码**

该 API 在 HTTPS Body 中返回请求的状态码，状态码详情请参考 [状态码](https://doc.yunxin.163.com/messaging/server-apis/TM5NTk2Mzc?platform=server)。

## <span id='根据标签查询在线成员列表'>根据标签查询在线成员列表</span>

```HTTP
POST https://api.yunxinapi.com/nimserver/chatroom/tagMembersQuery.action HTTP/1.1
Content-Type:application/x-www-form-urlencoded;charset=utf-8
```

### 请求头参数

请求头（Headers）参数的设置，请参考 [API 调用方式](/messaging/server-apis/jk3MzY2MTI)。

### 请求体参数

<table>
<thead>
<tr> <th>参数</th><th>类型</th><th>是否必需</th><th>说明</th></tr>
</thead>
<tr> <td> roomid </td><td> Long</td><td>是</td><td>聊天室 ID</td></tr>
<tr> <td> tag </td><td> string</td><td>是</td><td>标签</td></tr>
<tr> <td> endTime </td><td> Long</td><td>是</td><td>起始时间，逆序查询，若传 0 则表示从当前时间往前查</td></tr>

## 根据标签查询历史消息

根据聊天室标签（Tags）查询聊天室历史消息。

### 请求 URL

```HTTP
POST https://api.yunxinapi.com/nimserver/history/queryTagHistoryMsg.action HTTP/1.1
Content-Type: application/x-www-form-urlencoded;charset=utf-8
```

### 请求头参数

请求头（Headers）参数的设置，请参考 [API 调用方式](/messaging/server-apis/jk3MzY2MTI)。

### 请求体参数

<table>
  <thead>
    <tr> <th>参数</th><th>类型</th><th>是否必需</th><th>说明</th></tr>
    </thead>
    <tr> <td> roomId </td><td> Long</td><td>是</td><td>聊天室 ID</td></tr>
    <tr> <td> tags </td><td> String</td><td>是</td><td>标签 JSON 字符串 ["tag1", "tag2"]</td></tr>
    <tr> <td> types </td><td> String</td><td>否</td><td>消息类型 JSON 字符串 ["0","1","2"]，默认查询全部消息类型</td></tr>
    <tr> <td> fromTime </td><td> Long</td><td>否</td><td>检索开始时间</td></tr>
    <tr> <td> toTime </td><td> Long</td><td>否</td><td>检索结束时间</td></tr>
    <tr> <td> limit </td><td>Integer</td><td>否</td><td>查询数量，默认值和最大值都为 200</td></tr>
    <tr> <td> reverse </td><td>Integer </td><td>否</td><td>是否反向，0：否（默认），1：是</td></tr>
</table>

### **请求示例**

```cURL
curl -X POST 'https://api.yunxinapi.com/nimserver/history/queryTagHistoryMsg.action' -H 'AppKey: go9dnk49bk**glw0803mgq3' -H 'Nonce: 4tggger**3t23t' -H 'CurTime: 1443592222' -H 'CheckSum: 9e9db3b6c9abb2e1962cf3e6f7316fcc55583f86' -H 'Content-Type: application/x-www-form-urlencoded' -d 'roomId=10000&tags=["tag1","tag2","tag3"]&types=["0","1","2"]&fromTime=1666241788000&toTime=1666251788000&limit=100&reverse=0'
```

### **响应示例**

**请求成功返回示例**

```JSON
"Content-Type": "application/json; charset=utf-8"
{
    "msgs": [
        {
            "msgid": "0fb86152-f9fb-9cbe-bcd9-9caed878dade", //消息 ID
            "from": "jiajia01", //消息发送者 ID
            "type": 0,//消息类型
            "sendtime": 1666072906119,//消息发送时间
            "body": {
                "msg": "this is message attachment 3" //消息内容
            },
            "fromclienttype": 64 //消息客户端类型
        },
        {
            "msgid": "15459c99-666a-444b-3d50-3175f8cf5145",
            "from": "jiajia01",
            "type": 0,
            "sendtime": 1666072876187,
            "body": {
                "msg": "this is message attachment 2"
            },
            "fromclienttype": 64
        }
    ],
    "code": 200,
    "size": 2
}
```

**请求失败返回示例**

```
"Content-Type": "application/json; charset=utf-8"
{
"code": 414,
"desc": "error roomId" //聊天室 ID 不合法
}
```

```
"Content-Type": "application/json; charset=utf-8"
{
"code": 403,
"desc": "search not enable" //搜索未开启
}
```

### **状态码**

该接口在 HTTPS Body 中返回请求的状态码，以下仅列出与接口业务相关的状态码。完整状态码请参考 <a href="https://doc.yunxin.163.com/messaging/server-apis/TM5NTk2Mzc?platform=server" target="_blank">状态码</a>。

| 状态码 | 说明 | 处理建议 |
:--- | :---- | :----
200 | 请求成功 | -
414 | 参数错误 | 根据提示信息，检查传入参数的格式和限制条件
403 | 禁止操作 | 请开启云端消息检索权限

## 修改聊天室用户标签

修改聊天室用户的标签，修改 tag 后会通知被修改人的所有在线端并广播通知聊天室内所有用户。

### 请求 URL

```HTTP
POST https://api.yunxinapi.com/nimserver/chatroom/updateChatRoomRoleTag.action HTTP/1.1
Content-Type: application/x-www-form-urlencoded;charset=utf-8
```

### 请求头参数

请求头（Headers）参数的设置，请参考 [API 调用方式](/messaging/server-apis/jk3MzY2MTI)。

### 请求体参数

<table>
  <thead>
    <tr> <th>参数</th><th>类型</th><th>是否必需</th><th>说明</th></tr>
    </thead>
    <tr> <td> roomId </td><td> Long</td><td>是</td><td>聊天室 ID</td></tr>
    <tr> <td> tags </td><td> String</td><td>是</td><td>可设置多个标签，JSON 字符串 ["tag1", "tag2"]</td></tr>
    <tr> <td> accid </td><td> String</td><td>是</td><td>用户 ID</td></tr>
    <tr> <td> notifyTargetTags </td><td> String</td><td>否</td><td>通知目标用户的标签，是 <a href="https://doc.commsease.com/messaging/docs/TMxOTI0MDA?platform=android#%E6%A0%87%E7%AD%BE%E8%A1%A8%E8%BE%BE%E5%BC%8F" target="_blank">标签表达式</a>，同时也会改变该连接掉线时的通知对象</td></tr>
</table>

### **请求示例**

```cURL
curl -X POST 'https://api.yunxinapi.com/nimserver/chatroom/updateChatRoomRoleTag.action' -H 'AppKey: go9dnk49b**lw0803mgq3' -H 'Nonce: 4tggge**3t23t' -H 'CurTime: 1443592222' -H 'CheckSum: 9e9db3b6c9abb2e1962cf3e6f7316fcc55583f86' -H 'Content-Type: application/x-www-form-urlencoded' -d 'roomId=10000&tags=["tag1","tag2","tag3"]&accid=tyuiu987655787987fs7f98sd&notifyTargetTags={"tag":"tag1"} and {"tag":"tag2"}'
```

### **响应示例**

**请求成功返回示例**

```
"Content-Type": "application/json; charset=utf-8"
{"code":200}
```

**请求失败返回示例**

```
"Content-Type": "application/json; charset=utf-8"
{
"code": 414,
"desc": "error roomId" //聊天室 ID 不合法
}
```
```
"Content-Type": "application/json; charset=utf-8"
{
"code": 404,
"desc": "chatroom role not exists" //聊天室角色不存在
}
```

### **状态码**

该接口在 HTTPS Body 中返回请求的状态码，以下仅列出与接口业务相关的状态码。完整状态码请参考 <a href="https://doc.yunxin.163.com/messaging/server-apis/TM5NTk2Mzc?platform=server" target="_blank">状态码</a>。

| 状态码 | 说明 | 处理建议 |
:--- | :---- | :----
200 | 请求成功 | -
414 | 参数错误 | 根据提示信息，检查传入参数的格式和限制条件
404 | 聊天室用户不存在 | 检查用户的账号 ID（`accid`）
