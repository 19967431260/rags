网易云信 IM 服务端支持用户之间的好友关系管理。

## <span id='添加好友'>添加好友</span>

每个账号的好友数上限为 3000。

### API 使用限制


单个应用默认最高调用频率请参考 [频控说明](https://doc.yunxin.163.com/messaging/server-apis/zk4Mzg0MjE?platform=server)。

### URL 
```
POST  https://api.yunxinapi.com/nimserver/friend/add.action  HTTP/1.1
Content-Type: application/x-www-form-urlencoded;charset=utf-8
```

### 请求参数

- POST 请求中 Headers 的设置请参考[接口概述](/docs/TM5MzM5Njk/jk3MzY2MTI)。
- POST 请求中 Body 的设置如下：
<table>
            <thead>
        <tr> <th style="width:160px" >参数</th><th style="width:100px">类型</th><th>字符串长度上限</th><th>是否必填</th><th>示例</th><th>描述</th></tr></thead>
          <tr> <td>accid</td><td>String</td><td>32</td><td>必填</td><td>zhangsan</td><td>
          添加好友的发起者accid</td></tr>
          <tr> <td>faccid</td><td>String</td><td>32</td><td>必填</td><td>lisi</td><td>
          添加好友的接收者accid</td></tr>
          <tr> <td>type</td><td>int</td><td>/</td><td>必填</td><td>1</td><td>
          1 直接加好友（无需对方同意）；2 请求加好友（需要对方同意）；3 同意加好友；4 拒绝加好友</td></tr>
          <tr> <td>msg</td><td>String</td><td>256</td><td>选填</td><td>我是xx</td><td>
          加好友对应的请求消息</td></tr>
          <tr> <td>serverex</td><td>String</td><td>256</td><td>选填</td><td>/</td><td>
          服务器端扩展字段<br/> 此字段 client 端只读，server 端读写</td></tr>
            
</table>

### 返回参数
<table>
            <thead>
        <tr> <th>参数</th><th>类型</th><th>示例</th><th>描述</th></tr></thead>
          <tr> <td>code</td><td>int</td><td>200</td><td>状态码</td></tr>
</table>

### 示例
#### **请求示例（curl）**

```curl
curl -X POST -H "AppKey: go9dn***803mgq3" -H "Nonce: 4tgggerg***23t23t" -H "CurTime: 1443592222" -H "CheckSum: 9e9db3b6c***f7316fcc55583f86" -H "Content-Type: application/x-www-form-urlencoded" -d 'accid=zhangsan&faccid=lisi&type=1' 'https://api.yunxinapi.com/nimserver/friend/add.action'
```

#### **返回示例**

```json
"Content-Type": "application/json; charset=utf-8"
{
    "code":200
}
```

## <span id='获取好友关系'>获取好友关系</span>

### 功能描述
	
根据用户账号查询好友关系。

### API 使用限制


单个应用默认最高调用频率请参考 [频控说明](https://doc.yunxin.163.com/messaging/server-apis/zk4Mzg0MjE?platform=server)。

### URL
```
POST  https://api.yunxinapi.com/nimserver/friend/getByAccid.action  HTTP/1.1
Content-Type: application/x-www-form-urlencoded;charset=utf-8
```
### 请求参数
- POST 请求中 Headers 的设置请参考[接口概述](/docs/TM5MzM5Njk/jk3MzY2MTI)。
- POST 请求中 Body 的设置如下：
<table>
            <thead>
        <tr> <th style="width:160px">参数</th><th style="width:100px">类型</th><th>字符串长度上限</th><th>是否必填</th><th>示例</th><th>描述</th></tr></thead>
          <tr> <td>accid</td><td>String</td><td>32</td><td>必填</td><td>zhangsan</td><td>
          查询好友关系的发起者accid</td></tr>
          <tr> <td>faccid</td><td>String</td><td>32</td><td>必填</td><td>lisi</td><td>被查询好友关系的用户accid</td></tr>     
</table>

### 返回参数




<div style="width:160px">参数</div> | <div style="width:100px">类型</div> | 示例| 描述
---- | -------------- | ---------|----
code | int | 200 | 状态码
friend | JASON | {"createtime": 1440037706987,"ex": null,"bidirection": true, "faccid": "t2", "serverex": null,"updatetime": 1440037706987} | 好友信息，包括：<br><ul><li>createtime：好友关系创建时间</li><li>ex：扩展字段</li><li>bidirection：双向好友标志</li><li>faccid：好友的云信IM账号</li><li>serverex：服务端扩展字段</li><li>updatetime：好友关系更新时间</li></ul>
  

### 示例

#### 请求示例（curl）
```curl
curl -X POST -H "AppKey: go9d***3mgq3" -H "Nonce: 4tgg***t23t" -H "CurTime: 1443592222" -H "CheckSum: 9e9db3b***583f86" -H "Content-Type: application/x-www-form-urlencoded" -d 'accid=zhangsan&faccid=lisi' 'https://api.yunxinapi.com/nimserver/friend/getByAccid.action'
```

#### 好友关系存在的返回示例

```json
"Content-Type": "application/json; charset=utf-8"
{
    "code":200,
    "friend":
      {
        "createtime": 1440037706987,
        "ex": null,
        "bidirection": true,
        "faccid": "t2",
        "serverex": null,
        "updatetime": 1440037706987
      }
}
```

#### 好友关系不存在的返回示例

如果非好友关系，则返回`404`状态码。

```json
"Content-Type": "application/json; charset=utf-8"
{
    "code":404,
}
```


## 状态码

上述接口在 HTTPS Body 中返回请求的状态码，状态码详情请参见[状态码](/docs/TM5MzM5Njk/TM5NTk2Mzc)。
