网易云信 IM 服务端支持用户名片的查询和修改。

## <span id='获取用户名片'>获取用户名片</span>

### 功能描述

查询用户名片信息，包括昵称、年龄、性别等，同时可以查询用户的禁言状态（全局或者某功能模块）。

### API 使用限制


单个应用默认最高调用频率请参考 [频控说明](https://doc.yunxin.163.com/messaging/server-apis/zk4Mzg0MjE?platform=server)。

### URL
```
POST https://api.yunxinapi.com/nimserver/user/getUinfos.action  HTTP/1.1
Content-Type: application/x-www-form-urlencoded;charset=utf-8
```
### 请求参数

- POST 请求中 Headers 的设置请参考 [API 调用方式](https://doc.yunxin.163.com/messaging/server-apis/jk3MzY2MTI)。
- POST 请求中 Body 的设置如下：
<table>
            <thead>
        <tr> <th style="width:160px">参数</th><th style="width:100px">类型</th><th style="width:80px">必须</th><th>说明</th></tr></thead>
           <tr> <td> accids </td><td>String</td><td>是</td><td>用户帐号 ID 列表，格式为 JSONArray，例如：["id1", "id2", "id3"]。格式错误会返回 414 参数错误。<br>一次最多查询 200 个账号。</td></tr>
           <tr> <td> muteStatus </td><td>Boolean</td><td>否</td><td>是否返回功能模块的禁言状态，默认为 false</td></tr>      
</table>

### 返回参数

|参数|说明|
|:---|:----|
|code|状态码，200 表示请求成功|
|uinfos|查询的用户信息列表，JSON|

uinfos 中的参数说明

|参数|说明|
|:---|:----|
|accid|云信 IM 账号
|name|昵称
|icon|头像
|sign|签名
|email|邮箱
|birth|生日
|mobile|手机号码
|ex|自定义扩展信息
|gender|性别
|valid|是否有效
|mute|是否全局禁言
|muteP2P|账号在单聊模块中是否被禁言
|muteQChat|账号在群组模块中是否被禁言
|muteTeam|账号在群组模块中是否被禁言
|muteRoom|账号在聊天室模块中是否被禁言

### 示例

#### **请求示例（curl）**
```curl
curl -X POST -H "AppKey: go9dnk49bk***0803mgq3" -H "Nonce: 4tgggerg**23t" -H "CurTime: 1443592222" -H "CheckSum: 9e9db3b6c**fcc55583f86" -H "Content-Type: application/x-www-form-urlencoded" -d 'accids=["t1","t2"]&muteStatus=true' 'https://api.yunxinapi.com/nimserver/user/getUinfos.action'
```

          "ex": "",//扩展信息
          "gender": 1,//性别：0表示未知，1表示男，2表示女

- 入参 `muteStatus` 为 true，返回如下示例：

:::note notice
返回的用户资料信息中，如果未设置对应信息，则不返回（例如：用户未设置`email`信息，则返回的信息中不包括`email`字段）。
:::

#### **请求失败返回示例**

```
"Content-Type": "application/json; charset=utf-8"
{
    "code":414
    "desc":"limit max accids size: 200"  // 传入的accid数量过多
}
```

## <span id='更新用户名片'>更新用户名片</span>
### 功能描述
更新用户名片。
::: note notice
用户名片中包含的用户信息，在群组、聊天室等场景下，会暴露给群组、聊天室内的其他用户。这些字段里mobile，email，birth，gender等字段属于非必填、可能涉及隐私的信息，如果您的业务下，这些信息为敏感信息，建议在通过扩展字段ex填写相关资料并事先加密。
:::

### API 使用限制


单个应用默认最高调用频率请参考 [频控说明](https://doc.yunxin.163.com/messaging/server-apis/zk4Mzg0MjE?platform=server)。
