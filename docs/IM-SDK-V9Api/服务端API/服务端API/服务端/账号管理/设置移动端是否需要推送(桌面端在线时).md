设置桌面端在线时，移动端是否需要推送。


## API 使用限制

单个应用默认最高调用频率请参考 [频控说明](https://doc.yunxin.163.com/messaging/server-apis/zk4Mzg0MjE?platform=server)。

## URL
```http
POST https://api.yunxinapi.com/nimserver/user/setDonnop.action  HTTP/1.1
Content-Type: application/x-www-form-urlencoded;charset=utf-8
```



## 请求参数

- POST 请求中 Headers 的设置请参考[API调用方式](https://doc.yunxin.163.com/messaging/docs/jk3MzY2MTI?platform=server)。

- POST 请求中 Body 的业务参数设置如下：

<table>
            <thead>
        <tr> <th>参数</th><th>类型</th><th>必须</th><th>说明</th></tr></thead>
           <tr> <td>accid</td><td>String</td><td>是</td><td>
           用户的云信 IM 帐号
           </td></tr>
<tr> <td> donnopOpen </td><td> String </td><td>是</td><td>
           桌面端在线时，移动端是否不推送：</br> 
true：移动端不需要推送，false：移动端需要推送
           </td></tr>
</table>


## 示例

### **cURL请求示例**
```curl
curl -X POST -H "AppKey: go9dnk49**lw0803mgq3" -H "Nonce: 4tggg**23t23t" -H "CurTime: 1443592222" -H "CheckSum: 9e9db3b6c9abb2e1962cf3e6f7316fcc55583f86" -H "Content-Type: application/x-www-form-urlencoded" -d 'accid=zhangsan&donnopOpen=true'  'https://api.yunxinapi.com/nimserver/user/setDonnop.action'
```

### **返回示例**
http 响应:json
```json
"Content-Type": "application/json; charset=utf-8"
{
  "code":200
}

```
## 状态码

该接口在 HTTP Body 中返回状态码，状态码详情请参见[状态码](https://doc.yunxin.163.com/messaging/server-apis/TM5NTk2Mzc?platform=server)。
