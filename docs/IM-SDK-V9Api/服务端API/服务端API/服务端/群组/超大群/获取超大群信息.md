

获取超大群的详细信息，可以批量查询，一次最多查询 20 个超大群。

## API 使用限制

单个应用中 1 秒内所有的超大群操作 API 合计最多可调用 100 次，超过后限制调用，会返回 416 错误码。

:::note note
- 除超大群消息相关 API 外，其他所有超大群 API 都属于超大群操作 API。
- 超大群消息相关 API 包括：发送超大群消息，发送超大群自定义系统通知，撤回超大群消息，根据用户 ID/消息 ID 查询超大群历史消息。
:::

## URL
```
POST https://api.yunxinapi.com/nimserver/superteam/getTinfos.action  HTTP/1.1
Content-Type: application/x-www-form-urlencoded;charset=utf-8
```
## 请求参数

- POST 请求中 Headers 的设置请参考[接口概述](/docs/TM5MzM5Njk/jk3MzY2MTI)。
- POST 请求中 Body 的设置如下：
<table>
       <thead>
       <tr> <th>参数</th><th>类型</th><th>必填</th><th>说明</th></tr></thead>
       <tr> <td>tids</td><td>String</td><td>是</td><td>超大群 ID 列表（tids），如["3083","3084"]，一次最多查询 20 个超大群，最大长度 1024 位字符</td></tr>
</table>

## 返回参数

参数|类型|说明
:----|:----|:----
code|Integer|状态码
tinfos|String|超大群的详细信息

tinfos 中的参数说明
参数|类型|说明
:----|:----|:----
tid|Long|超大群 ID
tname|String|超大群名称
creator|String|群主，创建者
tlevel|Integer|超大群级别，群最大人数
size|Integer|超大群大小，即当前群人数
intro|String|超大群介绍
announcement|String|超大群公告
clientCustom|String|第三方客户端自定义字段
serverCustom|String|第三方服务器扩展字段
icon|String|超大群头像
muteType|Integer|群禁言类型
joinmode|Integer|申请入群的验证方式
beinvitemode|Integer|邀请入群的验证方式
invitemode|Integer|邀请入群的权限，即谁可以邀请他人入群
uptinfomode|Integer|修改群信息权限
upcustommode|Integer|修改群自定义属性的权限
createtime|Long|创建时间
updatetime|Long|更新时间
