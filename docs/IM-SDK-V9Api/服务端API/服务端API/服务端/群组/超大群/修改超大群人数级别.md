
IM 服务端支持修改超大群的人数级别（level）。

## API 使用限制

单个应用中 1 秒内所有的超大群操作 API 合计最多可调用 100 次，超过后限制调用，会返回 416 错误码。

:::note note
- 除超大群消息相关 API 外，其他所有超大群 API 都属于超大群操作 API。
- 超大群消息相关 API 包括：发送超大群消息，发送超大群自定义系统通知，撤回超大群消息，根据用户 ID/消息 ID 查询超大群历史消息。
:::

## URL
```
POST https://api.yunxinapi.com/nimserver/superteam/changeLevel.action  HTTP/1.1
Content-Type: application/x-www-form-urlencoded;charset=utf-8
```
## 请求参数

- POST 请求中 Headers 的设置请参考[接口概述](/docs/TM5MzM5Njk/jk3MzY2MTI)。
- POST 请求中 Body 的设置如下：
<table>
       <thead>
       <tr> <th>参数</th><th>类型</th><th>必填</th><th>说明</th></tr></thead>
       <tr> <td>tid</td><td>Long</td><td>是</td><td>云信服务器产生，超大群唯一标识，创建时会返回，最大长度 64 位长整型</td></tr>
       <tr> <td>owner</td><td>String</td><td>是</td><td>群主用户帐号，accid，最大长度 32 位字符</td></tr>
       <tr> <td>tlevel</td><td>Integer</td><td>是</td><td>群人数级别，该参数可设置的区间为 [2 ~ 控制台设置的群人数上限]<br>如何在云信控制台设置超大群的群人数上限（默认为 2000），请参考<a href="https://doc.yunxin.163.com/messaging/docs/TMwNjA4MjU?platform=android">配置超大群功能</a></td></tr>            
</table>

## 示例
### **请求示例（curl）**
```curl
curl -X POST -H "AppKey: go9dnk49***803mgq3" -H "Nonce: 4tggge**23t" -H "CurTime: 1443592222" -H "CheckSum: 9e9db3b6c9abb2e1962cf3e6f7316fcc55583f86" -H "Content-Type: application/x-www-form-urlencoded" -d 'tid=1513535&owner=zhangsan&tlevel=200' 'https://api.yunxinapi.com/nimserver/superteam/changeLevel.action'
```

### **请求成功返回示例**

```json
"Content-Type": "application/json; charset=utf-8"
{
  "code":200 
}
```

### **请求失败返回示例**
```
"Content-Type": "application/json; charset=utf-8"
{
"code": 801  // 超大群人数超过限制
}
```

状态码|说明|处理建议
:----|:----|:-----
200|请求成功|-
403|禁止操作：<ul><li>owner 不是群成员或 owner 不是群主</li><li>超大群功能未开通</li><ul>|根据对应提示信息做出处理
414|参数错误|根据提示信息，检查传入参数的格式和限制条件
416|调用频率超出限制|降低访问频率
500|服务出错|-
806|超出限制：<br>owner 加入的超大群数量超出限制|根据对应提示信息做出处理
