
用户可以查询存储在云信服务器中的超大群历史消息，只能查询在保存时间范围内的消息。 

## 根据用户 ID 查询超大群历史消息

### **API 使用限制**

单个应用最高调用频率：100次/秒，超过返回 416 错误码。

### **URL**
```
POST https://api.yunxinapi.com/nimserver/superteam/queryHistoryMsg.action HTTP/1.1
Content-Type: application/x-www-form-urlencoded;charset=utf-8
```
### **请求参数**

- POST 请求中 Headers 的设置请参考[接口概述](/docs/TM5MzM5Njk/jk3MzY2MTI)。
- POST 请求中 Body 的设置如下：
<table>
       <thead>
       <tr><th>参数</th><th>类型</th><th>必填</th><th>说明</th></tr></thead>
       <tr><td> tid </td><td>Long</td><td>是</td><td>云信服务器产生，超大群唯一标识，创建时会返回，最大长度 64 位长整型</td></tr>
       <tr><td>accid</td><td>String</td><td>是</td><td>需要查询的用户账号，accid，最大长度 32 位字符</td></tr>
       <tr><td> begintime</td><td>Long</td><td>是</td><td>开始时间，毫秒级别时间戳<br/>只能在应用可以查询的历史消息时间以内<br/>只能查询最新入群之后的消息</td></tr>
       <tr><td> endtime </td><td>Long</td><td>是</td><td>截止时间，毫秒级别时间戳</td></tr>
       <tr><td> limit </td><td>Integer</td><td>是</td><td>本次查询的消息条数上限，(0,100]</td></tr>
       <tr><td> reverse </td><td>Integer</td><td>否</td><td>查询方向<br/>1，按时间正序排列；2，按时间降序排列（默认）</td></tr>
       <tr><td> type </td><td>Integer</td><td>否</td><td>查询指定的消息类型，类型之间用","分割，不设置该参数则查询全部类型消息，最大长度 500 位字符<br/>类型支持:<br/>0，文本消息；1，图片；2，语音；3，视频；4，地理位置信息；6，文件；10，提示；100，自定义消息类型</td></tr>
       <tr><td> excludeMsgid </td><td>String</td><td>否</td><td>结束查询的最后一条消息的 msgid（不包含在查询结果中），用于定位锚点</td></tr>
</table>

### **返回参数**

<table>
    <thead>
        <tr> <th>参数</th><th>类型</th><th>说明</th></tr></thead>
        <tr> <td>code</td><td>Integer</td><td>状态码，请求成功则返回 200</td></tr>
        <tr> <td>size</td><td>Integer</td><td>历史消息数量</td></tr>
        <tr> <td>tid</td><td>Long</td><td>超大群 ID </td></tr>
        <tr> <td>msgs</td><td>String</td><td>历史消息的详细信息，JSON</td></tr>
</table>

msgs 中的参数说明
参数|类型|说明
:---|:---|:---
msgid|Long|服务端消息 ID
sendtime|Long|消息发送时间
from|String|消息发送方账号，accid
type|Integer|消息类型
ext|String|第三方扩展字段
body|Object|消息体，根据消息类型的不同而不同，格式参见[消息格式示例](https://doc.yunxin.163.com/messaging/docs/DQ2NTg4ODE?platform=server#%E6%B6%88%E6%81%AF%E6%A0%BC%E5%BC%8F%E7%A4%BA%E4%BE%8B)

### **示例**
#### **请求示例（curl）**
```curl
curl -X POST -H "AppKey: go9dnk***0803mgq3" -H "Nonce: 4tggg**3t23t" -H "CurTime: 1443592222" -H "CheckSum: 9e9db3b6c9abb2e1962cf3e6f7316fcc55583f86" -H "Content-Type: application/x-www-form-urlencoded" -d 'begintime=1443599631111&endtime=1443599639999&tid=1513535&accid=zhangsan&limit=50' 'https://api.yunxinapi.com/nimserver/superteam/queryHistoryMsg.action'
```
	            
#### **请求成功返回示例**

```json
"Content-Type": "application/json; charset=utf-8"
{
    "code": 200
    "tid": 123, 
    "size":1,
    "msgs":[
        {
            "msgid":1123, 
            "sendtime":1123123, 
            "from": "accid", 
            "type":0, 
            "ext": null, 
            "body":{  // （消息body根据消息类型的不同而不同，body格式见1.5.10）, Object
                "msg": "msg" // 消息体
            }
        }
    ]
}
```

#### **请求失败返回示例**
```
"Content-Type": "application/json; charset=utf-8"
{
    "code": 414,
    "desc": "bad reverse"  // reverse 不合法
}
```

### **状态码**

该接口在 HTTPS Body 中返回请求的状态码，以下仅列出与接口业务相关的状态码。完整状态码请参见[状态码](https://doc.yunxin.163.com/messaging/docs/TM5NTk2Mzc?platform=server)。

状态码|说明|处理建议
:----|:----|:-----
200|请求成功|-
403|禁止操作：<br>超大群功能未开通或 app 权限错误|根据对应提示信息做出处理
414|参数错误|根据提示信息，检查传入参数的格式和限制条件
416|调用频率超出限制|降低访问频率
500|服务出错|-


## 根据消息 ID 查询超大群历史消息

### **API 使用限制**

单个应用最高调用频率：100次/秒，超过返回 416 错误码。

### **URL**
```
POST https://api.yunxinapi.com/nimserver/superteam/queryHistoryMsgByIds.action  HTTP/1.1
Content-Type: application/x-www-form-urlencoded;charset=utf-8
```
### **请求参数**

- POST 请求中 Headers 的设置请参考[接口概述](/docs/TM5MzM5Njk/jk3MzY2MTI)。
- POST 请求中 Body 的设置如下：
<table>
       <thead>
       <tr><th>参数</th><th>类型</th><th>必填</th><th>说明</th></tr></thead>
       <tr><td> msgs </td><td>String</td><td>是</td><td>可转换成 JSONArray，最多只能有 100 个JSON，最大长度 20480 位字符<br/>每个 JSON 中包括一下四个字段：<br/>from：消息发送者账号，accid（String）<br/>to：超大群 ID，tid（Long）<br/>time：消息发送时间（Long）<br/>msgid：消息 ID（Long）</td></tr>
</table>

### **返回参数**

<table>
    <thead>
        <tr> <th>参数</th><th>类型</th><th>说明</th></tr></thead>
        <tr> <td>code</td><td>Integer</td><td>状态码，请求成功则返回 200</td></tr>
        <tr> <td>msgs</td><td>String</td><td>历史消息的详细信息，JSON</td></tr>
</table>

msgs 中的参数说明
参数|类型|说明
:---|:---|:---
msgid|Long|服务器消息 ID
sendtime|Long|消息发送时间
from|String|消息发送方账号，accid
type|Integer|消息类型
ext|String|第三方扩展字段
body|Object|消息体，根据消息类型的不同而不同，格式参见[消息格式示例](https://doc.yunxin.163.com/messaging/docs/DQ2NTg4ODE?platform=server#%E6%B6%88%E6%81%AF%E6%A0%BC%E5%BC%8F%E7%A4%BA%E4%BE%8B)
