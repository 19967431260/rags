
IM 服务端支持修改超大群成员信息，包括超大群成员的群昵称，自定义扩展信息，以及超大群消息的提醒开关，用户可按需开启或关闭超大群消息提醒。

## API 使用限制

单个应用中 1 秒内所有的超大群操作 API 合计最多可调用 100 次，超过后限制调用，会返回 416 错误码。

:::note note
- 除超大群消息相关 API 外，其他所有超大群 API 都属于超大群操作 API。
- 超大群消息相关 API 包括：发送超大群消息，发送超大群自定义系统通知，撤回超大群消息，根据用户 ID/消息 ID 查询超大群历史消息。
:::


## URL
```
POST https://api.yunxinapi.com/nimserver/superteam/updateTlist.action HTTP/1.1
Content-Type:application/x-www-form-urlencoded;charset=utf-8
```
## 请求参数

- POST 请求中 Headers 的设置请参考[接口概述](/docs/TM5MzM5Njk/jk3MzY2MTI)。
- POST 请求中 Body 的设置如下：
<table>
       <thead>
       <tr> <th>参数</th><th>类型</th><th>必填</th><th>说明</th></tr></thead>
       <tr> <td>tid</td><td>Long</td><td>是</td><td>云信服务器产生，超大群唯一标识，创建时会返回，最大长度 64 位长整型</td></tr>
       <tr><td>accid</td><td>String</td><td>是</td><td>需要设置的用户账号，accid，最大长度 32 位字符</td></tr>
       <tr> <td>silentType</td><td>Integer</td><td>否</td><td>1，关闭消息提醒；0，打开消息提醒</td></tr>
       <tr> <td>nick</td><td>String</td><td>否</td><td>用户对应的群成员昵称，最大长度 32 位字符</td></tr>
       <tr> <td>custom</td><td>String</td><td>否</td><td>自定义扩展字段，最大长度 32 位字符</td></tr>
       <tr> <td> bid </td><td>String</td><td>否</td><td>反垃圾业务ID，JSON字符串，{"textbid":"","picbid":""}，若不填则使用原来的反垃圾配置</td></tr>
</table>

## 示例
### **请求示例（curl）**
```curl
curl -X POST -H "CheckSum: e8c47bc1ede28***f7e7e8f2" -H "AppKey: fe41664**7ad2547" -H "Nonce: 1" -H "CurTime: 1452010720" -H "Content-Type: application/x-www-form-urlencoded" -d 'tid=4104&accid=zhangsan&silentType=1&nick=jack' 'https://api.yunxinapi.com/nimserver/superteam/updateTlist.action'
```

### **请求成功返回示例**

```json
"Content-Type": "application/json; charset=utf-8"
{
  "code":200 
}
```


### **请求失败返回示例**
```
"Content-Type": "application/json; charset=utf-8"
{
"code": 414,  // 参数错误
"desc": "tid not exist" //群不存在
}
```
