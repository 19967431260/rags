


群主和管理员可以禁言指定群成员，可以批量操作，但是一次最多禁言 10 人。

其中，群主可以禁言任何成员，管理员只能禁言普通成员。

## API 使用限制

单个应用中 1 秒内所有的超大群操作 API 合计最多可调用 100 次，超过后限制调用，会返回 416 错误码。

:::note note
- 除超大群消息相关 API 外，其他所有超大群 API 都属于超大群操作 API。
- 超大群消息相关 API 包括：发送超大群消息，发送超大群自定义系统通知，撤回超大群消息，根据用户 ID/消息 ID 查询超大群历史消息。
:::

## URL
```
POST https://api.yunxinapi.com/nimserver/superteam/muteTlist.action  HTTP/1.1
Content-Type: application/x-www-form-urlencoded;charset=utf-8
```
## 请求参数

- POST 请求中 Headers 的设置请参考[接口概述](/docs/TM5MzM5Njk/jk3MzY2MTI)。
- POST 请求中 Body 的设置如下：
<table>
       <thead>
       <tr> <th>参数</th><th>类型</th><th>必填</th><th>说明</th></tr></thead>
       <tr> <td>tid</td><td>Long</td><td>是</td><td>云信服务器产生，超大群唯一标识，创建时会返回，最大长度 64 位长整型</td></tr>
       <tr> <td>owner</td><td>String</td><td>是</td><td>群主或管理员帐号，accid，最大长度 32 位字符</td></tr>
       <tr> <td>muteAccids</td><td>String</td><td>是</td><td>需要禁言的群成员账号，\["aaa","bbb"\](JSONArray 对应的 accid，如果解析出错会报 414)，一次最多禁言 10 人</td></tr>
       <tr> <td>mute</td><td>Integer</td><td>是</td><td>是否禁言<br/>1，禁言；0，解禁</td></tr>
</table>

## 示例
### **请求示例（curl）**
```curl
curl -X POST -H "AppKey: go9dnk49b***03mgq3" -H "Nonce: 4tggger**3t23t" -H "CurTime: 1443592222" -H "CheckSum: 9e9db3b6c9abb2e1962cf3e6f7316fcc55583f86" -H "Content-Type: application/x-www-form-urlencoded" -d 'tid=1513535&owner=zhangsan&muteAccids=%5B%22lisi%22%2C%22wangwu%22%5D&mute=1' 'https://api.yunxinapi.com/nimserver/superteam/muteTlist.action'
```

### **请求成功返回示例**
```
"Content-Type": "application/json; charset=utf-8"
{
    "code":200
}
```

### **请求失败返回示例**
```
"Content-Type": "application/json; charset=utf-8"
{
    "code": 414,
    "desc": "muteAccids size exceed!"  // muteAccids 列表超过限制
}
```
