在群组（高级群）内发送消息的服务端 API 与发送单聊消息的 API 相同（<a href="https://doc.yunxin.163.com/messaging/docs/DQ2NTg4ODE?platform=server#发送消息" target="_blank">`msg/sendMsg.action`</a>）。该 API 通过`ope`参数区分发送的为单聊消息还是群聊消息。

本文介绍如何通过该 API 发送群聊消息，并在 SDK 端成功接收。


## API 使用限制

单个应用中 1 秒内默认最多调用该 API 100 次。如超限，将被屏蔽 10 秒。



## 前提条件
通过 IM 服务端 API 发送群消息前，请确保：
- 已通过 IM 服务端 API 或者 SDK API 创建高级群，获取群 ID。
- 群聊消息发送方和接收方已加入同一高级群。



## 实现群聊消息收发

**时序图**

以下时序图仅以 “发送方为高级群创建者，且创建群组和拉人入群均通过调用服务端 API 实现” 这个场景为例，展示实现群聊消息收发的流程。

@startuml
!theme cerulean
skinparam backgroundColor #FFFFFF


发送方 -> IM服务端: 创建高级群
接收方 -[#Darkorange]> "NIM SDK 实例": 监听群聊消息接收
接收方 -> "NIM SDK 实例": 登录
发送方 -> IM服务端: 将接收方加入高级群
发送方 -[#Darkorange]> IM服务端: msg/sendMsg.action
IM服务端 --[#Darkorange]>"NIM SDK 实例": 投递群聊消息
"NIM SDK 实例" --[#Darkorange]> 接收方: 触发回调投递群聊消息
@enduml

2. 发送方调用<a href="https://doc.yunxin.163.com/messaging/docs/DQ2NTg4ODE?platform=server#发送消息" target="_blank">`msg/sendMsg.action`</a> 发送群聊消息，调用时将参数`ope`设置为`1`（表示发送的为群聊消息），将参数`to`设置为群 ID。





3. IM 服务端将群聊消息投递至 SDK 端，SDK 端触发群聊消息接收回调，将群聊消息投递至接收方。
  

:::note notice
若消息的发送方不在群组内，会返回 414 错误码。
:::
