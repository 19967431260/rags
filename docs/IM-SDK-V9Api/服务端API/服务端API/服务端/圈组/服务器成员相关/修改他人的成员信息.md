


云信服务端提供 API 支持修改他人的服务器成员信息。


## 功能介绍

拥有“修改他人成员信息的权限”的服务器成员，可按需修改其他成员在服务器内显示的成员信息，如头像、昵称等。修改后，修改人和被修改人如果在线，都将收到“修改服务器成员信息”
（`SERVER_MEMBER_UPDATE(14)`）的系统通知。 



::: note note 
如果您的应用需要其他服务器成员也能够接收该系统通知，请通过云信官网提供的联系方式咨询商务经理开通。开通后，其他服务器成员接收该系统通知的条件，参见[服务器相关事件通知](https://doc.yunxin.163.com/messaging/docs/TkxMzc1NDg?platform=server#服务器相关事件通知)。
:::


## URL 
```http
POST https://api.yunxinapi.com/nimserver/qchat/updateOtherServerMember.action HTTP/1.1
Content-Type:application/x-www-form-urlencoded;charset=utf-8
```


## 请求参数


- POST 请求中 Headers 的设置请参考[API调用方式](https://doc.yunxin.163.com/TM5MzM5Njk/docs/jk3MzY2MTI?platform=server)。
- POST 请求中 Body 的设置如下：


<table>
    <thead><tr> <th style="width:160px">参数</th><th style="width:80px">类型</th><th>必须</th><th>说明</th></tr></thead>
        <tr> <td>accid</td><td>String</td><td>是</td><td>修改人的 IM 账号<note type=notice>修改人需要拥有“修改他人成员信息的权限”，否则将返回 403 错误码。</td></tr>
        <tr> <td>serverId</td><td>String</td><td>是</td><td>服务器 ID</td></tr>
        <tr> <td>otherAccid</td><td>String</td><td>是</td><td>被修改人的 IM 帐号</td></tr>
        <tr> <td>nick</td><td>String</td><td>否</td><td>被修改人的昵称，最大长度 64 字符</td></tr>
        <tr> <td>avatar</td><td>String</td><td>否</td><td>被修改人头像的 URL，最大长度 1024 字符</td></tr>
        <tr> <td>bid</td><td>String</td><td>否</td><td>安全通的自定义反垃圾（即内容审核）业务的 ID。自定义反垃圾业务主要用来针对单条消息或单次操作的资料信息进行除了默认反垃圾业务以外的内容审核。如需配置自定义反垃圾，请通过云信官网首页提供的微信、在线聊天和电话等方式联系商务经理进行配置，并获取对应的业务 ID。可转换为 JSON 格式，如：{"textbid":"","picbid":""}</td></tr>
</table>

## 返回参数



<div style="width:180px">参数</div> | <div style="width:90px">类型</div>  | 说明
---- | -------------- | ---------
code  | int  | 状态码
serverMember  | String  | 服务器成员信息，具体字段说明见[“修改自己的成员信息”接口返回参数中的 serverMember 字段说明](https://doc.yunxin.163.com/messaging/docs/DQyMDI4ODE?platform=server#返回参数)

  
## 示例
### cURL 请求示例
```curl
curl -X POST -H "AppKey: go9dnk49**0803mgq3" -H "Nonce: 4tggge**23t23t" -H "CurTime: 1443592222" -H "CheckSum: 9e9db3b6c9abb2e1962cf3e6f7316fcc55583f86" -H "Content-Type: application/x-www-form-urlencoded" -d 'accid=zhangsan&serverId=1513535&otherAccid=lisi&nick=update' 'https://api.yunxinapi.com/nimserver/qchat/updateOtherServerMember.action'
```

## 状态码


该接口在 HTTPS Body 中返回请求的状态码，以下仅列出与接口业务强相关的状态码。完整状态码请参见<a href="https://doc.yunxin.163.com/messaging/server-apis/TM5NTk2Mzc?platform=server" target="_blank">状态码</a>。


|状态码 | 说明 | 处理建议|
|---- | -------------- | ---------|
| 200 | 请求成功  | - |
| 403 | 非法操作或没有权限 |   <ul><li>检查是否已开通圈组功能</li><li>检查修改者是否拥有“修改他人成员信息的权限”</li><li>检查服务器资料中是否包含违规或敏感内容。如果已开通安全通且被检测到存在违规或敏感信息，可能会被安全通的内容审核判定为“不通过”</li></ul> |
| 404  |  对象不存在   |  <ul><li>检查是否存在必传参数为空的问题</li><li>检查传入的 IM 账号、服务器 ID 或自定义反垃圾业务 ID （bid）是否存在</li></ul>
| 414 | 参数错误 | 根据提示信息，检查传入参数的格式和限制条件|
| 416 |调用频率超限 | 降低调用频率 |
| 431 | HTTP 重复请求| -  |
