
云信服务端提供 API，支持用户按时间分页查询服务器的成员列表。


## URL 
```http
POST https://api.yunxinapi.com/nimserver/qchat/getServerMemberListPage.action HTTP/1.1
Content-Type:application/x-www-form-urlencoded;charset=utf-8
```

## 请求参数

- POST 请求中 Headers 的设置请参考[API调用方式](https://doc.yunxin.163.com/TM5MzM5Njk/docs/jk3MzY2MTI?platform=server)。
- POST 请求中 Body 的设置如下：


<table>
    <thead><tr> <th style="width:160px">参数</th><th style="width:80px">类型</th><th>必须</th><th>说明</th></tr></thead>
        <tr> <td>serverId</td><td>String</td><td>是</td><td>服务器 ID</td></tr>
        <tr> <td>timetag</td><td>String</td><td>是</td><td>用户加入服务器时间，单位毫秒，如果传 0 表示查询首页，按照成员加入服务器时间的逆序进行分页查询</td></tr>
        <tr> <td>limit</td><td>String</td><td>是</td><td>本次查询的数量上限，最大数量 200</td></tr>
        <tr> <td>accid</td><td>String</td><td>否</td><td>待查询用户的 IM 帐号，如果传入校验该用户是否为服务器成员</td></tr>
</table>


## 返回参数



<div style="width:160px">参数</div> | 说明
---- | -------------- 
code  | 状态码  
pageInfo | 分页信息，hasMore：是否查询下一页，nextTimetag：查询下一页时需要的时间戳
serverMembers  | 返回的服务器成员列表，具体字段说明参见[“修改自己的成员信息”接口返回参数中的 serverMember 字段说明](https://doc.yunxin.163.com/messaging/docs/DQyMDI4ODE?platform=server#返回参数)

## 状态码

该接口在 HTTPS Body 中返回请求的状态码，以下仅列出与接口业务强相关的状态码。完整状态码请参见<a href="https://doc.yunxin.163.com/messaging/server-apis/TM5NTk2Mzc?platform=server" target="_blank">状态码</a>。


|状态码 | 说明 | 处理建议|
|---- | -------------- | ---------|
| 200 | 请求成功  | - |
| 403 | 非法操作或没有权限 |检查是否已开通圈组功能 |
| 404  |  对象不存在   |  <ul><li>检查是否存在必传参数为空的问题</li><li>检查传入的 IM 账号或服务器 ID 是否存在</li></ul>
| 414 | 参数错误 | 根据提示信息，检查传入参数的格式和限制条件|
| 416 |调用频率超限 | 降低调用频率 |
| 431 | HTTP 重复请求| -  |
