

云信服务端支持服务器内的审批方调用 API 拒绝用户主动发起的“加入服务器的申请”。

## 功能介绍


申请方发起申请后，服务器内拥有“处理加入服务器申请的权限”的成员（以下简称为“审批方”），将收到一条“申请加入服务器”（`SERVER_MEMBER_APPLY(3)`）的系统通知。审批方可调用服务端 API 拒绝申请。审批方拒绝申请后，申请方将收到一条“拒绝申请”（`SERVER_MEMBER_APPLY_REJECT(4)`）的系统通知。

::: note notice
服务器内最多 100 位（随机 100 位）拥有“处理加入服务器申请的权限”的成员可收到（`SERVER_MEMBER_APPLY(3)`）系统通知。
:::






## URL 
```http
POST https://api.yunxinapi.com/nimserver/qchat/rejectServerApply.action HTTP/1.1
Content-Type:application/x-www-form-urlencoded;charset=utf-8
```



## 请求参数




- POST 请求中 Headers 的设置请参考[API调用方式](https://doc.yunxin.163.com/TM5MzM5Njk/docs/jk3MzY2MTI?platform=server)。
- POST 请求中 Body 的设置如下：


<table>
    <thead><tr> <th style="width:160px">参数</th><th style="width:80px">类型</th><th>必须</th><th>说明</th></tr></thead>
        <tr> <td>accid</td><td>String</td><td>是</td><td>审批方的 IM 账号</td></tr>
        <tr> <td>serverId</td><td>Long</td><td>是</td><td>服务器 ID</td></tr>
        <tr> <td>applyAccid</td><td>String</td><td>是</td><td>申请方的 IM 帐号</td></tr>
        <tr> <td>msg</td><td>String</td><td>是</td><td>拒绝申请的消息内容，最大长度 150 字符</td></tr>
        <tr> <td>requestId</td><td>Long</td><td>否</td><td>申请ID </td></tr>
</table>



## 示例
### cURL 请求示例
```curl
curl -X POST -H "AppKey: go9dnk49**803mgq3" -H "Nonce: 4tggge**3t23t" -H "CurTime: 1443592222" -H "CheckSum: 9e9db3b6c9abb2e1962cf3e6f7316fcc55583f86" -H "Content-Type: application/x-www-form-urlencoded" -d 'serverId=1513535&accid=zhangsan&applyAccid=lisi&msg=reject' 'https://api.yunxinapi.com/nimserver/qchat/rejectServerApply.action'
```

### 返回示例
HTTP 响应：JSON
```json
"Content-Type": "application/json; charset=utf-8"
{
    "code":200
}

```
