


云信服务端支持更新部分类型的系统通知。只有**存离线**的系统通知支持更新，并且只有**系统通知接收方**可更新。可更新的信息包括通知内容、通知扩展字段、通知状态等。


## 功能介绍


接收方可更新的系统通知类型如下表所示：

系统通知分类 ||  相关限制
---- | --------------
内置系统通知|邀请服务器成员<br>SERVER_MEMBER_INVITE(1) | 默认自动存离线。这四种通知每月总共至多存 1000 条离线，**超限后将无法更新通知的信息**
^^|拒绝邀请<br>SERVER_MEMBER_INVITE_REJECT(2)|^^
^^|申请加入服务器<br>SERVER_MEMBER_APPLY(3) | ^^
^^ |拒绝申请<br>SERVER_MEMBER_APPLY_REJECT(4) | ^^
自定义系统通知<br>CUSTOM(100) ||  只有**存离线**的自定义系统通知才能更新。每月至多存 1000 条离线，**超限后将无法更新通知的信息**

::: note notice
- 只有指明通知接收者的自定义系统通知才能设置存离线，即[发送自定义系统通知](https://doc.yunxin.163.com/TM5MzM5Njk/docs/jQ2MjQwNDg?platform=server)时， toAccids 不为空。
- 其他内置系统通知，不支持更新。具体的内置系统通知类型，请参见<a href="https://doc.yunxin.163.com/messaging/docs/TkxMzc1NDg?platform=server#内置系统通知类型" target="_blank">内置系统通知类型</a>。
:::

## **URL**
```
POST https://api.yunxinapi.com/nimserver/qchat/updateAttachMsg.action HTTP/1.1
Content-Type:application/x-www-form-urlencoded;charset=utf-8
```

## **示例**

### cURL 请求示例

```curl
curl -X POST -H "AppKey: go9dn***03mgq3" -H "Nonce: 4tgg***t23t" -H "CurTime: 1443592222" -H "CheckSum: 9e9d***c55583f86" -H "Content-Type: application/x-www-form-urlencoded" -d 'operatorAccid=zhangsan&msgIdServer=1&serverId=1&channelId=1&time=123&type=100&body=abc' 'https://api.yunxinapi.com/nimserver/qchat/updateAttachMsg.action'
```

### 返回示例

```json
"Content-Type": "application/json; charset=utf-8"
{
    "code":200
}

```

## 状态码
该接口在 HTTPS Body 中返回请求的状态码，以下仅列出与接口业务相关的状态码。完整状态码请参见<a href="https://doc.yunxin.163.com/messaging/server-apis/TM5NTk2Mzc?platform=server" target="_blank">状态码</a>。


|状态码 | 说明 | 处理建议|
|---- | -------------- | ---------|
| 200 | 请求成功  | - |
| 403 | 非法操作或没有权限 |   检查是否已开通圈组功能|
| 404  |  对象不存在   |  <ul><li>检查是否存在必传参数为空的问题</li><li>检查传入的 IM 账号、服务器 ID、频道 ID、消息 ID 等是否存在</li></ul>
| 414 | 参数错误 | 根据提示信息，检查传入参数的格式和限制条件|
| 416 |调用频率超限 | 降低调用频率 |
| 431 | HTTP 重复请求| -  |
