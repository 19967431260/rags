


分页查询频道分组黑白名单身份组列表。


## URL
```http
POST http://api.yunxinapi.com/nimserver/qchat/queryCategoryWhiteBlackIdentifyPage.action  HTTP/1.1
Content-Type: application/x-www-form-urlencoded;charset=utf-8
```


## 请求参数

- POST 请求中 Headers 的设置请参考[API 调用方式](https://doc.yunxin.163.com/messaging/docs/jk3MzY2MTI?platform=server)。

- POST 请求中 Body 的设置如下：

<table>
    <thead><tr> <th>参数</th><th>类型</th><th>必须</th><th>说明</th></tr></thead>
        <tr> <td>serverId</td><td>Long</td><td>是</td><td>服务器唯一标识</td></tr>
        <tr> <td>categoryId</td><td>Long</td><td>是</td><td>频道分组唯一标识</td></tr>
        <tr> <td>accid</td><td>String</td><td>否</td><td>操作者accid，如果传，则会校验该账号对该频道分组是否有权限，否则不校验</td></tr>
        <tr> <td>type</td><td>Long</td><td>是</td><td>1表示白名单，2表示黑名单</td></tr>
        <tr> <td>timetag</td><td>Long</td><td>是</td><td>时间戳，毫秒对应的字符串，0则表示查询最近的一页</td></tr>
        <tr> <td>limit</td><td>Long</td><td>是</td><td>本次查询的条数上限</td></tr>
</table>


## 返回参数


参数 |  说明
:-----|:--|:---
pageInfo |   分页信息，具体成员参数见下表
identifies | 黑名单身份组列表或白名单身份组列表，具体成员参数见下表


pageInfo 参数说明如下：

参数| 类型| 说明
:-----|:--|:---
hasMore | Boolean | 是否有下一页
nextTimetag |   Long   | 查询下一页时需要传入的时间戳

identifies 参数说明如下：

参数| 类型| 说明
:-----|:--|:---
ext | String  | 身份组的自定义扩展字段
membercount | Number| 身份组成员数量
createtime | Number | 创建时间
roleId | Integer | 身份组 ID 
auths |  String | 身份组的权限项，key为权限项的枚举值，value 为 1 开启（身份组成员拥有该权限），-1 关闭（身份组成员无该权限）
name  | String  | 身份组名称
icon   |  String  |  身份组图标的 URL
priority  | String  |  身份组的优先级，正数。数字越小，优先级越大，最小为1，everyone 身份组为 0，但不参与比较
type  |  Integer  |   身份组类型，1 为 @everyone 身份组，2 为自定义身份组
updatetime | String  | 更新时间
