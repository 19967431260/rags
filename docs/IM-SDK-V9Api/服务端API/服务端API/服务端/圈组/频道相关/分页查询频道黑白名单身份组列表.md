

云信服务端支持分页查询频道的黑白名单身份组列表。被加入黑名单或白名单的身份组即为**黑白名单身份组**。


## URL 
```http
POST https://api.yunxinapi.com/nimserver/qchat/queryWhiteBlackIdentify.action HTTP/1.1
Content-Type:application/x-www-form-urlencoded;charset=utf-8
```


## 请求参数

- POST 请求中 Headers 的设置请参考[API调用方式](/docs/TM5MzM5Njk/jk3MzY2MTI)。

- POST 请求中 Body 的设置如下：



<table>
    <thead><tr> <th style="width:160px">参数</th><th style="width="80px">类型</th><th>必须</th><th>说明</th></tr></thead>
        <tr> <td>serverId</td><td>Long</td><td>是</td><td>服务器 ID</td></tr>
        <tr> <td>channelId</td><td>Long</td><td>是</td><td>频道 ID </td></tr>
        <tr> <td>accid</td><td>String</td><td>是</td><td>操作者的 IM 账号，操作者需要拥有管理黑白名单的权限</td></tr>
        <tr> <td>type</td><td>int</td><td>是</td><td>1 表示白名单，2 表示黑名单</td></tr>
        <tr> <td>timetag</td><td>Long</td><td>是</td><td>时间戳，单位：毫秒，如传 0 则表示查询最近的一页</td></tr>
        <tr> <td>limit</td><td>String</td><td>是</td><td>本次查询的条数上限</td></tr>
</table>


## 返回参数



参数 | 说明
---- | -------------- 
code | 状态码
pageInfo | 分页信息，hasMore：是否查询下一页，nextTimetag：查询下一页时需要的时间戳
identifies  |  身份组列表，单个身份组所包含的字段信息参见[修复服务器身份组的返回参数中 identify 的字段说明](https://doc.yunxin.163.com/messaging/docs/DI4NzcyMTc?platform=server#修复服务器身份组的返回参数)

## 状态码


该接口在 HTTPS Body 中返回请求的状态码，以下仅列出与接口业务相关的状态码。完整状态码请参见[状态码](https://doc.yunxin.163.com/messaging/server-apis/TM5NTk2Mzc?platform=server)。


状态码 | 说明   | 处理建议
:---- | :---------|:-----
200 | 请求成功  | -
403 | 非法操作或没有权限 |   <ul><li>检查是否已开通圈组功能</li><li>检查操作者是否为当前服务器和频道的成员，如非成员则无权限</li><li>检查操作者是否拥有管理黑白名单的权限</li></ul>
404 | 对象不存在  | <ul><li>检查是否存在必传参数为空的问题</li><li>检查传入的服务器 ID 、频道 ID 或账号是否存在</li></ul>
414 | 参数错误 | 根据提示信息，检查传入参数的格式和限制条件
416 | 调用过于频繁  | 降低调用频率
