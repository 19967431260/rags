


云信服务端支持修改频道的分组相关信息，包括所属的频道分组和同步模式。 

::: note notice
服务器创建者以外的用户，需要拥有管理频道的权限，才可修改频道的分组相关信息。
:::

## 调用时机


圈组服务器内已存在频道。


## URL

```
POST  http://api.yunxinapi.com/nimserver/qchat/updateCategoryInfoOfChannel.action  HTTP/1.1
Content-Type: application/x-www-form-urlencoded;charset=utf-8
```

## 请求参数

- POST 请求中 Headers 的设置请参考[API调用方式](/docs/TM5MzM5Njk/jk3MzY2MTI)。

- POST 请求中 Body 的设置如下：

| <div style="width:160px">参数</div>      | <div style="width:90px">类型</div> | 是否必填 | 说明                                                                           |
|---- | -------------- | --------- |------------|
| accid      | String | 是 | 用户帐号                                   |
| channelId  | Long | 是 | 频道 ID，频道唯一标识                               |
| categoryId | Long | 否 | 频道分组 ID，频道分组唯一标识                           |
| syncMode   | int | 否 | 同步模式，0-不与频道分组同步数据，1-与频道分组同步数据 |

::: note notice
`syncMode`和`categoryId`不能同时为空。
:::


## 返回参数

| <div style="width:160px">参数</div>      | <div style="width:90px">类型</div> | 说明                                                                           |
---- | -------------- | ---------
code  | int | 状态码，表示 API 调用的结果状态
channelInfo | String  | 修改后的频道信息，具体字段信息参见[修改频道基础信息的返回参数中的 channelInfo](https://doc.yunxin.163.com/messaging/docs/zI1MTAzMzY?platform=server#返回参数)

## 状态码


该接口在 HTTPS Body 中返回请求的状态码，以下仅列出与接口业务相关的状态码。完整状态码请参见<a href="https://doc.yunxin.163.com/messaging/server-apis/TM5NTk2Mzc?platform=server" target="_blank">状态码</a>。


|状态码 | 说明 | 处理建议|
|---- | -------------- | ---------|
| 200 | 请求成功  | - |
| 403 | 非法操作或没有权限 |   <ul><li>检查是否已开通圈组功能</li><li>检查是否拥有相应的权限，如频道管理权限</li></ul> |
| 404  |  对象不存在   |  <ul><li>检查是否存在必传参数为空的问题</li><li>检查传入的账号、频道 ID、频道分组 ID 等是否存在</li></ul>
| 414 | 参数错误 | 根据提示信息，检查传入参数的格式和限制条件|
| 416 |调用频率超限 | 降低调用频率 |
| 431 | HTTP 重复请求| -  |
