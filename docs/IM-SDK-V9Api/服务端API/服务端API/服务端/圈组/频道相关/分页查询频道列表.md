
云信服务端支持分页查询频道列表。

## 调用时机

圈组服务器内已存在频道，用户进入服务器后需要获取频道列表时。



## URL
```http
POST https://api.yunxinapi.com/nimserver/qchat/getChannelListPage.action HTTP/1.1
Content-Type:application/x-www-form-urlencoded;charset=utf-8
```



## 请求参数




- POST 请求中 Headers 的设置请参考[API调用方式](/docs/TM5MzM5Njk/jk3MzY2MTI)。

- POST 请求中 Body 的设置如下：




<table>
    <thead><tr> <th style="width:160px">参数</th><th style="width:90px">类型</th><th>必填</th><th>说明</th></tr></thead>
        <tr> <td>serverId</td><td>Long</td><td>是</td><td>服务器 ID</td></tr>
        <tr> <td>timetag</td><td>Long</td><td>是</td><td>频道创建时间，单位毫秒，如果传 0 表示查询首页，分页查询按照创建时间倒序查询</td></tr>
        <tr> <td>limit</td><td>String</td><td>是</td><td>本次查询的数量上限，最大返回 200 条结果</td></tr>
        <tr> <td>accid</td><td>String</td><td>否</td><td>用户帐号，如果不传返回服务器下的频道，如果传入返回服务器下用户可查看的频道</td></tr>
</table>





## 返回参数


<div style="width:160px">参数</div> | 说明
---- | -------------- 
code  | 状态码  
pageInfo | 分页信息，hasMore：是否查询下一页，nextTimetag：查询下一页时需要的时间戳
channelInfos  | 返回的频道信息列表，具体字段说明参见[修改频道基础信息的返回参数中的 channelInfo](https://doc.yunxin.163.com/messaging/docs/zI1MTAzMzY?platform=server#返回参数)

## 状态码



该接口在 HTTPS Body 中返回请求的状态码，以下仅列出与接口业务相关的状态码。完整状态码请参见<a href="https://doc.yunxin.163.com/messaging/server-apis/TM5NTk2Mzc?platform=server" target="_blank">状态码</a>。


|状态码 | 说明 | 处理建议|
|---- | -------------- | ---------|
| 200 | 请求成功  | - |
| 403 | 非法操作或没有权限 |   <ul><li>检查是否已开通圈组功能</li><li>检查传入的 accid 是否为当前服务器和频道的成员，如果非成员则无权限查询</li></ul>  |
| 404 | 对象不存在  | <ul><li>检查是否存在必传参数为空的问题</li><li>检查传入的服务器 ID 或账号是否存在</li></ul>
| 414 | 参数错误 | 根据提示信息，检查传入参数的格式和限制条件|
| 416 |调用频率超限 | 降低调用频率 |
| 431 | HTTP 重复请求| -  |
