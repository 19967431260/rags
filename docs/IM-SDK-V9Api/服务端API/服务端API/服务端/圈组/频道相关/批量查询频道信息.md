
云信服务端支持传入最多 200 个频道 ID 查询这些频道的信息，如频道主题、频道类型、所属的频道分组的 ID 等。


## URL 
```http
POST https://api.yunxinapi.com/nimserver/qchat/getChannels.action HTTP/1.1
Content-Type:application/x-www-form-urlencoded;charset=utf-8
```



## 请求参数


- POST 请求中 Headers 的设置请参考[API调用方式](/docs/TM5MzM5Njk/jk3MzY2MTI)。

- POST 请求中 Body 的设置如下：


<table>
    <thead><tr> <th style="width:160px">参数</th><th style="width:90px">类型</th><th>必填</th><th>说明</th></tr></thead>
        <tr> <td>channelIds</td><td>JSON Array</td><td>是</td><td>频道 ID 列表，JSON Array 格式，最大数量 200，示例：["123","234"]</td></tr>
</table>


## 返回参数


<div style="width:160px">参数</div> | 说明
---- | -------------- 
code  | 状态码  
channelInfos  | 返回的频道信息列表，具体字段说明参见[修改频道基础信息的返回参数中的 channelInfo](https://doc.yunxin.163.com/messaging/docs/zI1MTAzMzY?platform=server#返回参数)

## 示例

### curl请求示例
```curl
curl -X POST -H "AppKey: go9dn**glw0803mgq3" -H "Nonce: 4tgg**3t23t" -H "CurTime: 1443592222" -H "CheckSum: 9e9db3b6c9abb2e1962cf3e6f7316fcc55583f86" -H "Content-Type: application/x-www-form-urlencoded" -d 'channelIds=["123","234"]' 'https://api.yunxinapi.com/nimserver/qchat/getChannels.action'
```

## 状态码



该接口在 HTTPS Body 中返回请求的状态码，以下仅列出与接口业务相关的状态码。完整状态码请参见<a href="https://doc.yunxin.163.com/messaging/server-apis/TM5NTk2Mzc?platform=server" target="_blank">状态码</a>。


|状态码 | 说明 | 处理建议|
|---- | -------------- | ---------|
| 200 | 请求成功  | - |
| 403 | 非法操作或没有权限 |   检查是否已开通圈组功能  |
| 404 | 对象不存在  | <ul><li>检查 channelIds 是否为空</li><li>检查 channelIds 中是否包含不存在的频道 ID</li></ul>
| 414 | 参数错误 | 根据提示信息，检查传入参数的格式和限制条件|
| 416 |调用频率超限 | 降低调用频率 |
| 431 | HTTP 重复请求| -  |
