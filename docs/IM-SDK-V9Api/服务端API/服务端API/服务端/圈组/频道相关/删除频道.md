
云信服务端支持删除频道。 

::: note notice 
删除频道需要拥有频道管理权限，否则将返回 403 错误码。
:::


## URL
```http
POST https://api.yunxinapi.com/nimserver/qchat/removeChannel.action HTTP/1.1
Content-Type:application/x-www-form-urlencoded;charset=utf-8
```


## 请求参数

- POST 请求中 Headers 的设置请参考[API调用方式](/docs/TM5MzM5Njk/jk3MzY2MTI)。

- POST 请求中 Body 的设置如下：

<table>
    <thead><tr> <th>参数</th><th>类型</th><th>必须</th><th>说明</th></tr></thead>
        <tr> <td>accid</td><td>String</td><td>是</td><td>用户帐号</td></tr>
        <tr> <td>channelId</td><td>Long</td><td>是</td><td>频道 ID，频道唯一标识</td></tr>
</table>

## 示例

### cURL 请求示例
```curl
curl -X POST -H "AppKey: go9dn**l1kglw0803mgq3" -H "Nonce: 4tg**23t23t" -H "CurTime: 1443592222" -H "CheckSum: 9e9db3b6c9abb2e1962cf3e6f7316fcc55583f86" -H "Content-Type: application/x-www-form-urlencoded" -d 'accid=zhangsan&channelId=1513535' 'https://api.yunxinapi.com/nimserver/qchat/removeChannel.action'
```

### 返回示例


http 响应:json
```json
"Content-Type": "application/json; charset=utf-8"
{
  "code":200
}

```

## 状态码



该接口在 HTTPS Body 中返回请求的状态码，以下仅列出与接口业务相关的状态码。完整状态码请参见<a href="https://doc.yunxin.163.com/messaging/server-apis/TM5NTk2Mzc?platform=server" target="_blank">状态码</a>。


|状态码 | 说明 | 处理建议|
|---- | -------------- | ---------|
| 200 | 请求成功  | - |
| 403 | 非法操作或没有权限 |   <ul><li>检查是否已开通圈组功能</li><li>检查是否拥有相应的权限，如频道管理权限</li></ul>  |
| 404 | 对象不存在  | <ul><li>检查是否存在必传参数为空的问题</li><li>检查传入的账号或频道 ID 是否存在</li></ul>
| 414 | 参数错误 | 根据提示信息，检查传入参数的格式和限制条件|
| 416 |调用频率超限 | 降低调用频率 |
| 431 | HTTP 重复请求| -  |
