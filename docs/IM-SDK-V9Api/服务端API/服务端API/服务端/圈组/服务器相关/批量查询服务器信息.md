

云信服务端支持传入最多 200 个服务器 ID 查询这些服务器的配置信息，包括包含的频道数量、邀请模式、申请模式等。

## URL 
```http
POST https://api.yunxinapi.com/nimserver/qchat/getServers.action HTTP/1.1
Content-Type:application/x-www-form-urlencoded;charset=utf-8
```
请求中Headers的设置请参考[接口概述](/docs/TM5MzM5Njk/jk3MzY2MTI)



## 请求参数


- POST 请求中 Headers 的设置请参考[API调用方式](https://doc.yunxin.163.com/TM5MzM5Njk/docs/jk3MzY2MTI?platform=server)。
- POST 请求中 Body 的设置如下：



<table>
    <thead><tr> <th style="width:160px">参数</th><th style="width:80px">类型</th><th>必须</th><th>说明</th></tr></thead>
        <tr> <td>serverIds</td><td><div>String[]</div></td><td>是</td><td>服务器 ID 列表，JSON Array 格式，最多可包含 200 个服务器 ID</td></tr>
</table>




## 示例
### cURL 请求示例
```curl
curl -X POST -H "AppKey: go9dnk49**0803mgq3" -H "Nonce: 4tggge**w323t23t" -H "CurTime: 1443592222" -H "CheckSum: 9e9db3b6c9abb2e1962cf3e6f7316fcc55583f86" -H "Content-Type: application/x-www-form-urlencoded" -d 'serverIds=%5B%22123%22%2C%22234%22%5D' 'https://api.yunxinapi.com/nimserver/qchat/getServers.action'
```

## 状态码


该接口在 HTTPS Body 中返回请求的状态码，以下仅列出与接口业务相关的状态码。完整状态码请参见<a href="https://doc.yunxin.163.com/messaging/server-apis/TM5NTk2Mzc?platform=server" target="_blank">状态码</a>。


|状态码 | 说明 | 处理建议|
|---- | -------------- | ---------|
| 200 | 请求成功  | - |
| 403 | 非法操作或没有权限 |   检查是否已开通圈组功能 |
| 404  |  对象不存在   |  <ul><li>检查是否存在必传参数为空的问题</li><li>检查传入的服务器 ID 是否存在</li></ul>
| 414 | 参数错误 | 根据提示信息，检查传入参数的格式和限制条件|
| 416 |调用频率超限 | 降低调用频率 |
| 431 | HTTP 重复请求| -  |
