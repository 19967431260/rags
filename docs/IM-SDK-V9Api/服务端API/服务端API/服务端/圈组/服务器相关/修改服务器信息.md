云信服务端支持修改服务器的配置信息，包括服务器名称、服务器图标、邀请模式等。

## **URL**
```http
POST https://api.yunxinapi.com/nimserver/qchat/updateServer.action HTTP/1.1
Content-Type:application/x-www-form-urlencoded;charset=utf-8
```
## **参数说明**

- POST 请求中 Headers 的设置请参考[API调用方式](https://doc.yunxin.163.com/TM5MzM5Njk/docs/jk3MzY2MTI?platform=server)。
- POST 请求中 Body 的设置如下：

## **示例**

### **请求示例（curl）**
```curl
curl -X POST -H "AppKey: go9dnk4**w0803mgq3" -H "Nonce: 4tggge**23t23t" -H "CurTime: 1443592222" -H "CheckSum: 9e9db3b6c9abb2e1962cf3e6f7316fcc55583f86" -H "Content-Type: application/x-www-form-urlencoded" -d 'serverId=1513535&accid=zhangsan&name=update' 'http://api.yunxinapi.com/nimserver/qchat/updateServer.action'
```

### **请求成功返回示例**

```json
"Content-Type": "application/json; charset=utf-8"
{
  "code":200, 
  "serverInfo": {
    "serverId": 123, //服务器唯一标识
    "name": "name", //名称
    "icon": "icon", //图标 
    "custom": "custom", //自定义扩展
    "owner": "zhangsan", //所有者的用户帐号
    "memberNumber": 1, //成员数量
    "inviteMode": 0, //邀请模式
    "applyMode": 0, //申请模式
    "channelNumber": 0, //频道数量
    "validFlag": 1, //有效标志：0-无效，1-有效
    "createTime": 1234, //创建时间
    "updateTime": 1234 //更新时间
  }
}

```
### **请求失败返回示例**

```
"Content-Type": "application/json; charset=utf-8"
{
    "code": 414,
    "desc": "accid not exist!" 
}
```




## 状态码


该接口在 HTTPS Body 中返回请求的状态码，以下仅列出与接口业务相关的状态码。完整状态码请参见<a href="https://doc.yunxin.163.com/messaging/server-apis/TM5NTk2Mzc?platform=server" target="_blank">状态码</a>。


|状态码 | 说明 | 处理建议|
|---- | -------------- | ---------|
| 200 | 请求成功  | - |
| 403 | 非法操作或没有权限 |   <ul><li>检查是否已开通圈组功能</li><li>检查是否拥有管理服务器的权限</li><li>检查服务器资料信息（包括图标和名称等）中是否包含违规或敏感内容。如果已开通安全通且被检测到存在违规或敏感信息，可能会被安全通的内容审核判定为“不通过”</li></ul> |
| 404  |  对象不存在   |  <ul><li>检查是否存在必传参数为空的问题</li><li>检查传入的账号、服务器 ID 或自定义反垃圾业务 ID （bid）是否存在</li></ul>
| 414 | 参数错误 | 根据提示信息，检查传入参数的格式和限制条件|
| 416 |调用频率超限 | 降低调用频率 |
| 431 | HTTP 重复请求| -  |
