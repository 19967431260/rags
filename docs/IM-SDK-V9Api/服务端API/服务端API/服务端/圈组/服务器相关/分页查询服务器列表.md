
云信服务端支持分页查询圈组中已创建的服务器。



## URL 
```http
POST https://api.yunxinapi.com/nimserver/qchat/getServerListPage.action HTTP/1.1
Content-Type:application/x-www-form-urlencoded;charset=utf-8
```



## 请求参数


- POST 请求中 Headers 的设置请参考[API调用方式](https://doc.yunxin.163.com/TM5MzM5Njk/docs/jk3MzY2MTI?platform=server)。
- POST 请求中 Body 的设置如下：


<table>
    <thead><tr> <th style="width:160px">参数</th><th style="width:80px">类型</th><th>必须</th><th>说明</th></tr></thead>
        <tr> <td>accid</td><td>String</td><td>是</td><td>操作者的 IM 帐号</td></tr>
        <tr> <td>timetag</td><td>Long</td><td>是</td><td>查询 timetag 之后加入的服务器，单位毫秒，传 0 表示查询首页。分页会按照加入时间倒序排序</td></tr>
        <tr> <td>limit</td><td>String</td><td>是</td><td>本次查询的数量上限，最大数量 200</td></tr>
</table>



## 示例
### cURL 请求示例
```curl
curl -X POST -H "AppKey: go9dnk**803mgq3" -H "Nonce: 4tgggergi**t23t" -H "CurTime: 1443592222" -H "CheckSum: 9e9db3b6c9abb2e1962cf3e6f7316fcc55583f86" -H "Content-Type: application/x-www-form-urlencoded" -d 'accid=zhangsan&timetag=1558938768520&limit=10' 'https://api.yunxinapi.com/nimserver/qchat/getServerListPage.action'
```

## 状态码

该接口在 HTTPS Body 中返回请求的状态码，以下仅列出与接口业务相关的状态码。完整状态码请参见<a href="https://doc.yunxin.163.com/messaging/server-apis/TM5NTk2Mzc?platform=server" target="_blank">状态码</a>。


|状态码 | 说明 | 处理建议|
|---- | -------------- | ---------|
| 200 | 请求成功  | - |
| 403 | 非法操作或没有权限 |   检查是否已开通圈组功能 |
| 404  |  对象不存在   |  <ul><li>检查是否存在必传参数为空的问题</li><li>检查传入的 IM 账号是否存在</li></ul>
| 414 | 参数错误 | 根据提示信息，检查传入参数的格式和限制条件|
| 416 |调用频率超限 | 降低调用频率 |
| 431 | HTTP 重复请求| -  |
