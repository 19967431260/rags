
云信服务端支持圈组服务器的创建者调用 API 删除服务器。 

::: note notice 
仅服务器创建者可删除服务器。
:::


## **URL**
```http
POST https://api.yunxinapi.com/nimserver/qchat/removeServer.action HTTP/1.1
Content-Type:application/x-www-form-urlencoded;charset=utf-8
```

## **参数说明**

- POST 请求中 Headers 的设置请参考[API调用方式](https://doc.yunxin.163.com/TM5MzM5Njk/docs/jk3MzY2MTI?platform=server)。
- POST 请求中 Body 的设置如下：


<table>
    <thead><tr> <th style="width:160px">参数</th><th style="width:90px">类型</th><th>必填</th><th>说明</th></tr></thead>
        <tr> <td>accid</td><td>String</td><td>是</td><td>服务器创建者的 IM 账号</td></tr>
        <tr> <td>serverId</td><td>Long</td><td>是</td><td>服务器 ID</td></tr>
</table>

## **示例**

### **cURL 请求示例**

```curl
curl -X POST -H "AppKey: go9dnk4**803mgq3" -H "Nonce: 4tggg**t23t" -H "CurTime: 1443592222" -H "CheckSum: 9e9db3b6c9abb2e1962cf3e6f7316fcc55583f86" -H "Content-Type: application/x-www-form-urlencoded" -d 'serverId=1513535&accid=zhangsan' 'http://api.yunxinapi.com/nimserver/qchat/removeServer.action'
```

### **请求成功返回示例**

```json
"Content-Type": "application/json; charset=utf-8"
{
  "code":200
}
```

### **请求失败返回示例**

```
"Content-Type": "application/json; charset=utf-8"
{
    "code": 414,
    "desc": "accid not exist!" 
}
```

## **状态码**



该接口在 HTTPS Body 中返回请求的状态码，以下仅列出与接口业务强相关的状态码。完整状态码请参见<a href="https://doc.yunxin.163.com/messaging/server-apis/TM5NTk2Mzc?platform=server" target="_blank">状态码</a>。


|状态码 | 说明 | 处理建议|
|---- | -------------- | ---------|
| 200 | 请求成功  | - |
| 403 | 非法操作或没有权限 |   <ul><li>检查是否已开通圈组功能</li><li>检查传入的 accid 是否为服务器创建者的 IM 账号</li></ul> |
| 404  |  对象不存在   |  <ul><li>检查是否存在必传参数为空的问题</li><li>检查传入的账号或服务器 ID 是否存在</li></ul>
| 414 | 参数错误 | 根据提示信息，检查传入参数的格式和限制条件|
| 416 |调用频率超限 | 降低调用频率 |
| 431 | HTTP 重复请求| -  |
