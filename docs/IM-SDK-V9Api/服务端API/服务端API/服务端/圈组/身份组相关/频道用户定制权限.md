

除了可以通过频道身份组对所有身份组成员在频道维度进行权限控制，也可以为单个频道成员专门定制权限，管控其在频道维度的操作。





## <span id='创建频道下某人的定制权限'>创建用户定制权限</span>


为某个成员创建定制权限。新创建的定制权限配置默认继承自频道身份组相应权限的配置。


::: note notice
服务器创建者以外的用户调用该 API 必须先拥有管理角色权限和管理频道的权限，且必须是该频道的成员。
:::

### URL
```http
POST https://api.yunxinapi.com/nimserver/qchat/createUserIdentify.action HTTP/1.1
Content-Type:application/x-www-form-urlencoded;charset=utf-8
```


### 请求参数

- POST 请求中 Headers 的设置请参考[API调用方式](/docs/TM5MzM5Njk/jk3MzY2MTI)。

- POST 请求中 Body 的设置如下：

<table>
    <thead><tr> <th><div style="width:160px">参数</div></th><th><div style="width:80px">类型</div></th><th><div style="width:30px">必须</div></th><th>说明</th></tr></thead>
        <tr> <td>accid</td><td>String</td><td>是</td><td>操作者accid</td></tr>
        <tr> <td>serverId</td><td>String</td><td>是</td><td>服务器唯一标识</td></tr>
	<tr> <td>channelId</td><td>String</td><td>是</td><td>频道的唯一标识</td></tr>
	<tr> <td>faccid</td><td>String</td><td>是</td><td>被操作的用户的accid</td></tr>
</table>



### 示例


#### curl请求示例
```curl
curl -X POST -H "AppKey: go9dnk49bk**03mgq3" -H "Nonce: 4tggge**323t23t" -H "CurTime: 1443592222" -H "CheckSum: 9e9db3b6c9abb2e1962cf3e6f7316fcc55583f86" -H "Content-Type: application/x-www-form-urlencoded" -d 'serverId=1513535&accid="accid"&channelId=10010&faccid="faccid"' 'http://api.yunxinapi.com/nimserver/qchat/createUserIdentify.action'
```

#### 返回示例
http 响应:json
```json
"Content-Type": "application/json; charset=utf-8"
{
	"code": 200,
	"identify": {
		"serverId": 2,//服务器唯一标识
		"channelId": 1,//频道唯一标识
		"createtime": 1,//创建时间
		"accid": "accid",//用户的accid
		"auths": "{\"1\":-1, \"2\":1}",//个人定制权限，key为具体权限项的枚举值，value为 1 时表示开启，为 -1 时表示关闭，为 0 时表示该权限项的开启状态继承自频道身份组的对应权限项
		"avatar": "name",//用户在服务器下的头像
		"inviter": "icon",//邀请者的用户账号
		"joinTime": 1,//用户加入服务器的时间
		"nick": "icon",//昵称
		"memberType": 1,//类型：0-普通成员，1-所有者
		"custom": "icon",//用户的自定义扩展字段
		"updatetime": 1//更新时间
	}
}

```

### 状态码
该 API 在 HTTPS Body 中返回请求的状态码，状态码详情请参见[状态码](https://doc.yunxin.163.com/messaging/server-apis/TM5NTk2Mzc?platform=server)。

该 API 的状态码主要如下：
200、403、414、416、431、500

## <span id='更新频道下用户定制权限'>修改用户定制权限</span>



::: note notice
服务器创建者以外的用户调用该 API 必须先拥有管理角色权限和管理频道的权限，且必须是该频道的成员。
:::



### URL 
```http
POST https://api.yunxinapi.com/nimserver/qchat/updateUserIdentify.action HTTP/1.1
Content-Type:application/x-www-form-urlencoded;charset=utf-8
```


### 请求参数

<table>
    <thead><tr> <th><div style="width:160px">参数</div></th><th><div style="width:80px">类型</div></th><th><div style="width:30px">必须</div></th><th>说明</th></tr></thead>
        <tr> <td>accid</td><td>String</td><td>是</td><td>操作者accid</td></tr>
        <tr> <td>serverId</td><td>String</td><td>是</td><td>服务器唯一标识</td></tr>
	<tr> <td>channelId</td><td>String</td><td>是</td><td>频道的唯一标识</td></tr>
	<tr> <td>faccid</td><td>String</td><td>是</td><td>被操作的用户的accid</td></tr>
	<tr> <td>auths</td><td>String</td><td>否</td><td> <p>用于修改权限的 JSON 字段, 其格式为 {resource1:type1,resource2:type2,resource3:type3}。</p>
         <ul>
          <li>resource 表示权限, 其取值代表的具体权限项如下：
            <ul>
<li>2：管理频道的权限，拥有该权限可创建、修改或删除频道 </li>
                <li>3：管理角色的权限，拥有该权限可创建、修改或删除身份组</li>
                <li>4：发送消息的权限</li>
                <li>9：撤回他人消息的权限</li>
                <li>10：删除他人消息的权限</li>
                <li>11：@ 他人的权限，拥有该权限可在发送消息时@他人</li>
                <li>12：@ 所有人的权限，拥有该权限可在发送消息时@所有人</li>
                <li>13：管理频道黑白名单的权限，拥有该权限可将服务器成员加入或移出频道黑名单</li>
                <li>15：实时互动频道：建立自己连接的权限 </li>
                <li>16：实时互动频道：断开他人连接的权限 </li>
                <li>17：实时互动频道：开启自己麦克风的权限 </li>
                <li>18：实时互动频道：开启自己摄像头的权限 </li>
                <li>19：实时互动频道：开启/关闭他人麦克风的权限 </li>
                <li>20：实时互动频道：开启/关闭他人摄像头的权限 </li>
                <li>21：实时互动频道：开启/关闭全员麦克风的权限 </li>
                <li>22：实时互动频道：开启/关闭全员摄像头的权限 </li>
                <li>23：实时互动频道：打开自己屏幕共享的权限 </li>
                <li>24：实时互动频道：关闭他人屏幕共享的权限 </li>
                <li>27：@身份组的权限，拥有该权限可在发送消息时@指定几个身份组的所有成员 </li>
            </ul>
        </li>
        <li> 
        <p>type 表示对权限项的状态类型，其取值代表的状态如下：
        <ul><li>-1：关闭权限（deny）</li>
        <li>1：开启权限（allow）</li>
        <li>0: 继承频道维度下对应权限项</li></ul></p>
        </li>
        </ul></td></tr>
</table>

## <span id='分页查询频道下用户的定制权限'>分页查询用户定制权限</span>



分页查询某些频道成员的定制权限，查询到的结果按照创建时间由大到小排序。




### URL 
```http
POST https://api.yunxinapi.com/nimserver/qchat/getUserIdentifyPages.action HTTP/1.1
Content-Type:application/x-www-form-urlencoded;charset=utf-8
```



### 请求参数


- POST 请求中 Headers 的设置请参考[API调用方式](/docs/TM5MzM5Njk/jk3MzY2MTI)。

- POST 请求中 Body 的设置如下：
