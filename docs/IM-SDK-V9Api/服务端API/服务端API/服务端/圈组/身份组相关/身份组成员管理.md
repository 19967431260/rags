



## <span id='把某些人拉进某服务器身份组'>添加服务器身份组成员</span>

将某些用户拉入某个服务器身份组。


::: note notice 
调用该 API 必须先拥有管理角色权限（服务器创建者默认拥有该权限）。
:::

### URL
```http
POST https://api.yunxinapi.com/nimserver/qchat/createServerIdentifyMember.action HTTP/1.1
Content-Type:application/x-www-form-urlencoded;charset=utf-8
```



### 请求参数


- POST 请求中 Headers 的设置请参考[API调用方式](/docs/TM5MzM5Njk/jk3MzY2MTI)。

- POST 请求中 Body 的设置如下：

<table>
    <thead><tr> <th><div style="width:160px">参数</div></th><th><div style="width:80px">类型</div></th><th><div style="width:30px">必须</div></th><th>说明</th></tr></thead>
        <tr> <td>accid</td><td>String</td><td>是</td><td>操作者accid</td></tr>
        <tr> <td>serverId</td><td>String</td><td>是</td><td>服务器唯一标识</td></tr>
		<tr> <td>roleId</td><td>String</td><td>是</td><td>服务器身份组唯一标识</td></tr>
		<tr> <td>faccids</td><td>String</td><td>是</td><td>被操作的用户的accid集合，jsonarray格式 ["accid1","accid2"]<note type='notice'>一次最多传入 100 个账号 ID。</note></td></tr>
</table>


### 示例

#### curl请求示例
```curl
curl -X POST -H "AppKey: go9dnk4**3mgq3" -H "Nonce: 4tggge**23t23t" -H "CurTime: 1443592222" -H "CheckSum: 9e9db3b6c9abb2e1962cf3e6f7316fcc55583f86" -H "Content-Type: application/x-www-form-urlencoded" -d 'serverId=1513535&accid="accid"&roleId=1&faccids="[\"accid1\",\"accid2\"]"' 'http://api.yunxinapi.com/nimserver/qchat/createServerIdentifyMember.action'
```

#### 返回示例
http 响应:json
```json
"Content-Type": "application/json; charset=utf-8"
{
    "code":200,
	"successAccids":"[\"accid1\",\"accid2\"]",//添加成功的accid集合
	"failAccids":"[\"accid1\",\"accid2\"]"//添加失败的acci集合
}

```

### 状态码
该 API 在 HTTPS Body 中返回请求的状态码，状态码详情请参见[状态码](https://doc.yunxin.163.com/messaging/server-apis/TM5NTk2Mzc?platform=server)。

该 API 的主要状态码如下：
200、403、414、416、431、500

## <span id='分页查询某个服务器身份组中有哪些人'>分页查询服务器身份组成员</span>

分页查询某个服务器身份组中的成员列表。

::: note notice 
- 调用该 API 必须先拥有管理角色权限（服务器创建者默认拥有该权限）。
- 仅自定义身份组（即 `type` 为 `2` 的身份组）支持调用该接口查询成员名单。创建服务器时默认创建的 @everyone 身份组 （即 `type` 为 `1` 的身份组）不支持调用该接口查询成员名单。如果调用该接口进行查询，将报错（错误码 403）。
:::
### URL 
```http
POST https://api.yunxinapi.com/nimserver/qchat/getServerIdentifyMemberPages.action HTTP/1.1
Content-Type:application/x-www-form-urlencoded;charset=utf-8
```



### 请求参数

- POST 请求中 Headers 的设置请参考[API调用方式](/docs/TM5MzM5Njk/jk3MzY2MTI)。

- POST 请求中 Body 的设置如下：

<table>
    <thead><tr> <th><div style="width:160px">参数</div></th><th><div style="width:80px">类型</div></th><th><div style="width:30px">必须</div></th><th>说明</th></tr></thead>
        <tr> <td>accid</td><td>String</td><td>是</td><td>操作者accid</td></tr>
        <tr> <td>serverId</td><td>String</td><td>是</td><td>服务器唯一标识</td></tr>
	<tr> <td>roleId</td><td>String</td><td>是</td><td>服务器身份组唯一标识</td></tr>
	<tr> <td>timetag</td><td>String</td><td>否</td><td>根据用户进入身份组的时间从大到小排序，分页的锚点，每页页尾数据的创建时间，不传或者传0默认查当前时间往前limit条数据</td></tr>
	<tr> <td>faccid</td><td>String</td><td>否</td><td>当一页数据页尾多条数据的创建时间相同时，根据用户的accid作为锚点来拉取下一页数据</td></tr>
	<tr> <td>limit</td><td>String</td><td>否</td><td>每页查询的条数，不填时为200，最大200</td></tr>
</table>

### 示例
#### cURL请求示例
```curl
curl -X POST -H "AppKey: go9dnk4**w0803mgq3" -H "Nonce: 4tggge**3t23t" -H "CurTime: 1443592222" -H "CheckSum: 9e9db3b6c9abb2e1962cf3e6f7316fcc55583f86" -H "Content-Type: application/x-www-form-urlencoded" -d 'serverId=1513535&accid="accid"&roleId=1&timetag=1111111111111&faccid="accid1"&limit=20' 'http://api.yunxinapi.com/nimserver/qchat/getServerIdentifyMemberPages.action'
```

### 返回示例
http 响应:json
```json
"Content-Type": "application/json; charset=utf-8"
{
    "code":200,
	"serverMembers":[{
	"accid": "lisi", //成员的用户帐号
	"nick": "nick", //昵称
	"avatar": "avatar", //头像
	"custom": "custom", //自定义扩展
	"memberType": 0, //类型：0-普通成员，1-所有者
	"joinTime": 1234, //加入时间
	"inviter": "zhangsan", //邀请者的用户帐号
	"createTime": 1234, //创建时间
	"updateTime": 1234 //更新时间
	}]
}

```

### 状态码
该 API 在 HTTPS Body 中返回请求的状态码，状态码详情请参见[状态码](https://doc.yunxin.163.com/messaging/server-apis/TM5NTk2Mzc?platform=server)。

该 API 主要状态码如下：
200、403、414、416、431、500

### URL 
```http
POST https://api.yunxinapi.com/nimserver/qchat/getUserServerIdentify.action HTTP/1.1
Content-Type:application/x-www-form-urlencoded;charset=utf-8
```



### 请求参数
