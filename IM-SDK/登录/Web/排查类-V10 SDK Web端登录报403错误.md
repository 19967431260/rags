## 问题：V10 SDK Web端登录报403错误

## 问题详情

**现象**：
客户使用V10 SDK Web端调用login方法报错403，但登录实际是成功的。

## 排查过程

1. 客户提供appkey和account → 客服查询服务器日志
2. 发现该时间点登录成功，无403记录 → 判断为SDK内部权限检查报错
3. 让客户开启debug参数并提供浏览器console日志
4. 发现是V10 SDK，报错原因是云端会话权限和客户端反垃圾权限未开启
**关键发现**：登录实际成功，403是SDK内部其他权限检查的报错

## 问题原因

V10 SDK需要开启云端会话权限和客户端反垃圾权限，否则SDK内部会报403错误，但不影响登录本身。

## 解决方案

V10 SDK Web端登录报403但实际登录成功时，需在控制台开启：1. 云端会话权限；2. 客户端反垃圾权限。开启后错误消失。注意：V10目前没有本地会话功能，预计1月20日左右版本会支持。
