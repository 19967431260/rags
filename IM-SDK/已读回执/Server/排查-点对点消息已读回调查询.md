## 问题：点对点消息已读回调查询

## 问题详情

**现象**：
客户反馈部分消息没有收到已读回调，需要查询是否触发。

**环境信息**：
- 平台：服务端

## 排查过程

1. 客户提供msgid_client查询已读回调情况
2. 解释已读回执不是客户端自动上报，需要调用接口发送
3. 查询后台日志发现已读回执抄送记录存在
4. 解释时间戳比对逻辑：timestamp比消息时间戳大代表该时间戳之前的消息都已读

## 根因分析

客户对已读回执的触发机制和回调内容理解有误

## 解决方案

已读回执需要客户端调用接口发送，不是自动触发。回调中的timestamp字段表示该时间戳之前的消息都已读，不是针对单条消息的回调。
