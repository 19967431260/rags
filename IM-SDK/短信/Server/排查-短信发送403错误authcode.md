## 问题：Server 短信发送403错误authcode sms no authorization

## 问题详情

**现象**：
客户签名模板审核通过后发送短信报错403,错误信息为authcode sms no authorization。调用接口返回200但未收到短信

## 排查过程

1. 检查签名模板状态 → 审核已通过
2. 检查资源包 → 资源包用完
3. 重新打开短信开关 → 接口调用成功返回200
4. 检查短信是否送达 → 未收到短信
5. 查询签名状态 → 运营商判定签名不活跃已剔除

**关键发现**：长期未发送导致运营商判定签名不活跃

## 问题原因

1. 资源包用完导致403错误
2. 太久未发送短信,运营商判定签名不活跃并剔除

## 解决方案

1. 购买资源包后重新打开短信开关
2. 技术支持重新向运营商报备签名(5-7个工作日)
3. 报备期间可先调整配置进行测试
