## 问题：Server 聊天室接口并发调用返回417错误

## 问题详情

**现象**：
客户并发调用/chatroom/toggleCloseStat.action接口操作同一个聊天室开闭状态，导致其中一个接口报错code 417。roomid:7332744790在2026-02-02 09:30:23.855和09:30:23.511两个时间点并发调用。

**相关日志**：
code 417

## 排查过程

1. 技术支持发现并发调用开关聊天室接口 → 其中一个返回417错误
2. 发现另一个时序问题 → 关闭聊天室后立即请求登录地址失败(errcode=13002)，随后又重新打开
3. 与客户沟通调用逻辑 → 客户之前在开启前先调用关闭，不等结果直接开启

**关键发现**：客户端接口调用时序问题，未等待前序接口返回结果

## 问题原因

客户端在开播前先调用关闭聊天室接口，不等返回结果直接调用开启接口，导致并发操作同一聊天室

## 解决方案

调整调用时序：将关闭操作挪到预览页面，在主播进入预览页面时先关闭，等设置完成后点开播按钮再开启，错开调用时间。code 417表示当前聊天室状态已经是入参中valid参数指定的状态。
