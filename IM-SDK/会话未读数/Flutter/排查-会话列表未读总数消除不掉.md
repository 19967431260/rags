## 问题：Flutter 会话列表未读总数消除不掉

## 问题详情

**现象**：
用户反馈会话列表消息未读总数一直有25个消除不掉，每个可见的会话都已读了。

## 排查过程

1. 询问是否有会话过滤或隐藏 → 客户反馈有系统账号消息但会删除
2. 拉取SDK日志分析 → 发现sessionid:1648843 unreadCount:3和sessionid:1387154 unreadCount:22
3. 客户反馈这两个会话不应该有未读数 → 最后聊天时间分别是9月8号和8月26号
4. 深入分析日志 → 发现是8月26号的消息在2025-09-24 21:07重新漫游下发了
5. 询问是否卸载重装 → 用户确实在09-24 21:07卸载重装
6. 分析原因 → 删除会话时调用了clearChattingHistory但没有调用clearSessionUnreadCount清空未读数
7. 检查配置 → 客户已配置shouldSyncUnreadCount: true

## 问题原因

用户卸载重装后漫游消息重新下发，之前删除会话时未清空未读数导致

## 解决方案

删除会话时同时调用clearSessionUnreadCount清空会话未读数；不要卸载app，通过控制台下发日志拉取任务保留现场日志
