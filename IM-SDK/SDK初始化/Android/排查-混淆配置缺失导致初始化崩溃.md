## 问题：Android Android SDK初始化崩溃NullPointerException

## 问题详情

**现象**：
Android端使用IM SDK 10.5.0版本，调用NimCore.instance.initialize初始化时崩溃，报错：java.lang.NullPointerException: Attempt to invoke interface method 'n4.a z1.k.r()' on a null object reference。崩溃发生在NimServiceV2创建时。

**环境信息**：
- SDK 版本：10.5.0
- 平台：Android

## 排查过程

1. 检查SDK版本 → 确认为10.5.0
2. 检查混淆配置 → 客户按文档配置后仍崩溃
3. 确认初始化线程 → 建议确保在主线程初始化

**关键发现**：缺少代码混淆配置或混淆配置不正确导致SDK内部类被混淆

## 问题原因

代码混淆配置缺失或不正确，导致SDK内部类被混淆，初始化时出现空指针异常

## 解决方案

添加SDK混淆配置，参考文档：https://doc.yunxin.163.com/messaging2/guide/DIyNzI1MDA?platform=client#防混淆配置-android，并确保初始化在主线程中进行
