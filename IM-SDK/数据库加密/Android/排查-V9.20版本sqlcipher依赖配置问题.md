## 问题：Android V9.20版本sqlcipher依赖配置问题

## 问题详情

**现象**：
客户询问云信9.20版本是否还需要sqlcipher-android依赖，以及找不到4.6.1版本的问题。

## 排查过程

1. 确认需要 → 需要4.6.1版本
2. 客户反馈找不到 → 客户使用阿里云镜像
3. 提供依赖配置 → implementation "net.zetetic:sqlcipher-android:4.6.1"
4. 建议仓库配置 → 把mavenCentral()放前面
5. 排查代理问题 → 看起来没走到mavenCentral()下载

## 问题原因

阿里云镜像可能没有该依赖，需要使用mavenCentral()

## 解决方案

依赖使用implementation "net.zetetic:sqlcipher-android:4.6.1"，仓库把mavenCentral()放前面
