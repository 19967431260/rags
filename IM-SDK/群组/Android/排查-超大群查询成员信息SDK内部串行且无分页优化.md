## 排查过程

1. 客户提供查询耗时截图 → 确认耗时问题
2. 客服确认SDK内部串行执行 → 定位到技术原因
3. 需判断每个群成员本地用户信息是否存在 → 确认具体原因
4. 分页查询queryMemberListByPage也有同样问题 → 确认分页也无法优化
5. 已给研发提优化需求 → 后续版本可能优化

**关键发现**：SDK内部queryMemberList串行执行，5596人×用户信息查询导致约1秒耗时

## 问题原因

SDK内部queryMemberList串行执行，且需要判断每个群成员的用户信息是否存在（不存在则查云端），5596人×用户信息查询=约1秒耗时

## 解决方案

超大群成员查询耗时问题：
1. SDK内部为串行逻辑，无分页优化
2. 进入群聊时同步群成员的建议：避免在进入群聊时同步全量成员信息
3. 可考虑分批拉取或延迟加载
4. 分页查询（queryMemberListByPage）也有相同耗时问题
