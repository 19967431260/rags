## 问题：Android V9覆盖安装V10后登录被踢KICKOUT

## 问题详情

**现象**：
客户从V9覆盖安装V10后，observeOnlineStatus回调KICKOUT被踢掉。调整了初始化顺序，v9 init改成了v10 config + initSdk，但问题依然存在。仅升级云信版本，没有代码变更，也会被踢。

**环境信息**：
- 平台：Android
- 版本：V9 -> V10

## 排查过程

1. 检查初始化顺序 → 调整后问题依然存在
2. 检查日志路径配置 → 确认日志路径调整与问题无关
3. 对比V9和V10日志 → 发现设备id一样，V9日志也有被踢情况
4. 分析安装顺序 → 发现是安装过v10又安装过v9，然后又安装V10才出的问题

**关键发现**：V9->V10->V9->V10的反复覆盖安装会导致被踢

## 问题原因

安装过v10又安装过v9，然后又安装V10才出的问题，存在版本覆盖安装的限制

## 解决方案

卸载重装，然后安装V9-登录成功-覆盖安装V10即可正常。避免V9->V10->V9->V10的反复覆盖安装。
