## 问题：Windows Windows清理会话未读数后未清零

## 问题详情

**现象**：
Windows端客户调用清理会话未读数接口后，有个会话未读数一直保持着未清零。

## 排查过程

1. 客户询问清理会话未读数是否有回调接口可以看状态
2. 技术支持回复清理会话未读数会触发会话变更
3. 客户询问是否会导致会话从最近会话列表清除
4. 确认不会清除
5. 客户反馈调用后未读数没清零
6. 技术支持解释清理未读会触发会话变更返回最新会话信息，UI需要展示最新会话（未读数为0）

## 问题原因

清理会话未读数后会触发会话变更回调，需要UI层更新展示最新的会话信息（未读数为0）。

## 解决方案

监听会话变更回调，更新UI展示最新的会话信息，其中未读数为0。
