## 问题：服务端API调用403错误时间戳校验失败

## 问题详情

**现象**：
调用云信服务端API时返回403错误，提示时间差问题，客户尝试调整时间加减5分钟、8小时均无效。

## 排查过程

1. 客户反馈API调用返回403时间差错误
2. 技术支持确认curtime需使用UTC时间戳（秒）
3. 建议每次请求重新获取当前UTC时间戳
4. 提供Apifox模板对比排查

## 问题原因

curtime参数使用UTC时间戳（秒），非毫秒，且需在请求瞬间前后5分钟内

## 解决方案

使用UTC时间戳（秒），每次请求时重新获取当前时间戳，确保服务器时间与标准时间同步
