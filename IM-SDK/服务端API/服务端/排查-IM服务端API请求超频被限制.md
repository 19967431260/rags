## 问题：服务端 IM服务端API请求超频被限制

## 问题详情

**现象**：
调用IM服务端API时出现请求超频错误，客户截图显示查询过快。当前频控上限为1000，业务场景需要提升到1500。

**环境信息**：
- SDK 版本：
- 系统版本 / 设备：
- 平台：服务端

## 排查过程

1. 客户反馈API调用报错 → 技术支持判断为请求超频
2. 客户确认当前频控为1000，希望提升到1500 → 技术支持提供控制台配置路径
3. 客户反馈1500频控费用较高 → 商务介入沟通折扣

## 问题原因

IM服务端API调用频率超过默认频控限制（1000次/秒）。

## 解决方案

两种解决方案：1）业务层做防抖节流，控制请求频率；2）在网易云信控制台-应用管理-频控管理中提升频控上限（会产生额外费用）。
