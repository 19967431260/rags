## 问题：HarmonyOS 鸿蒙端发送语音消息报错

## 问题详情

**现象**：
鸿蒙端发送语音消息时遇到错误，code==0但发送不成功。

## 排查过程

1. 分析日志发现是文件路径问题 → 定位问题
2. 建议参考demo实现，文件路径要copy到el2目录 → 提供方案
3. 发现sdk版本之间没对齐 → 进一步排查
4. 确认是areaMode设置问题 → 确认根因

**关键发现**：areaMode设置为el1导致

## 问题原因

文件路径设置问题，areaMode设置为el1导致，应使用el2的cacheDir。

## 解决方案

使用el2目录存储音频文件：
```typescript
const catchDir = getContext(this).getApplicationContext().cacheDir
```
不要修改context，文件路径不要放在el1的database里面。
