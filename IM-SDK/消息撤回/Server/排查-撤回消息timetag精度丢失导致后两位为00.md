## 问题：Server 撤回消息timetag精度丢失导致后两位为00

## 问题详情

**现象**：
客户反映群成员已不在群里，想撤回其之前发的消息时失败，客服排查发现timetag为1754440333004，疑似精度丢失导致后两位被自动设为00。

## 排查过程

1. 客户提供撤回失败截图 → 确认问题
2. 客服确认撤回与是否在群里无关，问题是message字段信息没匹配 → 排查方向
3. 客服发现timetag精度丢失，后两位自动设成00 → 根因定位
4. 客服展示raw格式数据对比，指出createtime和timetag不一致 → 数据验证
5. 客户确认自行排查数据问题 → 后续

**关键发现**：消息的timetag字段在传输或存储过程中精度丢失，后两位被自动置为00，导致撤回时message字段信息不匹配而失败

## 问题原因

消息的timetag字段在传输或存储过程中精度丢失，后两位被自动置为00，导致撤回时message字段信息不匹配而失败。

## 解决方案

撤回消息需要精确的timetag字段值，检查消息创建时传入的timetag是否有精度丢失（后两位被设为00），需确保timetag_raw与createtime精确对应后再发起撤回。
