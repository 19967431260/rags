## 问题：服务端 第三方回调超时导致违规图片放行

## 问题详情

**现象**：
用户发的图片被反垃圾系统拦截（errcode=1），但用户侧还是能看到图片。

**环境信息**：
- SDK 版本：
- 系统版本 / 设备：
- 平台：服务端

## 排查过程

1. 检查消息ID → null
2. 检查回调配置 → 发现callback.timeout为2000ms
3. 确认原因 → 第三方回调超过2秒未响应，走了默认放行
4. 提供解决方案 → 可改为默认拦截或提高超时时间

## 问题原因

第三方回调超时（2秒），触发默认放行机制

## 解决方案

在控制台修改第三方回调配置，可设置为默认拦截或适当提高超时时间（最高4-5秒）
