## 问题：Web Web端用户会话历史记录丢失部分消息

## 问题详情

Web端用户刷新页面后，部分时间段的会话历史记录丢失。客户端看到的消息数量少于实际发送的消息数量。使用getHistoryMsgs查询云端历史记录，但客户端显示数据不完整。

## 排查过程

1. 检查服务器日志 → 云端数据完整，getHistoryMsgs返回11条消息正确
2. 检查客户端调用 → 客户已全部改用getHistoryMsgs（纯云端查询）
3. 尝试在指定时间点查询服务器日志 → 未查到客户端调用getHistoryMsgs的记录
4. 要求提供SDK debug日志 → 客户提供的是业务层日志，无SDK日志
5. 对比无痕模式和正常模式 → 无痕模式数据正常，正常模式数据丢失
**关键发现**：怀疑本地数据库写入异常导致本地会话未更新，但getHistoryMsgs应返回正确数据。需要SDK debug日志确认实际调用情况。

## 问题原因

大概率是本地数据库写入异常导致本地最近会话没变更成功。无痕模式下无本地数据库，直接查云端数据正常。

## 解决方案

1. 关闭本地数据库初始化（V10已放弃数据库支持）；2. 本地会话根据漫游生成，列表展示时本地会话查完后用服务端会话补充；3. 如需继续使用数据库，必须提供SDK debug日志（debug:true）以便研发排查数据库异常。
