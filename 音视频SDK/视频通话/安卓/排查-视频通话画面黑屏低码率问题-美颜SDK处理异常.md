## 问题：安卓 视频通话画面黑屏低码率问题-美颜SDK处理异常

## 问题详情

**现象**：
多人视频通话中，部分用户看到对方画面黑屏，排查发现发送方码率极低（十kbps左右）。现象为某用户进房间后上行帧率正常但码率极低，类似发送了纯黑画面。

## 排查过程

1. 服务端排查确认上行帧率存在但码率异常低 → 确认发送端问题
2. 进一步分析确认是美颜SDK的setVideoCallback回调处理帧数据时出现异常 → 定位到美颜SDK问题
3. 导致发送出去的画面编码为极低码率的纯黑帧发送给对端 → 确认问题机制

**关键发现**：美颜SDK的setVideoCallback回调处理帧数据时出现异常，将原始正常画面处理成极低码率的纯黑帧

## 问题原因

使用setVideoCallback接入美颜SDK后，美颜侧帧数据处理异常，将原始正常画面处理成极低码率的纯黑帧发送给对端，导致对端看到黑屏

## 解决方案

1. 排查美颜SDK（特别是荣耀9X等特定机型）的setVideoCallback回调实现
2. 可尝试关闭美颜功能，确认视频通话是否恢复正常
3. 如果关闭美颜正常，说明是美颜SDK处理帧数据存在问题，需要美颜SDK侧修复
4. 如果仍有问题，需获取SDK日志进一步排查
