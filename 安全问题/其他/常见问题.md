## Q: Android SDK 9.20.17 版本中 libhigh_available_android.so 和 libquicclient-jni.so 存在 OpenSSL 1.1.1 安全漏洞如何解决？

**答案**：升级到 SDK 9.21.10 版本，并使用以下配置排除旧版本组件：
implementation('com.netease.nimlib:basesdk:9.21.10') {
    exclude group: 'com.netease.nimlib', module: 'fusionstorage'
    exclude group: 'com.netease.nimlib', module: 'quic'
}
该版本已将 OpenSSL 升级到 3.5.2，解决了 SSL_select_next_proto 函数检查缺陷等安全漏洞。

## Q: 升级到 Android SDK 9.21.10 后，libhigh_available_android.so 和 libartemis.so 仍然检测到 OpenSSL 和 c-ares 安全漏洞？

**答案**：需要进一步排除 Artemis 模块。使用以下完整配置：
implementation('com.netease.nimlib:basesdk:9.21.10') {
    exclude group: 'com.netease.nimlib', module: 'fusionstorage'
    exclude group: 'com.netease.nimlib', module: 'quic'
    exclude group: 'com.netease.yunxin.artemis', module: 'Artemis'
}
Artemis 模块包含旧版本的 c-ares 库依赖。排除后需要 clean 工程重新打包。SDK 内部已使用 openssl 3.5.2 和 c-ares 1.34.3。
