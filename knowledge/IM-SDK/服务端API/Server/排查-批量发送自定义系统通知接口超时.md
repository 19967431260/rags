## 问题：Server 批量发送自定义系统通知接口超时

## 问题详情

**现象**：
批量发送自定义系统通知sendBatchAttachMsg接口链接超时，单次发送约4900个用户

## 排查过程



## 问题原因

单次账号多，又存了离线，时间肯定长

## 解决方案

建议单次发送500或1000个接收方，不要一次性发送过多（API限制5000，但建议分批）
