## 问题：Web Web端会话列表被清空问题

## 问题详情

**现象**：
Web端IM出现会话列表被清空的情况，只有有新消息的会话会出现，查看数据库发现新会话的session数据是全的，其他session数据都不全，没有updatetime。此前正常使用，没有清理过缓存和数据库。

## 排查过程

1. 确认会话列表通过nim.db.getAllSessions()获取 → 确认获取方式
2. 检查nim实例和控制台调用返回内容 → 检查数据
3. 发现无痕模式正常，判断为数据同步问题 → 定位问题类型
4. 检查本地db中的漫游时间戳，发现存在但漫游消息未下发 → 定位具体原因
5. 确认为快速切换页面导致数据没同步完整 → 确认根因

**关键发现**：快速切换页面导致漫游消息同步不完整

## 问题原因

快速切换页面导致漫游消息同步不完整，本地db中的漫游时间戳存在但消息未正常下发。

## 解决方案

1. 首页初始化不要开db，防止快速操作导致db脏数据
2. 删除db中timetag表的漫游时间戳后重新登录
3. 服务端扩容漫游下发限制：15天200个会话，每个会话最近50条
