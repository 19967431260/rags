# 服务端发消息发送方查询不到问题

## 问题描述

服务端以用户A身份给用户B发送消息，用户B能查询到消息，但用户A（发送方）查询不到该消息

## 问题背景

1. 确认双方都使用queryMessageListEx查询
2. 分析消息发送参数 → 发现发送时设置了不存离线、不存漫游
3. 确认发送时用户A不在线

## 根因分析

服务端发送消息时设置了不存离线(option.persistEnable=0)、不存漫游(option.roamEnable=0)，发送方不在线且消息不存漫游，所以发送方无法查询到

## 解决方案

服务端发送消息时如需发送方也能查询到，需要设置option.persistEnable=1（存离线）和option.roamEnable=1（存漫游），确保消息能被双方查询
