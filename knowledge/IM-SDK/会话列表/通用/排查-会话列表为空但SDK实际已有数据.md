## 问题：通用 会话列表为空但SDK实际已有数据

## 问题详情

**现象**：
用户启动 App 后最近会话列表为空，表现为页面没有会话数据。客户已尝试重启 App、退出重登和切换网络，但问题仍存在。

**环境信息**：
- 产品：IM V9
- 现象：最近会话列表为空、消息似乎拉不到

## 排查过程

1. 技术支持先核对用户在线/离线时间，确认账号并非持续在线。
2. 客户补充问题为最近会话列表为空，并已尝试重启、重登、切网等动作。
3. 技术支持下发日志任务拉取 SDK 日志。
4. 日志中确认 SDK 实际已有 3 条会话数据，同时看到业务存在清理会话未读相关方法调用。
5. 结合日志与截图判断，问题不在云信数据拉取，而在业务侧 UI 展示或过滤逻辑。

**关键发现**：SDK 层已有会话数据，但业务 UI 未展示。

## 问题原因

业务侧 UI 展示或过滤逻辑导致会话列表未显示，非 SDK 拉取不到数据。

## 解决方案

本地用问题账号调试业务会话列表展示逻辑，重点排查会话过滤、清理未读或会话处理代码；若 SDK 日志已显示有会话数据，应优先排查 UI 侧。
