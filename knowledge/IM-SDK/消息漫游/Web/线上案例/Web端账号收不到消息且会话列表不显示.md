## 排查过程

1. 确认登录成功但消息收发异常 → 确认问题范围
2. 服务端协议日志显示有收到消息但会话列表不更新 → 服务端有消息但客户端未处理
3. 使用demo复现确认是SDK问题 → 排除业务代码
4. 定位到是SDK处理消息时to字段参数带空格导致漫游失败 → 根因定位
5. 找到具体消息：merchant_1005发给30382的消息，to字段值为'30382 '（带尾部空格） → 定位到具体数据
6. 临时方案：删除该云端消息+漫游消息 → 临时解决

**关键发现**：发消息时to字段（对方账号）带了空格，导致SDK在处理消息漫游时异常，消息无法正常写入本地数据库

## 问题原因

发消息时to字段（对方账号）带了空格，导致SDK在处理消息漫游时异常，消息无法正常写入本地数据库。

## 解决方案

1. 调用服务端消息删除接口删除该条云端消息（messageId: b6fec89999416c7d8a82379f4e772a17，from: merchant_1005，to: 30382）
2. 检查业务代码发消息时to字段确保不包含空格
