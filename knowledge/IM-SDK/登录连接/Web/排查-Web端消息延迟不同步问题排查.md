## 问题：Web端消息延迟不同步问题排查

## 问题详情

**现象**：
客户反馈云信PC端(Web)消息和APP端不同步，PC端总是延迟。用户云信ID：1175746285。11:20在APP发消息，11:22截图时PC端看不到该消息。

**环境信息**：
- 平台：Web

## 排查过程

1. 检查服务器日志发现WEB在该时间点处于掉线状态，直到2025-03-12 14:10:49才重新登录
2. 查看登录记录，发现最近7天有98次长链接被踢的情况
3. 分析发现用户开了多个tab，登录实例包含移动端超过了10个
4. 解释'悄悄被踢'机制：SDK先发起了链接A，因包超时等原因尝试断开A并发起B，但服务器先处理B再处理A，处理A时会给B下发'悄悄被踢'，但业务上无感知，因为仍会有一条有效长链接
5. 查看登录记录，发现三个操作系统登录过该账号（Windows Server 2008、OS X 10.15.7、Windows 10）

## 根因分析

用户账号被多设备共用，同时在线的长链接超过10个（云信后台配置的策略上限），导致新登录时最早的长链接被踢掉。

## 解决方案

云信登录策略是服务器累积达到10条长链接后，如有新的长链接创建，会踢掉最早的那一条。建议用户检查是否账号被共用，关闭其他电脑的浏览器页面，只在出问题的电脑登录观察。
