## 排查过程

1. 确认错误码 → 416是超频
2. 客户提供accid和时间点查询 → 未查到对应记录，只查到之前时间点的记录
3. 确认客户端限制规则 → 不是1s100次，而是连续一分钟300次上限
4. 建议按时间戳相减的一分钟计算，不是15:15~15:16这种整分钟

**关键发现**：

## 问题原因

客户端API调用限制：连续一分钟内最多300次，不是每秒100次

## 解决方案

客户端API调用限制为连续一分钟300次上限，需按时间戳相减的一分钟计算，避免连续一分钟内超过300次调用
