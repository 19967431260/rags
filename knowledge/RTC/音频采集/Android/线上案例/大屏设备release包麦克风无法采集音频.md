## 问题：Android 大屏设备release包麦克风无法采集音频

## 问题详情

**现象**：
在大屏设备上使用音视频通话功能，debug包麦克风可以正常收音，但release包不能收音。手机端可以听到大屏端声音，但大屏端无法采集音频

**环境信息**：
- SDK 版本：4.4.0
- 设备类型：大屏设备

## 排查过程

1. 检查服务器日志 → 日志显示设备有正常播放音量，但采集异常
2. 对比debug和release包日志 → 发现volume有区别
3. 检查混淆配置 → 确认debug和release的混淆配置一致
4. 使用release包+测试key测试 → 仍无声音，排除key配置问题
5. 设置日志等级为info，对比测试key和线上key的日志 → 发现3A算法配置差异
6. 调整3A算法配置，对齐测试环境参数 → 问题解决

**关键发现**：线上环境的3A算法默认参数在调整回声配置时被错误引入，导致大屏设备使用了不恰当的音频处理参数

## 问题原因

服务端针对特定设备型号下发不同的3A算法参数。在24年10月调整回声配置时，将900多个设备的回声参数复制到线上环境，导致默认参数被覆盖，大屏设备使用了不适配的音频采集参数

## 解决方案

调整服务端3A算法配置，将线上环境的默认参数对齐到测试环境的参数。针对定制设备建议在各场景充分测试，如有异常可联系技术支持针对性调整设备参数。
