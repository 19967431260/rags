## 问题：Web端RTC ICE连接失败问题

## 问题详情

**现象**：
Web端通话时一方听不到另一方声音，ICE连接建立失败。

**环境信息**：
- 平台：Web
- 部署方式：私有化环境
- 外网用户IP：121.36.124.33

**复现步骤**：
1. 外网用户加入会议
2. 通话建立
3. 出现听不到声音的情况

## 排查过程

1. **信息收集** → 确认通话双方信息
   - 获取会议ID
   - 确认双方uid
   - 检查accid和音视频uid对应关系

2. **日志分析** → 查看服务器日志
   - 发现用户11289网络连接一直失败
   - ICE连接建立失败
   - 用户219971和17862973013对应的音视频uid未找到

3. **网络排查** → 确认网络问题
   - 外网用户连接到私有化服务器超时
   - 排除VPN因素后仍有问题
   - 判断为网络网关层面问题

4. **对比测试** → 微信通话正常
   - 改用微信通话无问题
   - 确认是本地网络到私有化服务器的网络问题

**关键发现**：外网用户ICE连接超时，私有化环境网关可能存在异常

## 问题原因

1. **网络层面**：外网用户到私有化音视频服务器的ICE连接超时
2. **网关配置**：私有化环境网关可能存在配置问题
3. **网络限制**：用户网络可能存在未发现的限制

## 解决方案

1. **用户侧排查**：
   - 检查是否开启VPN
   - 尝试切换网络环境
   - 联系网管检查网络层配置

2. **服务端排查**：
   - 检查私有化环境网关配置
   - 确认端口开放情况
   - 检查防火墙设置

3. **信息收集**：
   - 收集双方console日志
   - 记录会议ID和uid
   - 保留网络抓包数据
