## 排查过程

1. 确认 rtc.hbzxb.com.cn 解析正确 → DNS 正常
2. 客户端 demo 测试正常，但客户侧无法连接 → 客户侧环境问题
3. 客服远程排查后确认：媒体流走的是 IP 地址 → 直接 IP 通信
4. 更换 IP 后 UDP 端口映射未同步更新，导致媒体流无法建立 → 根因确认
5. 客服修改 IP 下发配置后测试仍有问题 → 进一步排查
6. 最终确认 UDP 端口策略在 IP 变更后需要重新检查并配置，问题解决 → 问题解决

**关键发现**：服务器公网 IP 变更后，媒体流 UDP 端口映射未同步更新，导致媒体通道无法建立

## 问题原因

服务器公网 IP 变更后，媒体流 UDP 端口映射未同步更新，导致媒体通道无法建立

## 解决方案

更换服务器 IP 后，必须同步检查并更新 UDP 端口映射配置。媒体流走的直接 IP 通信，不仅仅是 DNS 解析。确认 UDP 端口在新的出口 IP 上有正确监听和转发配置后，问题才解决。
