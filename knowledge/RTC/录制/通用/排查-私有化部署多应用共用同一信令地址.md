## 问题：通用 私有化部署多应用共用同一信令地址

## 问题详情

**现象**：
客户部署私有化 RTC 环境，合同签的是部署一套供两个平台使用，但客户侧发现需要配置2个域名地址（yspt.okzhaobiao.com 和 rtc.hbzxb.com.cn），询问为什么需要2个域名地址接入。

## 排查过程

1. 客服确认一个服务器环境只支持配置一个地址下发 webrtc wss 连接地址 → 架构说明
2. 确认多个业务平台接入同一套私有化时，音视频平台会下发热一套媒体地址和信令地址 → 不同业务平台数据隔离，但接入地址可以共用
3. 客户后来更换了 rtc.hbzxb.com.cn 域名的 IP，导致 UDP 端口不通 → IP 变更问题

**关键发现**：客户不理解多应用共用同一信令地址的架构；另外更换服务器 IP 后媒体流 UDP 端口映射未同步更新导致连接失败

## 问题原因

客户不理解多应用共用同一信令地址的架构；另外更换服务器 IP 后媒体流 UDP 端口映射未同步更新导致连接失败

## 解决方案

多个应用接入同一套私有化 RTC 时只需提供一个公用的 webrtc 信令地址即可。更换服务器 IP 后需要同步更新媒体流 UDP 端口的映射配置。客户端需确保 wss 信令地址和媒体流 IP 均已正确配置。
