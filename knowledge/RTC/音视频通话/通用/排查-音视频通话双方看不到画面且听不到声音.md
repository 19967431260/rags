# 音视频通话双方看不到画面且听不到声音

## 问题描述

坐席端和客户端互相看不到视频画面，也听不到声音，坐席端能看到自己画面

## 问题背景

1. 确认坐席能看到自己画面 → 本地采集正常
2. 询问双方是否都在国内 → 确认都在国内
3. 查看服务端日志 → 发现用户一直在掉线重连
4. 客户提供SDK日志 → 最终发现是行内WiFi网络限制导致

## 根因分析

银行内部WiFi网络限制导致用户持续掉线重连，无法正常建立音视频连接

## 解决方案

检查银行内部网络策略，确保WiFi网络允许RTC流量通过，必要时联系网络管理员调整防火墙规则
