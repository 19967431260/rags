## Q: iOS医患沟通中患者发送的图片显示不出来

**问题描述**：医患沟通交流时，患者发送的图片显示不出来，辽宁地域用户出现问题。

**答案**：
1. 端午期间辽宁移动和联通地域运营商对 nim-nosdn.netease.im 域名有劫持，会影响该地域网络用户的图片下载。
2. 云信发现问题后针对该域名做了临时调度，但临时调度对SDK版本有要求。
3. 需要检查用户SDK版本是否支持临时调度。
4. 可以让用户提供accid信息，检查SDK日志确认具体问题。

## Q: 安卓4.0设备用SDK没法下载语音附件报证书错误

**问题描述**：客户反馈安卓4.0的设备使用SDK无法下载语音附件，报证书错误，咨询是否是因为上次云信升级G2证书导致的。

**答案**：
上次升级是服务端API域名的证书，没有动存储域名的证书。证书校验不是SDK侧校验的，理论上和SDK版本没有直接关系。

## Q: iOS Framework证书问题

**问题描述**：客户反馈iOS应用上传时提示framework证书有问题，按照网上处理方式都试过了还是报错。

**答案**：
1. 报错信息：`Invalid bundle structure. The "BreathTrain.app/Frameworks/YXAlog_iOS.framework/YXAlog_ios" binary file is not permitted`
2. 这是YXAlog库的问题，新版本已经去掉了该库。
3. 临时解决方案：在Podfile中添加post_install脚本，对YXAlog框架进行bitcode_strip处理：
```ruby
post_install do |installer|
  installer.pods_project.targets.each do |target|
    if target.name == 'YXAlog'
      `xcrun -sdk iphoneos bitcode_strip -r Pods/YXAlog/YXAlog_iOS.framework/YXAlog_iOS -o Pods/YXAlog/YXAlog_iOS.framework/YXAlog_iOS`
    end
  end
end
```
4. 参考链接：https://blog.csdn.net/shaobo8910/article/details/111315261
