# Playbook: 音视频服务端接口调用排查

- source file: `/Users/yxgc/.openclaw/workspace/tmp/popo-doc-export/docs/0b3ffacd5f7b4097bb9c9049f628f640__音视频服务端接口调用排查.md`
- scope: RTC 服务端接口 / 后端日志检索
- intent: 面向接口调用失败、录制文件未生成、404/400/401 等问题做快速反查

## 适用场景

适用于：
- 客户反馈服务端接口报错
- 录制/安全通/权限控制等接口调用异常
- 请求成功发出但客户不确定服务端是否收到

## 最小输入

至少要有：
- appkey
- 接口 URL 或接口路径
- 问题时间点

最好补齐：
- requestId
- cid / cname
- 原始请求参数
- 原始 response / 错误码

## 首先判断

1. 客户到底有没有实际发起该接口调用？
2. 服务端是否收到了请求？
3. 收到后是鉴权失败、请求体非法，还是资源不存在？
4. 该接口本身是否有“产品能力不保证”的预期限制？

## 分支排查路径

### A. 先查接口调用记录

文档建议使用 QS 内部 logsearch / kibana 查询。高频检索字段：
- `requestId`
- `appkey`
- `url`
- `cid`
- `cname`

优先动作：
- 用字段过滤，不要只全文搜索
- 若客户说“没生成录制文件”，先确认录制接口是否真的调用过、是否报错

### B. 常见错误码快速分流

#### B1. 401 `incomplete auth token`
含义：请求鉴权参数不全。

优先检查请求头是否包含：
- `AppKey`
- `Nonce`
- `CurTime`
- `CheckSum`

#### B2. 400 `invalid request`
含义：请求体 JSON 解析失败。

高频触发：
- v3 接口中字段类型不兼容
- 把数字字段传成 `null`

#### B3. 404 `NOT FOUND`
含义：请求对象不存在，或接口语义与目标对象不匹配。

文档案例：
- 查询房间状态接口不支持白板房间，只支持 RTC 房间。
- 某些接口必须带 query 参数（如 `?cname=xxx`），漏带时会表现为 404。

### C. 产品预期限制类问题

#### C1. 安全通检测任务不会可靠自动停止
不要把“未自动停止”直接判断成故障。
文档明确说明：
- 房间内无主播
- 最后一个人退出
- 房间自动关闭
- 主动关闭房间

以上都**不提供即时可靠的自动停止保证**。如需停止，应主动调停止接口。

### D. 案例化排查方法

当客户反馈“调用封禁某人视频流 API 一直 404”：
1. 查指定 URL
2. 查 request 内容
3. 对比接口要求是否缺失 query 参数（如 `cname`）
4. 再和客户确认 SDK/服务端封装层是否丢参

## 证据不足时必须补充

必须追补：
- requestId 或原始请求时间
- 完整 URL（含 query）
- 请求头关键鉴权字段是否带齐
- 请求体原文
- response 原文

## 不要轻易下的结论

- 不要只凭客户口述就认定“服务端没处理”；先查调用记录。
- 不要把所有 404 都当成对象不存在；很多时候是 query 参数缺失或接口对象不匹配。
- 不要把安全通“未自动停止”默认视为 bug；文档明确说产品不保证。
- 不要在没有请求体原文时判定 400 根因；字段类型错误非常常见。

## 升级条件

- 日志显示请求已到服务端，参数看起来正确，但响应仍与接口语义不符
- 同一接口批量客户集中报错
- 录制/安全通等后台任务状态与日志相互矛盾
