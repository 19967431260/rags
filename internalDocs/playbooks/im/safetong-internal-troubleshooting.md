# Playbook: 安全通综合排查（IM / RTC / 协作边界）

- source file: `/Users/yxgc/.openclaw/workspace/tmp/popo-doc-export/docs/f36564f2f70343ea95c8552f0bb14057__安全通内部排查文档.md`
- scope: 安全通全局总览，覆盖 IM、RTC、云信/易盾协作边界、策略语义与配置生效问题

## 适用场景

适用于以下类型的问题：
- 客户对安全通整体机制不理解，不清楚哪些内容走同步、哪些走异步。
- 用户反馈“命中了但结果看不懂”“疑似配置为什么没生效”“业务 id 怎么区分来源”。
- 需要判断问题属于云信接入配置、易盾审核效果、还是双方协作边界。
- 涉及 RTC/直播音视频安全审核、截图保留、断流联动等问题。

## 最小输入

### IM 场景
- appkey
- 业务场景（单聊/群聊/聊天室/圈组/用户资料/头像/自定义消息）
- 消息类型（文本/图片/音频/视频/自定义）
- 发送时间点
- 发送方 / 接收方 / 群 id / 聊天室 id
- taskid 或 guid（有一个就很好）
- 当前控制台配置截图：业务勾选、疑似处理、抄送配置、businessId 相关配置

### RTC/直播场景
- appkey
- 频道号 / 房间号
- 时间点
- 音视频类型、语种、审核频率配置
- 抄送地址 / 抄送样例
- guid 或 taskid

## 首先判断

1. **先分产品面**
   - IM 安全通
   - RTC / 直播音视频安全通

2. **再分同步 / 异步检测**
   - IM 文本、图片：通常按同步检测处理。
   - IM 音频、视频：通常按异步检测处理。
   - RTC/直播：按抄送和审核事件联动处理。

3. **再分问题类型**
   - 接入/配置问题
   - 检测结果理解问题
   - 疑似/删除策略理解问题
   - 配置生效延迟问题
   - 易盾侧效果/耗时问题

## 分支排查路径

### 路径 A：是否走了安全通、是否走了通用反垃圾

1. 判断是否已开通安全通。
2. 文档给出的经验口径：**安全通开启后不再走通用反垃圾**。
3. 若用户仍反馈像是通用反垃圾命中：
   - 先核实现网日志字段，而不是直接引用口径。
   - 如与日志矛盾，应升级确认版本行为差异。

### 路径 B：消息是否应送检、哪些字段可跳过

1. 服务端发送普通消息：默认会过安全通。
2. 普通消息可用 `useYidun` 控制不过。
3. 自定义消息通过字段 `antispam` 控制。
4. 若客户想过滤扩展参数：
   - 先确认该字段是否只是业务附带参数、无检测必要。
   - 某些字段（如 `ext` 内违规内容）易盾并不支持按预期检测，不要承诺一定能检出。

### 路径 C：怎么看检测结果

1. 客户要看文本/图片反垃圾结果与原因：
   - 指导开启消息抄送。
   - 重点看 `yidunRes` 中的 `action`、`label`、`suggestion`。
2. 解释建议结果：
   - `suggestion=0`：通过
   - `suggestion=1`：嫌疑
   - `suggestion=2`：删除
3. **疑似处理配置只对 `suggestion=1` 生效**。
   - 如果客户说“广告已设为通过，但还是被拦截”，必须先确认该样本是不是 `suggestion=2`。

### 路径 D：为什么配置了疑似处理，仍被拦截

1. 检查样本的 `suggestion`。
2. 若是 `1`：再继续检查疑似配置是否正确、是否命中对应业务场景。
3. 若是 `2`：直接说明**嫌疑配置不生效**，因为这不是嫌疑数据，而是删除级数据。

### 路径 E：配置变更为什么没有实时生效

1. 文档口径：后台配置有约 **10 分钟本地缓存**。
2. 若客户刚改完配置就复测失败：
   - 先让其等待缓存失效再测。
   - 不要马上判定“配置无效”。

### 路径 F：检测耗时 / 超时问题

1. 若客户反馈发送耗时长或审核结果慢：
   - 先核实客户给出的请求时间与结果时间是否一致。
2. 若确认是易盾检测耗时：
   - 提供 `taskid + 请求时间点` 给易盾技术支持。
3. 文档中给出的超时口径是 **3 秒**；与其他文档口径不一致时，应以具体日志和现网行为为准。

### 路径 G：白名单 / 差异化策略 / 多来源区分

1. 接收方白名单：
   - 可基于提交内容中的 `receiveUid` 在易盾侧做全放行。
2. 同类内容但不同来源希望采用不同检测标准：
   - 可通过不同业务和 `antiSpamConfigId` 区分到不同 `businessId`。
3. 若客户想强制文本/图片始终过检、绕过客户端传参：
   - 这是产品/研发级能力申请，不是现场配置即可解决的问题。

### 路径 H：资源删除、撤回、平台职责边界

1. 图片/视频命中后，易盾返回的是检测结果，不负责替客户删除其资源。
2. NOS 资源默认有效期 7 天，与同步/异步检测本身无直接因果关系。
3. 异步场景下的撤回/处置，通常仍需客户业务逻辑配合。

### 路径 I：RTC / 直播安全通

1. 判断客户是在问：
   - 审核频率
   - 是否可查看命中截图
   - 命中后是否可自动断流
   - 外语音频是否支持指定模型/语种
2. 对应处理：
   - 默认审核频率约 5 秒，可调。
   - 命中截图默认保留约 7 天。
   - 需要断流可联动调用禁用频道接口。
   - 多语种/指定语种能力需要后台配置，且受模型支持范围约束。

## 证据不足时必须补充

- appkey
- 时间点
- 问题样本的 taskid 或 guid
- 抄送原文或日志原文（尤其 `yidunRes` / `suggestion`）
- 控制台配置截图
- businessId / antiSpamConfigId / receiveUid 等关键配置项
- RTC 场景下的频道号、语种、审核频率配置

## 不要轻易下的结论

- 不要把“嫌疑配置没生效”直接归因为平台 bug，先看 `suggestion`。
- 不要承诺易盾会替业务方删除客户资源或自动撤回所有消息。
- 不要默认所有场景都实时生效，先考虑缓存延迟。
- 不要把效果问题、请求异常问题、产品配置问题混成一类。
- 不要在没有 taskid/guid 的情况下直接要求易盾做深度排查。

## 升级条件

- 云信日志与“安全通开启后不再走通用反垃圾”的经验口径冲突。
- 需要核实 businessId / antiSpamConfigId 绑定关系。
- 需要强制过检、默认兜底 businessId 等产品能力。
- RTC 场景涉及语种模型、断流联动、审核截图等后台能力配置。
- 需易盾确认检测异常、耗时异常、命中效果与预期不符。
