# Skill Market 系统架构

## 概述

Skill Market 是一个企业级的 Skill（技能/插件）管理和分发平台，支持多租户、权限控制、Skill 全生命周期管理。本文档介绍系统的整体架构、核心组件和技术栈。

## 2. 核心组件

### 2.1 Frontend (前端)

**技术栈：**
- **框架**：Next.js 14 (App Router)
- **UI 库**：React 18, shadcn/ui, Tailwind CSS
- **状态管理**：React Query (TanStack Query)
- **HTTP 客户端**：封装的 `@/lib/http`

**目录结构：**
```
frontend/
├── src/
│   ├── app/              # Next.js App Router 路由
│   │   ├── (auth)/       # 认证相关页面
│   │   ├── (dashboard)/  # 主应用页面
│   │   └── api/          # API 路由
│   ├── components/       # React 组件
│   │   ├── ui/           # shadcn/ui 基础组件
│   │   └── ...           # 业务组件
│   ├── lib/              # 工具库
│   │   ├── http.ts       # HTTP 客户端
│   │   └── utils.ts      # 通用工具
│   └── hooks/            # 自定义 Hooks
└── public/               # 静态资源
```

### 2.2 Backend (后端)

**技术栈：**
- **框架**：NestJS 10
- **ORM**：TypeORM
- **数据库**：MySQL 5.7+
- **缓存**：Redis

**目录结构：**
```
backend/
├── src/
│   ├── common/           # 公共模块
│   │   ├── decorators/   # 自定义装饰器
│   │   ├── guards/       # 认证/权限 Guard
│   │   ├── interceptors/ # 拦截器
│   │   └── filters/      # 异常过滤器
│   ├── modules/          # 业务模块
│   │   ├── auth/         # 认证模块
│   │   ├── users/        # 用户模块
│   │   ├── tenants/      # 租户模块
│   │   ├── skills/       # Skill 模块
│   │   ├── roles/        # 角色权限模块
│   │   └── ...
│   └── main.ts           # 入口文件
├── database/
│   └── migrations/       # 数据库迁移
└── test/                 # 测试文件
```

### 2.3 Agent Manager Core

**技术栈：**
- **语言**：Go 1.21+
- **HTTP 框架**：Gin
- **功能**：Agent 管理、任务分发

**目录结构：**
```
agent-manager-core/
├── cmd/
│   └── server/           # 服务入口
├── internal/
│   ├── api/              # API 处理器
│   ├── service/          # 业务逻辑
│   └── model/            # 数据模型
├── pkg/                  # 公共包
└── Makefile              # 构建脚本
```

### 2.4 ClawHive (桌面客户端)

**技术栈：**
- **框架**：Electron 28
- **UI**：React + Tailwind CSS
- **IPC**：Electron IPC

**功能：**
- 用户登录认证
- Skill 浏览和安装
- Agent (WEClawHelper) 管理
- 本地任务执行

### 2.5 WEClawHelper (Agent)

**技术栈：**
- **语言**：Go
- **功能**：本地任务执行、心跳上报

**功能：**
- 向后端注册和保持心跳
- 接收和执行任务
- Skill 下载和管理
- 结果上报

## 4. 数据流

### 4.1 用户登录流程

```
用户 → 前端 → 后端 /auth/login → 验证凭证 → 生成 JWT → 返回 Token
```

### 4.2 Skill 发布流程

```
开发者创建 Skill → 提交审核 → 运营审核 → 审核通过 → 发布到市场
                               ↓
                          审核拒绝 → 修改后重新提交
```

### 4.3 Agent 任务执行流程

```
用户触发任务 → 后端创建任务 → Agent 拉取任务 → 本地执行 → 上报结果 → 后端更新状态
```

## 6. 技术决策

### 6.1 为什么选择 NestJS？

- **模块化架构**：便于组织复杂业务逻辑
- **TypeScript 原生**：类型安全，开发体验好
- **依赖注入**：便于测试和解耦
- **装饰器模式**：声明式的路由和校验
- **生态丰富**：大量官方和社区模块

### 6.2 为什么选择 Go 做 Agent？

- **编译型语言**：分发简单，无需运行时
- **低资源占用**：适合常驻后台
- **并发友好**：goroutine 天然适合并发任务
- **跨平台**：一套代码编译多平台二进制

### 6.3 为什么选择 Electron 做桌面端？

- **跨平台**：一套代码支持 Windows/macOS/Linux
- **Web 技术栈**：团队技术复用
- **丰富的 API**：系统级操作支持好
- **自动更新**：electron-updater 方案成熟

## 7. 扩展性设计

### 7.1 水平扩展

- **后端服务**：无状态设计，可多实例部署
- **数据库**：支持读写分离
- **缓存**：Redis Cluster 支持

### 7.2 垂直扩展

- **模块化**：新功能以模块形式添加
- **插件化**：Skill 类型可扩展
- **多租户**：租户级别的资源隔离
