## 问题：首次安装接听立即挂断：未在 IM 登录后初始化呼叫组件

## 问题详情

**现象**：
首次安装App后，点击接听音视频通话立即挂断。杀掉进程重启App后，一切正常。复现步骤：安装App → 发起通话 → 点击接听 → 立即挂断。

**环境信息**：
- 呼叫组件版本：2.5.1
- IM V9 API
- APP使用了IM自动登录

## 排查过程

1. 获取xkit日志，确认报错码130001（joinRtc failed）
2. 确认是初始化呼叫组件时没有设置当前IM用户账号
3. APP使用IM自动登录，在登录成功前就尝试初始化呼叫组件，导致获取IM账号信息失败
4. 建议在IM登录成功回调中初始化呼叫组件

**关键发现**：joinRtc失败（130001）是因为呼叫组件初始化时拿不到IM登录账号信息，需要在IM登录流程完成后再初始化呼叫组件。

## 问题原因

初始化呼叫组件时未正确设置当前IM用户账号。IM走自动登录时，登录流程和呼叫组件初始化时序不匹配，导致呼叫组件初始化时获取不到IM账号，进而joinRtc失败。

## 解决方案

1. 把呼叫组件的初始化放到IM手动登录接口成功回调里
2. IM走自动登录时，也同时初始化呼叫组件
3. 切换账号的情况也需覆盖
4. 建议去掉sdkStorageRootPath自定义配置（使用默认值），这样后续拉日志时能同时拉取RTC和组件日志，方便排查问题
